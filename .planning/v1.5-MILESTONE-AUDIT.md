## Lessons Learned

- **Retroactive phase verification based on code reading alone missed a subtle GTK layout bug.** `Gtk.Picture`'s `measure()` returns source image natural size regardless of `set_size_request` (a minimum, not a cap) and regardless of `hexpand/vexpand=False` (which only govern extra space allocation). Future phases dealing with runtime layout should require live UAT, not static inspection, before being marked satisfied. Add "requires live display UAT" as an explicit gate for any phase touching GTK widget sizing or container allocation.
- **Audit verification artifacts should distinguish "retroactively satisfied by code inspection" from "verified by live UAT."** The initial audit used the same `human_needed` status for both, obscuring which requirements carried live-UAT risk.
- **The 51-row bookkeeping gap has persisted through an entire milestone** — consider having `/gsd-execute-phase` flip the REQUIREMENTS.md checkbox at phase-commit time rather than deferring the whole sweep to milestone close.
