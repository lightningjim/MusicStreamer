# Phase 65 Plan 04: Harden Win11 build to ship exactly one matching musicstreamer dist-info (VER-02-J fix)

<objective>
Close UAT gap VER-02-J — the Windows-bundled `MusicStreamer.exe` shows `v1.1.0` in the hamburger menu instead of the expected `v2.1.65`. Root cause (HIGH confidence per gap diagnosis): the Win11 VM build environment had a stale `musicstreamer-1.1.0.dist-info` left over from an old editable install made before the v2.x rewrites. When `copy_metadata("musicstreamer")` ran during PyInstaller's Analysis pass, it picked up BOTH the stale 1.1.0 dist-info AND the current 2.1.65 dist-info, bundled both, and at runtime `importlib.metadata.version("musicstreamer")` resolved to whichever appears first on `sys.path` — the stale 1.1.0 won.

Phase 65 RESEARCH §Landmine 2 maps this exact failure mode (editable-install double-distribution); the Linux dev `.venv` was verified clean during research, but the Win11 VM was not, so the bug slipped past Phase 65's automated dev-side tests and only surfaced under manual UAT.

This plan defends in depth at the build-script tier (the only tier that runs on the Win11 VM):

1. **Pre-bundle clean** — `build.ps1` uninstalls + reinstalls `musicstreamer` in the build env BEFORE invoking PyInstaller. The reinstall materializes a single fresh `musicstreamer-{pyproject_version}.dist-info` and removes any older stale dist-info(s) regardless of how they got there.
2. **Post-bundle assertion** — after PyInstaller produces `dist/MusicStreamer/`, `build.ps1` scans the bundle for `musicstreamer-*.dist-info` directories. Assert exactly one exists AND its `METADATA` `Version:` line equals `pyproject.toml [project].version`. On failure, exit non-zero with a clear error and dump the offending state to the build log. This catches future drift (any stale dist-info that survives the pre-bundle clean for any reason fails the build, not silently ships).
3. **Source-text drift-guards** — extend `tests/test_packaging_spec.py` with two regression-lock tests that verify both new steps are still present in `build.ps1`. Pure source-text tests (no PowerShell or PyInstaller invocation), matching the existing test pattern in this same file.

The runtime read sites (`musicstreamer/__main__.py` `setApplicationVersion` + `musicstreamer/ui_qt/main_window.py` menu footer) are NOT modified — they are correct on dev and will be correct on Win once the bundle stops shipping the stale dist-info. CONTEXT D-08's "no try/except fallback to a placeholder" contract is preserved because no runtime code is touched.

Purpose: Close UAT gap VER-02-J. Defend the build pipeline against future stale-dist-info drift.
Output: build.ps1 with two new defensive steps + 2 new drift-guard tests in tests/test_packaging_spec.py.

3. **Run the actual assertions** in pure PowerShell (no native command needed; `Get-ChildItem` + `Get-Content` are built in):

```powershell
    Write-Host "=== POST-BUNDLE ASSERTION: musicstreamer dist-info singleton + version match ==="

    # Bundle layout: PyInstaller --distpath ..\..\dist + spec name 'MusicStreamer'
    # produces dist/MusicStreamer/_internal/<dist-info>/.
    $bundleInternal = "..\..\dist\MusicStreamer\_internal"
    if (-not (Test-Path $bundleInternal)) {
        Write-Error "BUILD_FAIL reason=bundle_internal_not_found path='$bundleInternal' hint='pyinstaller produced an unexpected layout; check build.log'"
        exit 9
    }

    $msDistInfos = @(Get-ChildItem -Path $bundleInternal -Filter "musicstreamer-*.dist-info" -Directory -ErrorAction SilentlyContinue)
    if ($msDistInfos.Count -ne 1) {
        Write-Host "POST-BUNDLE ASSERTION FAIL: expected exactly one musicstreamer-*.dist-info, found $($msDistInfos.Count):"
        $msDistInfos | ForEach-Object { Write-Host "  - $($_.Name)" }
        Write-Error "BUILD_FAIL reason=post_bundle_distinfo_not_singleton found_count=$($msDistInfos.Count) hint='step 3c pre-bundle clean did not leave a single dist-info — investigate build env site-packages'"
        exit 9
    }

    $bundledDistInfo = $msDistInfos[0]
    $bundledMetadata = Join-Path $bundledDistInfo.FullName "METADATA"
    if (-not (Test-Path $bundledMetadata)) {
        Write-Error "BUILD_FAIL reason=bundled_metadata_missing path='$bundledMetadata' hint='dist-info shipped without METADATA file — corrupt install?'"
        exit 9
    }

    $bundledVersion = $null
    foreach ($line in (Get-Content $bundledMetadata)) {
        if ($line -match '^Version:\s*(.+?)\s*$') {
            $bundledVersion = $matches[1]
            break
        }
    }
    if (-not $bundledVersion) {
        Write-Error "BUILD_FAIL reason=bundled_metadata_no_version_line path='$bundledMetadata' hint='METADATA file present but has no `Version:` line'"
        exit 9
    }

    if ($bundledVersion -ne $appVersion) {
        Write-Host "POST-BUNDLE ASSERTION FAIL: dist-info version drift detected"
        Write-Host "  pyproject.toml [project].version : $appVersion"
        Write-Host "  bundled METADATA Version:        : $bundledVersion"
        Write-Host "  bundled dist-info dir name        : $($bundledDistInfo.Name)"
        Write-Error "BUILD_FAIL reason=post_bundle_version_mismatch bundled='$bundledVersion' expected='$appVersion' hint='step 3c pre-bundle clean did not refresh dist-info — investigate uv pip uninstall behavior'"
        exit 9
    }

    Write-Host "POST-BUNDLE ASSERTION OK — dist-info singleton: $($bundledDistInfo.Name) (version $bundledVersion matches pyproject)"
```
