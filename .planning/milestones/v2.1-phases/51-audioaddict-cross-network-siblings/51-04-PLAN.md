<objective>
Wire up the click-to-navigate flow that turns Plan 51-03's clickable links into actual dialog navigation. This plan delivers SC #2 (user can navigate from one sibling to another), enforces D-11 (Save / Discard / Cancel confirm on dirty navigation), and connects `EditStationDialog` to `MainWindow` via the new `navigate_to_sibling = Signal(int)` (D-09).

**Purpose:** Without this plan, the sibling list from Plan 51-03 is decorative — clicking a link does nothing. With this plan, clicking a link:
- on a clean dialog → opens the sibling's edit dialog directly
- on a dirty dialog → prompts Save / Discard / Cancel and routes accordingly

**Critical non-goal preserved (SC #4):** Sibling navigation must NOT introduce cross-network failover. The `_on_navigate_to_sibling` slot in MainWindow MUST NOT call any playback method (no `player.failover`, no `player.play`, no multi-stream queue manipulation). Only the edit dialog opens; playback is untouched (D-10).

**Output:** New `Signal(int)` declaration, two new methods on `EditStationDialog`, one new slot on `MainWindow`, two signal connections in MainWindow's existing dialog-open paths, plus 5 new pytest-qt tests.
</objective>

       ```python
       def _on_sibling_link_activated(self, href: str) -> None:
           """Phase 51 / D-09, D-11: handle a click on an 'Also on:' sibling link.

           Clean dialog → emit navigate_to_sibling(sibling_id) and accept().
           Dirty dialog → show Save | Discard | Cancel confirm:
             - Save: run _on_save; if it succeeds (validation passed), emit + accept.
                     If it fails (e.g. empty name), the validation QMessageBox.warning
                     in _on_save already informed the user — stay in the dialog,
                     do NOT emit navigate_to_sibling.
             - Discard: emit navigate_to_sibling, then reject() so the existing
                        _is_new placeholder-cleanup path (line 694-696) runs for
                        new-station dialogs.
             - Cancel: stay, do nothing.

           D-10 invariant: this method does NOT touch playback. No player calls,
           no failover queue manipulation. Sibling navigation is dialog-level only.

           Reads self._save_succeeded — populated by _on_save (set False at top,
           True immediately before accept()).
           """
           # Parse 'sibling://{int}'. Defensive against malformed hrefs (should
           # never happen since _render_sibling_html constructs them, but the
           # signal payload is a free-form string).
           prefix = "sibling://"
           if not href.startswith(prefix):
               return
           try:
               sibling_id = int(href[len(prefix):])
           except ValueError:
               return

           if not self._is_dirty():
               # Clean path: navigate immediately.
               self.navigate_to_sibling.emit(sibling_id)
               self.accept()
               return

           # Dirty path: confirm with Save | Discard | Cancel (D-11).
           answer = QMessageBox.question(
               self,
               "Unsaved changes",
               "You have unsaved changes. Save before navigating?",
               QMessageBox.StandardButton.Save
               | QMessageBox.StandardButton.Discard
               | QMessageBox.StandardButton.Cancel,
               QMessageBox.StandardButton.Cancel,  # default — matches accounts_dialog idiom
           )
           if answer == QMessageBox.StandardButton.Save:
               self._on_save()
               if not self._save_succeeded:
                   # Validation failed — stay; the warning dialog from _on_save
                   # already informed the user.
                   return
               # Save succeeded — _on_save already called accept(). Emit AFTER
               # accept() so MainWindow's slot runs synchronously after the
               # dialog teardown signals.
               self.navigate_to_sibling.emit(sibling_id)
               return
           elif answer == QMessageBox.StandardButton.Discard:
               # Discard path: emit FIRST (so MainWindow knows where to go),
               # then reject() to honor the existing _is_new placeholder cleanup.
               self.navigate_to_sibling.emit(sibling_id)
               self.reject()
               return
           else:
               # Cancel — stay in dialog, no signal.
               return
       ```
