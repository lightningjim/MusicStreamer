<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ZIP file (untrusted JSON) → commit_import → station_siblings | Crafted ZIP could attempt to inject sibling rows with malicious names |
| User-edited ZIP → name resolution | Names not in destination DB are silently dropped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-17 | T (Tampering) | Crafted siblings list with names matching unrelated stations | accept | Single-user desktop app; user controls both source and destination DBs; "name collision attack" is a degenerate self-inflicted case. Test `test_siblings_round_trip` confirms name-based resolution is exact-match (no fuzzy matching). |
| T-71-18 | T (Tampering) | siblings list contains non-string entries (e.g., dict, list, None) | mitigate | The `list(data.get("siblings") or [])` returns iterable; the inner loop calls `name_to_id.get(sibling_name)`. If sibling_name is not hashable (e.g., dict), `dict.get` raises TypeError. Plan adds `if not isinstance(sibling_name, str): continue` defensive guard. |
| T-71-19 | T (Tampering) | siblings list contains current station's own name | mitigate | `name_to_id` maps the current station's name to its id; lo, hi = min(sid, sid), max(sid, sid) → sid, sid → CHECK constraint a_id < b_id fires → INSERT OR IGNORE swallows. Effectively a silent no-op. Defensive: skip if `sibling_id == station_id` before INSERT. |
| T-71-20 | D (DoS) | Massive siblings list (e.g. 100k names) blocks UI | accept | ZIPs are user-curated; library size 50-200 stations means siblings list per station is in single digits. No mitigation. |

    Per 71-PATTERNS.md lines 723-743, the second pass:
    ```
    # Phase 71 D-07: resolve sibling names to IDs and write station_siblings rows
    name_to_id = {
        r["name"]: r["id"]
        for r in repo.con.execute("SELECT id, name FROM stations").fetchall()
    }
    for station_data in preview.stations_data:
        station_name = station_data.get("name", "")
        if not station_name:
            continue
        station_id = name_to_id.get(station_name)
        if station_id is None:
            continue
        sibling_names = list(station_data.get("siblings") or [])
        for sibling_name in sibling_names:
            if not isinstance(sibling_name, str):
                continue
            sibling_id = name_to_id.get(sibling_name)
            if sibling_id is None:
                continue   # silently drop unresolved names (D-07)
            if sibling_id == station_id:
                continue   # defensive: skip self-reference
            lo, hi = min(station_id, sibling_id), max(station_id, sibling_id)
            repo.con.execute(
                "INSERT OR IGNORE INTO station_siblings(a_id, b_id) VALUES (?, ?)",
                (lo, hi),
            )
    ```

    The iteration filter `detail_row.action in ("add", "replace")` is NOT required here — the second pass tolerates skipped stations (name_to_id.get returns None for those, and we skip). Simpler code path.
