## Album grouping render style

### Q1: How should the artist drill-down show album grouping?

| Option | Description | Selected |
|--------|-------------|----------|
| Section-header rows in the table | Insert visual divider rows between album groups (mirrors source HTML's `<tr class="albumTitle">`). Existing 4-column table stays the same width; rows get scannable section headers. | ✓ |
| Album column in every row | Add a 5th 'Album' column. Each row carries its album value. Sortable but every row repeats the album text — wider table, more visual noise. | |
| Both: column + section dividers | Maximum information density. Costs the most horizontal space; potentially redundant. | |

**User's choice:** Section-header rows in the table.

### Q2: How should section headers be rendered in the table?

| Option | Description | Selected |
|--------|-------------|----------|
| Spanning rows in QTableWidget | `setSpan(row, 0, 1, columnCount)` + bold + tinted single QTableWidgetItem, non-selectable/non-editable flags. Keeps existing widget; minimal diff. | ✓ |
| Swap to QTreeWidget for the drill view | Built-in expand/collapse. Inconsistent rendering between search-results and drill view of the same dialog area. | |
| QTableView + QStandardItemModel + custom delegate | Full per-row paint control. Heaviest change — not justified at this scope. | |

**User's choice:** Spanning rows in QTableWidget.

### Q3: What should each album-header row contain?

| Option | Description | Selected |
|--------|-------------|----------|
| Album name + song count | e.g. `The New Order (8 songs)`. Tiny scan-ability boost; one extra string format in render path. | ✓ |
| Just the album name | Simpler. Matches gbs.fm's source HTML (their `albumTitle` rows are bare names). | |
| Name + 'Add all' button | Bulk-add every song. NEW capability not in 60.2 title; deferred. | |

**User's choice:** Album name + song count.

**Notes:** No additional drill required after the three answers — parser-state carrying mechanism (per-row `album` field vs separate group markers) deferred to planner discretion.
