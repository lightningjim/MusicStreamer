# Phase 75 — UI Design Contract

> Visual and interaction contract for wiring `ToastOverlay` into the Phase 66 theme system. Generated by gsd-ui-researcher; verified by gsd-ui-checker. Lock-down for Claude's-discretion items left open in 75-CONTEXT.md.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native Qt — PySide6 6.11) |
| Preset | not applicable (Qt widget app, no shadcn/Tailwind) |
| Component library | PySide6 stock widgets (`QWidget`, `QLabel`, `QDialog`, `QColorDialog`) |
| Icon library | Qt stock (`QStyle.SP_DialogApplyButton`) — no new icons |
| Font | Qt default `QFont` (no size override anywhere in `toast.py`; system default) |

**Styling vehicle:** `QPalette` (primary) + per-widget `setStyleSheet()` QSS strings (only for ToolTipBase/ToolTipText interpolation on toast). No external design system to vendor.

|-------|-------|-------|

| sm | 8px | Toast border-radius; toast vertical padding; editor `root.setSpacing(8)`; QSS padding `8px 12px` (toast label) |

**Exceptions:** Toast `padding: 8px 12px` — 12px horizontal is the only non-4-multiple value, **inherited verbatim** from current `toast.py:50`. NOT modified by Phase 75. Listed for completeness only.

**Editor dialog growth:** 9 → 11 rows. Each `_ColorRow` ≈ 32px tall (4px top + 24px swatch + 4px bottom). 11 rows × 32px = ~352px + 8px×10 spacing + button-row (~32px) + 8px×2 margins ≈ 408px tall. No `setFixedHeight`, no `setMinimumHeight`, no `QScrollArea` — `QVBoxLayout` auto-sizes. Fits comfortably on the deployment target (Linux Wayland DPR=1.0, 1080p+).

**Document-as-default lock:** When checker/auditor diff this phase's QSS against today's, the only properties that may change are `background-color` and `color`. Specifically:
- `border-radius: 8px;` — UNCHANGED

- (NO `font-size:` property added)
- (NO `font-family:` property added)
- (NO `font-weight:` property added)

**WCAG body-text contrast target:** 4.5:1 at Qt default body size (which the toast renders at). Pairs below are validated against this target.

## Color

### Toast color contract (THIS PHASE'S CORE DELIVERABLE)

#### Per-preset ToolTipBase + ToolTipText hex pairs — LOCKED

These values land in `musicstreamer/theme.py:THEME_PRESETS`. Both keys are stored as 6-char hex (no alpha); alpha=220 is appended at QSS-build time per CONTEXT.md D-02. All pairs validated against WCAG AA (≥4.5:1) using the standard relative-luminance formula with sRGB linearization.

| Preset | ToolTipBase | ToolTipText | Contrast Ratio | Aesthetic Rationale |
|--------|-------------|-------------|----------------|----------------------|
| `vaporwave` | `#f9d6f0` | `#3a2845` | 9.6:1 PASS | Soft pink overlay LIGHTER than Window (`#efe5ff`); dark purple text matches the pastel-but-readable Vaporwave family. Tint sampled to feel "candy pink" without clashing with the lavender Window. |
| `overrun` | `#1a0a18` | `#ffe8f4` | 17.8:1 PASS | Near-black-magenta DARKER than the already-near-black Window (`#0a0408`); text inherits the same hot-pink-white as WindowText for family cohesion. |
| `gbs` | `#2d5a2a` | `#f0f5e8` | 9.5:1 PASS | Forest-green tinted overlay DARKER than the mint Window (`#A1D29D`); off-white text. Sampled to feel like a darker shade in the GBS brand-green family without going pure black. |
| `gbs_after_dark` | `#d5e8d3` | `#0a1a0d` | 15.2:1 PASS | Pale-green overlay LIGHTER than the very-dark-green Window (`#0a1a0d`) — direction-inverted from `gbs` per CONTEXT.md hint. Text reuses Window hex for family cohesion. |
| `dark` | `#181820` | `#f0f0f0` | 14.6:1 PASS | Slightly DARKER than Window (`#202020`) with a cool blue tint (~1% toward `#3584e4` accent family). Text matches WindowText for family cohesion. Preserves the "today's grey" feel but no longer pixel-identical to system. |
| `light` | `#2a2a32` | `#f5f5f5` | 13.5:1 PASS | Standalone dark overlay on light Window (`#f5f5f5`); same cool blue tint as `dark`. Text matches Window for family cohesion. Reads as a "subtle dark overlay" on the Light theme. |

**Validation methodology:** sRGB → linear (gamma 2.4 with 0.03928 threshold) → relative luminance L = 0.2126·R + 0.7152·G + 0.0722·B → contrast = (L_lighter + 0.05) / (L_darker + 0.05). All pairs exceed AA body-text threshold (4.5:1); most exceed AAA body-text threshold (7:1).

#### System-theme legacy fallback — IMMUTABLE QSS LOCK

When `theme == 'system'`, `ToastOverlay` ignores `palette().color(ToolTipBase)` and renders **this exact QSS string** (pixel-identical to today):

```css
QLabel#ToastLabel {
    background-color: rgba(40, 40, 40, 220);
    color: white;
    border-radius: 8px;
    padding: 8px 12px;
}
```

- `40, 40, 40` (RGB) and `220` (alpha) are immutable. **NOT** `rgba(40,40,40,220)` or `rgb(40, 40, 40)` — the literal string `"rgba(40, 40, 40, 220)"` must appear in the produced QSS so `tests/test_toast_overlay.py:143` continues to verify it (now gated to `theme='system'` only).
- `white` (not `#ffffff`, not `#FFF`) — verbatim today's string per `toast.py:48`.
- Border-radius and padding match the non-system path exactly.

#### Non-system QSS build template

For all 6 non-system presets AND custom, the rebuild produces (substituting `{r}, {g}, {b}` from `palette().color(QPalette.ToolTipBase)` and `{text_hex}` from `palette().color(QPalette.ToolTipText).name()`):

```css
QLabel#ToastLabel {
    background-color: rgba({r}, {g}, {b}, 220);
    color: {text_hex};
    border-radius: 8px;
    padding: 8px 12px;
}
```

- Alpha=`220` is appended **literally as the integer 220** (matches today's translucency). NOT `0.86`, NOT `86%`.
- `{text_hex}` is `#rrggbb` lowercase (Qt's `QColor.name()` default). NO alpha on text.
- `border-radius` and `padding` repeated verbatim — typography/geometry invariant.

### Color (60/30/10) — out-of-scope acknowledgment

This phase does **not** introduce new app-wide color tokens. The 60/30/10 split is owned by Phase 66's per-preset palette. Toast bg/text are **role-driven**, not in the 60/30/10 hierarchy — they reuse Qt's `ToolTipBase`/`ToolTipText` semantic roles.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | per-theme `Window` | Inherited from Phase 66 — Phase 75 does not modify |
| Secondary (30%) | per-theme `Base`/`AlternateBase` | Inherited from Phase 66 — Phase 75 does not modify |
| Accent (10%) | per-theme `Highlight` (+ accent-color override) | Inherited from Phase 59 + 66 — Phase 75 does not touch (accent independent of toast) |
| Toast overlay | per-theme `ToolTipBase`/`ToolTipText` | **NEW THIS PHASE** — reserved exclusively for `QLabel#ToastLabel` rendered inside `ToastOverlay` |
| Destructive | `ERROR_COLOR_HEX = "#c0392b"` (in `_theme.py`) | Phase 66 D-03 invariant — UNTOUCHED by Phase 75 |

**ToolTipBase/ToolTipText reserved for:** `QLabel#ToastLabel` inside `ToastOverlay` only. Qt may also use these roles for `setToolTip()` strings on other widgets; that's Qt's stock behavior and is not a regression — `setToolTip()` rendering on theme-driven palettes is a bonus side-effect, not a Phase 75 contract.

## Motion / Interaction Contract

### Retint motion — INSTANT, NO ANIMATION

Per CONTEXT.md D-03 + Deferred Ideas:

- On `QEvent.PaletteChange` or `QEvent.StyleChange`, `ToastOverlay.changeEvent()` calls `_rebuild_stylesheet()` which invokes `setStyleSheet(...)` synchronously. The next paint cycle renders the new colors.
- **NO `QPropertyAnimation` interpolating bg/text colors** during retint.
- **NO fade-out-old → fade-in-new** transition.
- **NO color-easing curve** on retint.
- The existing `_fade_in` (150ms) and `_fade_out` (300ms) animations on `_opacity` are **untouched**. They animate alpha 0↔1 of the entire widget — they do NOT touch bg/text color.

### Toast lifecycle — UNCHANGED

All animation/timing constants in `toast.py` are invariant:

| Token | Value | Source |
|-------|-------|--------|
| `_FADE_IN_MS` | 150 | `toast.py:25` |
| `_FADE_OUT_MS` | 300 | `toast.py:26` |
| `duration_ms` (default) | 3000 | `toast.py:82` (also `MainWindow.show_toast`) |
| `_MIN_WIDTH` | 240px | `toast.py:20` |
| `_MAX_WIDTH` | 480px | `toast.py:21` |
| `_SIDE_PADDING` | 64px | `toast.py:22` |
| `_BOTTOM_OFFSET` | 32px | `toast.py:23` |
| Border radius | 8px | QSS — `toast.py:49` |
| Padding | 8px 12px | QSS — `toast.py:50` |

### z-order — UNCHANGED

- `WA_TransparentForMouseEvents=True` — preserved.
- `WA_ShowWithoutActivating=True` — preserved.
- NO `WA_DeleteOnClose` — preserved.
- `parent.installEventFilter(self)` for resize repositioning — preserved.
- Coupling with `station_list_peek_overlay.py` (z-order siblings) — preserved.

### Editor 11-row layout

- `QVBoxLayout` `setSpacing(8)` + `setContentsMargins(8, 8, 8, 8)` — UNCHANGED from Phase 66.
- 11 rows iterate `EDITABLE_ROLES` (Window → WindowText → Base → AlternateBase → Text → Button → ButtonText → HighlightedText → Link → **ToolTipBase** → **ToolTipText**). New rows appear **at the bottom**, after Link.
- No `QScrollArea`. Dialog auto-sizes (no `setFixedHeight`/`setMinimumHeight`).
- Estimated height: ~408px (well within 1080p deployment target).
- Initial focus: stays on `_rows["Window"]._swatch_btn` (UI-SPEC §A11y from Phase 66 — unchanged).
- Tab order follows `EDITABLE_ROLES` order, naturally falls through to the new rows at end.

### Editor row labels (copywriting — LOCKED)

Append to `theme_editor_dialog.py:ROLE_LABELS`:

```python
"ToolTipBase": "Toast background",
"ToolTipText": "Toast text",
```

Rationale: user-facing label uses "Toast" (the app's term) over "ToolTip" (Qt's term). Two-word label fits in the existing 160px `label.setMinimumWidth(160)` slot.

### Picker live-preview retint — VERIFIED-AUTOMATIC

`theme_picker_dialog._on_tile_clicked()` calls `app.setPalette(build_palette_from_dict(THEME_PRESETS[theme_id]))`. Because `build_palette_from_dict` already iterates over arbitrary `{role_name: hex}` keys (no hardcoded role allow-list — verified at `theme.py:179-186`), the new `ToolTipBase`/`ToolTipText` keys flow through automatically.

`QApplication.setPalette()` fires a `PaletteChange` event on every widget. `ToastOverlay.changeEvent()` rebuilds its QSS. Next `show_toast()` call (or currently-visible toast) renders new colors instantly.
