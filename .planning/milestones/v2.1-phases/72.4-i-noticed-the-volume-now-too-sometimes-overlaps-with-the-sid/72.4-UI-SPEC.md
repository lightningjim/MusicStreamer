# Phase 72.4 — UI Design Contract (Delta Supplement)

> This is a **delta** UI-SPEC layered on top of Phase 72's parent contract and the Phase 72.1 / 72.3 sibling deltas. Read those first. Sections marked **Inherited** carry forward unchanged; only the **Delta** sections below describe new visual contract for Phase 72.4.
>
> **Scope:** ONE behavior — add a **second wrap tier** to the now-playing control row. When the post-72.1 row 1 still cannot fit at the current panel width, reparent the **volume slider + compact-mode toggle button** as a single "volume cluster" unit off row 1. The cluster moves to row 3 (a new `QHBoxLayout` beneath row 2) in the multi-stream very-narrow state, or to row 2 (the existing `_controls_row2`) in the single-stream narrow state. Volume uses `QSizePolicy.Expanding` on its wrap row; compact-toggle stays 28×28 fixed. Width-driven, instant, mode-agnostic. No new widgets, no new copy, no new colors.

### Target-Row Selection

Selection of row 2 vs row 3 as the wrap destination is driven by stream count (which IS already encoded in the picker's `isHidden()` state, per Phase 72.1 D-02 and the `_populate_stream_picker` single-source-of-truth at line 1135).

| Predicate | Target Layout |
|-----------|---------------|
| `should_wrap_volume_cluster == False` | `self.controls` (row 1 — restore Fixed 120px) |
| `should_wrap_volume_cluster == True AND self.stream_combo.isHidden() == True` (single-stream) | `self._controls_row2` (row 2 — share with the hidden picker; cluster owns row 2) |
| `should_wrap_volume_cluster == True AND self.stream_combo.isHidden() == False` (multi-stream) | `self._controls_row3` (row 3 — picker owns row 2; cluster gets row 3) |

Use `isHidden()` (intended-visibility semantics), NOT `isVisible()` — matches the Phase 72.1 invariant established at `now_playing_panel.py:1420` (isVisible returns False on any widget whose top-level ancestor has not been shown, which breaks the off-screen test pattern). The single-source-of-truth for picker visibility is `_populate_stream_picker` at line 1135, which calls `setVisible(len > 1)`.

### Idempotency Contract

| Scenario | Required behavior |
|----------|-------------------|
| `_reflow_volume_cluster()` called twice with no width change between calls | Second call is a no-op (no widget motion, no policy mutation). Driven by `indexOf(...) >= 0` membership checks at the start of `_move_volume_cluster_to`. |
| `_reflow_volume_cluster()` called during `__init__` before `volume_slider` or `_controls_row3` exists | Early-return via `hasattr` defensive guards. No exception, no partial state. |
| Spurious zero-width resize during teardown or first show | Early-return via `panel.width() <= 0` guard. |
| Resize event fires while widget is being constructed (Qt's "child added" callback inside `__init__`) | Early-return via `hasattr` guards. |
| Widen back across threshold | Cluster returns to row 1 via `insertWidget(self._volume_cluster_row1_index, ...)` for `volume_slider`, then `addWidget(compact_mode_toggle_btn)` (or `insertWidget` after volume — planner picks). Cluster sits between `eq_toggle_btn` and `addStretch(1)` — its original position. Size policy restored to `(Fixed, Fixed)` with `setFixedWidth(120)`. |
| Station change (multi → single) while cluster is on row 3 | Next `resizeEvent` re-evaluates: picker becomes hidden, target-row selection now picks row 2 (single-stream branch), cluster reparents row 3 → row 2 (or row 1 if also above threshold). Row 3 returns to empty/collapsed. |
| Station change (single → multi) while cluster is on row 2 | Next `resizeEvent` re-evaluates: picker becomes visible, picker reflow places it on row 2 (if narrow), volume-cluster reflow sees row 2 is now occupied by picker (target-row selection now picks row 3) and reparents row 2 → row 3. |
| Mid-resize transition (single resize event at exactly the threshold) | Idempotent — only state-changes trigger reparent. |

## Out-of-Scope Visual Decisions (explicit)

The following visual / interaction surfaces are explicitly NOT introduced by this phase. Adding any of them would expand scope.

- **No animations** on the reparent. Instant only. (CONTEXT mirrors 72.1 D-06)
- **No new color tokens.** Volume slider and compact-toggle use their existing palette on all rows. (Inherited from Phase 72)
- **No new fonts or font sizes.** Neither widget carries text in the new layout state.
- **No new hover / focus / pressed states** beyond Qt defaults. Existing styling carries over to the wrap rows.
- **No volume slider replacement** with a speaker-icon popover button or icon-only control. The slider remains a full `QSlider`. (CONTEXT Deferred)
- **No compact-toggle hide-on-narrow.** Compact-toggle stays visible in all states; wraps with the volume slider rather than disappearing. (CONTEXT D-01)
- **No `addStretch(1)`** on row 2 or row 3 on the wrap path. Volume's `Expanding` policy consumes free space; compact-toggle pins to the right edge. (CONTEXT D-11)
- **No label** above, beside, or below row 2 or row 3.
- **No notification or toast** when the volume cluster reparents. Silent. (Matches 72/72.1/72.3)
- **No persisted "user prefers three rows" toggle.** Width is the only trigger. (CONTEXT Deferred)
- **No hysteresis dead-band** for threshold flicker suppression. (CONTEXT Deferred)
- **No new keyboard shortcut** to force-wrap. Layout is fully resize-driven.
- **No FlowLayout** for the entire control row. Cluster-as-unit invariant requires deterministic row placement. (CONTEXT Deferred)
- **No restructuring** of star or eq button position. They remain on row 1 in every state. (CONTEXT D-01)
- **No change to the compact-toggle icon family** (`sidebar-{show,hide}-symbolic.svg`). Icon flip remains the responsibility of `_on_compact_btn_toggled`, never the reflow helper. (Phase 72 invariant)
- **No change to the volume slider's range, tick position, or orientation.** Slider is `Qt.Horizontal`, range `0..100`, `NoTicks` in every state. (`now_playing_panel.py:542-545`)
- **No change to `setMinimumWidth(560)`** on NowPlayingPanel or MainWindow. 72.4 makes that floor honored for the secondary cluster; it does not lower or raise the floor. (CONTEXT out-of-scope)
- **No coupling to Phase 72.2 initial-launch sizing bug.** 72.2 is a timing/state-machine bug (overlap on launch that resolves only after resize); 72.4 is a structural-overflow bug (overlap at narrow widths). They are independent; 72.4 ships without waiting for 72.2. (CONTEXT out-of-scope)
- **No touch-friendly hit-target adjustments.** Deployment is Wayland desktop with mouse (memory `project_deployment_target.md`). (CONTEXT out-of-scope)
- **No explicit tab-order curation between rows.** Qt's default top-down, left-to-right traversal is acceptable. (CONTEXT out-of-scope)
- **No third-tier wrap.** 72.4 introduces exactly ONE new wrap tier. Star, EQ, and the transport buttons (play/stop/edit) remain on row 1 in every state. A hypothetical fourth tier (e.g., star/eq wrap) is out of scope and not anticipated.
