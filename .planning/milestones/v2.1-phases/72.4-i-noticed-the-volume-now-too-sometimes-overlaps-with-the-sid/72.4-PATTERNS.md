#### Pattern 5 — Idempotent reparent helper (analog at lines 1338–1367)

**Existing (Phase 72.1):**
```python
# now_playing_panel.py:1338-1367
def _move_stream_picker_to(self, target_layout: QHBoxLayout) -> None:
    """Reparent stream_combo into target_layout idempotently. ..."""
    current_in_row1 = self.controls.indexOf(self.stream_combo) >= 0
    current_in_row2 = self._controls_row2.indexOf(self.stream_combo) >= 0
    # Idempotent early-returns (Pattern 2 -- avoids spurious reparent churn).
    if target_layout is self.controls and current_in_row1:
        return
    if target_layout is self._controls_row2 and current_in_row2:
        return
    # Remove from whichever layout currently owns the picker.
    if current_in_row1:
        self.controls.removeWidget(self.stream_combo)
    elif current_in_row2:
        self._controls_row2.removeWidget(self.stream_combo)
    # Add to target -- target-specific to preserve row-1 ordering.
    if target_layout is self.controls:
        self.controls.insertWidget(self._picker_row1_index, self.stream_combo)
    else:
        # target_layout is self._controls_row2
        self._controls_row2.addWidget(self.stream_combo)
```

**Same in 72.4:** Single helper `_move_volume_cluster_to(target_layout)` (per RESEARCH §Pattern 2 recommendation — single helper over two parallel helpers). Membership probe uses `volume_slider` (cluster-as-unit invariant D-03 means both members are always co-located). Idempotent early-return on already-in-target. removeWidget + addWidget pair preserves signal/slot bindings per Qt 6 semantics — same guarantee invoked by 72.1.

**Different in 72.4 (the delta):** Three target layouts instead of two (`self.controls`, `self._controls_row2`, `self._controls_row3`). Three idempotent early-return branches instead of two. Membership probe checks all three (`indexOf` on volume_slider in each). On reparent, **walk BOTH cluster widgets** in D-02 left-to-right order (`volume_slider` then `compact_mode_toggle_btn`):

```python
# Sketch — derived from 72.1 pattern + RESEARCH §Pattern 2
def _move_volume_cluster_to(self, target_layout: QHBoxLayout) -> None:
    current_in_row1 = self.controls.indexOf(self.volume_slider) >= 0
    current_in_row2 = self._controls_row2.indexOf(self.volume_slider) >= 0
    current_in_row3 = self._controls_row3.indexOf(self.volume_slider) >= 0
    # Idempotent early-returns
    if target_layout is self.controls and current_in_row1: return
    if target_layout is self._controls_row2 and current_in_row2: return
    if target_layout is self._controls_row3 and current_in_row3: return
    # Remove BOTH members from current layout
    current = self.controls if current_in_row1 else self._controls_row2 if current_in_row2 else self._controls_row3 if current_in_row3 else None
    if current is not None:
        current.removeWidget(self.volume_slider)
        current.removeWidget(self.compact_mode_toggle_btn)
    # Add to target in D-02 order
    if target_layout is self.controls:
        self.controls.insertWidget(self._volume_cluster_row1_index, self.volume_slider)
        self.controls.insertWidget(self._volume_cluster_row1_index + 1, self.compact_mode_toggle_btn)
    else:
        # target is row 2 or row 3 — both wrap-rows take cluster at the end
        target_layout.addWidget(self.volume_slider)
        target_layout.addWidget(self.compact_mode_toggle_btn)
```

**Landmines:**
- **`removeWidget` does NOT take a layout slot; it removes by widget pointer.** Both cluster members must be `removeWidget`'d from the same current layout in the same order they were added — Qt is forgiving here but the readability cost of swapped order is high.
- **D-02 order is load-bearing for the wrap row** (volume_slider FIRST, compact_mode_toggle_btn SECOND). On row 1 return, that ordering is preserved by inserting at `index` and `index + 1`. A swap here puts compact-toggle to the LEFT of volume on the wrap row — a silent visual regression that no Python type-check would catch.
- **Avoid `setParent(...)` directly.** Qt's `setParent` clears layout membership; the 72.1 precedent uses `removeWidget` + `addWidget` exclusively. Same applies here.
- **Cluster-as-unit invariant means a single membership probe is sufficient** — but if a future refactor splits the cluster (against D-03), the probe lies. A defensive `assert self.controls.indexOf(self.volume_slider) == self.controls.indexOf(...)` is overkill; the test idempotency assertion (RESEARCH §test surface) catches this regression.

    if expanding:

    else:
        self.stream_combo.setSizePolicy(QSizePolicy.Preferred, QSizePolicy.Preferred)

**Same in 72.4:** `_set_volume_size_policy(expanding: bool) -> None` toggles volume_slider's horizontal policy. Same Expanding ↔ static pattern. Belt-and-braces explicit policy in both branches (no implicit "preserve existing").

**Different in 72.4 (the delta) — THIS IS THE LANDMINE SECTION FROM RESEARCH:**

Volume slider has a `setFixedWidth(120)` at line 544 — this is a HARD width clamp (minWidth=maxWidth=120) that survives `setSizePolicy(...)` calls. Per UI-SPEC §Widget Size Policy Table and RESEARCH §Standard Stack, the canonical Qt 6 toggle is a **three-call sequence**:

```python
# Sketch derived from RESEARCH Standard Stack + UI-SPEC Widget Size Policy Table
def _set_volume_size_policy(self, expanding: bool) -> None:
    if expanding:
        # Clear the fixed-120 clamp, then set Expanding.
        self.volume_slider.setMinimumWidth(0)
        self.volume_slider.setMaximumWidth(16777215)  # QWIDGETSIZE_MAX
        self.volume_slider.setSizePolicy(QSizePolicy.Expanding, QSizePolicy.Fixed)
    else:
        # Restore fixed-120 + Fixed,Fixed policy (belt-and-braces).
        self.volume_slider.setFixedWidth(120)
        self.volume_slider.setSizePolicy(QSizePolicy.Fixed, QSizePolicy.Fixed)
```

The compact-toggle is **never** mutated by this helper (D-11: 28×28 fixed in every state).

**Landmines (THE round-trip pitfall RESEARCH flagged explicitly):**
- **`setFixedWidth(120)` cannot be undone by `setSizePolicy(Expanding, ...)` alone.** `setFixedWidth` internally clamps `minimumWidth = maximumWidth = 120`. Setting `QSizePolicy.Expanding` AFTER `setFixedWidth` is a no-op visually — the slider stays at 120px even though the policy reports `Expanding`. The three-call sequence (clear min, clear max, set policy) is the ONLY round-trip that works. This is the SINGLE biggest 72.4-specific landmine.
- **Use literal `16777215` or `QWIDGETSIZE_MAX`.** Per RESEARCH §Standard Stack, `QWIDGETSIZE_MAX` is `2^24 - 1 = 16777215` and may not be importable from PySide6.QtWidgets directly; the literal is verified safe. Document the literal in the helper comment.
- **The policy on row-1 return is `(Fixed, Fixed)`, not `(Preferred, Preferred)`.** Differs from the picker's row-1 policy (Preferred) — the picker has a 140px MIN floor that allows growth; the volume slider has a 120px FIXED clamp that disallows growth. Copying the picker's `(Preferred, Preferred)` here is wrong.

#### Pattern 7 — Width-driven reflow predicate (analog at lines 1385–1432)

**Existing (Phase 72.1):**
```python
# now_playing_panel.py:1385-1432
def _reflow_stream_picker_row(self) -> None:
    """Width-driven reflow predicate (D-01 / D-02 / D-05 / D-06). ..."""
    # Defensive guards (Pitfalls 3, 5; Open Question 5).
    if not hasattr(self, "stream_combo"):
        return
    if not hasattr(self, "_controls_row2"):
        return
    panel_width = self.width()
    if panel_width <= 0:
        return
    # D-02: single-stream stations stay one-row at all widths.
    if self.stream_combo.isHidden():
        self._move_stream_picker_to(self.controls)
        self._set_picker_size_policy(expanding=False)
        return
    # Width-driven branch (multi-stream).
    threshold = self._picker_row_min_width()
    should_wrap = panel_width < threshold
    if should_wrap:
        self._move_stream_picker_to(self._controls_row2)
        self._set_picker_size_policy(expanding=True)
    else:
        self._move_stream_picker_to(self.controls)
        self._set_picker_size_policy(expanding=False)
```

**Same in 72.4:** `_reflow_volume_cluster(self) -> None` — same defensive-guard preamble (`hasattr` on the new widgets, `panel_width <= 0` early-return). Same threshold-compute → predicate → dispatch shape.

**Different in 72.4 (the delta):**
1. **Three guards instead of two** — `hasattr(self, "volume_slider")`, `hasattr(self, "compact_mode_toggle_btn")`, `hasattr(self, "_controls_row3")`. Compact-mode-toggle hasattr is needed because the cluster invariant probe is via volume_slider but the move helper touches both.
2. **No single-stream "always row 1" branch.** Per D-08 (supersedes 72.1 D-02 for the cluster dimension), single-stream ALSO responds to width. There is no `if isHidden(): return` short-circuit.
3. **Three target layouts**, selected by stream-count after the wrap decision (per UI-SPEC §Target-Row Selection):

```python
# Sketch derived from 72.1 pattern + RESEARCH §Architecture
def _reflow_volume_cluster(self) -> None:
    # Defensive guards
    if not hasattr(self, "volume_slider"): return
    if not hasattr(self, "compact_mode_toggle_btn"): return
    if not hasattr(self, "_controls_row3"): return
    panel_width = self.width()
    if panel_width <= 0: return
    # Threshold: row 1 without the cluster, picker included iff multi-stream.
    threshold = self._row1_min_width(
        include_picker=not self.stream_combo.isHidden(),
        include_volume_cluster=False,
    )
    should_wrap = panel_width < threshold
    if not should_wrap:
        target = self.controls
    elif self.stream_combo.isHidden():
        target = self._controls_row2  # D-07: single-stream → row 2 (picker hidden)
    else:
        target = self._controls_row3  # D-04: multi-stream → row 3 (picker owns row 2)
    self._move_volume_cluster_to(target)
    self._set_volume_size_policy(expanding=should_wrap)
```

**Landmines:**
- **Use `isHidden()`, NOT `not isVisible()`.** Phase 72.1 burned a planning cycle on this at line 1420 — `isVisible()` returns False on any widget whose top-level ancestor has not been shown, which BREAKS the off-screen test pattern (`tests/test_phase72_1_stream_picker_reflow.py` Pattern I — direct `.resize()` form, no `.show()`). Per UI-SPEC §Target-Row Selection: "Use `isHidden()` (intended-visibility semantics)".
- **No `return` after the single-stream-hidden branch.** Unlike 72.1's predicate (which short-circuits when picker hidden), 72.4 must always fall through to the width-compare + dispatch.
- **Threshold caches per `(include_picker, include_volume_cluster)` tuple.** The `include_picker=not self.stream_combo.isHidden()` argument changes when station-count flips. The dict-cache shape handles this naturally; a single `Optional[int]` cache could not.

## Shared Patterns

### Cluster-as-unit invariant (D-03)
**Source:** CONTEXT.md D-01..D-03; UI-SPEC §Volume Cluster Composition.
**Apply to:** Pattern 5 (`_move_volume_cluster_to`), Pattern T1 (test layout-walk helper), Pattern R1 (REQUIREMENTS.md text).
**Excerpt (paraphrased contract):**
```
Both widgets move together or neither does. The left-to-right order on the
wrap row is volume_slider → compact_mode_toggle_btn (D-02). The membership
probe in _move_volume_cluster_to uses volume_slider alone because the cluster
invariant guarantees compact-toggle is co-located.
```

### Defensive guards (Pitfall 3 + 5 + OQ-5 from 72.1-RESEARCH)
**Source:** `_reflow_stream_picker_row` lines 1401–1408.
**Apply to:** Pattern 7 (`_reflow_volume_cluster`). Every reflow predicate MUST defensive-guard `hasattr` + `panel_width <= 0` BEFORE any layout access. Spurious resize events during `__init__` and zero-width resizes during teardown are real on this codepath.

### QA-05 bound-method connects (no lambdas)
**Source:** Project-wide; lines 565 (`self.compact_mode_toggle_btn.toggled.connect(self._on_compact_btn_toggled)`) and 774-775 (volume slider) are the canonical examples.
**Apply to:** All new helper methods. None of the new methods open a `lambda` for any purpose. The `test_no_lambda_in_reflow_wiring` source-grep test (Pattern T2 negative-2) enforces this.

### isHidden() not isVisible()
**Source:** Phase 72.1's comment at lines 1410–1419 and line 1420.
**Apply to:** Pattern 7 (target-row selection in `_reflow_volume_cluster`) and Pattern T2 (test predicates). `isVisible()` returns False on any widget whose top-level ancestor has not been shown — breaks the off-screen test pattern (Pattern I direct `.resize()` without `.show()`). Use `isHidden()` semantics.

### Lazy cache + `dict` keying
**Source:** Phase 72.1's `self._cached_row1_min` field + first-call population.
**Apply to:** Pattern 1 (cache field) and Pattern 4 (threshold helper). Cache key is now `(include_picker, include_volume_cluster)` tuple. Invalidation hook in `_populate_stream_picker` becomes `self._row1_min_cache.clear()`.

### `removeWidget` + `addWidget` (NEVER `setParent`)
**Source:** Phase 72.1's `_move_stream_picker_to`.
**Apply to:** Pattern 5 (`_move_volume_cluster_to`). Layout-level reparent only — `setParent` clears layout membership and risks dangling state. Qt's `removeWidget`/`addWidget` pair is the verified pattern.

### `panel.show()` + `qtbot.waitExposed(panel)` in test fixtures
**Source:** `tests/test_phase72_1_stream_picker_reflow.py` lines 131–141.
**Apply to:** Pattern T1 (`_make_panel`). Non-negotiable for any test that touches resize-driven layout — without it, `panel.width()` is 0 and the reflow's zero-width defensive guard early-returns.
