# Phase 72.4 Plan 02: Helper trio + cache generalization — Summary

Helper trio (`_row1_min_width`, `_move_volume_cluster_to`, `_set_volume_size_policy`) plus
cache generalization (`_cached_row1_min` → `_row1_min_cache: dict[tuple[bool, bool], int]`)
and construction-time `_volume_cluster_row1_index` capture. Turns 3 negative/invariant tests
GREEN in the 72.4 suite while keeping all 11 Phase 72.1 tests GREEN.

## What Was Built

### Task 1 — Rename cache to dict + capture _volume_cluster_row1_index + update invalidation hook (commit `611577d`)

Three surgical edits to `musicstreamer/ui_qt/now_playing_panel.py`:

1. **Cache field rename/reshape** (`__init__`, line 295 area):
   `self._cached_row1_min: Optional[int] = None`
   replaced with
   `self._row1_min_cache: dict[tuple[bool, bool], int] = {}`
   with a comment block citing both LAYOUT-02 and LAYOUT-04 and the four tuple-key
   variants.

2. **`_volume_cluster_row1_index` capture** inserted AFTER
   `self.controls.addWidget(self.volume_slider)` and BEFORE
   `self.compact_mode_toggle_btn = QToolButton(self)`:
   `self._volume_cluster_row1_index: int = self.controls.indexOf(self.volume_slider)`
   with a comment block mirroring the Phase 72.1 `_picker_row1_index` style.

3. **`_populate_stream_picker` invalidation hook** (line 1213):
   `self._cached_row1_min = None`
   replaced with
   `self._row1_min_cache.clear()`
   (PATTERNS §Pattern 1 Landmine 1 — missed rename re-introduces CR-01 stale-threshold bug).

**Deviation (auto-fix, Rule 1):** The `_picker_row_min_width` method body (not removed until
Task 2) referenced `self._cached_row1_min` at three points. With `_cached_row1_min` removed
from `__init__`, this caused an `AttributeError` in the 72.1 tests. Fixed inline by updating
the three internal references to use `self._row1_min_cache[(True, True)]` — keeping Task 1's
scope clean while maintaining GREEN 72.1 tests through the Task 1→2 transition period.

### Task 2 — Replace _picker_row_min_width with generalized _row1_min_width + update 72.1 caller (commit `7f80bff`)

**Method replacement** (`_picker_row_min_width` → `_row1_min_width`):

- New signature: `def _row1_min_width(self, include_picker: bool, include_volume_cluster: bool) -> int`
- Supports four threshold variants keyed by `(include_picker, include_volume_cluster)` tuple.
- **CR-01 layout-agnostic invariant extended to THREE add-back branches** (one per
  wrap-eligible widget: `stream_combo`, `volume_slider`, `compact_mode_toggle_btn`).
  Phase 72.1 had one; Phase 72.4 has three.
- Per-widget exclusion branches in the main loop:
  `if w is self.stream_combo and not include_picker: continue`
  `if w is self.volume_slider and not include_volume_cluster: continue`
  `if w is self.compact_mode_toggle_btn and not include_volume_cluster: continue`
- Outer-chrome walk (logo + cover + outer margins/spacing) reused verbatim from the 72.1 body.
- 16px safety buffer preserved.
- Old `_picker_row_min_width` symbol removed entirely — no shim.

**Caller update** in `_reflow_stream_picker_row`:
`threshold = self._picker_row_min_width()`
→
`threshold = self._row1_min_width(include_picker=True, include_volume_cluster=True)`
Semantically identical: row 1 with everything.

### Task 3 — Add _move_volume_cluster_to and _set_volume_size_policy helpers (commit `cd87539`)

**`_move_volume_cluster_to(self, target_layout: QHBoxLayout) -> None`** placed
AFTER `_move_stream_picker_to`:
- Membership probe via `volume_slider.indexOf()` across all three candidate layouts
  (`controls`, `_controls_row2`, `_controls_row3`).
- Defensive `hasattr(self, '_controls_row3')` before probing row 3 (Plan 03 adds that layout).
- Idempotent early-returns: target == controls and current_in_row1; etc.
- Removes BOTH cluster members from current layout (cluster-as-unit invariant D-03).
- Row-1 return: `insertWidget(self._volume_cluster_row1_index, self.volume_slider)` then
  `insertWidget(self._volume_cluster_row1_index + 1, self.compact_mode_toggle_btn)`.
- Wrap rows: `addWidget(volume_slider)` then `addWidget(compact_mode_toggle_btn)` — D-02 order.
- No `addStretch` anywhere in method body (D-11).
- No `lambda` anywhere.

**`_set_volume_size_policy(self, expanding: bool) -> None`** placed AFTER `_set_picker_size_policy`:
- `expanding=True`: canonical Qt 6 three-call sequence:
  `self.volume_slider.setMinimumWidth(0)`
  `self.volume_slider.setMaximumWidth(16777215)  # QWIDGETSIZE_MAX = 2^24 - 1`
  `self.volume_slider.setSizePolicy(QSizePolicy.Expanding, QSizePolicy.Fixed)`
- `expanding=False`: `self.volume_slider.setFixedWidth(120)` +
  `self.volume_slider.setSizePolicy(QSizePolicy.Fixed, QSizePolicy.Fixed)` (belt-and-braces).
- `compact_mode_toggle_btn` is NEVER mutated (D-11).
- No `lambda`.
