## RED-State Observation

Running `uv run --extra test pytest tests/test_phase72_4_volume_cluster_reflow.py -x`
today (against unchanged `NowPlayingPanel`) fails at the first behavior test
`test_wide_multi_stream_cluster_in_row_1` with:

```
AttributeError: 'NowPlayingPanel' object has no attribute '_controls_row3'. Did you mean: '_controls_row2'?
  File: tests/test_phase72_4_volume_cluster_reflow.py:240
  Path: _assert_volume_cluster_in_layout → `others = [layout for layout in (panel.controls, panel._controls_row2, panel._controls_row3) ...]`
```

This is the **genuine RED state** the plan requires — the AttributeError
fires on a missing 72.4 symbol (`_controls_row3` — the new row 3 layout
that Plan 02 will introduce in `__init__`), not on a syntax/collection error
or `pytest.fail` placeholder.

The negative tests (`test_no_lambda_in_reflow_wiring`,
`test_no_addstretch_in_wrap_path`) use a `getattr(NowPlayingPanel, name, None)`
guard so they surface clear `AssertionError: _reflow_volume_cluster does not
exist on NowPlayingPanel (Wave 1+)` messages instead of bare `AttributeError`.
This is the deliberate **asymmetric** RED state called out in the plan
(`behavior` point 9, negative-2).

Negative-1 (`test_no_repo_set_setting_for_cluster_wrap_state`) may PASS by
accident in Wave 0 because no production code is yet writing cluster-wrap
keys to repo — that is acceptable and the test remains a permanent
regression net for Plans 02/03.
