<objective>
Replace the atomic gain switch in `Player.set_eq_enabled` (which produces a brief IIR-coefficient-discontinuity click/pop on every toggle) with a smooth gain-ramping transition. Each band's gain interpolates from its current value to its target value over ~40ms in 8 ticks of 5ms via a QTimer on the GUI thread. Rapid re-toggles reverse from the current in-progress gains rather than snapping or queuing. Audio output transitions smoothly between EQ-applied and EQ-bypassed states with no audible artifact.

Purpose: Closes BUG-03. The dropout symptom is the textbook IIR-filter coefficient-discontinuity signature; smooth ramping is the textbook DSP fix. SC #1 + SC #2.

Output: `musicstreamer/player.py` gains a QTimer-driven ramp + state struct + tick handler. `set_eq_enabled` becomes the public-API trigger; `_apply_eq_state` is preserved unchanged for the profile-load path. `tests/test_player.py` gains 5 ramp behavior tests.

4. **Add three new helper methods** to `Player` (place them as a contiguous block immediately after `_apply_eq_state` ends at line 659, before `_rebuild_eq_element` at line 661):

   ```python
   # ------------------------------------------------------------------ #
   # Phase 52: EQ smooth gain ramp (D-02, D-03, D-04, D-05)
   # ------------------------------------------------------------------ #

   def _capture_current_gains(self) -> list[float]:
       """Read live per-band gains from the equalizer element (D-05).

       Authoritative source for ramp start_gain. Used both on fresh ramp
       start and on mid-ramp reverse-from-current.
       """
       n = self._eq.get_children_count()
       return [
           float(self._eq.get_child_by_index(i).get_property("gain"))
           for i in range(n)
       ]

   def _compute_target_gains(self) -> list[float]:
       """Compute per-band target gains from current (_eq_enabled,
       _eq_profile, _eq_preamp_db) state (D-04).

       Bypass (disabled or no profile) -> [0.0] * n. Profile-applied ->
       b.gain_db + preamp_db per band (Pitfall 5: ADD), padded with 0.0
       to children_count when profile has fewer bands.
       """
       n = self._eq.get_children_count()
       if self._eq_profile is None or not self._eq_enabled:
           return [0.0] * n
       gains = [0.0] * n
       for i, b in enumerate(self._eq_profile.bands):
           if i >= n:
               break
           # Pitfall 5: ADD preamp (do NOT subtract abs).
           gains[i] = float(b.gain_db) + self._eq_preamp_db
       return gains

   def _start_eq_ramp(self) -> None:
       """Begin or reverse a gain ramp toward the current target state.

       Fresh ramp: capture live start gains, write freq/bandwidth/type
       ONCE (Pitfall 4: bandwidth=Hz, not Q), seed ramp_state, start timer.
       In-progress ramp (D-05 reverse-from-current): re-capture live gains
       as new start_gain, replace target_gain, reset tick_index, keep timer
       running.
       """
       if self._eq is None:
           return
       target = self._compute_target_gains()
       start = self._capture_current_gains()
       # On fresh ramp into the profile-applied path, write the static
       # band properties (freq/bandwidth/type) once. Bypass path leaves
       # them untouched (matches existing _apply_eq_state bypass branch).
       if (
           self._eq_ramp_state is None
           and self._eq_profile is not None
           and self._eq_enabled
       ):
           n = self._eq.get_children_count()
           for i, b in enumerate(self._eq_profile.bands):
               if i >= n:
                   break
               band = self._eq.get_child_by_index(i)
               band.set_property("freq", float(b.freq_hz))
               # Pitfall 4: bandwidth (Hz) = freq_hz / max(Q, 0.01).
               band.set_property(
                   "bandwidth", float(b.freq_hz) / max(float(b.q), 0.01)
               )
               band.set_property(
                   "type", self._EQ_BAND_TYPE.get(b.filter_type, 0)
               )
       self._eq_ramp_state = {
           "start_gain": start,
           "target_gain": target,
           "tick_index": 0,
       }
       if not self._eq_ramp_timer.isActive():
           self._eq_ramp_timer.start()

   def _on_eq_ramp_tick(self) -> None:
       """Per-tick gain interpolation; final tick commits exact target (D-02)."""
       state = self._eq_ramp_state
       if self._eq is None or state is None:
           self._eq_ramp_timer.stop()
           self._eq_ramp_state = None
           return
       state["tick_index"] += 1
       k = state["tick_index"]
       n = self._eq.get_children_count()
       target = state["target_gain"]
       start = state["start_gain"]
       if k >= self._EQ_RAMP_TICKS:
           # Final tick: commit exact target (no lerp residual).
           for i in range(n):
               t_i = target[i] if i < len(target) else 0.0
               self._eq.get_child_by_index(i).set_property("gain", t_i)
           self._eq_ramp_timer.stop()
           self._eq_ramp_state = None
           return
       t = float(k) / float(self._EQ_RAMP_TICKS)
       for i in range(n):
           s_i = start[i] if i < len(start) else 0.0
           t_i = target[i] if i < len(target) else 0.0
           g = s_i + (t_i - s_i) * t
           self._eq.get_child_by_index(i).set_property("gain", g)
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GUI thread → GStreamer pipeline | `set_eq_enabled` is invoked from the Qt main thread; `_on_eq_ramp_tick` writes to `equalizer-nbands` band properties via GstChildProxy on the same (main) thread. No untrusted input crosses this boundary — the toggle is a single-user local UI. |
| Main thread (ramp QTimer) ↔ Main thread (`set_eq_profile`) | Both run on the GUI thread, but they can interleave because Qt's event loop dispatches each user click + each timer tick as separate events. A profile change triggered by `_rebuild_eq_element` replaces `self._eq` with a fresh element; a queued ramp tick that fires AFTER the rebuild would write to the new element with stale `start_gain[]` — not a memory bug (writes succeed), but produces a discontinuous gain on the new element, which is exactly the click symptom Phase 52 fixes. |

## STRIDE Threat Register

Phase 52 is GStreamer pipeline mutation in a single-user desktop app. No network input, no auth, no IPC. The only meaningful threat is the timer-vs-rebuild race.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01 | Tampering / Denial-of-Service (audio path) | `Player._eq_ramp_timer` ticking after `_rebuild_eq_element` replaces `self._eq` | mitigate | `set_eq_profile` (player.py:216) stops the ramp timer and clears `_eq_ramp_state` as its FIRST two body statements, BEFORE the band-count check that may call `_rebuild_eq_element`.
