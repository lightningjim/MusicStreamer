| sqlite3 OperationalError → except clause | Idempotent ALTER absorbs duplicate-column-name errors; other OperationalErrors (e.g., DB locked) ALSO swallowed — accepted trade-off matching Phase 47.2 D-02 precedent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-04 | Tampering | INSERT/UPDATE SQL with new columns | mitigate | Parameterized `?` placeholders for all values; column names are static literals (no user input flows into column names). |
| T-70-05 | Tampering | Idempotent ALTER absorbs OperationalError broadly | accept | try/except sqlite3.OperationalError matches Phase 47.2 precedent; alternative narrow-string-match would be brittle across sqlite versions. Risk: a non-duplicate OperationalError (e.g., DB lock) is also silenced — accepted as documented Phase 47.2 D-02 trade-off. |
| T-70-06 | Information disclosure | sample_rate_hz / bit_depth values | accept | Audio metadata only; no PII. Cached values are derived from public stream caps. |

ASVS L1 mapping: V5 Input Validation — parameterized SQL bindings prevent injection on the only attacker-influenceable inputs (the int values from caps detection).
