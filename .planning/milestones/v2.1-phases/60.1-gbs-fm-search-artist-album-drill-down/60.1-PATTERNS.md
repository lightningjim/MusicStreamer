### `musicstreamer/gbs_api.py` — `_AlbumPageParser` (NEW class, role=service, data flow=HTML scrape)

**Analog:** Same as `_ArtistPageParser` — `_QueueRowParser` shape — but with the D-02 belt-and-suspenders gate.

**D-02 gate pattern** (NEW, derived directly from CONTEXT.md D-02 + RESEARCH §Pitfall 2; the analog is the existing `_in_*_table` flag pattern in `_SongRowParser` line 321 + `_QueueRowParser` line 405, generalized to two flags):

```python
def __init__(self):
    super().__init__()
    self._in_playlist_div = False    # NEW outer gate (D-02 belt)
    self._in_album_table = False     # inner gate
    self._table_locked = False       # one-shot (only the FIRST matching table)
    ...

def handle_starttag(self, tag, attrs):
    ad = dict(attrs)
    if tag == "div" and "playlist" in (ad.get("class") or ""):
        self._in_playlist_div = True
        return
    if not self._in_playlist_div or self._table_locked:
        return
    if tag == "table" and ad.get("width") == "620":
        cls = ad.get("class") or ""
        # D-02 suspenders: explicitly reject artist-page and search-response tables
        if "artist" in cls or "songs" in cls:
            return
        self._in_album_table = True
        self._table_locked = True
        return
    # ... downstream row handling identical to _ArtistPageParser shape
```

**6-column row shape** (derived from `tests/fixtures/gbs/album_1488.html:386-394`, per RESEARCH §Pitfall 6): `td 0` = artist link, `td 1` = song link, `td 2` = codec (skip), `td 3` = bitrate (skip), `td 4` = duration, `td 5` = `/add/X` link. Output dict matches `_SongRowParser`'s `{songid, artist, title, duration, add_url}` so `_render_results()` works unchanged.

**`<div>` depth quirk:** GBS.FM HTML may not perfectly close `<div class="playlist">` (CONTEXT.md / RESEARCH does not pin this). Cheap defense: do not bother depth-tracking `</div>`; once `_table_locked = True` we ignore subsequent `<table>` tags. This matches the `_table_locked` one-shot in the analog `_SongRowParser._in_songs_table` flag (which also only handles the first matching table per page).

**Worker shape — typed signals + try/except auth-expired** (`_GbsSubmitWorker` lines 107-136):
```python
class _GbsSubmitWorker(QThread):
    """Phase 60 D-08 / GBS-01e: gbs_api.submit on a worker thread."""
    # finished payload: (message_text, row_idx)
    finished = Signal(str, int)
    # error: (msg, row_idx)
    error = Signal(str, int)

    def __init__(self, songid: int, cookies, row_idx: int,
                 search_version: int, parent=None):
        super().__init__(parent)
        self._songid = songid
        self._cookies = cookies
        self._row_idx = row_idx
        self.search_version = search_version

    def run(self):
        try:
            msg = gbs_api.submit(self._songid, self._cookies)
            self.finished.emit(msg, self._row_idx)
        except Exception as exc:
            if isinstance(exc, gbs_api.GbsAuthExpiredError):
                self.error.emit("auth_expired", self._row_idx)
            else:
                self.error.emit(str(exc), self._row_idx)
```

**Adapt for `_GbsArtistWorker`:**
```python
class _GbsArtistWorker(QThread):
    """Phase 60.1 / GBS-01e drill-down: gbs_api.fetch_artist_songs on a worker thread."""
    finished = Signal(list)   # results list (drilled artist's songs)
    error = Signal(str)       # 'auth_expired' OR raw msg

    def __init__(self, artist_id: int, artist_name: str, cookies, parent=None):
        super().__init__(parent)
        self._artist_id = artist_id
        self._artist_name = artist_name  # carried for breadcrumb
        self._cookies = cookies

    def run(self):
        try:
            out = gbs_api.fetch_artist_songs(self._artist_id, self._cookies)
            self.finished.emit(list(out.get("results", [])))
        except Exception as exc:
            if isinstance(exc, gbs_api.GbsAuthExpiredError):
                self.error.emit("auth_expired")
            else:
                self.error.emit(str(exc))
```

```python
_ARTIST_HREF_RE = re.compile(r"^/artist/(\d+)$")  # module-level near other regexes

def _on_artist_link_activated(self, item) -> None:
    """Phase 60.1: drill into /artist/<id> using the stored href (NOT item.text()).

    Snapshots pre-drill state ONCE per drill session (UI-SPEC FLAG-03 mitigation
    + RESEARCH §Pitfall 8). Subsequent clicks while drilling replace the drilled
    view but preserve the original snapshot.

    href = item.data(Qt.ItemDataRole.UserRole) or ""

    cookies = gbs_api.load_auth_context()
    if cookies is None:
        self._refresh_login_gate()
        return
```

### Pattern S-1: HTMLParser defensive wrapper (Pitfall 6 idiom)

**Source:** `gbs_api.py:478-493` (`_parse_adds_html`); also `gbs_api.py:559-573` (`_parse_artist_album_html`).

**Apply to:** `fetch_artist_songs(artist_id, cookies)` and `fetch_album_songs(album_id, cookies)` — every new top-level fetch wraps its parser in this idiom:
```python
def fetch_artist_songs(artist_id: int,
                      cookies: http.cookiejar.MozillaCookieJar) -> dict:
    """GET /artist/<id>; parse into _SongRowParser-shape results.

    Defensive: parse failure → empty results dict, never raises.
    """
    url = f"{GBS_BASE}/artist/{int(artist_id)}"
    with _open_with_cookies(url, cookies, timeout=_TIMEOUT_READ) as resp:
        html_str = resp.read().decode("utf-8", errors="replace")
    parser = _ArtistPageParser()
    try:
        parser.feed(html_str)
        parser.close()
    except Exception as exc:
        _log.warning("artist page parse failed for id=%s: %s", artist_id, exc)
        return {"results": []}
    return {"results": parser.results}
```

`fetch_album_songs` is identical with `_AlbumPageParser` and `/album/{id}` URL. The `_open_with_cookies` call propagates `GbsAuthExpiredError` automatically (no try/except needed at fetch level — workers catch it).
