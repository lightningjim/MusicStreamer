## Decisions Made

- **Fresh `LAYOUT-02` code, not LAYOUT-01 continuation.** Mirrors Phase 79 / BUG-11 precedent for follow-up phases. Keeps Traceability one-row-per-phase. Plan 01 Task 1 action specified this; Plan 02 + Plan 03 frontmatter both claim `requirements: [LAYOUT-02]`.
- **Direct `panel.resize(W, H)` instead of any `qtbot.resize` / `qtbot.wait` / `qtbot.waitExposed`.** Pattern I (`72.1-PATTERNS.md`) — `qtbot.waitExposed` is NOT used anywhere in the test suite (verified by grep across `tests/`). The offscreen pytest-qt platform delivers `resizeEvent` to non-exposed widgets in the existing Phase 72 tests. If a Wave-1 reproduction-flake ever surfaces, fixture work is Plan 03's concern.
- **Pre-emptively docstring-flagged the test-file lambda in `test_signal_survives_round_trip`** so it does not get mistaken for a QA-05 violation. QA-05 forbids lambdas in PRODUCTION `connect()` calls only; the `fired.append(i)` lambda is a test-side signal capture and is permitted (and indeed the same pattern the Phase 72 RESEARCH §Code Examples block uses).
