#### Pattern B — Reparent (`removeWidget` + `addWidget` round-trip)

**Analog:** `musicstreamer/ui_qt/station_list_peek_overlay.py` lines 102 and 140-141 (Phase 72 precedent — reparents `StationListPanel` between a QVBoxLayout and a QSplitter; not a pure HBox↔HBox swap, but the same Qt API surface).

**Adopt (move INTO a target container)** (line 102):
```python

```

**Release (move BACK to the original container)** (lines 140-141):
```python

```

**Codebase guarantee documented in this analog** (lines 88-89, 128-131): "Reparent `station_panel` into the overlay layout and show... Pitfall 6: use `insertWidget(0, ...)` — `addWidget` would append at index 1 and silently swap visual order with the now-playing pane."

Planner action: the picker reparent uses the same `removeWidget` / `addWidget` idiom. For row 1 (the destination on "back to wide"), use `addWidget` if the picker should go at the end of the row — but the picker's existing row-1 position is BETWEEN edit_btn (line 481, `controls.addWidget(self.edit_btn)`) and star_btn (line 500, `controls.addWidget(self.star_btn)`). Because all row-1 sibling widgets are themselves added by sequential `addWidget` calls, the picker MUST be re-inserted at the same index it was originally added, NOT just appended. Use `controls.insertWidget(<index>, self.stream_combo)` for the row-1-return path, where `<index>` is the picker's original position in row 1 (after edit_btn, before star_btn). Pitfall 6 from this analog applies directly.

def _sync_stream_picker(self, active_stream) -> None:
    """Sync stream picker to reflect failover-selected stream (D-22)."""
    if active_stream is None:
        return
    self.stream_combo.blockSignals(True)
    for i in range(self.stream_combo.count()):
        if self.stream_combo.itemData(i) == active_stream.id:
            self.stream_combo.setCurrentIndex(i)
            break
    self.stream_combo.blockSignals(False)

#### Pattern H — Layout-walk assertion (find which QHBoxLayout owns a widget)

**Analog:** `tests/test_phase72_now_playing_panel.py` lines 138-179.

**Walk + index-of pattern (lines 138-153 — the recursive layout walker that resolves "which sub-layout contains widget X"):**
```python
# Walk the layout containing volume_slider to verify ordering.
found_layout = None

def _walk_layouts(layout):
    nonlocal found_layout
    if found_layout is not None:
        return
    for i in range(layout.count()):
        item = layout.itemAt(i)
        if item.widget() is panel.volume_slider:
            found_layout = layout
            return
        sub = item.layout()
        if sub is not None:
            _walk_layouts(sub)

_walk_layouts(panel.layout())
assert found_layout is not None, "Could not locate layout containing volume_slider"
```

#### Pattern I — Resize-driven assertion (`window.resize(W, H)` direct call; no `qtbot.resize`, no `qtbot.wait`)

**Analog:** `tests/test_phase72_compact_toggle.py` lines 131, 211-215; `tests/test_phase72_peek_overlay.py` lines 65, 404; `tests/test_toast_overlay.py` lines 23, 81, 116. The codebase-wide pattern is **direct `.resize(W, H)` on the widget, followed immediately by an assertion** — no `qtbot.resize`, no `qtbot.wait(0)` calls anywhere in the repo (verified by grep).

**Round-trip pattern (test_phase72_compact_toggle.py lines 206-215, the closest analog for a resize-triggered behavioral assertion):**
```python
def test_resize_while_compact_keeps_compact(window):
    """D-07: manual toggle only; no auto-exit when window is resized wider."""
    btn = window.now_playing.compact_mode_toggle_btn
    btn.click()  # ON
    assert btn.isChecked() is True
    window.resize(2000, 800)
    assert btn.isChecked() is True
    window.resize(560, 800)
    assert btn.isChecked() is True
    assert window.station_panel.isHidden() is True
```

Planner action: follow this exact form. Call `panel.resize(W, H)` then immediately assert the picker's layout membership. Do NOT introduce `qtbot.wait(...)` or `qtbot.resize(...)` — neither is used anywhere in this repo. If a flake is observed, add `qtbot.waitExposed(panel)` (which Phase 72 integration tests use at `tests/test_phase72_integration.py:88`) on first show; subsequent resizes should be synchronous on Qt's main thread.

Note on widget-vs-window resize: Phase 72's analog `resize` calls target `window` (a top-level `MainWindow`). Phase 72.1's tests are panel-level (no `MainWindow` involvement per RESEARCH §Architectural Responsibility Map), so the call target is `panel.resize(W, H)` directly. Both forms dispatch a `QResizeEvent` to the affected widget's `resizeEvent()` override on the next event-loop turn, which on offscreen Qt is synchronous to the test function's continuation.

## Shared Patterns

### Bound-method connects, no lambdas (QA-05)

**Apply to:** Any `.connect(...)` call introduced by Phase 72.1 (if any).
```python
self.compact_mode_toggle_btn.toggled.connect(self._on_compact_btn_toggled)
```

### Single source of truth for `stream_combo.setVisible(...)`
**Source:** `musicstreamer/ui_qt/now_playing_panel.py` line 1135 (`_populate_stream_picker`).
**Apply to:** The reparent helper — it must NOT call `setVisible(...)` on `stream_combo` itself. It only owns layout membership and `setSizePolicy(...)`. `_populate_stream_picker` remains the sole writer of `stream_combo.setVisible(...)`.
```python

```

### No SQLite persistence for layout state
**Source:** Phase 72 D-09 precedent (no `repo.set_setting` for `compact_mode`). Verified by `tests/test_phase72_compact_toggle.py:149-165 test_compact_mode_toggle_does_not_persist_to_repo`.
**Apply to:** Phase 72.1's reparent helper. The row 1/row 2 state is recomputed on every `resizeEvent`; no `self._repo.set_setting(...)` call is added.

### Reparent via `removeWidget` + `addWidget`/`insertWidget` (signals + state preserved by Qt)
**Source:** `musicstreamer/ui_qt/station_list_peek_overlay.py` lines 102, 140-141. Documented Qt guarantee: `setParent` (implicit via `addWidget`) does not disconnect signal/slot connections. `QComboBox.currentIndex` / `itemData` / model survive reparent.
**Apply to:** The `_reflow_stream_picker_row` helper in `now_playing_panel.py`.
