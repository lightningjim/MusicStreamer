# Phase 60.4: Two GBS.FM Informationals — Token Counter (search dialog) + Playlist Summary (active-playlist widget) - Context

**Gathered:** 2026-05-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Surface two GBS.FM playlist-state pieces that exist in upstream data but are not currently rendered in the UI:

1. **Token counter (search dialog).** GBS.FM tracks per-user submission tokens; the count appears verbatim on the home page (`/`) inside `<th class="plinfo">You have N tokens, so you can add extra dongs to the playlist!</th>`. This data is NOT currently fetched by the app at all. Phase 60.4 adds a new fetcher (`gbs_api.fetch_user_tokens(cookies) -> Optional[int]`) that pulls `/`, parses the `<th class="plinfo">` text, and returns the integer. The search dialog (`gbs_search_dialog.py::GBSSearchDialog`) shows the count as a top-row label `Tokens: N`, refreshed on dialog open + after every successful submit. Color-coded: red at 0 (`ERROR_COLOR_HEX`), amber at 1–3 (new `WARNING_COLOR_HEX = '#d4a017'` constant), default theme color at 4+.

2. **Playlist summary (active-playlist widget).** The `pllength` /ajax event already populates `state["queue_summary"]` (e.g., `"Playlist is 11:34 long with 3 dongs"`) inside `_fold_ajax_events` (`gbs_api.py:279-280`). Phase 60-10 D-10c stopped rendering this line because of the gbs.fm in-joke "dongs". Phase 60.4 reverses that decision: jargon stays VERBATIM (it's part of gbs.fm's voice/charm), and the line renders as the first row of `_gbs_playlist_widget` with a `· ` (U+00B7) prefix to distinguish it from track rows.

**In scope:**

- `musicstreamer/gbs_api.py`: new `fetch_user_tokens(cookies) -> Optional[int]` capability. Pulls `GBS_BASE/` (home page), parses `<th class="plinfo">` via an HTMLParser sibling to `_ArtistAlbumParser` / `_ArtistPageParser`. Returns int on success, None on parse miss / network error / 5xx. `auth_expired` (302 to login or sentinel sentinel detection consistent with existing helpers) raised as the standard exception so callers can route to the existing auth-expired UX.
- `musicstreamer/ui_qt/gbs_search_dialog.py::GBSSearchDialog`:
  - New `_token_label` QLabel placed in the top row, right of `_breadcrumb_label`. PlainText. Hidden while `_refresh_login_gate()` is in the not-logged-in state.
  - New `_GbsTokenWorker(QThread)` mirroring `_GbsSubmitWorker` shape (QA-05 bound-method connections; `finished = Signal(int)` carrying the count, `error = Signal(str)` carrying `'auth_expired'` sentinel or message). Spawned on dialog `showEvent()` (or equivalent open-time hook) and re-spawned at the END of `_on_submit_finished` (success branch) so the post-submit refetch sees the just-spent token count.
  - Optimistic decrement: when `_on_submit_finished` enters the success branch, decrement `_token_label`'s integer state by 1 BEFORE spawning the refetch worker. The refetch result then overwrites; usually a no-op.
  - Failure UX: on token-fetch failure (network/parse/5xx), set label to `Tokens: —` (em-dash). Silent log; no toast (Phase 60 Pitfall 3).
  - Auth-expired: `_token_label` flips to `Tokens: —` and `_refresh_login_gate()` re-disables the dialog. Existing toast 'GBS.FM session expired — reconnect via Accounts' unchanged.
  - Color tier: `setStyleSheet(f"color: {ERROR_COLOR_HEX};")` at 0; `setStyleSheet(f"color: {WARNING_COLOR_HEX};")` at 1–3; clear stylesheet (default theme color) at 4+ AND on the `—` placeholder.
- `musicstreamer/ui_qt/_theme.py`: add `WARNING_COLOR_HEX: str = "#d4a017"` constant (sibling of `ERROR_COLOR_HEX`). No `WARNING_COLOR_QCOLOR` companion needed — only the QSS-string form is consumed for now.
- `musicstreamer/ui_qt/now_playing_panel.py::_on_gbs_playlist_ready`: insert a single guarded line BEFORE the `▶ {icy}` row — `if state.get("queue_summary"): self._gbs_playlist_widget.addItem(QListWidgetItem(f"· {state['queue_summary']}"))`. Plain-text (QListWidgetItem default). No new fields, no jargon scrubbing, no font/style changes.
- Tests: `tests/test_gbs_api.py` (new home-page fixture + `fetch_user_tokens` parser tests), `tests/test_gbs_search_dialog.py` (token label render, color transitions across 0/1–3/4+/`—`, optimistic-decrement, refetch-on-submit-success, auth-expired flip, login-gate hide), `tests/test_now_playing_panel.py` (summary row renders verbatim with `· ` prefix at index 0; skipped on missing/empty queue_summary; auth-expired hides via existing widget-visibility path).

**Out of scope:**

- Jargon sanitization / "safe-for-work" toggle. User explicitly chose to preserve gbs.fm voice (dongs, Farts, etc.). Deferred as opt-in setting (see `<deferred>`).
- Periodic token-count poll while the search dialog is open. Refresh is on-open + after-each-submit only.
- Token count on any surface other than the search dialog (e.g., main window status bar, station tile, tooltip).
- Aggregate playlist summary on any surface other than the active-playlist widget (e.g., search dialog, main window).
- Changes to the existing 15s `_gbs_poll_timer` cadence or the kick-on-track-change path from Phase 60.3.
- Exposing `fetch_user_tokens` on a polling/background schedule outside the search dialog.
- Wider jargon scrubbing in any rendered surface (column headers, button labels, etc.).
- Phase 60.5+ (any future phases) — separate scope.
- Migration of the meaning of `state["queue_summary"]` (e.g., parsing into `(duration_seconds, song_count)` tuple). Phase 60.4 ships the verbatim string only.

<decisions>
## Implementation Decisions

### Token source & refresh (Area 1)

- **D-T1 (refresh trigger):** Token count refetches on (a) dialog open (`showEvent` or equivalent first-render hook) and (b) the SUCCESS branch of `_on_submit_finished` (i.e., after a confirmed-non-error submit). No periodic poll. Rationale: spend events are the only thing that reliably mutates the count; gbs.fm doesn't expose tokens in /ajax so any poll would be a full home-page fetch. Rejected: open-only (counter goes stale immediately after first submit); periodic poll (extra HTTP for a slow-changing value).

- **D-T2 (fetch failure):** On any failure path of `_GbsTokenWorker` (network, HTML parse miss, server 5xx, malformed integer), set `_token_label.setText("Tokens: —")` and clear color stylesheet (default theme color). Silent `_log.warning` only. NO toast (Phase 60 Pitfall 3 — don't toast-spam transient failures). Rejected: hide entirely (layout shift the user explicitly didn't pick); error toast (noisy for an info display).

- **D-T3 (post-submit window):** On `_on_submit_finished` success branch, BEFORE spawning the refetch worker, decrement the label's currently-displayed integer by 1 (`Tokens: 48` → `Tokens: 47`). Apply color-tier transitions immediately based on the decremented value. The in-flight refetch overwrites the value when it lands; if server count disagrees, the refetch wins (server-is-truth, Phase 60 Pitfall 2). Rejected: '—' until refetch lands (UX flash for a known-successful action); keep-old-count (briefly looks like submit was free).

### Token placement & wording (Area 2)

- **D-T4 (placement):** New `_token_label` QLabel positioned in `_build_ui` top_row, AFTER `_breadcrumb_label.addWidget`, before `root.addLayout(top_row)`. Always added to the layout; visibility toggled by `_refresh_login_gate` (and by `setText('Tokens: —')` for failure/auth-expired states). Top row order: `_search_edit (stretch=1)` | `_search_btn` | `_page_label` | `_breadcrumb_label` | `_token_label`.

- **D-T5 (wording):** Compact `f"Tokens: {n}"` format. Singular handled implicitly: `Tokens: 1`, `Tokens: 0`. Plain-text invariant T-40-04 preserved trivially (`setTextFormat(Qt.TextFormat.PlainText)` matches `_page_label`/`_breadcrumb_label` precedent). Rejected: "48 tokens left" (singular/plural code with no UX gain); verbatim "You have 48 tokens, so you can add extra dongs to the playlist!" (too long for top row; would force layout reflow or wrap).

- **D-T6 (color tier):** Three-tier color via `setStyleSheet(f"color: {hex};")` at the call site (mirrors `_error_label` at line 349):
  - `n == 0` → `ERROR_COLOR_HEX` (`#c0392b`).
  - `1 ≤ n ≤ 3` → new `WARNING_COLOR_HEX` (`#d4a017`).
  - `n ≥ 4` → empty stylesheet `setStyleSheet("")` (default theme color).
  - `'—'` placeholder (any failure / auth-expired) → empty stylesheet (default theme color).

  Add `WARNING_COLOR_HEX: str = "#d4a017"` to `_theme.py` as a sibling of `ERROR_COLOR_HEX`. No `WARNING_COLOR_QCOLOR` (no `setForeground` consumer for it yet — single QSS-string call site).

### Playlist summary render & jargon policy (Area 3)

- **D-S1 (placement):** Summary row inserted as the FIRST row of `_gbs_playlist_widget`, ABOVE the `▶ {icy}` now-playing line, in `_on_gbs_playlist_ready` (now_playing_panel.py:~1115 region after `self._gbs_playlist_widget.clear()`). Single-widget rendering — no new QLabel, no new visibility logic, no layout surgery. Stays inside the existing 180px max-height; user can scroll the widget if the queue is long.

- **D-S2 (jargon policy — REVERSAL OF 60-10 D-10c):** Render `state["queue_summary"]` VERBATIM. gbs.fm in-jokes ("dongs", "Farts", any other casing variants) are preserved as part of the site's voice. Phase 60-10 D-10c's reasoning ("don't render because of dongs jargon") is reversed: the user finds the data useful AND the jargon charming. NO sanitization function, NO whole-word substitution, NO render-time replacement, NO new helper. Rejected: parse-time scrub (loses gbs.fm voice); render-time scrub (same loss); helper-for-future-reuse (premature abstraction with one would-be caller).

- **D-S3 (decorative prefix):** Summary row text format = `f"· {state['queue_summary']}"` — leading U+00B7 (`·`) middle-dot + space, distinguishing the summary from track rows (`▶ ` for now-playing, `1. ` etc. for queue, `Score: ` for score). Plain-text — T-40-04 preserved (U+00B7 is a printable character, not markup).

- **D-S4 (missing payload):** When `state.get("queue_summary")` is falsy (cold-start `/ajax` response before `pllength` event arrives, or `pllength` payload is empty string after `.strip()` per gbs_api.py:280), SKIP the summary row entirely. Pure additive: `if state.get("queue_summary"): self._gbs_playlist_widget.addItem(QListWidgetItem(f"· {state['queue_summary']}"))`. Row appears the moment the next /ajax tick carries `pllength`. Honest-data principle (60.2 D-10) preserved — no fake "loading…" placeholder.

### Login / auth-expired fallback (Area 4)

- **D-T7 (no-cookies open):** When `gbs_api.load_auth_context()` returns None at dialog open (or after cookie clear), the existing `_refresh_login_gate()` disables the search/submit UI. Phase 60.4 EXTENDS that gate: `_token_label.setVisible(False)` while the gate is active; `_token_label.setVisible(True)` is set when the gate transitions to logged-in. Token-fetch worker is NOT spawned while logged out.

- **D-T8 (auth-expired mid-session):** When `_on_submit_error` receives `'auth_expired'` (or any future code path that re-engages the login gate mid-session), `_token_label.setText("Tokens: —")` AND `setStyleSheet("")` (default theme color) BEFORE the existing `_refresh_login_gate()` call. The existing toast 'GBS.FM session expired — reconnect via Accounts' is unchanged. The label then hides via D-T7 once the login gate engages.

- **D-S5 (summary auth-expired):** Summary row inherits widget visibility — when `_gbs_playlist_widget.setVisible(False)` is called from `_on_gbs_playlist_error('auth_expired')` (now_playing_panel.py:1153) OR from `_refresh_gbs_visibility` (logged-out / non-GBS branches), the summary row is hidden along with everything else. ZERO new visibility code in 60.4. When the widget reappears (login restored, GBS bound, /ajax returns), the next render cycle adds the summary row at the top via D-S4's `if state.get("queue_summary")` guard. Already symmetric with the existing widget-hide-on-auth-expired pattern from Phase 60.

### Claude's Discretion

- **Worker shape vs. inline asyncio:** Whether `_GbsTokenWorker` is its own QThread subclass (mirroring `_GbsSubmitWorker`) or extends `_GbsSubmitWorker` with a mode flag, or whether the existing `_GbsPollWorker` could be re-used by extending the home-page-aware fetch path. Recommended: dedicated `_GbsTokenWorker` QThread subclass, ~30 LoC, mirrors existing dialog-side worker convention exactly. Planner picks based on whether the file's worker count would create churn.

- **HTMLParser shape vs. regex for `<th class="plinfo">`:** The text shape `"You have N tokens, so you can add extra dongs to the playlist!"` is regex-friendly (`r"You have (\d+) tokens?"`) AND HTMLParser-friendly (sibling of `_ArtistAlbumParser`). Planner picks based on which keeps `gbs_api.py` consistent. Slight preference for regex since the one capture is a digit run inside known prose; HTMLParser is overkill for one-class extraction. But if any future home-page scrape lands here, HTMLParser scales better.

- **Dialog open-time hook:** `showEvent` vs. constructor-tail vs. first-`_refresh_login_gate`-success — the choice depends on PySide6 lifecycle timing. Planner verifies existing dialog open-time hooks (e.g., for the existing login-gate refresh) and picks the matching pattern.

- **Optimistic decrement: where to track the integer state.** Three options: (a) parse it back out of `_token_label.text()` ("Tokens: 47" → 47), (b) store on `self._last_known_tokens` instance attr, (c) carry through worker signal payloads. Recommended (b) — simple instance attribute, no fragile string parsing, no extra signal field.

- **Color-application helper:** Whether to wrap the three-tier color logic in a small `_apply_token_color(n: Optional[int])` helper or inline it at the two-or-three call sites. Planner picks for minimum diff.

- **Test fixture for home page:** The existing `tests/fixtures/gbs/home_playlist_table.html` has the `<th class="plinfo">` line. Use it directly OR create a smaller `home_tokens_only.html` fragment focused on the parser unit. Planner picks based on which keeps `tests/test_gbs_api.py` readable.

- **Logging on parse miss:** Whether `_log.warning` includes the raw HTML snippet (helps diagnosability) or just the URL (avoids log size bloat). Planner picks based on existing `_log` conventions in `gbs_api.py`.

### Established Patterns

- **Plain-text invariant T-40-04:** All `QListWidget` / `QLabel` items use plain text (`Qt.TextFormat.PlainText`). 60.4 preserves: `_token_label.setTextFormat(Qt.TextFormat.PlainText)` mirrors `_breadcrumb_label`; the summary `QListWidgetItem(f"· …")` is plain-text by default (QListWidgetItem default). Grep baseline (`setTextFormat|setHtml|RichText`) stays at 4 (Phase 60.3-06 lock).
- **QA-05 bound-method connections:** Every Qt signal/slot connection in 60.4 uses bound method references (`worker.finished.connect(self._on_token_fetched)`), never lambdas (Phase 999.x lifecycle invariant).
- **Pitfall 1 / token-guard:** `_GbsTokenWorker` callbacks must be guarded against stale workers if the dialog can be reopened. Pattern: `_token_request_id` int, increment on each kick; slot discards if mismatch. Mirrors `_search_version` from `_GbsSearchWorker` shape (gbs_search_dialog.py:891+).
- **Pitfall 3 / no-toast-spam:** Token-fetch failures log silently; no toast. Mirrors `_on_gbs_playlist_error` non-`auth_expired` branch (now_playing_panel.py:1167-1168) and the existing 60.x convention.
- **Pitfall 5/7 / no-retry:** Single-shot fetch on each trigger. Failure → '—' placeholder; the next user-driven trigger (next submit, next dialog open) is the retry.
- **Pitfall 11 / PlainText only:** QListWidgetItem default. The summary row's `QListWidgetItem(f"· …")` is plain-text by default; no `setTextFormat` override needed.
- **Honest-data principle (60.2 D-10):** No fabricated counts; no fabricated summary text. Empty/missing → skip or '—'.
- **Provider-gating idiom (`now_playing_panel.py:882`):** `is_gbs = (self._station is not None and self._station.provider_name == "GBS.FM")`. Summary row inherits this gate via the existing `_gbs_playlist_widget.setVisible` flow — no new gating in 60.4.

### Integration Points

- **Search dialog open → `_GbsTokenWorker.run()` → `fetch_user_tokens(cookies)` → home-page GET → parse `<th class="plinfo">` → `finished.emit(int)` → `_on_token_fetched(count)` → `_token_label.setText(f"Tokens: {count}")` + color-tier setStyleSheet.**

- **Submit error 'auth_expired' (lines 936+) → toast + `_token_label.setText('Tokens: —')` + `_token_label.setStyleSheet('')` + `_refresh_login_gate()` (which hides `_token_label` via D-T7).**
- **`_refresh_login_gate` (line 381+) → on logged-in transition, kick token fetch + `_token_label.setVisible(True)`. On logged-out transition, `_token_label.setVisible(False)`.**
