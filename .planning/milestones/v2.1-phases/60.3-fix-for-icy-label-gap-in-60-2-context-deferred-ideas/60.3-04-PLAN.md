Phase 60.3 — Plan 04 of N (Wave 1, gap closure for **CR-01 / Verification Gap 1**).

Close the D-05 over-fire defect identified by the code review and confirmed by `/gsd-verify-work`: the no-downgrade guard at `on_title_changed:542-543` was implemented as a coarse `_gbs_label_source == 'ajax'` early-return with no track-identity component. After Track A is /ajax-stamped, the flag stays `'ajax'` across the server-side track transition, so when Track B's ICY tag arrives:

1. `icy_label` keeps showing Track A's full Artist - Title for up to 15 s (until the next 15 s `_gbs_poll_timer` tick).
2. `_last_icy_title` is not updated to the new track.
3. The D-03 kick at `:582-583` never fires — it lives INSIDE the same function body that just early-returned.

CONTEXT.md D-05 (line 41) explicitly required "title-suffix match OR entryid coupling" — the implementation has neither. This plan closes the gap with the smallest diff that satisfies the contract: combine **option (b)** (reset flag on track change) AND **option (c)** (split the kick out of the early-return) from the REVIEW. Together they are robust and complementary:

- **(b)** Track-change reset ensures that a new track's ICY arrival sees `_gbs_label_source != 'ajax'` and stamps the bare title normally.
- **(c)** Splitting the kick out of the early-return ensures D-03 fires on EVERY ICY arrival in GBS context (when the worker is idle), regardless of source-flag state — so even if the kick path is later misdiagnosed as same-track, the /ajax round-trip still kicks.

**Why both, not one or the other:** Either fix alone would close the verification gap on paper, but each leaves a sharp edge:
- (b) alone: relies on `_on_gbs_playlist_ready` to fire for every track transition. If the user is in a state where /ajax is failing transiently (network blip), the flag never resets and we regress.
- (c) alone: the kick fires, but the local label/`_last_icy_title` writes are still suppressed, so the user sees Track A's label until /ajax responds (~200 ms with the kick, but visibly stale).

Combined, the local writes happen immediately on track change AND the kick guarantees a /ajax upgrade even if the same-track guard mis-fires.

Purpose: close VERIFICATION.md Gap 1 + REVIEW.md CR-01 with a minimal-diff change that preserves the existing same-track no-downgrade behaviour (the legitimate D-05 mechanism) while removing the new-track over-fire.
