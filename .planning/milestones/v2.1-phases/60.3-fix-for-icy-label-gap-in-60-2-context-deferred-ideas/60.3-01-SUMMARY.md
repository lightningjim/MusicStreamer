## Accomplishments

- **Source-of-truth flag plumbed.** `self._gbs_label_source: Optional[str] = None` declared adjacent to `_gbs_current_entryid` in `__init__`. Tri-state semantics: `'ajax'` (canonical /ajax stamp owns label), `'icy'` (bare ICY tag owns label OR auth_expired fallback), `None` (no station / non-GBS / bridge-window pre-/ajax).
- **Lifecycle resets in both branches of `_refresh_gbs_visibility`.** Entry branch (`should_show=True`) resets alongside `_gbs_poll_cursor = {}` so a fresh GBS-context entry starts with no source. Exit branch (`not should_show`) resets alongside `_gbs_current_entryid = None` so leaving GBS context fully clears state.
- **`_gbs_poll_in_flight()` predicate added.** Single-line boolean reading the SYNC-05 retention slot (`self._gbs_poll_worker is not None and self._gbs_poll_worker.isRunning()`). Placed adjacent to `_is_gbs_logged_in` to match the existing predicate-method cluster.
- **Three scaffolding tests pin the contract.** `test_gbs_label_source_default_none_outside_gbs`, `test_gbs_label_source_resets_on_leaving_gbs_context`, `test_gbs_poll_in_flight_predicate_truth_table` (None / not-running / running states). All pass; full panel test file now 79/79 green (76 pre-existing + 3 new).

## Decisions Made

- **Tri-state `Optional[str]` over `Enum`.** String literals `'ajax'`/`'icy'` match the project idiom (`vote_value` button property is a bare int; `_gbs_poll_cursor` dict keys are bare strings). Readability wins over type-system purity at this scale.
- **Method, not property, for `_gbs_poll_in_flight`.** Mirrors `_is_gbs_logged_in` (line 871) — single-line `def ... -> bool` predicate. Properties would be subtly different to mock in tests (where Plans 02/03 may need to override the in-flight check).
- **Two reset sites instead of one (Pitfall 4 redundancy accepted).** Resetting on the entry branch (alongside `_gbs_poll_cursor = {}`) gives a same-station rebind a fresh start; resetting on the exit branch (alongside `_gbs_current_entryid = None`) handles context departure. The redundant reset on entry is intentional for clarity.
- **Plan locks Open Question 1 by deferral.** PATTERNS §C raised "icy_disabled-on-GBS interaction" as a planner choice; Plan 01 doesn't touch that surface (no `_apply_gbs_icy_label` here). Plan 02 will lock it when it adds the helper.
