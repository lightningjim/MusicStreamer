# Phase 60.3: Fix for ICY label gap in 60.2 Context Deferred Ideas - Context

**Gathered:** 2026-05-06
**Status:** Ready for planning

<domain>
## Phase Boundary

When a GBS.FM station is the bound provider, the now-playing track-title label (`now_playing_panel.icy_label`) currently shows only the title because GBS.FM's ICY `StreamTitle` tag carries the title alone (no artist component). SomaFM and other ICY streams send `Artist - Title` and the label correctly renders both. Concurrently, the GBS active-playlist widget's `/ajax` poll already returns the full `Artist - Title` as `state["icy_title"]` and renders it on the playlist's now-playing row. This phase brings `icy_label` to parity with SomaFM-style streams when on GBS.FM by sourcing the missing artist component from `/ajax`.

**In scope:**
- `now_playing_panel.py`: change icy_label sourcing for GBS.FM-bound stations so `_on_gbs_playlist_ready` writes the full `Artist - Title` to `icy_label`, with the bare ICY tag acting as a transient pre-/ajax bridge.
- `now_playing_panel.py`: kick `_on_gbs_poll_tick` from `on_title_changed` when in GBS context (debounced) so a track-change ICY tag triggers an immediate /ajax refresh instead of waiting up to 15s.
- `now_playing_panel.py`: no-downgrade guard on `on_title_changed` so a bare ICY title doesn't overwrite an already-/ajax-stamped full-string label.
- Couple `_last_icy_title`, favorites-key, and cover-art-query string to the same source: full `Artist - Title` once /ajax stamps; bare ICY before that.
- During the bridge window (bare ICY shown, /ajax not yet stamped, user is logged in to GBS), star button is disabled and cover-art lookup is suppressed; both come alive when /ajax stamps.
- Auth-expired / not-logged-in fallback: when /ajax is unavailable, the bare ICY tag is the truth — label updates from `on_title_changed` normally, star/cover-art enabled (today's behavior continues; back-compat for any existing GBS favorites stored title-only).
- Tests: cover the source-of-truth ordering (ICY-then-/ajax, /ajax-then-ICY race), the kick-on-tag-change path with worker-in-flight skip, the no-downgrade guard, the bridge-state star/cover-art gating, and the auth-expired fallback path.

**Out of scope:**
- ICY-source labeling for non-GBS providers (SomaFM/AudioAddict/etc. unchanged — they already deliver full `Artist - Title`).
- Polling cadence changes for the existing 15s `/ajax` timer (D-03 kicks an additional poll on ICY tag change; the 15s baseline remains, matches gbs.fm web UI DELAY=15000).
- New auth-expired UX surfaces (toast, banner, login prompt) — fallback is the existing widget-hide + bare-ICY behavior.
- Migration of pre-existing GBS favorites stored under title-only keys — D-06 explicitly accepts the trade.
- Phase 60.4 (token-counter informational + playlist-summary informational) — separate phase, separate decisions.
- icy_disabled-on-GBS interaction (left to Claude's Discretion — planner picks consistency).
- Cover-art junk detection / iTunes hit-rate changes — out of scope; the lookup string just changes shape.

</domain>

<decisions>
## Implementation Decisions

### Source-of-truth strategy (Area 1)
- **D-01:** When `_station.provider_name == "GBS.FM"`, `_on_gbs_playlist_ready` is the authoritative writer for `icy_label`, sourcing from `state["icy_title"]` (which gbs.fm misleadingly names but in fact delivers as full `Artist - Title`). For non-GBS stations, the existing `on_title_changed` path is unchanged. Rejected: merge-on-every-write (creates two equal writers, brittle); accept-and-document only (visible defect persists, parity with SomaFM never lands).
- **D-02:** ICY title-only acts as a transient bridge before /ajax stamps. Last-writer-wins; /ajax overwrites the bridge value once it arrives. `on_title_changed` continues to write in GBS context but its value is transient until /ajax responds (subject to D-05). Rejected: hold-previous-/ajax-label (stale up to 15s on track change, jarring); blank-until-/ajax (visibly empty during loading); station-name fallback (less informative than the partial title).

### Track-change timing (Area 2)
- **D-03:** ICY tag change in GBS context kicks an immediate `_on_gbs_poll_tick` to shorten the title-only window from up to 15s down to one round-trip. The 15s `_gbs_poll_timer` continues to run unchanged — the kick is additive, not a cadence change. Pitfall: do not restart the 15s timer (would reset its next-tick). Rejected: accept-15s-lag (short tracks may upgrade after they've already ended); compress-poll-cadence (more network/CPU, divergence from gbs.fm DELAY=15000).
- **D-04:** Debounce by skipping the kick if `self._gbs_poll_worker` exists AND its thread is still running. Reuses the existing token machinery (`_gbs_poll_token`) for stale-discard; no new state needed. Rejected: time-based debounce (new state variable for marginal benefit); no-debounce-rely-on-tokens (occasional duplicate polls; extra network).
- **D-05:** No-downgrade guard. `on_title_changed` in GBS context skips writing `icy_label` if the bare ICY title matches the title-suffix of the current /ajax-stamped label (i.e., the substring after `' - '` in the current `icy_label.text()`), or, equivalently, if a label-source flag indicates the label is currently /ajax-owned for the active entryid. Prevents `/ajax → ICY` ordering races from briefly downgrading a perfectly-good `Artist - Title` to title-only. Rejected: entryid-coupling (cleaner state model but more wiring); accept-the-flicker (visible regression on initial bind and on kicked-/ajax-arrives-before-ICY paths).

### Favorites + cover-art coupling (Area 3)
- **D-06:** `_last_icy_title` holds the same string the user sees in `icy_label` — full `Artist - Title` once /ajax stamps; bare ICY title before that. All three downstream uses (display, favorites key, cover-art query) read the same value. Cover-art lookup against iTunes benefits from the more specific string. Existing favorites stored under bare-title GBS keys won't migrate; user explicitly accepts this trade-off. Rejected: display-only change (loses the better cover-art match; introduces a second string variable and synchronization); split-cover-but-not-favorites (more moving parts).
- **D-07:** During the bridge window (bare ICY title shown, /ajax not yet stamped, user is logged in to GBS), `star_btn` is disabled and `_fetch_cover_art_async` is suppressed. Both come alive when `_on_gbs_playlist_ready` stamps the label. Implementation: gate via the same source flag introduced for D-05 — star/cover-art enabled iff `source == 'ajax'` (logged-in path) OR no /ajax is possible (D-08 fallback). Rejected: fire-on-bare-then-refire-on-/ajax (double-fire pattern, redundant iTunes calls, harder test contract); fire-once-on-bare-only (conflicts with D-06's "all three driven by the full string").

### Auth-expired / not-logged-in fallback (Area 4)
- **D-08:** When the user is not logged in to GBS.FM (no cookies file, validator returned False) OR auth has expired during runtime (`_on_gbs_playlist_error` with `msg == "auth_expired"`), the bare ICY tag is the truth: `icy_label` updates from `on_title_changed` normally, star/cover-art remain enabled. The D-07 "hold until /ajax" gate is relaxed when no /ajax is possible (source flag treated as `'icy'` rather than `None`). The existing GBS playlist widget hide + timer stop on auth_expired (line 1001) is unchanged — this phase doesn't add any new auth-expired UX. Rejected: hide/blank label (forces login for any track display, breaks back-compat); visible auth-hint copy (new UX surface; 60.3 stays surgical); block + prompt re-login (new toast/banner pattern; out of scope).

### Claude's Discretion
- **icy_disabled-on-GBS interaction:** Whether per-station `icy_disabled=True` on a GBS.FM station also suppresses /ajax-driven label stamping (consistent with the existing on_title_changed icy_disabled gate, but makes the /ajax path slightly more conditional) OR /ajax stamping continues regardless (since the GBS playlist widget itself ignores icy_disabled). Planner picks consistency. Likely consistent: skip /ajax stamping too, since the user has explicitly disabled track display for that station.
- **Source-flag state machine:** Whether the source flag is a tri-state enum (`'ajax' | 'icy' | None`) on the panel, a method on the panel that derives source from current state, or stored implicitly via comparing `icy_label.text()`. All three satisfy D-05/D-07. Tri-state enum is most explicit and testable.
- **Where exactly the icy_label.setText call is added inside `_on_gbs_playlist_ready`:** In-place near the playlist-row rendering (line ~977) or in a small helper method `_apply_gbs_icy_label(state)` called from there. Helper is cleaner once unit-tested in isolation; in-place is simpler if the diff stays small.
- **Test mocking shape:** Whether to extend the existing `mock_gbs_api` fixture (Phase 60 conftest extension) with parametrized /ajax responses for the source-of-truth tests, or add a dedicated `gbs_ajax_responses` fixture file. Planner picks based on which keeps `tests/test_now_playing_panel.py` readable.
- **Logging:** Whether to add a one-off `_log.debug` line on the kick-skip path (D-04 worker-in-flight) for diagnosability, or leave silent. Probably silent — matches existing poll-tick conventions.

</decisions>

## Existing Code Insights

### Reusable Assets
- **`/ajax` already delivers the missing data.** `_fold_ajax_events` (`gbs_api.py:243+`) maps the `metadata` event to `state["icy_title"]` which is the full `Artist - Title` string. Naming is unfortunate (gbs.fm calls it `icy_title` even though it carries more than ICY does) but the semantics are correct — no parser work needed.
- **GBS provider gate already idiomatic.** `is_gbs = (self._station is not None and self._station.provider_name == "GBS.FM")` is the established detection pattern (line 882). 60.3 reuses it for the icy_label branch and the kick-/ajax debounce.
- **Token-discard machinery (Pitfall 1).** `_gbs_poll_token` + `if token != self._gbs_poll_token: return` (lines 922, 941, 997) handles stale workers. D-03's kick reuses it — no new race-handling needed.
- **`_GbsPollWorker` retention slot (SYNC-05).** `self._gbs_poll_worker = worker` (line 928) is the source of truth for "is a poll in flight". D-04's debounce reads it directly: skip if exists and `isRunning()`.
- **`_last_icy_title` is the single coupling point** (line 530). Already drives display, favorites, and cover-art. D-06's "drive all three" decision keeps the coupling — just changes what gets written into it for GBS-context tracks.
- **`bind_station` resets `_last_icy_title`** (line 503) and calls `_refresh_gbs_visibility` (line 517). Source flag resets here too.

### Established Patterns
- **Plain-text invariant T-40-04:** All `QListWidget`/`QTableWidget`/`QLabel` items use plain text by default; no markup. 60.3 preserves this — `state["icy_title"]` from /ajax is plain text (`PlainText` is QListWidgetItem's default per line 972 comment). icy_label is a QLabel; setText() on plain string is plain by default.
- **Honest-data principle (Phase 60.2 D-10):** Don't fabricate. If /ajax doesn't provide an artist component (it always does on gbs.fm, per the metadata event semantics), the bare ICY title is shown — no synthetic "Unknown Artist" prefix.
- **Server-is-truth (Phase 60 Pitfall 2 / D-07d):** /ajax wins over locally-derived state. D-01 follows this.
- **No-retry (Phase 60 Pitfall 5/7):** Don't retry on transient /ajax failure; just log. D-08 follows this — auth_expired hides the widget once and stops the timer; ICY takes over silently.
- **Don't toast-spam on every poll tick (Phase 60 Pitfall 3):** Auth-expired surfaces only as widget hide; D-08 preserves this — no new toast/banner.

### Integration Points
- **`title_changed` signal (player.py:446) → `on_title_changed` slot (now_playing_panel.py:523):** The instant ICY-tag entry point. D-03 kicks /ajax from inside this slot when in GBS context. D-05 guards the icy_label.setText call here.
- **15s `_gbs_poll_timer` → `_on_gbs_poll_tick` → `_GbsPollWorker.run` → `playlist_ready.emit(token, state)` → `_on_gbs_playlist_ready`:** The /ajax data path. <specifics>
## Specific Ideas

- **The verbatim defect (from Phase 60.2 deferred-idea entry, 2026-05-06):** SomaFM Drone Zone shows `Mount Hibiki - Fortress Mist, Pt. 2`; GBS.FM shows just `La Frontière de la Nuit` even though the active-playlist widget below the player has the full `Cimerion & Oublieth (Arcanes) - La Frontière de la Nuit` from the `/ajax` poll. The data is already in the panel — 60.3 wires it to the right surface.
- **Bridge example:** User clicks GBS.FM station → bind_station fires → `_refresh_gbs_visibility` shows widget + kicks first poll → ~5-second window where label may briefly show the bare ICY title (e.g., `La Frontière de la Nuit`) before /ajax response arrives → label upgrades to `Cimerion & Oublieth (Arcanes) - La Frontière de la Nuit`. Star button and cover-art lookup come alive with the upgrade (D-07).
- **Track-change example:** Mid-stream, a new GBS.FM track starts. Within 1-2s, GStreamer parses the new ICY tag → `on_title_changed` fires with bare title → kicks `_on_gbs_poll_tick` (D-03) since worker is idle → /ajax round-trip ~200ms → label upgrades to full Artist - Title. Worst case `_gbs_poll_worker` is mid-tick when the track changes: kick is skipped (D-04) but the in-flight poll *will* see the new entryid (since /ajax is event-stream-based and the new metadata event is in the pending events the moment the track changed server-side) — so the in-flight poll resolves the upgrade anyway, no second round-trip needed.
- **No-downgrade example (the subtle race the guard prevents):** During the bridge OR after a kicked /ajax has already won, ICY tag arrives slightly later for the same track. `on_title_changed` sees label currently reads `Cimerion & Oublieth (Arcanes) - La Frontière de la Nuit`; ICY brings `La Frontière de la Nuit`. Title-suffix match → skip the write. Label stays correct.

</specifics>
