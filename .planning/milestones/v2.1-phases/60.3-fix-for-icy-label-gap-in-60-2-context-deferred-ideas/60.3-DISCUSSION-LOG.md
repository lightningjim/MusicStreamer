## Source-of-truth strategy

### Q1: Where does icy_label get its text from when GBS.FM is active?

| Option | Description | Selected |
|--------|-------------|----------|
| /ajax replaces ICY for GBS | When provider == GBS.FM, on_title_changed becomes a no-op for the label, and _on_gbs_playlist_ready sets icy_label from state['icy_title']. Cleanest semantics; gbs.fm /ajax is authoritative. Label is blank/station-name until first /ajax response. | ✓ |
| Merge ICY + /ajax | ICY tag still drives instant updates; /ajax adds the artist prefix when available. Faster initial display, but creates two writers and a brief 'partial' state. | |
| Accept + document only | No code change to the title path. Add a tooltip / sublabel noting that GBS.FM ICY is title-only. Cheapest. Visible defect remains. | |

**User's choice:** /ajax replaces ICY for GBS (Recommended).
**Notes:** D-01 — _on_gbs_playlist_ready is the authoritative writer when in GBS context. state["icy_title"] is already the full Artist - Title.

### Q2: Pre-/ajax window — what does icy_label show before /ajax stamps?

| Option | Description | Selected |
|--------|-------------|----------|
| Show ICY title-only as bridge | Use the GStreamer ICY tag as a temporary bridge: shows bare title immediately, then upgrades to full Artist - Title when /ajax arrives. Last-writer-wins, /ajax winning. | ✓ |
| Hold previous /ajax label until next poll | Don't update icy_label from ICY at all. Stale label up to 15s after track changes. | |
| Blank label until first /ajax response | icy_label is empty until first /ajax response. Honest but visibly empty. | |
| Station name fallback (current bind_station behavior) | icy_label shows station name (e.g., 'GBS.FM') until /ajax stamps. | |

**User's choice:** Show ICY title-only as bridge (Recommended).
**Notes:** D-02 — bridge UX preserves continuity; on_title_changed continues writing; /ajax overwrites once it arrives.

## Track-change timing

### Q1: Does an ICY title-change in GBS context kick an immediate /ajax poll?

| Option | Description | Selected |
|--------|-------------|----------|
| Kick /ajax on ICY tag change | When in GBS context, on_title_changed schedules an immediate _on_gbs_poll_tick (debounced). Closes the title-only window to ~one round-trip. Don't restart the 15s timer; just kick a one-shot poll. | ✓ |
| Accept 15s lag | No code change to poll cadence. Title-only label persists up to ~15s after a track change. | |
| Compress poll cadence | Drop the 15s interval to something shorter (e.g., 5s) only while a GBS station is bound. More network/CPU; divergence from gbs.fm DELAY=15000. | |

**User's choice:** Kick /ajax on ICY tag change (Recommended).
**Notes:** D-03 — additive kick; 15s baseline timer keeps running.

### Q2: Debounce rule for the kicked poll

| Option | Description | Selected |
|--------|-------------|----------|
| Skip if worker in-flight | If self._gbs_poll_worker exists AND its thread is still running, skip the kick. Reuses existing token machinery. | ✓ |
| Skip if any /ajax landed in last N seconds | Track last_ajax_response_at; only kick if more than N seconds elapsed. New state variable. | |
| Always kick (no debounce) | Token mechanism handles stale results; semantically safe but occasional duplicate polls. | |

**User's choice:** Skip if worker in-flight (Recommended).
**Notes:** D-04 — read existing _gbs_poll_worker retention slot; no new state.

### Q3: No-downgrade guard — handling /ajax-then-ICY ordering races

| Option | Description | Selected |
|--------|-------------|----------|
| No-downgrade guard | on_title_changed in GBS context writes only if the label is currently empty/station-name OR the ICY title's normalized form differs from the title-suffix of the current label. Prevents downgrade. | ✓ |
| Track entryid coupling | Track which /ajax entryid currently owns the label. on_title_changed writes only if owner is None or doesn't match. | |
| Accept the brief flicker | No guard. Brief downgrade to title-only when ICY arrives after /ajax, then re-upgrades. | |

**User's choice:** No-downgrade guard (Recommended).
**Notes:** D-05 — compare ICY against substring after ' - ' in current icy_label, OR track via source flag.
