Phase 60.3 — Plan 05 of N (Wave 2, gap closure for **CR-02 / Verification Gap 2 + WR-02 + IN-01**).

Close the D-08 auth_expired bridge-gate defect identified by the code review and confirmed by `/gsd-verify-work`: when `_on_gbs_playlist_error("auth_expired")` fires, the source flag flips to `'icy'` correctly, but the bridge gate predicate in both `_update_star_enabled` and `on_title_changed` ALSO checks `_is_gbs_logged_in()`, which only inspects cookie-file existence. The cookie file is NOT deleted on auth_expired (it persists on disk as a stale artifact), so:

- `is_gbs=True`, `_is_gbs_logged_in()=True`, `_gbs_label_source='icy' != 'ajax'` → `has_track=False` → **star DISABLED**.
- Same predicate in `on_title_changed:564-568` suppresses cover-art lookup.

CONTEXT.md D-08 (line 48) explicitly promised "star/cover-art remain enabled" when /ajax is impossible. The current implementation only delivers this for the logged-out-from-start case (no cookies file), not the auth-expired-mid-runtime case.

**Decision: Option (b) `_gbs_ajax_disabled` flag, NOT cookie deletion.**

Per the planning context, cookie deletion (Option a) was explicitly rejected because it invalidates Phase 60 D-04 (cookie-file existence as the login-state ladder #3 signal). Adopting a separate panel-local boolean `_gbs_ajax_disabled` preserves that invariant and creates a clean separation of concerns:

- `_is_gbs_logged_in()` answers "do we have credentials on disk?" — unchanged.
- `_gbs_ajax_disabled` answers "did we last hear back that /ajax doesn't work?" — new.
- Bridge gate combines both: `ajax_possible = _is_gbs_logged_in() AND NOT _gbs_ajax_disabled`.

**WR-02 follow-on:** With Option (b), the existing `_refresh_gbs_visibility:944` line `self._gbs_label_source = None` no longer needs to wipe a session-state-relevant flag (cookie still on disk → should_show stays True after auth_expired, and the user's rebind via `_on_play_pause_clicked` re-enters this branch). However, `_gbs_ajax_disabled = True` set in `_on_gbs_playlist_error` MUST survive a same-station re-entry — otherwise the flag is wiped and the bridge gate re-engages. The fix: reset `_gbs_ajax_disabled = False` ONLY when binding a fresh GBS station (i.e., in `bind_station`, not in `_refresh_gbs_visibility`). For symmetry, also reset in the `should_show=False` branch (leaving GBS context entirely) so a non-GBS rebind clears the state cleanly.

**IN-01 cleanup:** Once Option (b) lands, the 13-line prose comment in `test_gbs_auth_expired_relaxes_bridge_gate` (lines 1532-1544) becomes obsolete. Replace it with `assert panel.star_btn.isEnabled() is True` and `assert panel._gbs_ajax_disabled is True`.

**Why this plan depends on 60.3-04:** Plan 04 restructures `on_title_changed` (moves the kick out of the early-return). Plan 05 modifies the SAME function's bridge-window cover-art predicate. Sequential ordering avoids a merge conflict.

Purpose: close VERIFICATION.md Gap 2 + REVIEW.md CR-02 + WR-02 + IN-01 with a clean panel-local-flag implementation that preserves Phase 60 D-04 (cookie-file existence as login signal).
