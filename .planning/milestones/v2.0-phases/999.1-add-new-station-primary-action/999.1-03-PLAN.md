<pitfalls>
- **Pitfall 1 (menu order):** D-01 locks "New Station" as Group 1 position 0, ABOVE Discover and Import. Simplest implementation: move the new `act_new = self._menu.addAction("New Station")` block to execute BEFORE the Discover/Import blocks — actions are appended in call order, so literal reordering of the `addAction` calls achieves this without `insertAction`.
- **Pitfall 2 (QA-05 — no self-capturing lambdas):** The existing `_on_edit_requested` uses `lambda: self._sync_now_playing_station(fresh.id)` — this is accepted precedent. For the new slot, capturing `new_id` in a lambda `lambda: self.station_panel.select_station(new_id)` mirrors that precedent. Alternative (preferred for strictness): `functools.partial(self.station_panel.select_station, new_id)`. Either works; match existing file conventions — if `functools.partial` is already imported, prefer it.
- **Pitfall 3 (delete-on-reject is the dialog's job):** The dialog (post Plan 01) handles placeholder cleanup on reject/close. MainWindow's slot must NOT pre-emptively delete — it only creates the placeholder and opens the dialog. If the dialog rejects, the dialog itself calls `repo.delete_station`. Do NOT add cleanup logic in the MainWindow slot.
- **Pitfall 4 (station_deleted wiring):** Connect `dlg.station_deleted` to `self._on_station_deleted` (same as edit flow). This handles the case where a user in "new" mode clicks the explicit Delete button — existing `_on_station_deleted` refreshes the list and clears now-playing if needed. Harmless if never invoked for a brand-new station.
- **Pitfall 5 (no auto-play):** D-07 explicitly forbids auto-play. Do NOT connect `station_saved` to `_sync_now_playing_station` for the new-station dialog — a brand-new station is never the currently-playing one and a no-op is fine, but adding the connection makes the test `test_new_station_save_does_not_auto_play` ambiguous. Only connect `station_saved` to (a) `_refresh_station_list` and (b) `select_station(new_id)`.
- **Pitfall 6 (test monkeypatch pattern):** Integration tests in Plan 00 monkeypatch `EditStationDialog.exec` to simulate accept/reject without showing a real modal. Ensure `_on_new_station_clicked` calls `dlg.exec()` as the final step so the monkeypatch can intercept it.
- **Pitfall 7 (id capture):** `new_id` must be captured in the closure/partial BEFORE `dlg.exec()`. If the dialog emits `station_saved` from within exec, the connection must already exist.
</pitfalls>
