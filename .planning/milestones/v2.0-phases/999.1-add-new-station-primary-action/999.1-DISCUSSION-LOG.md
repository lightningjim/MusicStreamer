## New-station Flow

| Option | Description | Selected |
|--------|-------------|----------|
| Pre-create empty row; delete on cancel (Recommended) | `repo.create_station()` for real id, open dialog normally, `repo.delete_station(id)` on Cancel. Existing update/asset code unchanged. | ✓ |
| Add 'new mode' to dialog — defer DB write until save | Refactor EditStationDialog to accept placeholder Station and branch in _on_save. Cleaner model but touches asset copy flow (which needs real id). | |
| Pre-create on first asset op; skeleton until then | Hybrid — more branching logic. | |

**User's choice:** Pre-create empty row; delete on cancel

### Cancel / force-close

| Option | Description | Selected |
|--------|-------------|----------|
| Treat close as cancel — delete the row (Recommended) | Override reject()/closeEvent to delete placeholder so orphans cannot accumulate. | ✓ |
| Treat close as cancel — keep the empty row | Less code, orphan empty-name stations possible. | |

**User's choice:** Treat close as cancel — delete the row

## Save Validation

| Option | Description | Selected |

**User's choice:** Name only (current behavior)
