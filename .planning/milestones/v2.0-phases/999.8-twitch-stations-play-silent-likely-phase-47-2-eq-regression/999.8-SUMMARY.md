# Phase 999.8 Summary

**Outcome:** Resolved — but not by the planned work.
**Original hypothesis:** Phase 47.2 equalizer-nbands on `playbin3.audio-filter` silences Twitch's HLS audio format → REFUTED.
**Actual root cause:** `streamlink` 8.x dropped `Streamlink.set_plugin_option`. The Twitch resolution worker crashed with `AttributeError` before assigning a URI to playbin3. Symptom (timer ticks, no audio) was indistinguishable from EQ-induced silencing because the crash happened in a `daemon=True` thread without surfacing as a `playback_error`.

## What shipped

- `9df84de` — `fix(streamlink): use prefixed session.set_option for Twitch OAuth header`
  - One-line replacement: `session.set_plugin_option("twitch", "api-header", v)` → `session.set_option("twitch-api-header", v)`
  - Confirmed via local probe that the prefixed key round-trips and is read by the Twitch plugin.
  - Verified end-to-end: Twitch stations now play with audio.

## What did NOT ship

- The 3 PLAN.md files (D-06 diagnostic gate, D-07 GstBin wrapper fix, D-04 matrix harness) and `999.8-VALIDATION.md` were deleted as stale. They were planned against the refuted hypothesis. The audit-all-stream-types matrix harness was discussed as defensive scaffolding but dropped per single-user pragmatic policy.

## Key lesson

Daemon threads in Qt-driven workers can mask bugs that look like downstream playback issues. The `_twitch_resolve_worker` (and any sibling `_youtube_resolve_worker` / `_aa_resolve_worker`) should consider emitting `playback_error` on unhandled exceptions rather than letting tracebacks land silently in stderr. Logged for future consideration; not in scope for this phase.
