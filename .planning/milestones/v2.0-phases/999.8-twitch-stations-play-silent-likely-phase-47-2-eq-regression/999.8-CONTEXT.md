# Phase 999.8: Twitch stations play silent — likely Phase 47.2 EQ regression - Context

**Gathered:** 2026-04-24
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the silent-playback regression for Twitch stations (streamlink-resolved HLS URLs play with audio under standalone `gst-play-1.0` but produce no audio in MusicStreamer). Strong suspicion: the Phase 47.2 `equalizer-nbands` element permanently attached to `playbin3.audio-filter` (player.py:102-105) breaks audio format negotiation for Twitch's HLS variant. Because the phase goal calls for audit-all-stream-types, this phase also sweeps every stream flavor under every EQ state to catch latent format-negotiation bugs that simply have not tripped yet.

In scope:
- Reproduce the Twitch silent-audio symptom with instrumentation (caps before/after the EQ element).
- Determine the root cause (EQ audio-filter negotiation vs. another playbin3 interaction).
- Ship a fix that makes Twitch audible with or without EQ / preamp / profile, and preserves correct behavior for ShoutCast, YouTube-live, AudioAddict, and Radio-Browser.
- Run a 15-cell audit matrix (5 stream types × 3 EQ states) with preamp and band-count axes and format-caps evidence per cell.
- Ship regression guards: (a) unit test for audio-filter caps negotiation in bypass, (b) a local-only manual harness `scripts/eq_audit.sh` (or `tests/integration/audio_filter_smoke.py`) that runs the matrix on demand before releases.

Out of scope:
- Changing EQ UX (profile picker, preamp slider, band-count selector) — Phase 47.2 UX stays.
- Adding runtime pad-probe logging — captured as a deferred idea.
- Adding manual UAT checklist to CLAUDE.md — captured as a deferred idea.
- CI-integrated integration tests — user chose local-only manual harness.
- Phase 999.9 (yt-dlp "No video formats found" regression) — separate bug.

</domain>

<decisions>
## Implementation Decisions

### Scope
- **D-01:** Full audit-all-stream-types approach. Fix Twitch AND run a systematic matrix across every stream type before declaring the phase done. Justification: surfaces latent format-negotiation bugs that have not triggered yet, and format-level fixes (if that's the cause) naturally benefit the whole matrix.
- **D-02:** Matrix dimensions (every cell must produce captured format-caps evidence, not just a pass/fail):
  - **Stream types (5):** Twitch (streamlink HLS), YouTube-live (yt-dlp resolved HLS), ShoutCast/mp3 (direct HTTP), AudioAddict (PLS-resolved direct HTTP), Radio-Browser (url_resolved direct HTTP).
  - **EQ states (3):** disabled-no-profile (fresh install), disabled-with-profile (profile loaded but toggled off), enabled-with-profile.
  - **Preamp axis (3):** -30 dB, 0 dB, +12 dB — tested against the enabled-with-profile state on each stream type.
  - **Band-count axis (4):** 1, 5, 10, 20+ bands — tested against the enabled-with-profile state on each stream type. Covers the `_rebuild_eq_element` NULL-transition path.
  - **Format-caps inspection:** every cell captures caps at the EQ input pad and output pad (Gst.Pad.get_current_caps serialized via `Gst.caps_to_string`) so the audit produces grep-able evidence, not just behavioral outcomes.

### Regression guard / testing
- **D-03:** Unit test — build a Player-adjacent GstBin that mirrors the production audio-filter wiring, feed a synthetic audio source via `appsrc` with several explicit caps (at minimum: `audio/x-raw,format=S16LE,rate=44100,channels=2`, `audio/x-raw,format=F32LE,rate=48000,channels=2`, and Twitch-style `audio/mpeg` AAC variant), and assert the filter passes data through in bypass. Runs under `uv run --with pytest pytest` with the rest of the suite. No network required.
- **D-04:** Manual harness — ship `scripts/eq_audit.sh` (or `tests/integration/audio_filter_smoke.py`) that iterates the 15-cell matrix + preamp + band-count axes against a user-supplied set of representative URLs (from the live station list), prints pass/fail and caps strings per cell, exits non-zero on any silence/failure. Runs on demand, NOT part of default `uv run pytest`. Becomes the release gate for anything touching `player.py` EQ code or the `audio-filter` slot.
- **D-05:** Do NOT integrate the harness into CI. User explicitly chose local-only to keep CI fast.

### Claude's Discretion — Diagnostic-first
- **D-06:** Phase 1 of the plan is a diagnostic task: reproduce the Twitch silent-playback in-app, toggle `set_eq_enabled(False)` at runtime, and confirm audio returns. If EQ-off restores audio → confirm hypothesis and proceed to fix. If EQ-off does NOT restore audio → hypothesis is wrong, research must widen to other playbin3 interactions (sink, buffering, buffer-duration conflict with HLS's segment boundary) before fix tasks are written. This is a hard gate before writing any fix code.

### Claude's Discretion — Fix shape (deferred until diagnostic confirms)
- **D-07:** If EQ-off fixes silence, apply the GStreamer-canonical remedy: wrap the equalizer in a `GstBin` containing `audioconvert ! equalizer-nbands ! audioconvert` (plus `audioresample` if evidence shows rate mismatch) and set the bin as `audio-filter`. Audioconvert handles F32LE ↔ S16LE negotiation that bare equalizer-nbands cannot. Do NOT switch to plug/unplug dynamic filter mutation — too invasive, creates race conditions during PLAYING→NULL→PLAYING transitions.
- **D-08:** If caps evidence shows the `pulsesink` negotiation itself rejects Twitch's format post-EQ, also add `audioresample` to the bin. Do not add prematurely — only add on evidence.
- **D-09:** Preamp D-18 pattern (gain = b.gain_db + preamp_db) stays as-is. No change to EQ semantics.

</decisions>

## Existing Code Insights

### Reusable Assets
- **Gst.ElementFactory.make + Gst.Bin wiring pattern** — already used elsewhere in player.py for pipeline construction.
- **`restore_eq_from_settings`** — already handles preamp / profile / enabled state restoration; audit harness can drive Player state through this entry point rather than reconstructing everything.
- **`scripts/` directory convention** — project already has `run_local.sh`; a new `scripts/eq_audit.sh` fits the existing top-level ergonomics (MusicStreamer follows the pattern of shell scripts for ops tasks).
- **`tests/` directory** — `tests/test_player.py` exists and owns player-adjacent unit tests; the caps-negotiation unit test belongs here alongside the existing player tests (same conftest fixtures).

### Established Patterns
- **GstChildProxy band mutation** (`self._eq.get_child_by_index(i).set_property(...)`) — already the canonical way to write band state; any fix that swaps in a bin must preserve this API through either the bin's child navigation or a stored reference to the inner equalizer element.
- **Graceful degrade** (`if self._eq is None`) at every EQ touchpoint — the fix's bin-based replacement must preserve the same "missing plugin → all EQ calls become no-ops" contract.
- **READY-state rebuild** on band-count change — the existing `_rebuild_eq_element` pattern (NULL→READY→rebuild→apply) is the template for any bin-swap during runtime.

### Integration Points
- **`Player.__init__` audio-filter assignment** — the single line at player.py:105 is where the bin-wrapper would replace the bare equalizer element.
- **`_rebuild_eq_element`** — must rebuild the bin (not just the inner equalizer) on band-count change; OR swap to rebuilding just the inner equalizer inside the stable bin, which is simpler.
- **`_apply_eq_state`** — child-by-index access pattern may need to be updated to navigate `bin.get_by_name("eq")` or a stored handle, depending on bin structure.

<specifics>
## Specific Ideas

- Reproduction conditions (from phase goal, 2026-04-24): fresh install + fresh Twitch OAuth import (post-Phase 999.3 fix). Twitch stations resolve via streamlink, elapsed-time counter ticks, but speakers produce nothing. Same URL played standalone via `gst-play-1.0 <url>` produces audio. YouTube and ShoutCast stations on the same install play with audio end-to-end.
- Suspected element silencing the chain: `equalizer-nbands` plugged into `playbin3.audio-filter` at player.py:105, constructed at Player init and never unplugged even when `_eq_enabled=False` and `_eq_profile=None`.
- The YouTube-resolved HLS path plays audio despite the same EQ element being attached — meaning the bug is not "EQ breaks HLS" but "EQ breaks Twitch's specific HLS variant." Likely negotiation of AAC codec caps or segment-audio format that the bare equalizer-nbands cannot passthrough without an audioconvert.
- Phase 47.2 Pitfall 1 (num-bands realloc unreliable, rebuild whole element) suggests `equalizer-nbands` is sensitive to downstream caps changes — consistent with a negotiation fault hypothesis.

</specifics>

<deferred>
## Deferred Ideas

- Runtime pad-probe logging at EQ output — valuable instrumentation but adds runtime overhead and log noise. Revisit if this class of bug recurs.
- Manual UAT checklist in CLAUDE.md / contributing notes — single-user project, Kyle owns all commits; checklist adds ceremony without reviewer to enforce it.
- CI integration of the `scripts/eq_audit.sh` harness — user explicitly chose local-only. Revisit only if the project ever grows contributors or adopts CI gating.
- Plug/unplug dynamic `audio-filter` mutation — superseded by the bin-wrapper approach per D-07; revisit only if bin-wrapper fails caps negotiation in some cell.
- Auditing other GStreamer pipeline slots (video-sink, audio-sink, text-sink) for analogous format-negotiation bugs — Twitch video is already faked (`fake-video`) and text-sink is unused; audio-sink is `pulsesink` which has its own well-understood negotiation story.

</deferred>
