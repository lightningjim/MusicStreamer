<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local IPC (QLocalSocket) | Bytes received from another process on the same user account cross into single_instance._drain |
| OS PATH → process | shutil.which reads system PATH; node binary path enters check_node return value |
| Qt UI → user shell | QDesktopServices.openUrl invokes the user's default browser with NODEJS_INSTALL_URL constant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-02-01 | S (Spoofing) | single_instance _drain accepts arbitrary bytes from local socket | mitigate | Strict equality check `data == b"activate"`; unknown payloads silently dropped; no privileged action exposed (only emits a Qt signal that raises a window). |
| T-44-02-02 | S (Spoofing) | Linux stale socket file lets unprivileged actor pre-bind SERVER_NAME | mitigate | `setSocketOptions(QLocalServer.SocketOption.UserAccessOption)` restricts socket to current user; `removeServer()` cleans stale files from prior crashes (Pitfall 2). |
| T-44-02-03 | T (Tampering) | Malicious node.exe earlier in PATH could hijack YT resolution | accept | Out of Phase 44 scope — user runs the Node.js they chose to install (per RESEARCH §Security Domain). Documented in EULA (Plan 04). |
| T-44-02-04 | I (Information Disclosure) | check_node logs the resolved node path | accept | Local logfile only; path is non-secret; helpful for diagnosing PATHEXT issues (Pitfall 3). |
| T-44-02-05 | D (Denial of Service) | QLocalServer.listen failure leaves app running without single-instance guard | mitigate | Graceful degradation: `_log.warning(...)` + return None → caller continues without the guard. App startup never blocks (V7 ASVS L1). |
| T-44-02-06 | I (Information Disclosure) | NODEJS_INSTALL_URL launches in default browser | accept | Hardcoded constant — not user-controllable; no PII exposure. |
</threat_model>
