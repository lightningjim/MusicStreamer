## Component Inventory

### ExportSettingsHandler (no dialog — direct file save)
- Trigger: hamburger menu "Export Settings" action
- Flow: `QFileDialog.getSaveFileName` → default filename `musicstreamer-export-YYYY-MM-DD.zip` → default directory `QStandardPaths::DocumentsLocation`
- On success: `ToastOverlay` success message
- On error: `ToastOverlay` error message (no modal dialog)
- No new widget required — handler method on `MainWindow`

### ImportSummaryDialog (new QDialog)
- `setMinimumWidth(480)` — matches `cookie_import_dialog.py` baseline
- `setModal(True)`
- Layout: `QVBoxLayout` with `setContentsMargins(16,16,16,16)` and `setSpacing(16)`

**Section 1 — Mode selector**
- `QLabel` "Import mode:" at 10pt Regular
- `QRadioButton` "Merge — add new + update matches" (default selected)
- `QRadioButton` "Replace All — wipe library and restore from ZIP"
- Warning `QLabel` beneath Replace All radio, visible only when Replace All selected: "This will replace all stations, streams, and favorites." styled `color: #c0392b;` at 9pt

**Section 2 — Summary counts** (shown after dry-run, before commit)
- `QLabel` at 13pt DemiBold: "{N} added, {M} replaced, {K} skipped, {L} errors"
- `QTreeWidget` expandable list, hidden until user clicks "Show details" / auto-shown if errors > 0
  - Columns: Station name | Action (Added / Replaced / Skipped / Error)
  - Max visible height: 200px; scrollable if more entries
  - 10pt Regular text

**Section 3 — Button box**
- `QDialogButtonBox(QDialogButtonBox.Ok | QDialogButtonBox.Cancel)`
- OK label overridden to "Import" (commit changes)
- Cancel aborts — no partial import (D-11 all-or-nothing)

### MainWindow wiring
- Enable `act_export` and `act_import_settings` placeholders (lines 100–107 `main_window.py`)
- Connect to `_on_export_settings()` and `_on_import_settings()` handlers
- Both handlers use `ThreadPoolExecutor` for ZIP I/O (matches AA import pattern from `import_dialog.py`)

## Interaction States

### Export flow
| State | UI response |
|-------|-------------|
| User clicks "Export Settings" | `QFileDialog.getSaveFileName` opens |
| User cancels file picker | No action, no toast |
| Export in progress | Hamburger menu item disabled; cursor change optional |
| Export success | `ToastOverlay`: "Settings exported to {filename}" |
| Export error (I/O failure) | `ToastOverlay`: "Export failed — {error message}" |

### Import flow
| State | UI response |
|-------|-------------|
| User clicks "Import Settings" | `QFileDialog.getOpenFileName` opens (filter: `*.zip`) |
| User cancels file picker | No action |
| Invalid ZIP / missing settings.json | `ToastOverlay`: "Invalid settings file" — abort, no dialog |
| Valid ZIP, dry-run complete | `ImportSummaryDialog` opens with counts + expandable list |
| User clicks Cancel in summary | Dialog closes, no changes written |
| User clicks Import in summary | Changes committed; dialog closes; station list refreshes; `ToastOverlay`: "Import complete — {N} added, {M} replaced" |
| Import write error | `ToastOverlay`: "Import failed — {error}" |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (export) | "Export Settings" (hamburger menu label, matches D-13) |
| Primary CTA (import commit) | "Import" (QDialogButtonBox OK override) |
| Export success toast | "Settings exported to {filename}" |
| Export error toast | "Export failed — {error message}" |
| Invalid ZIP toast | "Invalid settings file" |
| Import success toast | "Import complete — {N} added, {M} replaced" |
| Import error toast | "Import failed — {error message}" |
| Summary count line | "{N} added, {M} replaced, {K} skipped, {L} errors" |
| Replace All warning label | "This will replace all stations, streams, and favorites." |
| Merge radio label | "Merge — add new stations and update matches" |
| Replace All radio label | "Replace All — wipe library and restore from ZIP" |
| Show details toggle | "Show details" / "Hide details" |
| Dialog window title | "Import Settings" |

No empty state — import dialog only appears after a valid file is selected. No zero-station case to handle in this phase.

Destructive confirmation (D-11): "Replace All" mode uses **both** patterns together:

1. **Inline warning label** beneath the Replace All radio — `#c0392b`, 9pt: "This will replace all stations, streams, and favorites." Sets expectation before the user clicks Import.
2. **Modal confirmation** (`QMessageBox.warning`, Yes / Cancel with Cancel as default) fired from `_on_import` only when `replace_all` is selected: "This will erase your entire station library and replace it with the import. Continue?"

Rationale for the extra modal (approved deviation from the otherwise-inline-only app convention): wiping the user's entire station library + favorites + providers is irreversible, happens in a single transaction, and cannot be undone via application UI. The modal is a deliberate second-touch safeguard unique to this action. Merge mode does not show the modal.

All other destructive actions in the codebase (edit-station delete, cookie clear) remain inline-only per the app convention; this modal is scoped to the Settings Import Replace All path.
