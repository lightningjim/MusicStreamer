## Decisions Made

- **UAT scope collapse (B inline, D skipped):** Scenario B was verified inline by the executor via `ls /tmp/ms-cookies-* 2>/dev/null` returning empty — the proof is byte-level and does not require human judgment. Scenario D was skipped because the corruption-detect + auto-clear + toast path is identical code between player and import (both call `cookie_utils.is_cookie_file_corrupted` → `constants.clear_cookies` → `toast_callback`). Unit coverage `test_scan_playlist_auto_clears_corrupted` (green in Plan 02) already exercises the import path end-to-end; duplicating it as a manual UAT step adds zero confidence.
- **PROJECT.md entry phrasing:** Wrote the Key Decisions row to explicitly name Plan 35-06 as the regression-introducing plan and yt-dlp's `__exit__`/`save_cookies()` as the mechanism. Rationale: `/gsd-plan-phase` ingests PROJECT.md context when planning future phases that touch cookie I/O, so the entry needs to teach the mechanism, not just announce the fix.
- **Did NOT expand scope to Phase 999.9 side finding:** YouTube playback itself is failing with `yt-dlp: No video formats found` errors — but Scenario A proves this is NOT caused by Phase 999.7 (cookies are byte-unchanged). Filed as **Phase 999.9** in backlog (commit `4b2f8cc`) rather than derailing 999.7 closure.
