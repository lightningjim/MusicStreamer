## Info

### IN-01: Stale "extractor_args" phrasing in `tests/test_cookies.py` module docstring

**File:** `tests/test_cookies.py:1-13` (specifically line 8)
**Issue:** The module docstring still says cookies are passed "alongside the EJS JS challenge solver extractor_args". After Phase 999.9, the EJS solver is wired via `js_runtimes`, not `extractor_args`. The phrasing is now misleading for future readers grepping for how EJS is enabled.
**Fix:**
```python
# Replace line 8:
#   extractor_args, and yt_dlp resolves the direct HLS URL which is fed
# with:
#   js_runtimes (Phase 999.9), and yt_dlp resolves the direct HLS URL which
#   is fed
```

### IN-02: Stale "EJS extractor_args" in test function docstrings

**File:** `tests/test_cookies.py:122, 163`
**Issue:** Both `test_youtube_resolve_passes_cookiefile_when_present` (line 122) and `test_youtube_resolve_omits_cookiefile_when_absent` (line 163) still describe the behavior as "alongside the EJS extractor_args" / "still includes the EJS extractor_args". The body of each test now correctly asserts `js_runtimes` and the absence of `youtubepot-jsruntime`, so the docstrings contradict the assertions.
**Fix:** Update both docstrings to reference `js_runtimes` instead of `extractor_args`, e.g.:
```python
"""_youtube_resolve_worker passes cookiefile=<path> to yt_dlp.YoutubeDL
when cookies.txt exists, alongside the js_runtimes entry (Phase 999.9)."""
```

### IN-03: Test-only `qtbot` fixture parameter is unused in `make_player`

**File:** `tests/test_cookies.py:21-30`
**Issue:** `make_player(qtbot)` accepts `qtbot` but never uses it. Callers pass `qtbot` purely to ensure pytest-qt's event loop fixture is active, but the helper itself does not need it. Not a bug — just noise that a future reader may try to "fix". This is pre-existing to Phase 999.9 (not introduced by the fix), included only because the file is in review scope.
**Fix:** Either accept `_qtbot` to signal "intentionally unused" or drop the parameter and let call sites request `qtbot` in their own signatures for side-effect setup:
```python
def make_player(_qtbot):  # param kept for caller convention; unused here
    ...
```
Optional — safe to leave as-is.
