# Phase 40.1 Plan 04: Station Logos Summary

Restore station-logo rendering (D-11) and add v1.5-parity logo management (D-12/13/14) to EditStationDialog via a shared `_art_paths.abs_art_path()` resolver.

## What Shipped

**Shared resolver (`musicstreamer/ui_qt/_art_paths.py`)**
- New public helper `abs_art_path(rel_or_abs) -> Optional[str]` — resolves relative `station_art_path` entries against `paths.data_dir()`; passes absolute paths through; returns `None` for falsy input.
- No cross-module private imports; both panels and the edit dialog import the same public function.

**NowPlayingPanel (`now_playing_panel.py`)**
- `_load_scaled_pixmap` now resolves `path` via `abs_art_path` before `QPixmap` construction. Existing fallback icon behavior preserved.

**StationListPanel (`station_list_panel.py`)**
- `_load_station_icon` resolves the station path via `abs_art_path` and uses the resolved absolute path as both the `QPixmapCache` key and the `QPixmap` load target. Cache-key change (`station-logo:{abs_path}`) prevents stale entries when a station's logo changes.

**EditStationDialog (`edit_station_dialog.py`)**
- New `_LogoFetchWorker` (QThread) — YT thumbnail auto-fetch on debounced URL change; emits `finished(str)` with a temp file path or `""`. AA branch deliberately no-ops this phase (planner note: revisit when channel-key derivation is feasible).
- Logo row above form: 64×64 `_logo_preview` QLabel, `_choose_logo_btn` (Choose File…, `QFileDialog` filtered to `*.png *.jpg *.jpeg *.webp *.svg`), `_clear_logo_btn`.
- `_refresh_logo_preview`, `_on_choose_logo`, `_on_clear_logo`, `_on_url_timer_timeout`, `_on_logo_fetched`, `_invalidate_cache_for` methods.
- Both picker and auto-fetch route through `assets.copy_asset_for_station(station.id, src, "station_art")` (D-14 write path).
- `_on_save` now persists `self._logo_path` (not stale `station.station_art_path`) and updates the in-memory `Station.station_art_path` so any `station_saved` consumer sees the new value.
- `QPixmapCache.remove(f"station-logo:{abs}")` called on every logo change — paired with the station_list_panel cache-key rewrite to guarantee refresh shows new art.
- `closeEvent` and `reject` disconnect the worker's `finished` signal if still running — prevents signal delivery to a deleted widget (Pitfall 5).
