# Phase 40.1 Plan 02: Discovery Icon Buttons Summary

Replaced text "Play"/"Stop" buttons in the Discovery dialog results table with icon-only toggles using `media-playback-start-symbolic`/`media-playback-stop-symbolic` SVG resources, restoring UI-06 polish consistent with the rest of the Qt port.

## What Changed

**`musicstreamer/ui_qt/discovery_dialog.py`:**
- Imported `QIcon` from `PySide6.QtGui`
- Added side-effect import `from musicstreamer.ui_qt import icons_rc` so `:/icons/*` resources resolve at runtime (matches now_playing_panel pattern)
- Added module-level `_play_icon()` and `_stop_icon()` helpers using the established theme-with-SVG-fallback pattern
- `_on_search_finished`: per-row play button now constructed with `setIcon(_play_icon())` + `setAccessibleName("Play preview")` + `setToolTip("Play preview")` instead of `QPushButton("Play", …)`
- `_on_play_row`: all `setText("Play"|"Stop")` calls replaced with icon + accessibleName + toolTip swaps (both toggle-off branch and cross-row preview-reset branch)
- Save column untouched — still a text `QPushButton("Save", …)` per D-06

**`tests/test_discovery_dialog.py`:**
- `test_play_button_uses_icon` — asserts empty text, non-null icon, accessibleName == "Play preview"
- `test_play_button_toggles_icon_on_click` — asserts icon.cacheKey() changes on toggle and accessibleName flips between "Play preview" and "Stop preview"
