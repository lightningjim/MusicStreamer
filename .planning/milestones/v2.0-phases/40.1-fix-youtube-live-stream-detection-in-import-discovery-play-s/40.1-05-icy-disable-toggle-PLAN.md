<objective>
Root-cause and fix the "Disable ICY" toggle regression. Per RESEARCH.md the flag IS persisted correctly — `repo.update_station()` writes `icy_disabled` and `list_stations()` reads it back. The bug is on the consumption side:

1. `Player._on_gst_tag()` fires `title_changed` unconditionally (no station reference on the GLib thread — scope creep to fix here).
2. `NowPlayingPanel.on_title_changed()` runs on the Qt main thread and has `self._station`, but does NOT check `icy_disabled` before updating `icy_label` or calling `_fetch_cover_art_async` (which is what triggers iTunes lookups).
3. The panel's bound `self._station` can go stale after `EditStationDialog` save — if MainWindow doesn't rebind, the toggle has no effect until restart.

Fix all three per D-15, D-16, D-17.

Purpose: Restore per-station ICY disable (full suppress: no ICY titles, no iTunes cover art lookup) without architectural scope creep into `Player`.
Output: Panel gate + MainWindow rebind + regression test asserting no emit + no iTunes call.
</objective>
