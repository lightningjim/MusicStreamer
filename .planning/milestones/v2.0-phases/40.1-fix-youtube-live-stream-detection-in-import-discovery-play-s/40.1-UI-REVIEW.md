## Top 3 Priority Fixes

1. **Centralize the error-red color** — `#c0392b` is hardcoded at 6+ sites (`import_dialog.py:264,292,339,433,466`, `edit_station_dialog.py:131`, `cookie_import_dialog.py:106`, `accent_color_dialog.py:166`). User-visible impact is low now, but any future theme/dark-mode work will regress. Extract to a module-level constant (e.g., `musicstreamer.ui_qt.theme.ERROR_COLOR`) or a named QSS class and reference everywhere.

2. **AA auto-fetch silently no-ops** — `EditStationDialog._LogoFetchWorker.run` at `edit_station_dialog.py:86-104` does handle AA URLs now, but per Plan 04 SUMMARY the AA CDN auto-fetch branch was documented as no-op. Current code actually attempts it, yet if `slug` or `channel_key` is None the user gets "Fetch not supported for this URL" even on a recognizably-AA URL. Distinguish "AA URL but channel-key not derivable" from "unsupported URL entirely" so the user understands why and can use Choose File instead.

3. **Logo fetch status stays sticky** — `edit_station_dialog.py:533` sets `"Fetched"` on the status label with no timer to clear it. Users who then type a new URL see "Fetched" stale for 500ms (debounce) before "Fetching…" replaces it, and a subsequent no-op URL leaves "Fetched" from an earlier session. Add a `QTimer.singleShot(3000, lambda: self._logo_status.clear())` after success, or clear it when `textChanged` fires.

## Detailed Findings

### Pillar 1: Copywriting (4/4)

- `_format_import_summary` at `import_dialog.py:53-64` nails D-10 wording: `"Imported N new"`, `"All M already in library"`, `"Imported N new, M skipped (already in library)"`. Avoids the bug-reading "Imported 0 channels".
- Delete confirm at `edit_station_dialog.py:651` is specific: `"Delete '{name}'? This cannot be undone."` — not generic.
- AA error branching at `import_dialog.py:436-442` correctly distinguishes `no_channels`, `invalid_key`, and generic network errors with different human copy.
- Accessibility names mirror tooltips on Discovery play buttons (`discovery_dialog.py:363-364`, `474-476`) — consistent screen-reader and mouseover labels.
- No generic "Submit", "OK", "Click Here". "Save Station" / "Discard" / "Delete Station" are specific (edit_station_dialog.py:290-297).
- Minor nit: `"Fetching\u2026"` / `"Choose File\u2026"` use Unicode ellipsis — good, not three dots.

### Pillar 2: Visuals (3/4)

- Icon-only toggle on Discovery play column done correctly: `setIcon()` + `setAccessibleName()` + `setToolTip()` swap on each state transition (`discovery_dialog.py:436-476`). Screen-reader parity maintained.
- Station logo render restored via `abs_art_path` resolver; both panels + EditStationDialog share one helper. Fallback chain is explicit (`now_playing_panel.py:52-64`): resolved path → fallback `:/icons/audio-x-generic-symbolic.svg`, never null.
- 64×64 logo preview in EditStationDialog provides immediate visual feedback for the picker.
- Focal point on NowPlayingPanel: 180×180 logo + 13pt DemiBold ICY title anchors the eye. Good hierarchy.
- Gap: while `Fetch from URL` button is disabled during in-flight work (`edit_station_dialog.py:450`), there's no progress indicator. Status label shows "Fetching…" but no spinner/animation. For a 500ms debounce + network fetch this could read as frozen on slow networks.
- Gap: `_logo_preview.clear()` when no logo means the 64×64 slot goes completely blank (`edit_station_dialog.py:477`) — no placeholder glyph. Contrast with NowPlayingPanel's fallback icon. Inconsistent visual language for "no art".

### Pillar 3: Color (3/4)

- Chip QSS uses `palette(base)`, `palette(mid)`, `palette(highlight)`, `palette(highlighted-text)` (`edit_station_dialog.py:115-129`, `station_list_panel.py`) — theme-respecting, Wayland-friendly.
- Accent color is driven by system palette for chips and selection, appropriate for a desktop app.
- **Hardcoded red** `#c0392b` at:
  - `edit_station_dialog.py:131` (delete button)
  - `import_dialog.py:264,292,339,433,466` (error status x5)
  - `cookie_import_dialog.py:106`, `accent_color_dialog.py:166`
  Six-plus sites, same literal. Would break a dark theme and contradicts the palette-based chip approach.
- Toast and accent color dialog use `setStyleSheet` with additional custom colors (see `toast.py:45`, `accent_color_dialog.py:87,221,230`) — not audited in scope but worth noting for a future theme pass.
- No rainbow of accent colors; red is the only branded literal.

### Pillar 4: Typography (4/4)

- NowPlayingPanel defines three intentional text roles with distinct fonts:
  - Name·Provider: 9pt Normal (`now_playing_panel.py:125-130`)
  - ICY title: 13pt DemiBold (`:132-140`) — primary focal text
  - Elapsed: 10pt Normal + `QFont.TypeWriter` hint (`:143-149`) — tabular digits so "1:05" → "1:06" doesn't shift layout
- Icon sizes: 24×24 for main transport, 20×20 for star — two sizes, matches role hierarchy.
- No use of arbitrary font sizes in the audited files; everything goes through `QFont.setPointSize` with deliberate values (9/10/13).
- `Qt.PlainText` enforced on ICY label (`:139`) — security AND typographic consistency (no rich-text font overrides from upstream).

### Pillar 5: Spacing (4/4)

Reviewed spacing patterns across all four files:

| File | Dialog/Panel contentsMargins | Inner spacing |
|------|------------------------------|---------------|
| `import_dialog.py:194,241` | `(16,16,16,16)` per tab | 8 |
| `discovery_dialog.py:192` | `(16,16,16,16)` | 8 |
| `edit_station_dialog.py:168` | `(16,16,16,16)` | 8 |
| `now_playing_panel.py:104` | `(16,16,16,16)` | 24 (three-column outer), 8 (center) |

- Inner rows use `setContentsMargins(0,0,0,0)` + `setSpacing(4)` or `8` — two-tier scale, consistent.
- Logo preview fixed at `(64,64)`; cover at `(160,160)`; logo at `(180,180)` — a 64/160/180 step is generous but intentional per UI-SPEC precedent (not seen here).
- Transport buttons fixed at `36×36` with `24×24` icons — 6px visual padding, correct for Qt HIG.
- No `[arbitrary]` inline values; no magic numbers outside the 4/8/16/24 ladder.

### Pillar 6: Experience Design (3/4)

**State coverage — strong:**
- Loading: `QProgressBar` (indeterminate then determinate in AA flow), `setRange(0,0)` for indeterminate — `import_dialog.py:301-302, 402-403, 416-417`.
- Error: dedicated status labels with red styling; error branching on AA with 3 distinct messages — `import_dialog.py:430-442`.
- Empty: `"No live streams found in this playlist."` explicit — `import_dialog.py:314`.
- Disabled: play/import buttons gated via `_set_yt_busy` / `_set_aa_busy` — `import_dialog.py:386-390, 471-474`.
- Destructive confirmation: delete uses `QMessageBox.question` with `Cancel` default — `edit_station_dialog.py:649-656`.
- ICY disable gate at consumption side correctly prevents both label overwrite and cover-art fetch (`now_playing_panel.py:289-315`). Plan 05 root-cause analysis is exemplary — Player left untouched, gate at Qt layer.
- Elapsed timer plan 06 has correct pause/stop/failover semantics laid out in the table.

**Gaps:**
- `_logo_status` sticky-state bug (see Top Fix #3).
- AA auto-fetch discovered-AA-but-no-key path reports "Fetch not supported" — misleading (see Top Fix #2).
- `closeEvent` / `reject` on EditStationDialog waits 2s for logo worker (`edit_station_dialog.py:563`). Reasonable but undocumented to user — if the user hits Discard during a fetch, they see a 2s hang. A `QProgressDialog` or at least cursor-override would signal the wait.
- Toast + inline label duplication on import complete (`import_dialog.py:374-378, 456-460`) is per-spec D-07, but note the inline label has no dismiss affordance and persists until the next action.
- No keyboard accelerators audited (`&` mnemonics) in the dialogs I read — not necessarily a regression, but worth flagging for a future a11y pass.
