# Phase 999.3 Plan 02: Failure UX upgrade + persistent OAuth log Summary

**One-liner:** Replace the generic "Try again" QMessageBox with a category-aware
QDialog that parses oauth_helper's structured stderr events (D-12), offers an
inline Retry button (D-09), and persists scrubbed diagnostic events to a
size-rotated `~/.local/share/musicstreamer/oauth.log` (D-10, D-11).

## What was built

### Task 1 — paths helper + OAuthLogger module (commit `8019f64`)

- Added `paths.oauth_log_path()` returning `<root>/oauth.log`, honoring
  `_root_override` for test isolation. Pure — no I/O on call.
- Created `musicstreamer/oauth_log.py` with the `OAuthLogger` class:
  - Wraps `logging.handlers.RotatingFileHandler(maxBytes=64*1024, backupCount=2)`
    — file rotates at 64 KB; at most three files (`oauth.log`, `oauth.log.1`,
    `oauth.log.2`) ever exist.
  - Tightens file permissions to `0o600` on the first `log_event` call
    (T-40-03 parity with `clear_twitch_token`).
  - `_scrub()` helper rejects any detail that contains `"://"` (URL) or
    `"access_token"` (OAuth fragment key), starts with `state=` / `code=` /
    `token=`, or exceeds 200 chars — replaced with `"<scrubbed>"`. Rejection
    is also defensively applied to non-string input.
  - Each event is serialized as a single JSON line with a fixed schema:
    `{ts, category, detail, provider}`. Category and provider are length-capped
    (64 / 32 chars) at write time.
- Added 13 unit tests in `tests/test_oauth_log.py` covering permissions, JSON
  schema, all scrub rules (URL / access_token / state= / code= / token= /
  oversize), benign short detail, rotation at 64 KB, and the `backup 3 never
  exists` invariant.
- Extended `tests/test_paths.py` with 2 new tests + the `oauth_log_path`
  accessor added to the shared override / purity probes.

### Task 2 — AccountsDialog failure UX + stderr parser + log writes (commit `3c43fa5`)

- Added module-level `_CATEGORY_LABELS` mapping all six categories from the
  Plan 01 stderr contract (`InvalidTokenResponse`, `TwitchRejectedRequest`,
  `LoginTimeout`, `WindowClosedBeforeLogin`, `PortBusy`, `SubprocessCrash`) to
  user-facing strings.
- Rewrote `_on_oauth_finished` to:
  1. Parse stderr byte-by-byte. Each non-empty line is `json.loads`'d inside a
     `try/except JSONDecodeError`; malformed lines are skipped. Only dicts with
     a `"category"` key are accepted (T-999.3-05). The LAST valid event wins.
  2. Read stdout for the token with the same tolerant-decode semantics.
  3. Success branch (exit 0 + non-empty token + Success/no event): write token
     with 0o600, log a Success event, update status. Unchanged write behavior.
  4. Failure branch: if exit 0 with empty token → synthesize
     `InvalidTokenResponse empty_stdout`; else if no parseable event →
     synthesize `SubprocessCrash exit=<code>`. Log the event (logger failure
     swallowed), update status, invoke `_show_failure_dialog(category, detail)`.
- Added `_show_failure_dialog(category, detail)` — a `QDialog` with:
  - Title label (11pt bold, PlainText) showing `_CATEGORY_LABELS[category]`,
    falling back to "Unknown error".
  - Detail label (9pt, PlainText, WordWrap) showing the scrubbed detail, or
    `"(no details provided)"` when empty.
  - `Retry` and `Close` buttons in an HBox row. Retry connects to `accept`,
    Close to `reject`. T-40-04: every label is `PlainText` — user-visible
    detail cannot inject HTML.
  - After `exec()`, `Accepted` triggers `_launch_oauth_subprocess()` inline
    without closing AccountsDialog (D-09).
- Extracted `_launch_oauth_subprocess()` from `_on_action_clicked`'s
  not-connected branch. Single authoritative launch path shared by the
  initial Connect click and the failure-dialog Retry path.
- Added `_get_oauth_logger()` — lazy `OAuthLogger(paths.oauth_log_path())`
  wrapped in `try/except OSError` (and a broader `except Exception` for
  defensive coverage). On failure returns `None`; callers `if logger is not
  None:` to gate every log call. Logging is supplementary per D-11.
- Phase 48 AudioAddict code (`_on_aa_clear_clicked`, `_aa_status_label`,
  `_aa_clear_btn`, `_is_aa_key_saved`, the `_update_status` AA section, and
  the AA groupbox in `__init__`) preserved verbatim.

## Deviations from Plan

**[Rule 1 - Bug] Corrected failure classification precedence**

- **Found during:** Task 2 test review (`test_oauth_finished_synthesizes_invalid_on_empty_token`)
- **Issue:** The plan's failure-branch pseudocode classified `last_event is
  None` FIRST (→ `SubprocessCrash`), then `exit_code == 0 and not token`
  (→ `InvalidTokenResponse`). For the realistic case of `exit_code == 0` +
  empty stderr + empty stdout, this would incorrectly yield `SubprocessCrash
  exit=0` — but the subprocess exited cleanly; it just produced no token.
  That is semantically an invalid response, not a crash.
- **Fix:** Swapped precedence in `_on_oauth_finished` — `exit_code == 0 and
  not token` is classified first (regardless of whether last_event is
  present), then the `last_event is None` fallback only applies to the
  non-zero-exit case.
- **Files modified:** `musicstreamer/ui_qt/accounts_dialog.py`
- **Commit:** `3c43fa5`

No architectural deviations, no auth gates, no checkpoints.
