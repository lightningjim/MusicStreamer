<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → loopback listener | Untrusted HTTP requests cross into subprocess via 127.0.0.1:17823 |
| Twitch redirect JS → /capture | Token crosses from browser JS context to subprocess via POST body |
| HTTP handler thread → main Qt thread | Captured state crosses thread boundary via threading.Event |
| Subprocess stderr → parent QProcess | Diagnostic events cross process boundary |

| T-999.3-02 | Tampering (CSRF) | `_RedirectHandler.do_POST` | mitigate | `_generate_state()` uses `secrets.token_urlsafe(16)`; `_TwitchLoopbackServer.expected_state` stored on init; POST handler rejects 400 on mismatch AND emits `InvalidTokenResponse state_mismatch`. Test `test_capture_rejects_state_mismatch` enforces. |
| T-999.3-04 | Information Disclosure (token exfil via page content) | `_BOUNCE_PAGE_HTML` | mitigate | Self-contained HTML — no `<script src=>`, no `<img src=https://...>`, no `<link href=https://...>`.
