## Mid-UAT Pivots (documented for future reference)

1. **UA rejection (Twitch "browser not supported"):** Resolved with `QTWEBENGINE_CHROMIUM_FLAGS="--user-agent=<Chrome 140 UA>"` set at module-import time (commit `1017c41`). Profile-level `setHttpUserAgent` alone was applied too late for Twitch's initial page-load detection.
2. **OAuth-redirect architecture abandoned:** The original Wave 1 plan used an authorization-code + loopback listener flow. Research had already flagged that the piggyback web client_id's registered redirect URIs don't include `http://127.0.0.1:*`, but the full implication only surfaced during UAT when Twitch silently refused our `redirect_uri`. Pivoted to cookie-harvest (mirror of `_GoogleWindow`): log in to `twitch.tv`, watch `QWebEngineCookieStore.cookieAdded` for the `auth-token` cookie on `.twitch.tv`, extract its value (commit `10896de`, `a4c8b32`). This yields the exact token Twitch's web client uses, which is what streamlink needs.
