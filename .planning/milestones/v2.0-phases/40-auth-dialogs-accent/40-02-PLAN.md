Create `musicstreamer/oauth_helper.py` — a standalone subprocess helper per D-02, D-04, D-07.

This is launched via `python -m musicstreamer.oauth_helper --mode twitch|google` from the main process.

Structure:
```python
"""Subprocess OAuth helper — isolated QWebEngineView for Twitch/Google login.

Runs in a SEPARATE process to avoid loading QtWebEngine (130MB) in the main app.
Communicates results back via stdout (token for Twitch, cookie text for Google).

Usage:
  python -m musicstreamer.oauth_helper --mode twitch
  python -m musicstreamer.oauth_helper --mode google
"""
```

`--mode twitch`:
- Creates a `QApplication` (standalone — this is a separate process)
- Opens `QWebEngineView` to `https://id.twitch.tv/oauth2/authorize?client_id=...&redirect_uri=http://localhost&response_type=token&scope=`
  - Note: the actual Twitch client_id should be read from an env var `MUSICSTREAMER_TWITCH_CLIENT_ID` or hardcoded as the app's registered client ID. For now, use the same client ID pattern as v1.5. Check if there's an existing constant.
- Intercepts the redirect URL containing `#access_token=...`
- Override `QWebEngineView.urlChanged` signal: when URL contains `access_token=`, extract token, print to stdout, call `QApplication.quit()`
- If the window is closed without token: exit with code 1 (no stdout)
- Writes nothing to disk — the main process handles file write + permissions

`--mode google`:
- Opens `QWebEngineView` to `https://accounts.google.com/ServiceLogin`
- After login completes (URL navigates to `myaccount.google.com` or similar), extract cookies from `QWebEngineCookieStore`
- Connect to `page().profile().cookieStore().cookieAdded` signal to collect all cookies
- On a "Done" button click (add to the window) or after detecting YouTube domain cookie:
  - Format collected cookies in Netscape format
  - Print to stdout
  - Call `QApplication.quit()`
- If window closed without cookies: exit with code 1

Both modes:
- `argparse` with `--mode` required argument
- Window title: "Twitch Login" or "Google Login"
- Window size: 800x600
- Use `if __name__ == "__main__":` guard AND also support `-m` invocation

IMPORTANT security:
- Use `sys.executable` for interpreter path (no PATH injection per T-40-05)
- Do NOT use `shell=True` anywhere
- Token/cookies only go to stdout, never logged

NOTE: The import `from PySide6.QtWebEngineWidgets import QWebEngineView` will fail if `python3-pyside6.qtwebenginewidgets` is not installed. The script should catch `ImportError` and print a helpful error message to stderr, then exit with code 2.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Subprocess stdout → token file | Untrusted data from child process written to disk |
| OAuth redirect URL → token extraction | Token parsed from URL fragment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-03 | Information disclosure | Token file world-readable | mitigate | `os.chmod(path, 0o600)` immediately after write in `_on_oauth_finished` |
| T-40-04 | Spoofing | QLabel showing external text | mitigate | `setTextFormat(Qt.PlainText)` on status label |
| T-40-05 | Elevation of privilege | Subprocess PATH injection | mitigate | Use `sys.executable` (absolute path) as QProcess program; never `shell=True` |
| T-40-06 | Information disclosure | Token logged to console | accept | Token only read via QProcess.readAllStandardOutput — not logged; low risk for desktop app |
</threat_model>
