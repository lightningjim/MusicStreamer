<decisions_made>

- `add_signal_watch` MUST run on the thread whose thread-default MainContext is iterated. Marshaling via `GstBusLoopThread.run_sync` (GLib idle source → bridge thread) is the only correct path; `push_thread_default` from main fails with `acquired_context` assertion because the bridge's loop has already acquired the context.
- AUMID ctypes call uses explicit `LPCWSTR` argtypes — default marshaling can pass narrow (A) pointer on some CPython builds, silently failing.
- `detach_stream()` is mandatory before `RandomAccessStreamReference.create_from_stream(stream)` when DataWriter still holds ownership — otherwise readers see zero-length stream and overlay thumbnail is blank despite non-None reference.
</decisions_made>

## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `add_signal_watch` on main thread | PyGObject attaches GSource to caller's thread-default MainContext. Default context is not iterated on Windows; handlers silently never fire. | blocking | `_bridge.run_sync(lambda: bus.add_signal_watch())` pattern in player.py / gst_bus_bridge.py. Verify any new bus-watch wiring follows this. |
| `push_thread_default` across threads | Context is already acquired by bridge thread's MainLoop; pushing from another thread fails silently (GLib warning, no exception). | advisory | Always marshal onto owning thread via idle source; never try to borrow a running context. |
| `store_async().get()` on Qt STA main thread | Windows raises 'Cannot call blocking method from single-threaded apartment'; exception is swallowed into warning log, thumbnail silently falls back to None. | advisory | Use `asyncio.run(await ...)` for winrt IAsyncOperation when on STA thread. Sync `.get()` is only safe on MTA threads. |
| Forgetting `detach_stream()` on DataWriter | Reader through `RandomAccessStreamReference` gets unreadable stream; no error, just missing thumbnail. | advisory | Always detach before wrapping in a stream reference. |
| `ctypes.windll` calls without explicit argtypes | Narrow (A) pointer marshaling can silently succeed on some CPython builds while actually corrupting the string. HRESULT=S_OK is not enough to trust; readback-verify during UAT. | advisory | Set `argtypes` / `restype` explicitly for every Win32 function used. |
