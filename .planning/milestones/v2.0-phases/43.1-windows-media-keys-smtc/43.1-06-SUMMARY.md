## What was NOT just validation — three root causes surfaced and fixed

Execution deviated from "run UAT, paste results" because the first-pass Windows launch surfaced three functional bugs that had never been seen on Linux (the automated suite had not exercised them). All three were diagnosed end-to-end with the diagnostic/fix/strip loop and fixed against `phase-43.1-uat-diag` before UAT sign-off.

1. **GStreamer bus handlers never fired on Windows.** `bus.add_signal_watch()` was called inline on the main thread. Its `GSource` attached to the main thread's default `MainContext`, which no one iterates on Windows (PySide6 has no GLib integration there). Result: `message::error`, `message::tag`, `message::buffering` silently dropped — no ICY titles, no error recovery, no buffering UI. **Fix:** marshal `add_signal_watch` onto the `GstBusLoopThread` via a new `run_sync(callable)` helper so the watch attaches to the thread that actually iterates its context.

2. **AUMID not bound to the process.** `SetCurrentProcessExplicitAppUserModelID` called after `QApplication()` is a no-op — the AUMID binds at first window creation. Default `ctypes` marshaling can also pass Python `str` as a narrow (A) pointer. **Fix:** call with explicit `LPCWSTR` argtypes on `SetCurrentProcessExplicitAppUserModelID`, BEFORE `QApplication()`. Readback via `GetCurrentProcessExplicitAppUserModelID` confirms S_OK + correct value. (Shell still renders "Unknown app" because the display name requires a registered Start Menu shortcut — deferred to Phase 44 installer.)

3. **Thumbnail rendered nothing in the overlay.** Two compounding bugs: (a) `store_async().get()` raises "Cannot call blocking method from single-threaded apartment" on Qt's STA main thread (RESEARCH Pitfall #3); (b) `RandomAccessStreamReference.create_from_stream` sees an unreadable stream because `DataWriter` still owns it. **Fix:** `asyncio.run(await writer.store_async())` + `writer.detach_stream()` before wrapping. Title/artist had been rendering via the same code path — only the thumbnail needed the detach.

Fourth bug, discovered during UAT-6/7 testing — **cross-OS regression introduced by the fix for #1.**

4. **Every Shoutcast/Icecast stream died at exactly 10 s, Linux + Windows.** Diagnosed from a GStreamer debug log showing a clean external teardown with no error message, tracked to `_on_gst_tag` calling `QTimer.singleShot(0, self._cancel_timers)` from the bus-loop thread — which posts to the calling thread's Qt event loop, and the pure-GLib bridge thread has none. The cancel vanished, the `BUFFER_DURATION_S = 10` failover timer fired, `_try_next_stream` tore down playback. Latent pre-43.1 because bus handlers ran on main (on Linux) or never fired (on Windows). **Fix:** replace bare-callable `QTimer.singleShot` with a `Qt.Signal` + `QueuedConnection` — same pattern already used for `title_changed` / `twitch_resolved`. Linux verified (SomaFM, DI.fm), Windows verified (unblocks UAT-6, UAT-7).
