| Property | Value |

## Interaction Contract

### Provider Label (NP-01)

- On `_play_station(st)`: if `st.provider_name` is non-empty, set `station_name_label.set_text(f"{st.name} · {st.provider_name}")`. If `provider_name` is empty or None, set `station_name_label.set_text(st.name)`.
- On `_stop()`: set `station_name_label.set_visible(False)`.
- The middle dot separator is U+00B7 (·), surrounded by single spaces: `" · "`.
- ICY title updates do NOT change `station_name_label` — only `title_label` is updated by ICY.

### Volume Slider (AUDIO-01, AUDIO-02)

- Initialize slider value from `repo.get_setting("volume", "80")` on window construction.
- Apply initial volume to player immediately after player construction: `player.set_volume(int_val / 100.0)`.
- Signal: `Gtk.Scale` emits `value-changed` — connect to `_on_volume_changed`.
- `_on_volume_changed`: read `int(slider.get_value())`, call `player.set_volume(value / 100.0)`, call `repo.set_setting("volume", str(value))`.
- Volume change is applied immediately (no debounce — local operation, no network call).
- Slider increments: `set_increments(1, 10)` — arrow keys move by 1, page up/down by 10.
- Slider digits: `set_digits(0)` — display integer only, no decimal.
- Draw value: `set_draw_value(False)` — no inline numeric label (keeps panel compact).

### GStreamer Volume

- `Player.set_volume(float)` calls `self._pipeline.set_property("volume", clamp(value, 0.0, 1.0))`.
- GStreamer `playbin3` volume property accepts 0.0–1.0 float.
- Volume must apply even when no stream is currently playing (sets the property for next play).

### mpv (YouTube) Volume

- When launching mpv subprocess, pass `--volume={int_val}` where `int_val` is `int(slider.get_value())`.
- If slider changes while mpv is running: send `set volume {int_val}` via mpv IPC socket if IPC is available, or store value and apply on next mpv launch.
