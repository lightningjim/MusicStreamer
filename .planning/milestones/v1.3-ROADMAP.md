## Overview

Added station discovery and a favorites system. Users can star ICY track titles, browse Radio-Browser.info stations in-app, and bulk-import AudioAddict network channels or YouTube playlist live streams.

## Milestone Summary

**Key Decisions:**
- `Adw.ToggleGroup` over linked `Gtk.ToggleButton` pair for Stations/Favorites switcher — native Adwaita segmented control, HIG-compliant
- `url_resolved` preferred over `url` from Radio-Browser API — `url` is often a PLS/M3U playlist, not a direct stream
- `entry.get('is_live') is True` strict identity check for yt-dlp — non-live entries return `None` not `False` in flat-playlist mode
- Open thread-local DB connection in import worker — SQLite connections cannot be shared across threads
- Use `ch['key']` not `ch['name']` for AudioAddict PLS URL slug — channel names have spaces, keys are lowercase slugs
- `ValueError('no_channels')` when all AudioAddict networks return empty — catches expired keys returning 200+empty instead of 401
- Resolve PLS to direct stream URL in `aa_import.fetch_channels` — GStreamer cannot play PLS playlists directly
- `INSERT OR IGNORE` for favorites dedup — UNIQUE(station_name, track_title) constraint; silent dedup

**Issues Resolved:**
- SQLite cross-thread errors in import workers fixed via thread-local `db_connect()` pattern
- AudioAddict PLS URLs not playable by GStreamer — resolved PLS to direct stream URL at import time

**Issues Deferred / Tech Debt:**
- DISC-03 cosmetic: stale now-playing state after preview close when nothing was playing prior (bypasses `main_window._stop()`)
- `cover_art._parse_artwork_url` (lines 53–64) — dead code, superseded by `_parse_itunes_result` (Phase 12). Safe to delete.
- Two silent `except Exception: pass` in `discovery_dialog.py` `_load_tags`/`_load_countries` — intentional graceful degradation
- `except Exception: pass` in `aa_import._resolve_pls` — intentional fallback on PLS resolution failure
- Nyquist compliance partial for phases 13–15 (VALIDATION.md exists but wave_0 incomplete)
