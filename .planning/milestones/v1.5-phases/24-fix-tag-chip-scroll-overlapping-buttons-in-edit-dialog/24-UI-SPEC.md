## Layout Contract (Phase-Specific)

This phase is exclusively a layout fix — the primary deliverable is constraining the chip scroll width within the dialog's form column.

### Problem

`chip_scroll` (a `Gtk.ScrolledWindow`) has no max width constraint and no `hexpand=False` set explicitly. When the dialog is wide, the ScrolledWindow expands beyond the grid column 1 boundary, overlapping adjacent controls.

### Required Fix: FlowBox (Discretion D-01 + D-02 resolved)

**Decision:** Replace horizontal `Gtk.Box` + `Gtk.ScrolledWindow` with `Gtk.FlowBox` (no ScrolledWindow wrapper).

Rationale: Wrapping to multiple lines is the correct Adwaita pattern for tag chips. Horizontal scroll is a poor UX for a small set of discrete toggleable chips — it hides options. FlowBox naturally constrains to column width with zero max-width plumbing.

**FlowBox properties:**
- `orientation = Gtk.Orientation.HORIZONTAL`
- `selection_mode = Gtk.SelectionMode.NONE` (selection handled by ToggleButton state, not FlowBox)
- `column_spacing = 8` (matches existing chip box spacing)
- `row_spacing = 4` (xs token — tight vertical rhythm)
- `homogeneous = False`
- `hexpand = False` (critical — prevents column overflow)
- `margin_top = 4`, `margin_bottom = 4`, `margin_start = 8`, `margin_end = 8` (preserve existing chip_scroll margins)
- Min height: not set (natural height from rows)

**Chip widget (Discretion D-02 resolved):** Keep `Gtk.ToggleButton`. Do not restyle to pill shape — Adwaita ToggleButtons in a FlowBox already render as compact chips. Pill styling would require custom CSS which this project does not use.

**Remove:** `chip_scroll` (`Gtk.ScrolledWindow`) and the wrapping horizontal `Gtk.Box` (`_chip_box`). Replace with `self._chip_box` as `Gtk.FlowBox` directly attached to grid row 3, column 1 (via `tags_box`).

### Grid Attachment

`tags_box` (vertical `Gtk.Box`) attaches to `form` at row 3, column 1. No change to grid structure. `tags_box` contains: `self._chip_box` (FlowBox) then `self.new_tag_entry` (unchanged).
