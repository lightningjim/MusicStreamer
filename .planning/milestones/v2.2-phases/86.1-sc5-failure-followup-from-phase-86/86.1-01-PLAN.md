<objective>
Stop the privacy/security defect at its root: in the Flatpak sandbox the narrow :ro host mount (D-01) makes `src != dest`, so `migration.run_migration()` silently copies ALL host data — including 0600 `cookies.txt` / `twitch-token.txt` secrets — into the sandbox without user consent, preempting the consent-based import wizard.

This plan introduces ONE shared sandbox-detection helper (`is_sandboxed()` via `/.flatpak-info`) in `flatpak_first_launch.py` and makes `run_migration()` sandbox-aware: when sandboxed it skips the auto-copy entirely while STILL creating the dest dir and writing the migration marker (the `buffer_log.install_buffer_events_handler` ordering invariant at `__main__.py:199` depends on DATA_DIR existing after `run_migration()`).

Purpose: Eliminate the silent, non-consensual ingestion of host secrets into the sandbox. The wizard (Plan 02) becomes the SOLE path for host data to enter the sandbox.
Output: `is_sandboxed()` helper, sandbox-aware `run_migration()`, and unit tests that simulate the sandbox condition and prove no copy occurs.
</objective>
