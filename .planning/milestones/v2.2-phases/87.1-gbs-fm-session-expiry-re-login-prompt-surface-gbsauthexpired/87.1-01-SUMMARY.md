# Phase 87.1 Plan 01: GbsReloginHandler Summary

**One-liner:** Single-flight QObject handler launching `oauth_helper --mode gbs` with 4-condition Netscape cookie validation and 0o600 write, emitting `relogin_succeeded` / `relogin_failed` signals (GBS-AUTH-EXP-02).

## What Was Built

`musicstreamer/ui_qt/gbs_relogin_handler.py` — new `GbsReloginHandler(QObject)` class that:

- Accepts expiry notifications from any consumer via `notify_expiry_detected()` — main-thread-only, safe to call from multiple workers
- Single-flight de-dup (WR-03): `if self._login_proc is not None: return` in `notify_expiry_detected` blocks concurrent subprocess spawns
- Launches `oauth_helper --mode gbs` via `_make_oauth_launch_args("gbs")` (handles frozen vs. source path resolution, no PATH injection)
- Wires both `finished` → `_on_finished` and `errorOccurred` → `_on_error` (QA-05 bound methods; `errorOccurred` prevents FailedToStart from permanently jamming the guard)
- Four-condition success gate in `_on_finished` (mirrors `accounts_dialog._on_gbs_login_finished`): `exit_code == 0` AND `netscape_text` non-empty AND stderr category absent-or-Success AND `_validate_gbs_cookies()` True
- Writes cookies to `paths.gbs_cookies_path()` with `os.chmod(path, 0o600)` immediately after write (T-40-03)
- Resets `_login_proc = None` FIRST in both `_on_finished` and `_on_error` (FailedToStart path) before any signal emission
- No GBS cookie/credential bytes in any log call (T-87.1-05)

`tests/test_gbs_relogin_handler.py` — 4 TDD unit tests covering all security-relevant paths:

1. `test_single_flight_dedup` — two `notify_expiry_detected()` calls produce exactly 1 `QProcess.start`
2. `test_relogin_succeeded_on_valid_cookies` — `relogin_succeeded` emitted after valid cookie write; asserts cookie file exists and `stat.S_IMODE == 0o600`
3. `test_relogin_failed_on_nonzero_exit` — `relogin_failed` emitted on non-zero exit; no cookie file written
4. `test_failed_to_start_resets_guard` — `FailedToStart` emits `relogin_failed`, resets `_login_proc = None`, subsequent `notify_expiry_detected()` launches fresh subprocess
