## Gap-Closure Status

The BLOCKER from 87.1-HUMAN-UAT Test 1 was: `_open_with_cookies` auto-followed the server's `302 → /accounts/login/` (urllib's default `HTTPRedirectHandler`) and returned the 200 login page, so `fetch_active_playlist` raised `JSONDecodeError` instead of `GbsAuthExpiredError`, and `_GbsPollWorker.run` took the silent-fallback branch — the inline prompt was unreachable.

**Plan 04 gap closure verified in codebase:**

1. `_open_with_cookies` (gbs_api.py:158-228) now installs a `_NoRedirect` inner class (mirroring the existing `_open_no_redirect` pattern) that returns `fp` instead of following any redirect. After `opener.open()`, the stopped response is inspected: if `Location` contains `/accounts/login/`, `GbsAuthExpiredError` is raised. Non-login 3xx redirects are re-followed with a clean opener. The previous `HTTPError`/302 belt-and-braces block is retained.

2. `_fetch_marquee` (gbs_marquee.py:355-362) now has a narrow `except gbs_api.GbsAuthExpiredError: raise` inserted BEFORE the generic `except Exception` handler that was previously swallowing the error (confirmed by Task 3 AUDIT finding).

3. Five real-chain regression tests in `tests/test_gbs_api.py` patch at `urllib.request.AbstractHTTPHandler.do_open` (not at `_open_with_cookies` itself) so the full CPython opener chain runs. All 5 pass.

4. All three commits exist in git history: a66f9c10 (RED tests), f8d07bfd (GREEN fix), 831801c4 (AUDIT fix).
