# Phase 87.1 Plan 05: GBS Re-login Detection-vs-Launch Separation Summary

Closed the major UAT defect (87.1-HUMAN-UAT.md `## Gaps`, Test 1, severity major): the
inline "GBS session expired" prompt now appears on real expiry without auto-launching the
`oauth_helper --mode gbs` login window. Both detection paths (playlist poll + marquee
worker) route to a new reveal-only slot; the subprocess launch remains user-gated on the
"Log in again" button.

## What Was Built

**`_on_gbs_expiry_detected(self) -> None`** — a new bound-method slot on `NowPlayingPanel`
that only calls `self._gbs_expiry_widget.setVisible(True)`. It is idempotent (safe to call
from multiple detection paths), takes zero arguments (matches the no-payload
`GbsMarqueeWorker.auth_expired` Signal), and explicitly documents the detection-vs-launch
contract in its docstring (D-01 / GBS-AUTH-EXP-01 / USER DECISION 2026-06-17).

**Detection path rewiring (two sites):**

1. `_on_gbs_playlist_error` auth_expired branch: removed the
   `self._gbs_relogin_handler.notify_expiry_detected()` call (~lines 3171-3172); replaced
   the bare `setVisible(True)` with `self._on_gbs_expiry_detected()`. The station-poll
   state mutations (hide playlist widget, stop poll timer, clear, `_gbs_label_source='icy'`,
   `_gbs_ajax_disabled=True`) are unchanged — they are correct and playlist-specific.

2. `attach_gbs_marquee_worker`: changed `worker.auth_expired.connect(...)` target from
   `self._gbs_relogin_handler.notify_expiry_detected` to `self._on_gbs_expiry_detected`,
   keeping `Qt.QueuedConnection` (cross-thread delivery). The handler-presence guard was
   removed since the panel slot has no handler dependency. Comment updated to cite USER
   DECISION 2026-06-17.

**`_on_gbs_relogin_clicked` — UNCHANGED.** Remains the sole caller of
`notify_expiry_detected()`. `gbs_relogin_handler.py` — UNCHANGED. Single-flight WR-03
guard intact.

## Marquee Test Approach

The `test_marquee_auth_expired_shows_prompt_no_auto_launch` test used a **real
`GbsMarqueeWorker`** instance (imported from `musicstreamer.gbs_marquee`). The constructor
touches no network — it only sets up internal state and a `cadence_changed_internal` signal
connection. `worker.auth_expired.emit()` followed by `QApplication.processEvents()` drains
the `Qt.QueuedConnection` queue and delivers the signal to the panel's main-thread slot.
