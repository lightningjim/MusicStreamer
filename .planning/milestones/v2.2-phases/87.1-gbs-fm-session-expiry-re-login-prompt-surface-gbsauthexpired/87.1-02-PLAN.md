<objective>
Add an `auth_expired` Qt Signal to `GbsMarqueeWorker` and emit it from `_fetch_marquee` when `gbs_api.GbsAuthExpiredError` is raised, so the marquee poller can feed the shared `GbsReloginHandler` instead of failing silently (GBS-AUTH-EXP-02 — "shared handler reused by marquee").

Purpose: The marquee worker (Phase 87) currently catches `GbsAuthExpiredError` and only logs a warning (D-18, `gbs_marquee.py:355`). For the shared handler to be genuinely shared, the marquee must surface expiry as a cross-thread signal that Wave 2 will connect to `handler.notify_expiry_detected` with `Qt.QueuedConnection`. This plan adds ONLY the signal + emission (and its test); the connection lives in Plan 03 (panel) which owns the wiring site.

Output: modified `musicstreamer/gbs_marquee.py` + new test in `tests/test_gbs_marquee.py`.
