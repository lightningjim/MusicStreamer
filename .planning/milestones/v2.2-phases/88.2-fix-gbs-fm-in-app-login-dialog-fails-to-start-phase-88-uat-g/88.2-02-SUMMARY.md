# Phase 88.2 Plan 02: Launch-Failure UX and Regression Guards Summary

**One-liner:** `errorOccurred` handlers on GBS/Twitch QProcess reset the WR-03 re-entrancy guard and surface the cookie-import fallback on FailedToStart; `build.ps1` OAUTH HELPER GUARD (exit 12) + drift-guard lock the frozen-exe dispatch contract.

## Objective

Add the launch-failure UX layer (D-02/D-03) on top of Plan 01's launcher fix: a GBS login `errorOccurred` handler that resets the WR-03 guard, logs visibly at WARNING, and auto-surfaces the `CookieImportDialog` fallback on `FailedToStart`; a Twitch `errorOccurred` handler (reset + log, no fallback); the `build.ps1` D-05 runtime guard (`--oauth-helper --self-test`, exit 12) with a `SkipOauthGuard` bypass; and the D-05 source-text drift-guard locking both `build.ps1` and `__main__.py`.
