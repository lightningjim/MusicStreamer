# Phase 88.2 Plan 01: Frozen-safe OAuth Subprocess Launcher Summary

**One-liner:** Frozen-safe `_make_oauth_launch_args` helper and `--oauth-helper` argv dispatch in `__main__.py` repair GBS.FM + Twitch + Google login on PyInstaller-frozen Windows build by replacing the broken `-m musicstreamer.oauth_helper` form with `--oauth-helper` re-exec.

## Objective

Repair all three oauth-helper subprocess launch sites so they work when `sys.executable` is `MusicStreamer.exe` (frozen build). Root cause: `QProcess.start(sys.executable, ["-m", "musicstreamer.oauth_helper", "--mode", m])` is silently ignored by the frozen exe because it cannot interpret `-m`. Fix: shared helper `_make_oauth_launch_args` that branches on `getattr(sys, "frozen", False)`.

## What Was Built

### `musicstreamer/subprocess_utils._make_oauth_launch_args(mode)`

Returns `(program, args)` for launching `oauth_helper --mode <mode>`:
- Frozen branch: `(sys.executable, ["--oauth-helper", "--mode", mode])`
- Non-frozen branch: `(sys.executable, ["-m", "musicstreamer.oauth_helper", "--mode", mode])`

T-40-05 contract preserved: `sys.executable` as program (no PATH lookup), args as list (never shell string), `mode` is a hardcoded call-site literal.

### `musicstreamer/__main__._run_oauth_helper(argv)`

New dispatch arm added to `main()` before `_run_gui`. Routes `--oauth-helper` flag to:
- `--self-test` in argv: returns 0 immediately (no QApplication, no window — D-05 headless smoke guard for build.ps1 exit 12)
- Normal path: strips `--oauth-helper` from argv, sets `sys.argv`, lazy-imports and calls `oauth_helper.main()` (which builds its own QApplication)

Pitfall 1 mitigated: `oauth_helper.main()` uses `argparse.parse_args()` (not `parse_known_args`), so `--oauth-helper` must be stripped before forwarding.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test 4 sys.modules injection instead of monkeypatch reload**

- **Found during:** Task 2 (TDD GREEN phase)
- **Issue:** The initial implementation of test 4 used `importlib.reload(musicstreamer.oauth_helper)` after patching `sys.exit`. However, `oauth_helper.py` calls `sys.exit(2)` at module scope when `PySide6.QtWebEngineWidgets` is absent (Linux CI), and the `from PySide6.QtCore import ...` line that follows the guarded `sys.exit` still raises `ModuleNotFoundError` even with `sys.exit` patched to a no-op, because `QtCore` *is* available but the reload context confuses the module state.
- **Fix:** Replace the reload approach with a `sys.modules` injection: a `types.ModuleType("musicstreamer.oauth_helper")` fake with a stub `main` function is inserted into `sys.modules` before calling `_run_oauth_helper`. This prevents `oauth_helper.py`'s module-level code from running entirely while still letting the lazy `from musicstreamer.oauth_helper import main` in `_run_oauth_helper` resolve to the stub.
- **Files modified:** tests/test_launcher_frozen_dispatch.py
- **Commit:** b58daf94
