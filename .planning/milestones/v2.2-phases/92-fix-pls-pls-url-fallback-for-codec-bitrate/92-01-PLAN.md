<objective>
Add URL-fallback to the PLS/M3U/XSPF codec and bitrate extractors so that when a
playlist's title field carries no codec/bitrate token (e.g. SomaFM prose titles),
the extractor inspects the entry's stream URL instead. Title-derived metadata still
wins when present (backwards-compatible). Closes the Phase 58 carry-over todo and
satisfies FIX-PLS-01.

Purpose: SomaFM and other ICY providers put descriptive prose in TitleN; their codec
and bitrate live only in the URL (groovesalad-256-mp3). Today both columns come up
blank on import. This populates them from the URL on a title miss.

Output: Extended _extract_codec / _extract_bitrate (new optional url param), a
delimiter-bounded URL bitrate regex, the 3 call sites updated to pass the entry URL,
and 4 new unit tests — written RED-first.
</objective>
