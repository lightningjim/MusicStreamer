<objective>
Close UAT Gap G1: the Windows installer leaves stale version-named `dist-info`
directories in `{app}\_internal` across upgrades, so at runtime
`importlib.metadata.version("musicstreamer")` resolves to the LOWEST version
present and the app mislabels itself (observed: reports 2.1.68 while running
2.2.86 code).

Root cause: `MusicStreamer.iss` relies solely on Inno Setup's AppId-triggered
auto-uninstall to remove the prior version. That removes shortcuts (already
covered by the existing `[InstallDelete]`) but is not removing the
version-named `_internal\musicstreamer-*.dist-info` directories, so they pile
up. `build.ps1` already defends the BUILD side (pre-bundle clean + post-bundle
singleton assertion `exit 9`). This plan adds the missing INSTALL-side analog.

Purpose: make every consumer keyed off the app version correct again —
`app.setApplicationVersion` (musicstreamer/__main__.py:211) and the User-Agent
strings in cover_art_mb.py and soma_import.py — by guaranteeing exactly one
dist-info survives an upgrade.

Output: a one-line `[InstallDelete]` extension in MusicStreamer.iss (plus an
extended safety comment) and a VM-only UAT verification row.
</objective>

Existing [InstallDelete] in packaging/windows/MusicStreamer.iss:

  ; WIN-02-A: Remove the previous version's Start-Menu .lnk shortcuts on upgrade.
  ; [InstallDelete] runs AFTER Inno auto-uninstalls the prior version (via AppId) but
  ; BEFORE [Icons] creates the new shortcuts. ...
  ; Scope is intentionally minimal: exactly the two .lnk paths — no wildcards, no
  ; filesandordirs, no paths under {userappdata}/{localappdata}/{userpf} data dirs.
  [InstallDelete]
  Type: files; Name: "{userprograms}\MusicStreamer.lnk"
  Type: files; Name: "{userprograms}\Uninstall MusicStreamer.lnk"

Key Inno Setup constants in play:
  {app}          -> {userpf}\MusicStreamer  (the install directory; install-managed, replaced every build)
  {userappdata}  -> %APPDATA%  (USER DATA — SQLite DB, cookies, tokens, EQ profiles; D-03 MUST survive)

[InstallDelete] timing: runs at the START of installation, before [Files] copies
the new bundle. So deleting old version-named dirs here leaves exactly one
freshly-copied musicstreamer-<X.Y.Z>.dist-info after [Files].
