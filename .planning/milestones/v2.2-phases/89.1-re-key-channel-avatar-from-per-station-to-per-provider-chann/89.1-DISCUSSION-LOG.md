## Existing avatar migration

| Option | Description | Selected |
|--------|-------------|----------|
| Backfill at db_init | One-time idempotent copy/rename of per-station PNG → {provider_id}.png + set providers.avatar_path; leave old files | |
| Lazy re-fetch per provider | Don't touch files; NULL on upgrade; re-fetch once on next play/edit | |
| Backfill + clean orphans | Backfill, then delete old per-station {station_id}.png files | ✓ |

**User's choice:** Backfill + clean orphans (→ D-01)
**Notes:** Preserves existing avatars with zero network re-fetch on upgrade and avoids stale duplicate files accumulating in channel-avatars/.

### Backfill conflict resolution

| Option | Description | Selected |
|--------|-------------|----------|
| Most recently updated | Sibling with newest updated_at (or highest station_id proxy) wins | ✓ |
| First / lowest station_id | Earliest-created sibling wins | |
| You decide | Planner picks cleanest given the query | |

**User's choice:** Most recently updated wins (→ D-02)
**Notes:** Reflects freshest fetch of the channel's current picture; siblings are normally identical anyway.

## Refresh button semantics

### Auto-fetch on open (provider already has avatar)

| Option | Description | Selected |
|--------|-------------|----------|
| Reuse, no network fetch | Show existing provider avatar immediately; auto-fetch only fires when provider has none | ✓ |
| Always re-fetch on open | Re-fetch every open, overwriting | |

**User's choice:** Reuse, no network fetch (→ D-07)
**Notes:** Core dedup goal — one fetch per channel, not per station.

| Option | Description | Selected |

**User's choice:** Refresh provider, update all siblings, with explicit shared-effect hint (→ D-08)
**Notes:** User should not be surprised that other stations of the channel changed.
