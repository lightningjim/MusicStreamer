**Test:** Open EditStationDialog on a YouTube station that already has a stored provider avatar (providers.avatar_path set, file present on disk). Confirm the avatar preview populates immediately from the cache and NO avatar fetch worker starts (no network request).

**Why human:** The D-07 gate is implemented at `edit_station_dialog.py:1319-1323` and wired correctly, but no automated test covers the "skip" path because the default test fixture has `provider_avatar_path=None`. The test `test_debounce_launches_avatar_worker_on_yt_url` always exercises the fetch path.

#### 2. D-08 Refresh Avatar End-to-End: Shared-Effect Hint

**Test:** With a YouTube station open in EditStationDialog, click the Refresh Avatar button. Observe that a fresh network fetch fires, the per-provider file is overwritten, and the status text shows the shared-effect hint naming the channel.

**Expected:** `_avatar_status` shows "Avatar saved — all {channel_name} stations updated"; a second sibling stream of the same YouTube channel (if any) shows the updated avatar in the now-playing panel without requiring its own dialog open.

**Why human:** `test_on_avatar_fetched_success_updates_preview_and_persists` tests `_on_avatar_fetched` directly (method-level) and confirms persist + hint. The full Refresh button click → `_force_avatar_refresh=True` → `_on_url_timer_timeout` → worker → `_on_avatar_fetched` chain is not tested end-to-end.

#### 3. Real-DB Upgrade: Backfill from Existing Per-Station PNGs

**Test:** Start the application with a database that contains stations with `channel_avatar_path` pointing to existing `assets/channel-avatars/{station_id}.png` files. Run the upgraded application (or call `db_init` on an upgraded build). Verify the migration:

**Expected:** `assets/channel-avatars/{provider_id}.png` created for each provider that had sibling stations with avatars; `providers.avatar_path` set in the DB; old `{station_id}.png` files deleted; avatar displays correctly in the now-playing panel using the new provider-keyed file; no network re-fetch occurs.

**Why human:** The backfill is tested with in-memory SQLite and `tmp_path` fixtures. A real upgrade involving production-layout disk files and a live SQLite database with multiple sibling streams (e.g., actual Lofi Girl streams) requires manual validation.
