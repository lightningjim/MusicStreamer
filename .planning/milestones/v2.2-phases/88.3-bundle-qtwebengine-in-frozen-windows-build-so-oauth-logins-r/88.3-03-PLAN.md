<objective>
Rewire the OAuth launch path for B1 and fix the Twitch UA. Phase 88.2's
_make_oauth_launch_args re-execs the conda main exe with `--oauth-helper`
(`sys.executable` IS MusicStreamer.exe when frozen) — which forces QtWebEngine
into the conda bundle. Under B1, WebEngine lives in a SEPARATE oauth_helper.exe
(built in 88.3-04, Inno-installed to `{app}\oauth_helper\oauth_helper.exe`), so
the frozen branch must launch THAT binary instead of re-exec'ing itself.

Also fix the carried Twitch follow-up: oauth_helper.py hardcodes a Linux UA
(`X11; Linux x86_64`), but on Windows Chromium emits `Sec-CH-UA-Platform:
"Windows"` which can't be spoofed by the UA string, so Twitch rejects the login.
Make _CHROME_UA platform-appropriate.

Purpose: make the conda main exe spawn the separate WebEngine helper, and make
Twitch login succeed on Windows. Output: B1 launch target + platform-aware UA.
</objective>
