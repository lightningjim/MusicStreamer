<objective>
Revert the stale same-bundle WebEngine changes shipped by 88.3-01. The conda
main bundle CANNOT host QtWebEngine (conda-forge ships zero PySide6 WebEngine
bindings; layering pip Addons over conda qt6-main causes the G6 DLL-load
failure). Under the spike-validated B1 architecture, WebEngine lives in a
SEPARATE oauth_helper.exe artifact (built in 88.3-04), never in the conda exe.

This plan removes every conda-side WebEngine assumption: spec hiddenimports,
the `--check-webengine` guard that imports WebEngine in the conda process,
build.ps1's step-0 pip preflight + step-4e bundle guard, and re-points the 5
drift-guard tests at B1 invariants.

Purpose: leave the conda artifact WebEngine-free so 88.3-04's separate helper
build is the only WebEngine carrier — eliminating the ABI conflict at its root.
Output: WebEngine-free conda spec, reverted __main__.py/build.ps1, B1 drift-guards.
</objective>
