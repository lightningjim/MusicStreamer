<objective>
Land all Linux-runnable source-text changes that bundle QtWebEngine into the frozen
Windows build and add a build-time guard so a missing WebEngine runtime fails the
BUILD, not the user. This covers fix_shape parts 2 (spec bundling), 3 (deepen guard),
4 (documentation), plus the Wave 0 drift-guard test file.

Purpose: Phase 88 UAT G6 — the oauth_helper subprocess crashes exit=2 because
QtWebEngine is neither in the build env nor collected by MusicStreamer.spec. The
main app deliberately never imports QtWebEngine (130MB-startup-avoidance decision)
and the only import (oauth_helper.py:62-71) is wrapped in try/except, so PyInstaller's
modulegraph never collects the runtime. Explicit hiddenimports fix it; a new
--check-webengine guard catches regressions at build time.

Output: New drift-guard test file + edited spec, __main__.py, build.ps1, README.
All changes are source-text and verifiable on Linux dev (no Windows build required).
The actual VM env provisioning + build + UAT smoke is Plan 02 (Wave 2).
</objective>
