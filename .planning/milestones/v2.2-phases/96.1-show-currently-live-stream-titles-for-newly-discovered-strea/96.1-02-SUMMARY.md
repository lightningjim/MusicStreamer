## Accomplishments

- `_BLANK_TITLES` frozenset + `_is_blank_title` predicate catches empty, whitespace-only, and any-case "Untitled" titles (D-01)
- `_fallback_title` derives `"Untitled — watch?v=<id>"` from URL query param via `urllib.parse` (D-03, em dash U+2014)
- `resolve_live_title` performs per-video yt-dlp `extract_info` without `extract_flat` or `playlist_items`, with `socket_timeout=10`, `temp_cookies_copy()`, and `build_js_runtimes(node_runtime)` (D-01/D-02/D-03)

## Decisions Made

- `resolve_live_title` omits `playlist_items` (unlike `fetch_channel_avatar`) because we need per-video title metadata, not channel-level thumbnails
- Exception catch is broad (`Exception`, not just `DownloadError`) to satisfy D-03 "any failure" requirement — covers timeout, network, or unexpected errors
- `_fallback_title` uses a function-local `urllib.parse` import to avoid adding a module-level import for a single small utility
