> Visual and interaction contract for the single new "Add a song" QPushButton in
> the now-playing panel GBS widget cluster.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** This is a PySide6/Qt Widgets desktop application, not a web
> app. There is no CSS, no shadcn, and no web design system. The UI surface is
> ONE new QPushButton reusing the existing GBS widget cluster. Nearly all
> decisions are locked by CONTEXT.md. This document pins the visual and
> interaction contract for that control.

## Spacing Scale

Qt Widgets uses pixel margins set via `setContentsMargins` and `setSpacing`.
The values below are observed from the GBS widget cluster in
`now_playing_panel.py` and matched for the new button.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Expiry widget vertical padding (`_expiry_layout.setContentsMargins(0,4,0,4)`) |
| sm | 4px | Vote row `setSpacing(4)` — sibling spacing |
| md | 0px | Default VBoxLayout content margin (center column uses addWidget with no extra margin) |

**Exceptions:** The GBS cluster does not use a normalized 8-point scale. It
uses the panel's default QVBoxLayout margins. The "Add a song" button must NOT
introduce custom margins that break the existing cluster rhythm. Use
`center.addWidget(self._gbs_add_btn)` with no explicit `setContentsMargins`
override on the button — Qt default margins apply, same as `_gbs_relogin_btn`
(line 717) and the vote buttons.

This is a Qt desktop app. "Typography" for the button is the system font Qt
applies to QPushButton by default. No custom font sizes or weights are set on
any GBS cluster control in the current codebase.

| Role | Size | Weight | Notes |
|------|------|--------|-------|
| Button label | System default (Qt) | Regular (system default) | Same as `_gbs_relogin_btn` ("Log in again") and the vote buttons ("1"–"5"). Do NOT set a custom font or size. |

**QLabel copy in the expiry widget** (sibling reference):
- Primary line ("GBS session expired"): system default size, regular weight
- Sub-line ("Log in again to keep the playlist updating"): system default size, regular weight

alongside the button must follow the same rule.

## Color

The panel uses no explicit color hex codes on GBS cluster buttons. Color is
fully delegated to the platform Qt style (Fusion on Linux Wayland). The "Add
a song" button must match this delegation — no `setStyleSheet` on
`_gbs_add_btn`. Applying a custom color would produce visual inconsistency with
every sibling control.

| Role | Value | Usage |
|------|-------|-------|
| Dominant | Qt platform surface color | Panel background — unchanged |
| Secondary | Qt platform button background | Button idle state — system default |
| Accent | Qt platform highlight color | Button :focus / :pressed — system default |
| Destructive | not applicable | No destructive action in this phase |

**Accent reserved for:** button :focus and :pressed pseudo-states (platform
default). No custom accent is introduced.

**Constraint:** Do NOT call `setFlat(True)` on `_gbs_add_btn`. The vote
buttons and `_gbs_relogin_btn` are not flat. The only flat button in the
cluster vicinity is `_similar_collapse_btn` (line 788), which is a collapse
toggle — a different affordance class. "Add a song" is a primary action
button; it should use the platform default raised appearance.

## Widget Placement Contract

### Insertion point

`now_playing_panel.py` center column (`QVBoxLayout`), immediately AFTER
`_gbs_expiry_widget` (line 722) and BEFORE `_gbs_vote_row` (line 767).

Exact order in center layout after this phase:

```
_gbs_playlist_widget   — line 695  (QListWidget, max 180px height)
_gbs_expiry_widget     — line 722  (Phase 87.1 expiry prompt)
_gbs_add_btn           — NEW       (QPushButton("Add a song"))
_gbs_vote_row          — line 767  (QHBoxLayout, 5 vote buttons)
```

Source: RESEARCH.md §Pattern 2, verified at `now_playing_panel.py:689-767`.

### Button construction

```python
self._gbs_add_btn = QPushButton("Add a song", self)
self._gbs_add_btn.setVisible(False)  # hidden-when-empty pattern
self._gbs_add_btn.clicked.connect(self._on_add_song_clicked)  # QA-05 bound method
self._gbs_add_btn.setToolTip("Add a song to the GBS.FM queue")
center.addWidget(self._gbs_add_btn)
```

**Label:** exactly `"Add a song"` — locked by CONTEXT.md D-06 / GBS-TOKEN-02.
**Tooltip:** exactly `"Add a song to the GBS.FM queue"` — does NOT contain the
word "token" per GBS-TOKEN-02. No alternative wording is permitted.

### No sizing overrides

The vote buttons set `setMinimumWidth(32)` and `setMaximumWidth(48)` because
they are compact score buttons. The "Add a song" button is a full-width action
button — do NOT set min/max width constraints. Let it expand to fill the
center column width, which is the natural behavior of `addWidget` in a
`QVBoxLayout`. This matches the `_gbs_relogin_btn` ("Log in again") which also
has no width constraints.

## Visibility and State Matrix

### Visibility predicate

The button follows the `should_show = is_gbs and logged_in` predicate already
used by `_gbs_playlist_widget` and the vote buttons in
`_refresh_gbs_visibility()` (`now_playing_panel.py:2962`).

```
is_gbs    = self._station is not None and self._station.provider_name == "GBS.FM"
logged_in = self._is_gbs_logged_in()
should_show = is_gbs and logged_in
```

| Condition | Button visible? | Notes |
|-----------|----------------|-------|
| No station bound | No | Default initial state |
| Non-GBS station bound | No | `is_gbs` is False |
| GBS station bound, NOT logged in | No | `logged_in` is False |
| GBS station bound AND logged in | **Yes** | Regardless of token count or queue state — D-05 |
| GBS session expired (expiry widget shown) | No | `logged_in` returns False when session expired; `_gbs_expiry_widget` is shown instead |
| Post-add (dialog closed, re-poll in flight) | **Yes** | Button persists after add — D-08 |

**Token count does NOT affect visibility.** The original REQUIREMENTS.md
GBS-TOKEN-01 text (`tokens==0 AND queue==0`) is superseded by CONTEXT.md D-05.
Do not add any `fetch_user_tokens()` call in the visibility path.

### In-flight / debounce state

While the `GBSSearchDialog` is open (`dlg.exec()` is blocking), the button is
effectively non-interactable because `exec()` blocks the event loop modal-style.
No explicit `setEnabled(False)` is needed during the dialog lifetime — the
modal dialog captures input.

For the network request itself (inside the dialog's `_GbsSubmitWorker`): the
existing dialog already disables its own submit button during the in-flight
request. The panel's "Add a song" button does not need a separate disabled
state for this case.

**Discretion default (CONTEXT.md):** Mirror existing dialog submit-button
behavior — no panel-level disabled state during submit. The dialog owns that
feedback.

### Post-add state

After `submission_completed` emits and `trigger_gbs_repoll()` fires:
- Button remains visible (D-08 — no hide-after-add).
- `_on_gbs_poll_tick()` is called; playlist widget updates to show the new song.
- No visual change on the button itself (no checkmark, no color change).

## Copywriting Contract

| Element | Copy | Constraint |
|---------|------|-----------|
| Button label | `Add a song` | Locked — GBS-TOKEN-02, CONTEXT.md D-06 |
| Tooltip | `Add a song to the GBS.FM queue` | No "token" word — GBS-TOKEN-02 |
| Success feedback | Server's `messages`-cookie text surfaced verbatim by the existing `GBSSearchDialog` inline | Not new copy — do NOT invent success copy on the panel |
| Limit / rejection feedback | Server's `messages`-cookie text surfaced verbatim by `GBSSearchDialog._on_submit_finished()` | Not new copy — server is truth (D-07) |
| Auth-expiry feedback | Existing dialog toast ("GBS.FM session expired — reconnect via Accounts") from `GBSSearchDialog._on_submit_error("auth_expired")` | Not new copy |

### Forbidden words (GBS-TOKEN-02)

The following word is banned in the button label, tooltip, and any surrounding
QLabel added to the GBS cluster as part of this phase:

- `token` (case-insensitive, any form: "token", "tokens", "Token")

The source-grep drift-guard (`test_gbs_token_drift_guard.py`) enforces this on
the `add_song_zero_token()` function body string literals and the new widget
construction code.

### Empty / no-data state

There is no empty state for this button. The button appears only when GBS is
bound and logged in; in that state, the `GBSSearchDialog` always has the
ability to search. There is no "nothing to add" state requiring copy.

### Error states — handled by existing dialog

| Error | Where surfaced | New copy needed? |
|-------|---------------|-----------------|
| Network error on submit | `GBSSearchDialog` inline error area | No |
| Song already queued | `GBSSearchDialog` inline — server messages cookie text | No |
| Zero-token limit (one at a time) | `GBSSearchDialog` inline — server messages cookie text | No |
| Auth expired during add | `GBSSearchDialog` toast + login gate refresh | No |

All error UX is owned by the existing dialog and the existing expiry prompt.
This phase introduces no new error display widgets.

## Accessibility Contract

- The button is a standard `QPushButton` — Qt provides default keyboard focus
  behavior (Tab navigation, Space/Enter activation).
- `setToolTip("Add a song to the GBS.FM queue")` satisfies Qt's tooltip
  accessibility mechanism. No additional `setAccessibleName` is required unless
  the checker flags it; the button label text is already descriptive.
- The button must be in the natural tab order of the GBS cluster (after the
  expiry widget's "Log in again" button, before the vote buttons). Do NOT call
  `setFocusPolicy(Qt.NoFocus)` or `setTabOrder` overrides.
