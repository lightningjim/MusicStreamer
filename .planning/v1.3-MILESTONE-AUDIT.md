### Integration Gap: DISC-03 Preview Close (cosmetic)

`discovery_dialog.py` `_on_close_request` (~line 416) calls `player.stop()` directly when `_prior_station is None`, bypassing `main_window._stop()`. When a preview is playing and the dialog is closed with nothing previously playing, the title label and stop button remain in a stale state. Data is unaffected; clicking any station resolves it immediately.

### Notable Findings (non-blocking)

**FAVES-01 conditional path:** Star button only reveals after `fetch_cover_art` iTunes callback fires. Stations with no ICY metadata never trigger `_on_title`, so star remains hidden for non-ICY stations. Intended per spec ("gated on non-junk ICY title").

**Phase 14→15 ImportDialog backward compat:** YouTube tab fully preserved in unified ImportDialog. All handlers isolated from AudioAddict tab. Confirmed compatible.

**Dedup consistency:** All three import paths (discovery save, `yt_import.import_stations`, `aa_import.import_stations`) call `repo.station_exists_by_url` before insert. Consistent across all surfaces.
