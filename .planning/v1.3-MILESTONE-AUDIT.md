### Integration Defect: reload_list not view-mode-aware

**Severity:** Low (degraded UX, not crash or data loss)  
**File:** `musicstreamer/ui/main_window.py:810`  
**Impact:** If user is in Favorites view when an import or discovery save completes, `reload_list()` is called but `_render_list()` immediately returns early (correct guard exists inside). Station list is not visually updated. User must switch to Stations view manually.  
**Fix:** Add `if self._view_mode == "stations":` guard before calling `_render_list()` in `reload_list()`, or switch view mode automatically after import.
