# Phase 40.1 Plan 01: YouTube Import Fix Summary

Restored YouTube playlist import by deleting an incorrect secondary `is_live` filter in `_YtScanWorker.run()` that was silently dropping every scan result.

## Bug

`yt_import.scan_playlist()` already filters to live entries via `_entry_is_live()` and returns dicts with only `title`/`url`/`provider` keys — no `is_live` field. The worker then applied:

```python
live = [e for e in results if e.get("is_live") is True]
```

Since `is_live` was never present, this comprehension always produced `[]`, leaving the import checklist empty regardless of playlist contents.

## Fix

Removed the comprehension. Worker now emits `scan_playlist` results directly:

```python
def run(self):
    try:
        results = yt_import.scan_playlist(self._url)
        self.finished.emit(results)
    except Exception as exc:
        self.error.emit(str(exc))
```

Commit: `7a282a2`

## Test

Added `test_yt_scan_passes_through` (commit `29b5eb5`) covering the exact failure mode:

- Monkeypatches `scan_playlist` to return dicts without `is_live` keys (mirrors real output shape)
- Runs a real `_YtScanWorker` thread and waits on its `finished` signal via `qtbot.waitSignal`
- Asserts `_yt_list.count() == 2` and that both titles populate the checklist

Before fix: fails with `0 == 2`. After fix: passes.
