## Fixed Issues

### WR-01 / WR-02 / WR-03: `_LogoFetchWorker` lifecycle and tmp-file cleanup

**Files modified:** `musicstreamer/ui_qt/edit_station_dialog.py`
**Commit:** `bdca8ae`
**Applied fix:** Adopted the monotonic fetch-token pattern that mirrors
`NowPlayingPanel._cover_fetch_token`:

- `_LogoFetchWorker.finished` signal changed from `Signal(str)` to
  `Signal(str, int)` and constructor now takes `(url, token, parent)`.
- `_on_url_timer_timeout` increments `self._logo_fetch_token` and passes
  it into each worker. No disconnect is performed; stale emissions are
  dispatched to `_on_logo_fetched` which discards and unlinks them.
- `_on_logo_fetched(tmp_path, token=0)` now (a) unlinks stale-token tmp
  files (WR-01), (b) wraps the consumption in `try/finally` so tmp files
  are unlinked even if `copy_asset_for_station` raises (WR-03), and (c)
  defensively unlinks truthy-but-missing tmp_path paths.
- Added `_shutdown_logo_fetch_worker` helper called from `closeEvent` and
  `reject` that disconnects and waits up to 2s for the in-flight worker,
  preventing `QThread: Destroyed while running` warnings and allowing
  tmp-file cleanup (WR-02).
- The default `token=0` in the slot signature preserves backward
  compatibility with direct-call tests (e.g. `dialog._on_logo_fetched(path)`).

Incidentally resolves IN-01 (manual fetch race) via the same token scheme.

### WR-04: `_on_cover_art_ready` crashes event loop on non-numeric token

**Files modified:** `musicstreamer/ui_qt/now_playing_panel.py`
**Commit:** `afc6343`
**Applied fix:** Wrapped `int(token_str)` in `try/except ValueError: return`
so a malformed payload is silently dropped instead of propagating out of a
Qt queued-connection slot.
