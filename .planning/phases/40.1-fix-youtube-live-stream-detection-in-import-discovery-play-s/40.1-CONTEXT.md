**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Targeted bug-fix and polish sweep folding the original Phase 40.1 scope (3 items) plus the entirety of the original Phase 45 scope (2 items). All five issues are regressions or oversights in the v2.0 Qt port surfaced during daily use and Phase 39 UAT.

**Five fixes in scope:**
1. **YT import** — `_YtScanWorker` double-filters live entries; every live stream gets dropped, user sees "No live streams found" on valid playlists.
2. **Discovery icons** — Play/Stop text buttons → icons in `DiscoveryDialog` results table.
3. **AA import feedback** — `aa_import.import_stations_multi()` returns `(imported, skipped)` but UI only shows imported count; user sees "Imported 0 channels" with no signal that everything was deduped.
4. **Station logos broken** — Defined logos don't render in the station list or now-playing panel; placeholder/note shows for every station. EditStationDialog also missing logo upload AND auto-fetch on URL paste (both worked in v1.5).
5. **ICY disable doesn't suppress** — Toggling "Disable ICY" on a station has no effect; iTunes cover art still pulls and ICY track titles still update.

Out of scope (explicit cut-lines — DO NOT PULL FORWARD):
- Platform media keys → Phase 41 (MEDIA-01..05)
- Settings export/import → Phase 42 (SYNC-01..05)
- GStreamer Windows spike → Phase 43
- Windows installer → Phase 44

</domain>

<decisions>
## Implementation Decisions

### YT Import Fix
- **D-01:** Remove the `_YtScanWorker.run()` post-filter at `musicstreamer/ui_qt/import_dialog.py:64` (`live = [e for e in results if e.get("is_live") is True]`). `yt_import.scan_playlist()` already filters via `_entry_is_live()` — the worker's re-filter drops every entry because the returned dicts carry no `is_live` key. Pass `results` straight through to `finished.emit`.
- **D-02:** Add a regression test in `tests/test_import_dialog.py` (or sibling) that monkeypatches `yt_import.scan_playlist` to return entries without an `is_live` key and asserts those entries reach `_on_yt_scan_complete`.

### Discovery Icons
- **D-03:** Single toggle button per row in the `Play` column. Icon swaps between play and stop based on `_previewing_row` state. Removes any need for a separate `Stop` column.
- **D-04:** Use `QIcon.fromTheme("media-playback-start", QIcon(":/icons/media-playback-start.svg"))` and `"media-playback-stop"` per Phase 36 pattern. Bundle SVG fallbacks via `icons.qrc`; regenerate `icons_rc.py`.
- **D-05:** Drop button text — set the icon only via `QPushButton.setIcon()`. Keep accessible name set via `setAccessibleName("Play preview")` / `"Stop preview"` for screen readers.
- **D-06:** Save column stays as text "Save" button (not in scope).

### AA Import Feedback (and YT for parity)
- **D-07:** `_on_aa_import_complete` shows BOTH a toast AND an inline label: `"Imported {N} new, {M} skipped (already in library)"`. Inline label persists in the dialog; toast surfaces it briefly.
- **D-08:** `_AaImportWorker.finished` signal must change from `Signal(int)` to `Signal(int, int)` carrying `(imported, skipped)`. Same change for `_YtImportWorker.finished` to enable matching YouTube feedback.
- **D-09:** YouTube import gets the same detailed summary in `_on_yt_import_complete` for consistency: `"Imported {N} new, {M} skipped (already in library)"`. Both as toast and inline status label.
- **D-10:** When `M == 0` and `N > 0`: show `"Imported {N} new"` (drop the skip clause). When `N == 0` and `M > 0`: show `"All {M} already in library"` (no "imported" framing). Avoid the awkward "Imported 0 new, 12 skipped" wording.

### Station Logos
- **D-11:** Investigate why defined logos don't render — likely a missing `QPixmap` load path in `now_playing_panel.py` and `station_list_panel.py` after the GTK→Qt migration. Logo paths are stored on `Station.station_art_path` (still populated by `assets.copy_asset_for_station()`); the bug is on the Qt render side, not data.
- **D-12:** EditStationDialog gets a logo input row above the form. Two controls side-by-side:
  - **"Choose File…"** button → `QFileDialog.getOpenFileName()` filtered to `*.png *.jpg *.jpeg *.webp *.svg`. Picked file is copied via `assets.copy_asset_for_station()` (existing helper) and `station.station_art_path` is updated.
  - **"Auto-fetch from URL"** behavior — re-trigger the v1.5 daemon-thread fetch pattern when the URL field changes. YouTube URL → `yt-dlp` thumbnail; AudioAddict URL → `_normalize_aa_image_url` pattern. Match Phase 39 D-07.
- **D-13:** Show a small (e.g., 64×64) preview pixmap of the current logo next to the picker buttons so user can see what's set.
- **D-14:** Both the file-picker path and the auto-fetch path go through `assets.copy_asset_for_station()` so storage location is consistent.

### ICY Disable Toggle
- **D-15:** When `station.icy_disabled` is True:
  - `Player` must NOT route ICY tag bus messages to the title signal (suppress `track_title_changed.emit`)
  - Track title slot shows the station name (existing fallback)
  - Cover art slot shows the station logo (existing fallback) — NO iTunes lookup is fired
- **D-16:** Investigate the actual gap — likely either: (a) the toggle isn't being persisted to the DB, (b) `Player` reads `station.icy_disabled` once at play-start and not on tag bus messages, or (c) the cover art module isn't checking the flag at all. Planner/researcher to root-cause.
- **D-17:** Regression test: monkeypatch a station with `icy_disabled=True`, simulate an ICY TAG bus message, assert (a) no `track_title_changed` emission, (b) no iTunes API call.
