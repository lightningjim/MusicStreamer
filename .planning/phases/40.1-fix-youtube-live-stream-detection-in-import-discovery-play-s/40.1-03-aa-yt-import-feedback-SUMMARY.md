# Phase 40.1 Plan 03: AA/YT Import Feedback Summary

One-liner: Replace misleading "Imported 0 channels" wording with context-sensitive (imported, skipped) summary rendered as both an inline QLabel and a toast, with AA/YT parity driven by a shared `_format_import_summary` helper.

## What Was Built

- Added module-level `_format_import_summary(imported, skipped)` in `musicstreamer/ui_qt/import_dialog.py` implementing D-10 wording rules:
  - `(N, 0)` → "Imported N new"
  - `(0, M)` → "All M already in library"
  - `(N, M)` → "Imported N new, M skipped (already in library)"
- Changed `_AaImportWorker.finished` and `_YtImportWorker.finished` from `Signal(int)` to `Signal(int, int)`. Both `run()` bodies now tuple-unpack the backend's `(imported, skipped)` result before emitting.
- Updated `_on_aa_import_complete(imported, skipped)` and `_on_yt_import_complete(imported, skipped)` to format the summary and render it to BOTH the tab's existing status QLabel (`self._aa_status` / `self._yt_status`) and the toast — achieving AA/YT parity (D-07, D-09).
- Handler connections (`worker.finished.connect(self._on_*_import_complete)`) left unchanged — PySide6 auto-splats positional signal args.
