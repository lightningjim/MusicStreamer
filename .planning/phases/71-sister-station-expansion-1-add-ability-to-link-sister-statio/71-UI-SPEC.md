# Phase 71 — UI Design Contract

> Visual and interaction contract for the Sister Station Expansion (manual sibling linking) in MusicStreamer. Generated by gsd-ui-researcher 2026-05-12. Consumed by gsd-ui-checker, gsd-planner, gsd-executor, gsd-ui-auditor.

This is a **native Qt application** — vocabulary throughout is Qt-idiomatic (QLabel, QPushButton, QComboBox, QListWidget, QDialog, FlowLayout, QPalette, QSS). There is no web frontend, no shadcn, no Tailwind. The "design system" is the Phase 66 theme architecture (preset QPalette palettes + user-editable Custom) layered with the Phase 59 accent override. Phase 71 introduces **three new visual elements**: (1) the sibling chip row in `EditStationDialog`, (2) the `AddSiblingDialog` two-step picker modal, and (3) no new NowPlaying widget — the existing `_sibling_label` gains merged content, no structural change.

## Typography

Phase 71 introduces **no new font sizes or weights**. All text inherits the surrounding widget's font (Qt platform default ≈ 9–10 pt Normal on Linux DPR=1.0).

| Role | Size | Weight | Phase 71 Usage |
|------|------|--------|----------------|
| Chip label (sibling name) | inherits Qt platform default (≈ 9–10 pt) | Normal | Sibling chip button text in `EditStationDialog` chip row; same as existing tag chips at `edit_station_dialog.py:657-665` |

## Color

Phase 71 uses **only QPalette tokens and the existing `_theme.py` constants**. **No new hex literals.**

### Palette role usage

| Role | QPalette source | Phase 71 Usage |
|------|-----------------|----------------|
| Dominant (60% surface) | `QPalette.Base` / `QPalette.Window` | Inherited from parent widgets — unchanged from Phase 66. `AddSiblingDialog` background = `Window`. |
| Secondary (30% chrome) | `QPalette.Button` / `QPalette.AlternateBase` | QListWidget alternating rows via `AlternateBase`; `QComboBox` dropdown via `Button`. |
| Accent (10%) — chip selected fill | `QPalette.Highlight` (`palette(highlight)`) | The `+ Add sibling` chip has no selected state; no accent needed for it. Manual sibling chips: unselected only (chips are not toggleable — they represent linked items). Accent NOT used for Phase 71 chips. |
| Accent text | `QPalette.HighlightedText` | Only applies if a chip is in selected state (see above — chips are not toggleable; accent color is effectively unused in Phase 71 chip row). |
| Chip unselected fill | `QPalette.Base` (`palette(base)`) | Background of all sibling chips and the `+ Add sibling` button in their default (non-interactive) state — inherits `_CHIP_QSS` unselected rule at `edit_station_dialog.py:190-195` |
| Chip border | `QPalette.Mid` (`palette(mid)`) | Border of all sibling chips — inherits `_CHIP_QSS` unselected rule `border: 1px solid palette(mid)` |
| Text (chip label) | `QPalette.Text` (inherited from parent) | Station names and `×` glyph inside chips |
| Destructive (unlink action) | **none — no color change** | The `×` button within a manual chip uses the same chip border/text palette as the chip name. The unlink action is NOT rendered in `ERROR_COLOR_HEX`. Rationale: unlink is reversible (user can re-link), so destructive-red would be misleading. The `×` glyph is the sole affordance; color escalation is unnecessary. |
| AA chip visual distinction | Same `palette(base)` / `palette(mid)` / `palette(text)` as manual chips | AA auto-detected chips are visually identical to manual chips EXCEPT they have no `×` button. This is the **only** visual difference (D-15 contract). No tint, no italic, no opacity change. Rationale: simplicity wins — the absence of `×` is unambiguous without adding visual noise. |

**Accent is NOT reserved for any new Phase 71 element.** Chips in the sibling row are not toggleable selectors; they are display-only items with one action (`×`). No accent needed.

**`QListWidget` in `AddSiblingDialog` selection:** uses the native Qt selected-row highlight which draws `palette(highlight)` background + `palette(highlighted-text)` text. This is the standard `QListWidget` behavior — no override needed.

### Theme compatibility

All 7 Phase 66 preset themes inherit the `palette(base)` / `palette(mid)` / `palette(highlight)` / `palette(highlighted-text)` contrast guarantees that Phase 68/70 verified. Phase 71 reuses the identical QSS tokens from `_CHIP_QSS`. No new theme-specific mitigation needed.

### NowPlayingPanel (no new copy — unchanged surface)

The existing `_sibling_label` rendering via `render_sibling_html(siblings, current_name)` is unchanged. The merged output of AA auto-detected + manual siblings continues to use the established "Also on: NetworkA • NetworkB" format (Phase 51 D-07, D-08). Phase 71 adds no new copy to the NowPlaying surface.

### Toast notifications

| Trigger | Copy | Duration |
|---------|------|----------|
| Manual sibling link added successfully | `f"Linked to {other_station_name}"` | 3000 ms (default `show_toast` duration) |
| Manual sibling link removed successfully | `f"Unlinked from {other_station_name}"` | 3000 ms |

**Toast emit point:** `EditStationDialog._refresh_siblings()` or the `×` click slot calls `Repo.add_sibling_link` / `Repo.remove_sibling_link`, then emits a new `Signal(str)` (e.g. `sibling_toast = Signal(str)`) that `MainWindow` routes to `show_toast(text)`. Alternatively: `EditStationDialog` accepts a `toast_callback: Callable[[str], None]` kwarg at construction and calls it directly. Planner picks; bound-method (QA-05) preferred.

**Station name truncation in toast:** if `other_station_name` exceeds 40 characters, use `other_station_name[:37] + "…"` — same convention as any future long-name case. Most station names are under 40 characters in practice.

### No destructive confirmation dialog

Unlinking is **reversible** (user can re-add the link via `+ Add sibling`). Therefore, clicking `×` does NOT open a `QMessageBox.question` confirmation. The action fires immediately on click. This matches Phase 51 D-11 precedent where the "discard" path prompts, but here the action is undo-able through a simple re-link, not a permanent data loss.

### 2. `AddSiblingDialog` (two-step picker modal)

| Property | Value |
|----------|-------|
| Class | `AddSiblingDialog(QDialog)` in new file `musicstreamer/ui_qt/add_sibling_dialog.py` |
| Modal | `setModal(True)` — always shown as application-modal dialog |
| Window title | `"Add Sibling Station"` |

| Resize behavior | Resizable vertically (station list stretches to fill available height). Minimum width 480 px enforced. |
| Outer layout | `QVBoxLayout` with `setContentsMargins(16, 16, 16, 16)` and `setSpacing(8)` |

**Internal layout (top to bottom):**

```
┌─────────────────────────────────────────────┐
│  Provider:  [QComboBox ─────────────────▼]  │  ← provider_row (QFormLayout or QHBoxLayout)
│  Station:   [QLineEdit filter───────────]   │  ← search_row
│  ┌──────────────────────────────────────┐   │
│  │  QListWidget                         │   │  ← station_list (stretch=1)
│  │  Station A                           │   │
│  │  Station B  ← selected              │   │
│  │  Station C                           │   │
│  └──────────────────────────────────────┘   │
│  [Cancel]                  [Link Station]   │  ← QDialogButtonBox
└─────────────────────────────────────────────┘
```

| Widget | Details |
|--------|---------|
| `_provider_combo` (`QComboBox`) | Populated from `Repo.list_providers()`. Default = editing station's `provider_id`. Items sorted alphabetically by provider name for easy scanning. `currentIndexChanged` signal connected to `_on_provider_changed` (bound method, QA-05). |
| `_search_edit` (`QLineEdit`) | Placeholder: `"Filter stations…"`. `textChanged` signal connected to `_on_search_changed` (bound method). Live-filter-as-you-type against the current provider's station list. No debounce — the station list is in-memory (no network calls). |
| `_station_list` (`QListWidget`) | `setSelectionMode(QAbstractItemView.SingleSelection)`. Populated by `_repopulate_station_list()`. Each item: `QListWidgetItem(station_name)` with `setData(Qt.UserRole, station_id)`. `itemSelectionChanged` signal connected to `_on_selection_changed` (bound method) to enable/disable the OK button. `itemDoubleClicked` connected to `_on_accept` (bound method) — double-click equivalent to clicking OK. |
| `_button_box` (`QDialogButtonBox`) | `QDialogButtonBox(QDialogButtonBox.Ok \| QDialogButtonBox.Cancel)`. OK button label overridden to `"Link Station"` via `button_box.button(QDialogButtonBox.Ok).setText("Link Station")`. OK disabled by default; enabled when exactly one item is selected in `_station_list`. `accepted.connect(self._on_accept)` (bound method). `rejected.connect(self.reject)`. |

**Provider QComboBox rationale:** A native `QComboBox` is sufficient for the typical provider count of 5–15. A left-pane `QListWidget` alternative would consume too much horizontal space at 480 px minimum width. The QComboBox selects a provider, immediately filters the station list below — this is the same two-step pattern described in D-12.

**Station list filtering logic (`_on_search_changed` + `_on_provider_changed`):**

Exclusion rules applied in `_repopulate_station_list()`:
1. Exclude `current_station.id` (self).
2. Exclude all station IDs already linked (both AA auto-detected and manual) — obtained by merging `find_aa_siblings(all_stations, current_id, current_url)` + `Repo.list_sibling_links(current_id)` into a set of excluded IDs.
3. Filter by selected provider (provider match on `station.provider_name`).
4. Apply `_search_edit` text: case-insensitive substring match on `station.name` using `str.lower()` — no regex, no external library.

**Empty-state rendering in `_station_list`:**

When the filtered list is empty, clear the `QListWidget` and add one non-selectable item:
- All linked already: `QListWidgetItem("All stations in this provider are already linked.")`
- No stations at all: `QListWidgetItem("No other stations found for this provider.")`

Both items use `item.setFlags(Qt.NoItemFlags)` to make them non-selectable and visually muted. The OK button remains disabled.

**`_on_accept()` slot:**

1. Reads selected `station_id` from `_station_list.currentItem().data(Qt.UserRole)`.
2. Calls `self._repo.add_sibling_link(self._current_station_id, station_id)`.
3. Stores `self._linked_station_name = selected_item.text()` (for toast copy, readable by the caller after `exec()`).
4. Calls `self.accept()`.

The caller (`EditStationDialog`) reads `dialog._linked_station_name` on `Accepted` result to fire the toast.
