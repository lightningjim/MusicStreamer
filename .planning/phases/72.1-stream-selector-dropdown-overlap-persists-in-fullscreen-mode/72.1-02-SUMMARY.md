## Decisions Made

- **Predicate `isHidden()` not `isVisible()`** for the D-02 single-stream branch in `_reflow_stream_picker_row`. Documented inline in the source. Rationale: Qt's `isVisible()` returns False on any widget whose top-level ancestor has not been shown, regardless of explicit setVisible state. Plan 01's deliberate omission of `panel.show()` in the test fixture made the plan's `isVisible()` predicate unworkable; `isHidden()` gives the intended-visibility semantics the plan was reaching for.
- **Threshold includes outer chrome.** `_picker_row_min_width` walks both `self.controls` (row 1) AND `self.layout()` (outer QHBoxLayout = logo + center + cover). Required because `panel.width()` (what `resizeEvent` sees) includes the entire panel, not just the center column where row 1 lives.
- **`# noqa: N802` marker on `resizeEvent`** — Qt method names are camelCase; project lint expects snake_case. The marker codifies the documented Qt-override exception (Pattern 5 in 72.1-RESEARCH lines 360-391, Pattern D in 72.1-PATTERNS).
- **Method placement: cluster near picker-related methods.** `resizeEvent` immediately before `_populate_stream_picker` (the visibility writer); 4 helpers immediately after `_sync_stream_picker`. Spatial cohesion keeps the stream-picker concern in one neighborhood of the class body.
- **`addWidget` for row-2 entry, `insertWidget(snapshot_index, ...)` for row-1-return.** Row 2 only ever has the picker (no ordering concern); row 1 has 8 siblings (transport, picker, secondary cluster, compact-toggle, stretch) where ordering is the visual contract.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Plan predicate `isVisible()` replaced with `isHidden()`**
- **Found during:** Task 3 (running test 1 after methods landed)
- **Issue:** Plan body specified `if not self.stream_combo.isVisible():` for the D-02 single-stream early-return. But Qt's `isVisible()` returns False whenever the widget's ancestor hierarchy is unshown — and Plan 01's `_make_panel` fixture deliberately omits `panel.show()`. For multi-stream stations the picker's `setVisible(True)` was honored by `_populate_stream_picker`, but `isVisible()` still returned False, causing the reflow predicate to misclassify multi-stream panels as single-stream (returned early to row 1 without reaching the width branch). Tests 1, 2, 5 failed.
- **Fix:** Replaced `if not self.stream_combo.isVisible():` with `if self.stream_combo.isHidden():`. `isHidden()` reflects only the explicit-setVisible-False call (which `_populate_stream_picker` writes when `len(streams) <= 1`), independent of hierarchy show state. Comment block in the method body documents the rationale.
- **Files modified:** musicstreamer/ui_qt/now_playing_panel.py (in `_reflow_stream_picker_row`)
- **Verification:** Tests 1, 2, 3, 4, 5, 8, neg-1, neg-2 all GREEN after the change.
- **Committed in:** `bf4dfcb` (Task 3 commit)

**2. [Rule 1 - Bug] `_picker_row_min_width` extended to walk OUTER layout**
- **Found during:** Task 3 (running test 2 after isHidden fix)
- **Issue:** Plan body specified threshold = sum of row 1 widgets + spacing + margins + 16. Initial implementation computed 524px. But the test asserts the picker is in row 2 at `panel.resize(560, 800)`. Since `560 > 524`, the predicate evaluated should-wrap = False → picker stayed in row 1 → test 2 failed. Root cause: `panel.width()` (what `resizeEvent` sees and what the predicate compares against) includes the entire panel — the outer layout is `logo(180) + outer.spacing(24) + center + outer.spacing(24) + cover(160) + outer.margins(16+16) = 420 + center`. At panel.width()=560, center.width()=140 which cannot hold row 1's 524px content. The threshold needed to be panel-relative, not center-relative.
- **Fix:** Extended `_picker_row_min_width` to also walk `self.layout()` (the outer QHBoxLayout) and accumulate the minimum widths of every outer-layout widget (logo + cover) plus outer margins + outer spacing for each gap. New threshold sits around 944px (524 + 420), so panel.width()=560 triggers wrap and panel.width()=1200 doesn't.
- **Files modified:** musicstreamer/ui_qt/now_playing_panel.py (in `_picker_row_min_width`)
- **Verification:** Tests 1 (wide 1200 → row 1), 2 (narrow 560 → row 2), 5 (round-trip) all GREEN. Threshold value verified via direct runtime print.
- **Committed in:** `bf4dfcb` (Task 3 commit)

**3. [Rule 1 - Bug] Test fixture `_make_panel` missing `.show()` + `qtbot.waitExposed`**
- **Found during:** Task 3 (running test 1 after methods landed and isHidden fix applied)
- **Issue:** Test 1 asserts `panel.stream_combo.isVisible()` returns True after `bind_station` with a multi-stream station. But Qt's `isVisible()` requires the widget's top-level ancestor to be shown. Plan 01's `_make_panel` only calls `qtbot.addWidget(panel)` (which does NOT show); test 1 hit the second assertion at the `isVisible()` check and failed. Plan 01 never observed this because it stopped at the first RED-state AttributeError on `panel.controls`.
- **Fix:** Added `panel.show()` + `qtbot.waitExposed(panel)` in `_make_panel` after `qtbot.addWidget(panel)`. Mirrors the precedent in `tests/test_phase72_integration.py:86-88` which the plan body explicitly cited.
- **Files modified:** tests/test_phase72_1_stream_picker_reflow.py (in `_make_panel`)
- **Verification:** All 7 target tests (1, 2, 3, 4, 5, neg-1, neg-2) GREEN; tests 3 and 4 still correctly assert `isVisible() is False` for single-stream (because setVisible(False) is honored even after show + waitExposed).
