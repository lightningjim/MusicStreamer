All three plan-time threats in the LAYOUT-02 register have been verified CLOSED in
the implemented code at `musicstreamer/ui_qt/now_playing_panel.py`. No unregistered
attack surface was introduced.

## Boundary Check (Defense-in-Depth — Not a Mitigation Substitute)

The phase scope is intentionally tight: a single widget class in a single file
(`musicstreamer/ui_qt/now_playing_panel.py`). The baseline-diff confirms:
- No new files in `musicstreamer/` (the only production directory).
- No new imports beyond `QSizePolicy` (a layout class, no I/O surface).
- No new signal/slot connections (the `resizeEvent` override is a Qt method
  override, not a `connect()` call — verified by `grep -c "lambda" ... reflow methods` == 0).
- No new event handlers beyond the `resizeEvent` Qt override (which itself
  delegates only to `super().resizeEvent(event)` and `self._reflow_stream_picker_row()`).
- No SQLite writes added (verified by test 9 `test_no_repo_set_setting_for_wrap_state` GREEN —
  locked at AST level).
- No network I/O, no IPC, no DBus, no subprocess spawn, no file system access
  added by any of the 5 new methods (resizeEvent / _reflow_stream_picker_row /
  _picker_row_min_width / _move_stream_picker_to / _set_picker_size_policy).

The plan-time trust-boundary table ("(none introduced)") accurately reflects the
implemented code.
