# Phase 72.1 — UI Design Contract (Delta Supplement)

> This is a **delta** UI-SPEC layered on top of Phase 72's parent contract. Read Phase 72's UI-SPEC first. Sections marked **Inherited** carry forward unchanged; only the **Delta** sections below describe new visual contract for Phase 72.1.
>
> **Scope:** ONE behavior — reparent the `QComboBox` stream picker between row 1 of the now-playing controls and a new row 2 below it, driven by panel width, for multi-stream stations only. Picker stretches on row 2; reverts to 140px-min Preferred on row 1. Single-stream stations and wide windows are unaffected.

### Picker Size Policy Table

The picker's `QSizePolicy.horizontalPolicy()` changes based on which row it lives in. This is the ONLY widget property the reparent helper toggles (besides layout membership).

| State | Parent Layout | `setSizePolicy().horizontal` | Effective Width | Source |
|-------|---------------|------------------------------|-----------------|--------|
| Multi-stream + wide (panel ≥ wrap threshold) | row 1 (`controls`) | `QSizePolicy.Preferred` (Qt default for `QComboBox`) | ≥ 140px (existing `setMinimumWidth(140)` floor), grows to content `sizeHint()` | CONTEXT D-08; preserves Phase 39 D-19..D-22 sizing |
| Multi-stream + narrow (panel < wrap threshold) | row 2 (`controls_row2`) | `QSizePolicy.Expanding` | Full row 2 width minus `center` margins (no other widgets in row 2 to share width with) | CONTEXT D-07 |
| Single-stream + any width | row 1 (`controls`) | Preferred — but picker is `setVisible(False)` so width is collapsed to 0 by Qt | 0px (hidden widget contributes no width per Qt `QHBoxLayout` behavior) | Existing Phase 39 gate at `now_playing_panel.py:1135`; CONTEXT D-02 |
| Mid-resize transition (single resize event at exactly the threshold) | unchanged from prior tick until reparent fires | unchanged | unchanged | RESEARCH — `_reflow_stream_picker_row` is idempotent; only state-changes trigger a reparent |

Notes:
- The 140px `setMinimumWidth(140)` from `now_playing_panel.py:485` is NOT removed; it stays as the row-1 floor. On row 2, `Expanding` overrides it upward, so the minimum is irrelevant in practice on row 2 (the picker grows past 140px).
- Size policy is restored on every reparent, not just first reparent — guarantees idempotency if a stale policy is ever observed.

### State Truth Table (the four combinations)

| Stream Count | Panel Width | Row 1 Contents | Row 2 Contents | Row 2 Visible (visually) | Picker Visible | Picker In Row |
|--------------|-------------|----------------|----------------|--------------------------|----------------|---------------|
| Multi-stream (`len(streams) > 1`) | Wide (≥ threshold) | play, stop, edit, **stream_combo**, star, eq, volume, compact, stretch | (empty) | No (zero-height collapsed `QHBoxLayout`) | Yes | 1 |
| Multi-stream | Narrow (< threshold) | play, stop, edit, star, eq, volume, compact, stretch | **stream_combo** (Expanding) | Yes | Yes | 2 |
| Single-stream (`len(streams) ≤ 1`) | Wide | play, stop, edit, (stream_combo hidden, 0px), star, eq, volume, compact, stretch | (empty) | No | No | 1 |
| Single-stream | Narrow | play, stop, edit, (stream_combo hidden, 0px), star, eq, volume, compact, stretch | (empty) | No | No | 1 |

**Invariants enforced by this table:**
- Row 1's *non-picker* widgets are always in the same order, always in row 1. (D-03)
- The picker is the only widget that moves. (D-03)
- Single-stream stations never trigger reparent, regardless of width. (D-02)
- Row 2 is only visually present when it owns the picker (i.e., multi-stream + narrow). (D-05)

|----------|-------|--------|
| Animation | None — instant | CONTEXT D-06 + Phase 72 UI-SPEC §"animation: instant" precedent |

| Z-order change | None — picker stays at the same z as row 1 widgets (both rows are children of `center`, same z-plane) | RESEARCH §Architecture |

| Focus retention | If `stream_combo` had keyboard focus at reparent time, Qt preserves focus on the widget across `setParent()` / layout transfer (verified Qt 6.11 — RESEARCH §Qt API). No explicit `setFocus()` call. | RESEARCH §Qt Reparenting Semantics |
| Focus fallback | If for any reason focus cannot be retained (e.g., picker becomes hidden mid-reparent because the active station changed to single-stream), Qt moves focus to the next focusable widget in tab order (default behavior). | Qt default |
| Signal preservation | `stream_combo.currentIndexChanged.connect(self._on_stream_selected)` survives reparent. Qt signal/slot bindings are not affected by `setParent()` or layout transfer. | RESEARCH §Qt Reparenting Semantics |

## Out-of-Scope Visual Decisions (explicit)

The following visual / interaction surfaces are explicitly NOT introduced by this phase. Adding any of them would expand scope.

- **No animations** on the reparent. Instant only. (CONTEXT D-06)
- **No new color tokens.** Picker uses its existing palette on both rows. (Inherited from Phase 72)
- **No new fonts or font sizes.** Picker items use existing dropdown typography. (No new text)
- **No new hover / focus / pressed states beyond Qt defaults.** The picker already has them from Phase 39.
- **No visual treatment for the bug-repro state** (the pre-fix overlapping picker). The fix removes the overlap; documenting the bad state is unnecessary.
- **No label above or below row 2.** Picker is self-describing.
- **No notification or toast** when the picker reparents. Silent. (Matches Phase 72 D-Discretion)
- **No persisted "user prefers two rows" toggle.** Width is the only trigger. (CONTEXT Deferred Ideas)
- **No hysteresis dead-band** for threshold flicker suppression. (CONTEXT Deferred Ideas)
- **No new keyboard shortcut** to force-wrap. Layout is fully resize-driven. (CONTEXT Deferred Ideas)
- **No visual badge** in row 1 indicating "picker is on row 2" (e.g., a 320kbps chip next to compact-toggle). Out of scope. (CONTEXT Deferred Ideas)
- **No generalized FlowLayout** for the entire control row. (CONTEXT D-04)
- **No restructuring** of any other row 1 widget — star, eq, volume, compact-toggle keep their existing positions. (CONTEXT D-03)
