## Decisions Made

- **`if not self.stream_combo.isVisible():` (not `isHidden()`) for the extension hook predicate.** This branch runs immediately AFTER `setVisible(len(streams) > 1)` at line 1165 — the just-set state is reflected by `isVisible()` exactly as intended for the single-stream case. (Plan 02's `_reflow_stream_picker_row` uses `isHidden()` because it runs on resizeEvent at indeterminate times relative to show-state; that distinction doesn't apply here.) Both predicates would work for the single-stream branch in this exact location, but `isVisible()` reads naturally with the plan's pseudo-code intent.
- **Multi-stream case defers to the next resizeEvent tick (no explicit reflow call from `_populate_stream_picker`).** Pitfall 1 in 72.1-RESEARCH: `_populate_stream_picker` does not know `panel.width()` with reliable timing (it runs on station bind, before paint). Wayland coalesces resize events; the picker reaches its correct row within one frame via the natural resize cycle.
- **Sample (g) test-only fix (no production change).** Test 7 was written in Plan 01 before Plan 02's `_make_panel` deviation surfaced the show + waitExposed requirement. Test 7 replicated the omission on its MainWindow construction path. The fix is one block of code in the test; production behavior is verified by Plan 02's GREEN tests 1-5 and Plan 03's GREEN test 6 — no production gap exists.
- **Sample (h) regression net activated (not just passes-by-accident).** Plan 02's SUMMARY flagged test 8 as "passes-by-accident." Plan 03 confirms: the reparent design intentionally preserves QObject parent identity (`QHBoxLayout.addWidget` does not change `QObject.parent()` — only layout membership), and Qt 6 propagates explicit palette roles to children. The test is a live regression net for any future refactor that swaps `addWidget` for an `setParent`-then-`addWidget` sequence.
