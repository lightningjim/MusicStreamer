<objective>
Close the Phase 95 UAT gap (live repro, 2026-06-20): editing a YouTube station
whose stream had already ended, then saving a new working URL, flashes a single
"Stream exhausted" toast before the new stream plays. The Phase 95-02
`_recovery_seq` generation guard structurally cannot suppress this because the
triggering bus error carries the CURRENT generation stamp.

Add a YouTube-resolve-in-flight gate (NOT another generation guard): while the
CURRENT generation's YouTube resolution is still pending, exhaustion
(`failover.emit(None)` / "Stream exhausted") is not declared. When the resolve
settles it either plays (success) or legitimately advances/exhausts (failure).

Purpose: Eliminate the residual false-exhaustion toast at its root cause (a
premature exhaustion verdict during an unfinished async resolve) without
over-suppressing genuine all-streams-failed exhaustion.

Output:
- `_youtube_resolve_in_flight` gate in `musicstreamer/player.py`, additive to the
  intact `_recovery_seq` (95-02) and `_youtube_resolve_seq` (95-01) guards.
- New regression tests V14/V15/V16 in
  `tests/test_player_edit_invalidation.py`, and V12 reconciled to the gated
  behavior.
</objective>
