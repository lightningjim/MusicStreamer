<objective>
Close CR-01 (BLOCKER, 95-REVIEW.md, verified against source 2026-06-20): the
Phase 95-03 `_youtube_resolve_in_flight` gate has two correctness holes that make
the same "spurious Stream exhausted" / "stuck gate" symptom the gate was added to
eliminate still reachable.

1. PERMANENT LEAK: editing a playing YouTube station to a non-YouTube/direct URL
   mid-resolve takes the `_set_uri` branch (never re-arms or clears the gate); the
   old worker's failure is correctly classified stale and returns WITHOUT clearing
   the gate, so `_youtube_resolve_in_flight` is stuck True for the session and every
   future failover is silently suppressed. `stop()` does not recover it.
2. SPURIOUS EXHAUSTION: `play()` does not bump `_youtube_resolve_seq` (only
   `invalidate_for_edit` does) and `youtube_resolution_failed = Signal(str)` carries
   no generation stamp, so a rapid same-generation YouTube A->B switch mis-classifies
   the old worker's failure as current, clears B's fresh gate, and emits
   `failover.emit(None)` while B is still resolving.
3. FALSE-GREEN TEST: V16d bumps `_youtube_resolve_seq` WITHOUT calling `_play_youtube`
   — a state production never produces — so the leak is invisible to CI.

Fix direction (refined into tasks below):
- Give the failure path a real per-worker generation stamp by widening
  `youtube_resolution_failed` to `Signal(str, int)` (mirroring the success-path
  `youtube_resolved = Signal(str, bool, int)`), captured at worker-spawn and carried
  through. Decide staleness by carried-seq vs current-seq, not the overwrite-prone
  instance attribute. Remove the now-dead `_youtube_resolve_in_flight_seq`.
- Bump `_youtube_resolve_seq` in `play()` too, so plain station switches invalidate a
  prior in-flight resolve (covers both success WR-01 and failure CR-01 paths).
- Guarantee the gate cannot leak across a restart that bypasses `_play_youtube`: clear
  `_youtube_resolve_in_flight = False` on every (re)start path that does NOT spawn a
  YouTube resolve — at minimum the `_set_uri` direct/non-YouTube branch and `stop()`.
- Update `tests/_fake_player.py` to mirror the new arity (FakePlayer parity guard).

Purpose: Eliminate the gate leak and the spurious-exhaustion mis-classification at
their root (an overwrite-prone instance stamp + a missing generation bump), while
preserving every prior Phase 95 win (D-01..D-05).

Output:
- `musicstreamer/player.py`: widened failure Signal, carried-seq staleness,
  `play()` resolve-seq bump, gate clears on `_set_uri` direct branch + `stop()`,
  `_youtube_resolve_in_flight_seq` removed.
- `tests/_fake_player.py`: `youtube_resolution_failed = Signal(str, int)` parity.
- `tests/test_player_edit_invalidation.py`: V17/V18/V19 + V16d reconciled to drive
  the real `_play_youtube` arming path.
