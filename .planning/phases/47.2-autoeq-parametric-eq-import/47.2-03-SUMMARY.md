## Implementation

### musicstreamer/ui_qt/eq_response_curve.py (NEW)

- `compute_response(profile, preamp_db, n_points=256)` — stdlib log-spaced (20 Hz – 20 kHz) combined biquad magnitude response. Verbatim RBJ cookbook coefficients for `PK`, `LSC`, `HSC`. Pitfall 5 additive preamp (`b.gain_db + preamp_db`).
- `ResponseCurve(QWidget)` — minimum height 180 px. Draws palette-driven grid (decade verticals at 100 / 1 k / 10 kHz, horizontals at ±6 dB + 0), axis labels, and the curve path. All colors via `self.palette()` — zero hex literals. `changeEvent(PaletteChange|StyleChange)` triggers repaint on theme flip (matches `_MutedLabel` idiom).
- `set_profile(profile, preamp_db)` — flat reference line when profile is None or has no bands; otherwise computes and stores 256 points, calls `update()`.

### musicstreamer/ui_qt/equalizer_dialog.py (NEW, ~415 lines)

Implements the full D-11 layout top-to-bottom. Constructor signature `(player, repo, toast_callback, parent=None)` mirrors `DiscoveryDialog`.

Slider precision: `_SLIDER_SCALE = 10` → 0.1 dB per integer tick. Preamp range –20.0 to +6.0 dB, band range ±15 dB.

Security (threat model):
- **T-47.2.03-01 (tampering / bad file on import):** `parse_autoeq` is called BEFORE `shutil.copyfile` so bogus content never reaches `eq-profiles/`.
- **T-47.2.03-02 (tampering / filename):** both `_on_import` and `_on_save_as_new` route through `_sanitize` with a `"profile"` fallback. `grep -c _sanitize equalizer_dialog.py` = 2.
- **T-47.2.03-04 (DoS on startup restore):** belt-and-braces `try/except` in `MainWindow.__init__` wraps `restore_eq_from_settings` on top of the one already in Player.
- **T-47.2.03-06 (overwrite via save-as-new):** destination is always under `eq_profiles_dir()` after `_sanitize`, so directory traversal is impossible. Overwrite of same-name profile is intentional per D-20.
