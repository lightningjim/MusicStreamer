# Phase 47.2: In-app parametric EQ with AutoEQ profile import — Research

**Researched:** 2026-04-19
**Domain:** GStreamer audio-filter DSP + PySide6 dialog + AutoEQ text parser
**Confidence:** HIGH

## Summary

Phase 47.2 inserts a GStreamer `equalizer-nbands` element into the existing
`playbin3.audio-filter` slot and adds a PySide6 dialog that imports AutoEQ
`ParametricEQ.txt` profiles and drives the filter's band properties. All
moving pieces are first-party: no new library dependencies (GStreamer good
plugins already ship the element, PySide6 already renders dialogs, QPainter
handles the frequency-response curve). The AutoEQ text format is a
regular-language grammar that a ~15-line regex parser decodes.

The phase is unusually clean from a risk standpoint:
- `equalizer-nbands` is already on the system (verified `1.26.5`, gst-plugins-good)
- `playbin3.audio-filter` accepts the element at construction time (verified by probe)
- Per-band properties are `gdouble` freq/gain/bandwidth + a 3-value enum type, all
  live-mutable through the `GstChildProxy` interface that the element implements
- The AutoEQ format has ~5 grammatical elements; no Unicode, no nesting
- Dialog patterns (QDialog + hamburger-menu launch + pytest-qt qtbot) are already
  in use by 5+ existing dialogs in `musicstreamer/ui_qt/`

**Primary recommendation:** Build per CONTEXT.md decisions as-written, with two
small corrections surfaced in "Corrections to CONTEXT.md" below — the AutoEQ
filter-type tokens are `LSC`/`HSC` (not `LS`/`HS`), and the `equalizer-nbands`
element natively supports shelf bands (so D-03's "approximate shelves as wide
peaking" is a fallback, not a necessity).

## User Constraints (from CONTEXT.md)

### Locked Decisions

**EQ Engine**
- **D-01:** Use GStreamer `equalizer-nbands` (from `gstreamer-plugins-good`) as the audio-filter element. Zero new dependencies. Cross-platform (Linux + Windows).
- **D-02:** Map AutoEQ filter rows 1:1 onto bands — `N bands` where `N` = number of filter rows parsed from `ParametricEQ.txt` (typically 5–10). Each band is configured with the AutoEQ row's `frequency`, `bandwidth` (derived from `Q`: `bw = freq / Q`), and `gain`.
- **D-03:** Treat AutoEQ `LS` (low-shelf) and `HS` (high-shelf) rows as wide peaking (`PK`) bands at the corner frequency — imperfect but keeps the engine single-path. Log a debug note when shelves are encountered so sound-quality regressions can be traced.
- **D-04:** Rebuild the `equalizer-nbands` element whenever the band count changes (new profile with different filter count). Within a single profile, mutate band properties on the existing element rather than recreating.

**EQ Lifecycle**
- **D-05:** Hot-toggle while playing — construct the `equalizer-nbands` element once at pipeline setup and leave it resident in `audio-filter`. Enable/disable is implemented by zeroing every band gain (bypass) rather than rebuilding the pipeline. Profile swaps and preamp/band tweaks update band gains live.
- **D-06:** On parse failure, show a toast (`Failed to parse EQ profile: <reason>`) and keep the previous active profile intact. Never auto-disable EQ on a bad file.

**UI Surface**
- **D-07:** New `EqualizerDialog` (Qt) launched from a "Equalizer" hamburger-menu action, inserted alongside Discover/Import/Accent/Cookies/Accounts.
- **D-08:** Add a small icon-only "EQ" toggle button in `now_playing_panel.controls_box` (next to Star/Pause/Stop) so the user can A/B-compare corrected vs. flat without opening the dialog. Button reflects `eq_enabled` state visually.
- **D-09:** EQ dialog centerpiece is a **static frequency-response curve** drawn with QPainter. Compute the combined magnitude response of all bands over a fixed grid of log-spaced points (20 Hz – 20 kHz, ~256 points) using standard biquad peaking-EQ magnitude formulas. Redraw on profile load, preamp change, or band edit (Manual mode).
- **D-10:** Curve is **display-only** — no click/drag interaction. Controls for preamp and (in Manual mode) per-band gain live as separate widgets below the curve.
- **D-11:** Dialog layout (top to bottom): profile switcher dropdown + Import + delete-trash icon for active row · active profile name · response-curve canvas · preamp slider (with dB label) · per-band slider row (read-only unless Manual mode toggled) · Manual/Reset toggle · Save-as-new-profile button (Manual mode only) · Close.

**Profile Storage**
- **D-12:** Imported profiles are **copied into** `{user_data_dir}/eq-profiles/<safe_headphone_name>.txt` on import. Parse at load time. Survives restart and is user-inspectable. Uses the existing `paths.py` / `platformdirs.user_data_dir("musicstreamer")` root.
- **D-13:** Library model with switcher — the dialog lists all files in `eq-profiles/` in a dropdown; user picks active. Active profile is a single setting (see D-15). Multiple profiles (different headphones) supported from day one.
- **D-14:** Trash-icon delete per row in the switcher, with confirmation — deletes the file and clears `eq_active_profile` if the deleted one was active.
- **D-15:** Persist EQ state in SQLite `settings` table via the existing `set_setting`/`get_setting` pattern:
  - `eq_active_profile` — filename (basename only; full path reconstructed from `paths.user_data_dir() / eq-profiles /`)
  - `eq_enabled` — `"1"` / `"0"`
  - `eq_preamp_db` — float as string (e.g., `"-6.5"`)
- **D-16:** Extend Phase 42 settings export/import ZIP to include an `eq-profiles/` folder. `settings_export.py` writes the directory into the ZIP; `settings_import_dialog.py` + importer restores it alongside `settings.json` and `logos/`.

**Band Editing + Preamp**
- **D-17:** Auto-apply the `Preamp: X.X dB` header from the AutoEQ file into the preamp slider on import. User can adjust afterward. Preamp persists per session in `eq_preamp_db`. No per-profile preamp history.
- **D-18:** Preamp implementation: **uniform offset subtracted from every band's gain** before the gain is written to the `equalizer-nbands` band. Keeps all attenuation inside the EQ element — no conflict with `playbin3.volume` (AUDIO-01 slider).
- **D-19:** Per-band sliders are **read-only when an AutoEQ profile is loaded**. A "Manual" toggle in the dialog unlocks them for editing; toggling Manual on does NOT detach from the active file automatically — edits become pending until the user hits "Save as new profile".
- **D-20:** "Save as new profile" prompts for a profile name (`QInputDialog.getText`), writes a new `ParametricEQ.txt`-formatted file to `eq-profiles/<name>.txt`, and switches the active profile to the new file. Original imported profile is untouched. Manual tweaks are NEVER written back over the source file.

**Parser**
- **D-21:** AutoEQ parser lives in a new module — `musicstreamer/eq_profile.py` — with two pure functions:
  - `parse_autoeq(text: str) -> EqProfile` — raises `ValueError` on malformed input.
  - `serialize_autoeq(profile: EqProfile) -> str` — for D-20 save-as-new.
- **D-22:** `EqProfile` is a dataclass: `{ preamp_db: float, bands: list[EqBand] }` where `EqBand = { filter_type: Literal["PK","LS","HS"], freq_hz: float, gain_db: float, q: float }`.
- **D-23:** Parse regex handles the canonical AutoEQ line format: `Filter N: ON PK Fc <freq> Hz Gain <gain> dB Q <q>` (and `OFF` filters are skipped). Case-insensitive on `PK`/`LS`/`HS`. Tolerate leading/trailing whitespace. Reject files with zero `ON` filters.

## Corrections to CONTEXT.md

Two small factual drifts that the planner should incorporate as Wave 0 clarifications:

### C-1: AutoEQ shelf tokens are `LSC` / `HSC`, not `LS` / `HS`

[VERIFIED: minidsp.com AutoEQ doc + asymptotic.io PipeWire AutoEQ post + WebSearch
of canonical AutoEQ README examples]

Real AutoEQ `ParametricEQ.txt` files use three filter-type tokens:
- `PK` — peaking
- `LSC` — low-shelf
- `HSC` — high-shelf

D-23 specifies `LS`/`HS`. The regex in `eq_profile.py` should accept
`PK|LSC|HSC` (case-insensitive). The `EqBand.filter_type` Literal in D-22
should be updated to `Literal["PK", "LSC", "HSC"]`.

**Why this matters:** a profile with shelf bands will parse as "malformed"
under D-23's literal regex and trip D-06's toast path, leaving the user with
no usable EQ from any AutoEQ file that contains `LSC`/`HSC` rows
(which is the majority of published profiles).

### C-2: `equalizer-nbands` natively supports shelf band types

[VERIFIED: Python probe of `Gst.ElementFactory.make("equalizer-nbands")` —
see Code Examples §1]

Each child band exposes a `type` property of enum
`GstIirEqualizerBandType` with three values:

| Enum value | Meaning |
|-----------|---------|
| `0` | Peak filter (default inner band) |
| `1` | Low shelf filter |
| `2` | High shelf filter |

D-03 decides to approximate `LS`/`HS` as wide peaking bands "to keep the engine
single-path." That approximation is **unnecessary** on the GStreamer side —
the element already speaks shelves. Setting `band.set_property("type", 1)` for
LSC and `2` for HSC produces an honest shelf filter.

**Recommendation:** The planner should flip D-03 from "wide peaking
approximation" to "set band `type=1` for LSC, `type=2` for HSC, `type=0` for
PK" unless the user has a reason to prefer the single-path engine. It's the
same number of property sets, and sound quality improves on shelf-heavy
profiles (which are common).

If the user wants D-03 kept as-is for simplicity, the research still
supports it — both paths work — but the preamble of the curve graph must
still compute shelf magnitude correctly for the displayed response curve
(§Code Examples §3 covers this).

Flag this to the user during plan review; if ambiguous, default to the
native-shelf implementation (lower risk of audibly wrong output).

### Pattern 1: `equalizer-nbands` construction and live mutation

**What:** Construct once at `Player.__init__`, mutate band properties via
`GstChildProxy` at runtime. Band count is fixed at construction; profile
swaps with a different band count require a NULL/rebuild.

**When to use:** Any time a profile is loaded, preamp changes, or the enable
flag flips.

**Source:** Probed against local `gstreamer-1.0/1.26.6` + `gst-plugins-good 1.26.5`.

```python
# In Player.__init__, AFTER audio_sink assignment, BEFORE set_state(PLAYING):
self._eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
if self._eq is not None:
    self._eq.set_property("num-bands", 5)          # default; rebuilt per-profile
    self._pipeline.set_property("audio-filter", self._eq)
    # All bands default to gain=0 → bypass on first construct (D-05)
else:
    self._eq = None                                  # graceful no-op
```

Live mutation through child proxy:
```python
band = self._eq.get_child_by_index(band_idx)  # band_idx: 0..num_bands-1
band.set_property("freq", freq_hz)            # float
band.set_property("bandwidth", freq_hz / q)   # D-02: bw = freq / Q
band.set_property("gain", gain_db)            # float, dB (-24.0..+12.0 typical range)
band.set_property("type", 0)                  # 0=PK, 1=LSC (low shelf), 2=HSC (high shelf)
```

Band count change (new profile with different N):
```python
# num-bands is CONSTRUCT (not CONSTRUCT_ONLY) but IIR-equalizer's realloc
# path is unreliable in the wild — rebuild the element per D-04.
self._pipeline.set_state(Gst.State.READY)
self._pipeline.get_state(Gst.CLOCK_TIME_NONE)
self._eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
self._eq.set_property("num-bands", new_n)
self._pipeline.set_property("audio-filter", self._eq)
# Then repopulate every band's freq/bandwidth/gain/type and return to PLAYING.
```

constructed playbin3 in the probe; `band.set_property("gain", -3.5)` read

### Pattern 2: AutoEQ parser regex

**What:** One regex matches a filter line; Preamp line is a separate one-liner.

**When to use:** `parse_autoeq(text)` iterates `text.splitlines()`, matches
each against the regex, builds an `EqBand` per `ON` row.

**Source:** [VERIFIED: minidsp.com AutoEQ doc + asymptotic.io example + remote
Sennheiser HD 650 ParametricEQ.txt sample on github.com/jaakkopasanen/AutoEq]

```python
# musicstreamer/eq_profile.py
import re
from dataclasses import dataclass, field
from typing import Literal

FilterType = Literal["PK", "LSC", "HSC"]   # C-1: LSC/HSC, not LS/HS

_PREAMP_RE = re.compile(
    r"^\s*Preamp\s*:\s*(-?\d+(?:\.\d+)?)\s*dB\s*$",
    re.IGNORECASE,
)

_FILTER_RE = re.compile(
    r"""^\s*Filter\s+\d+\s*:\s*
         (?P<state>ON|OFF)\s+
         (?P<type>PK|LSC|HSC)\s+
         Fc\s+(?P<freq>-?\d+(?:\.\d+)?)\s*Hz\s+
         Gain\s+(?P<gain>-?\d+(?:\.\d+)?)\s*dB\s+
         Q\s+(?P<q>-?\d+(?:\.\d+)?)\s*$""",
    re.IGNORECASE | re.VERBOSE,
)

@dataclass
class EqBand:
    filter_type: FilterType
    freq_hz: float
    gain_db: float
    q: float

@dataclass
class EqProfile:
    preamp_db: float = 0.0
    bands: list = field(default_factory=list)  # list[EqBand]

def parse_autoeq(text: str) -> EqProfile:
    profile = EqProfile()
    any_on = False
    for raw in text.splitlines():
        line = raw.strip()
        if not line or line.startswith("#"):
            continue
        m = _PREAMP_RE.match(line)
        if m:
            profile.preamp_db = float(m.group(1))
            continue
        m = _FILTER_RE.match(line)
        if m:
            if m.group("state").upper() == "OFF":
                continue
            any_on = True
            profile.bands.append(EqBand(
                filter_type=m.group("type").upper(),  # type: ignore[arg-type]
                freq_hz=float(m.group("freq")),
                gain_db=float(m.group("gain")),
                q=float(m.group("q")),
            ))
            continue
        # Unknown but non-fatal — silently skip (AutoEQ files occasionally
        # have commentary or blank headers).
    if not any_on:
        raise ValueError("AutoEQ file contains no ON filters")
    return profile
```

Serializer (for D-20 save-as-new):
```python
def serialize_autoeq(profile: EqProfile) -> str:
    lines = [f"Preamp: {profile.preamp_db:.1f} dB"]
    for i, b in enumerate(profile.bands, start=1):
        lines.append(
            f"Filter {i}: ON {b.filter_type} "
            f"Fc {b.freq_hz:g} Hz Gain {b.gain_db:.1f} dB Q {b.q:.2f}"
        )
    return "\n".join(lines) + "\n"
```

### Pattern 4: QPainter response-curve widget

**What:** Custom QWidget subclass; paintEvent walks the 256-point grid,
converts amplitude dB to y-pixel, draws a QPainterPath.

```python
class ResponseCurve(QWidget):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.setMinimumHeight(180)
        self._points: list[tuple[float, float]] = []   # (freq_hz, mag_db)

    def set_profile(self, profile: EqProfile, preamp_db: float) -> None:
        self._points = compute_response(profile, preamp_db)  # §Code §3
        self.update()  # triggers paintEvent

    def paintEvent(self, event):
        from PySide6.QtGui import QPainter, QPainterPath, QPen
        p = QPainter(self)
        p.setRenderHint(QPainter.Antialiasing)
        p.fillRect(self.rect(), self.palette().window())
        # Grid lines at 100/1k/10k Hz and ±6/±12 dB here — see discretion §
        path = QPainterPath()
        for i, (freq_hz, mag_db) in enumerate(self._points):
            x = self._freq_to_x(freq_hz)   # log scale: 20Hz..20kHz → 0..w
            y = self._db_to_y(mag_db)      # ±12dB → bottom..top
            if i == 0: path.moveTo(x, y)
            else:      path.lineTo(x, y)
        pen = QPen(self.palette().highlight().color(), 2)
        p.setPen(pen)
        p.drawPath(path)
```

**When to use:** Once, in `EqualizerDialog`. Do not reuse as a general chart widget.

### Anti-Patterns to Avoid
- **Recreating the EQ element on every gain tweak:** The element is live-mutable
  through GstChildProxy. Only rebuild when band count changes (D-04).
- **Mutating `num-bands` at runtime:** CONSTRUCT-flag property; the property is
  technically writable but the internal band array realloc is fragile. Rebuild
  the whole element instead.
- **Writing preamp back to the source AutoEQ file on manual edits:** D-20
  forbids — save-as-new only.
- **Connecting GStreamer bus emission to any GUI widget directly:** Use the
  existing Qt-signal bridge in `player.py`; response-curve redraw is driven by
  `EqualizerDialog` state, not bus traffic.
- **Using absolute paths to `~/.local/share/musicstreamer/eq-profiles`:** Always
  go through `paths.eq_profiles_dir()` so tests monkeypatch cleanly (PORT-05
  / existing convention).
- **Hand-rolling a Pango-style filter grammar:** The AutoEQ format is regex-
  regular. One compiled regex + one stripped-line loop covers the whole file.

## Common Pitfalls

### Pitfall 1: `num-bands` cannot be mutated at runtime safely
**What goes wrong:** Setting `num-bands` on a live element may appear to work
(the property write succeeds), but the internal band array is not fully
reallocated and subsequent band writes hit stale memory / segfault in some
plugin versions.
**Why it happens:** `num-bands` is `G_PARAM_CONSTRUCT` (set at construction)
but not `G_PARAM_CONSTRUCT_ONLY` — GObject accepts the write and forwards it
to the class's property setter, but `GstIirEqualizer`'s setter is not a full
realloc path in all release branches.
**How to avoid:** Rebuild the element (set pipeline to READY, create a new
`equalizer-nbands` with the new `num-bands`, set as `audio-filter`, return to
PLAYING). This is D-04's decision and it is correct.
**Warning signs:** Any change of active profile where `len(new.bands) !=
len(current.bands)`.

### Pitfall 2: Setting `audio-filter` while in PLAYING state
**What goes wrong:** The audio-filter change is only picked up when the
streaming thread is not running; setting it while PLAYING either ignores the
change silently or causes a pad-linking race.
**Why it happens:** `audio-filter` is a playbin3 construction-time slot.
**How to avoid:** For the hot-toggle path (D-05), DO NOT unset or replace the
element. Keep it resident from `__init__` forward. Toggle by zeroing gains
only. If a full rebuild is necessary (band-count change, Pitfall 1), set
pipeline to READY first, swap, and return to PLAYING.
**Warning signs:** "EQ is silent" or "no change from band tweaks" after a
profile swap.

### Pitfall 3: `GstIirEqualizerBandType` enum maps 0/1/2 (not 1/2/3)
**What goes wrong:** Setting `type=3` raises a GObject warning and leaves
the band in the previous type.
**Why it happens:** Enum starts at 0.
**How to avoid:** Use a constant map:
```python
_BAND_TYPE = {"PK": 0, "LSC": 1, "HSC": 2}
band.set_property("type", _BAND_TYPE[eq_band.filter_type])
```
**Warning signs:** `g_warning: value "((GstIirEqualizerBandType) 3)" of type
'GstIirEqualizerBandType' is invalid`.

### Pitfall 4: GStreamer bandwidth vs AutoEQ Q
**What goes wrong:** AutoEQ Q and `equalizer-nbands` bandwidth are different
parameters. Writing AutoEQ's Q value directly to `bandwidth` gives totally
wrong-sounding filter widths.
**Why it happens:** `bandwidth` is a frequency range in Hz, not a quality
factor. The conversion is `bandwidth_hz = freq_hz / q` (D-02).
**How to avoid:** Always compute `bw = freq / q` at the boundary. Never pass
Q directly. Log the computed bandwidth alongside the input Q during debug.
**Warning signs:** Profiles with high-Q peaks (Q > 3) sound washy/smeared.

### Pitfall 5: AutoEQ's `Preamp` is negative and must be ADDED, not SUBTRACTED
**What goes wrong:** AutoEQ's `Preamp: -6.2 dB` means "attenuate all output
by 6.2 dB to make headroom." D-18 says "uniform offset subtracted from every
band's gain" — which is correct ONLY if preamp is interpreted as
`gain[i] = band.gain_db + preamp_db` (preamp_db is usually negative → net
attenuation). Confusing sign convention flips the curve upside-down.
**Why it happens:** "Subtract the preamp" reads as `gain - abs(preamp)`.
The AutoEQ convention is `final_gain = band_gain + preamp` where preamp is
typically negative.
**How to avoid:** Spell it out in code: `effective_gain = b.gain_db +
profile.preamp_db + user_preamp_adjust`. Add a test with `preamp=-6, band=+4
→ effective=-2`.
**Warning signs:** Boosted-looking curves for profiles whose AutoEQ file has
`Preamp: -6 dB` (should instead look attenuated globally).

### Pitfall 6: Pathological profile names on file copy
**What goes wrong:** User imports a profile named `../../../evil.txt` or one
containing path separators.
**Why it happens:** D-12 constructs `eq-profiles/<safe_headphone_name>.txt`
from the user-visible name. Without sanitization this is an arbitrary-write.
**How to avoid:** Mirror `settings_export._sanitize()` — it already has the
fallback for `.` / `..` and strips non-word chars. Reuse that function:
```python
from musicstreamer.settings_export import _sanitize
```
Or expose it as a module-level helper if reuse feels awkward.
**Warning signs:** None at runtime — this is a latent security issue; flag
in test.

### Pitfall 7: Dialog holding a reference to a stale Station after stop
**What goes wrong:** EqualizerDialog is not tied to the currently-playing
station (EQ is per-user, not per-station). It must survive a stop → play
cycle.
**Why it happens:** Dev habit from existing dialogs that bind to a station.
**How to avoid:** Dialog takes `player` + `repo`; never takes a `Station`.
The dialog's state is sourced from `repo.get_setting("eq_*", ...)` only.

### Pitfall 8: ZIP member `eq-profiles/` path traversal
**What goes wrong:** A hostile settings.zip with `eq-profiles/../../evil.txt`
extracts to the wrong place.
**Why it happens:** `_validate_zip_members` in settings_export.py is already
defensive (posixpath.normpath, backslash rejection). The extension just needs
to go through the same gate.
**How to avoid:** Add `eq-profiles/` to the normal ZIP walk — do NOT open
members with a custom `os.path.join` path. Use the existing validation
harness. See `settings_export._validate_zip_members`.

### §3 Biquad magnitude response (stdlib-only)

Computes |H(e^jω)| in dB for one band at one frequency, from RBJ cookbook.
Needed for the frequency-response curve D-09.

[CITED: webaudio.github.io/Audio-EQ-Cookbook/audio-eq-cookbook.html (W3C spec)]

```python
import math

_FS = 48000.0  # reference sample rate for curve display (pulsesink default)

def _biquad_coeffs(filter_type: str, freq_hz: float, q: float, gain_db: float):
    """RBJ cookbook coefficients, normalized by a0 (returns (b0,b1,b2,a1,a2))."""
    A = 10.0 ** (gain_db / 40.0)
    w0 = 2.0 * math.pi * freq_hz / _FS
    cos_w0 = math.cos(w0)
    sin_w0 = math.sin(w0)
    alpha = sin_w0 / (2.0 * max(q, 1e-6))

    if filter_type == "PK":
        b0 =  1.0 + alpha * A
        b1 = -2.0 * cos_w0
        b2 =  1.0 - alpha * A
        a0 =  1.0 + alpha / A
        a1 = -2.0 * cos_w0
        a2 =  1.0 - alpha / A
    elif filter_type == "LSC":
        sA = math.sqrt(A)
        b0 = A * ((A + 1) - (A - 1) * cos_w0 + 2 * sA * alpha)
        b1 = 2 * A * ((A - 1) - (A + 1) * cos_w0)
        b2 = A * ((A + 1) - (A - 1) * cos_w0 - 2 * sA * alpha)
        a0 = (A + 1) + (A - 1) * cos_w0 + 2 * sA * alpha
        a1 = -2 * ((A - 1) + (A + 1) * cos_w0)
        a2 = (A + 1) + (A - 1) * cos_w0 - 2 * sA * alpha
    elif filter_type == "HSC":
        sA = math.sqrt(A)
        b0 = A * ((A + 1) + (A - 1) * cos_w0 + 2 * sA * alpha)
        b1 = -2 * A * ((A - 1) + (A + 1) * cos_w0)
        b2 = A * ((A + 1) + (A - 1) * cos_w0 - 2 * sA * alpha)
        a0 = (A + 1) - (A - 1) * cos_w0 + 2 * sA * alpha
        a1 = 2 * ((A - 1) - (A + 1) * cos_w0)
        a2 = (A + 1) - (A - 1) * cos_w0 - 2 * sA * alpha
    else:
        raise ValueError(f"Unknown filter_type: {filter_type}")

    return (b0 / a0, b1 / a0, b2 / a0, a1 / a0, a2 / a0)


def _biquad_magnitude_db(b0, b1, b2, a1, a2, freq_hz: float) -> float:
    """|H(e^jw)| in dB, where H(z) = (b0 + b1 z^-1 + b2 z^-2) / (1 + a1 z^-1 + a2 z^-2)."""
    w = 2.0 * math.pi * freq_hz / _FS
    cos1, sin1 = math.cos(w),     math.sin(w)
    cos2, sin2 = math.cos(2 * w), math.sin(2 * w)
    num_re = b0 + b1 * cos1 + b2 * cos2
    num_im =   - b1 * sin1 - b2 * sin2
    den_re = 1.0 + a1 * cos1 + a2 * cos2
    den_im =     - a1 * sin1 - a2 * sin2
    num_mag2 = num_re * num_re + num_im * num_im
    den_mag2 = den_re * den_re + den_im * den_im
    return 10.0 * math.log10(max(num_mag2 / max(den_mag2, 1e-30), 1e-30))


def compute_response(profile, preamp_db: float,
                     n_points: int = 256) -> list[tuple[float, float]]:
    """Compute log-spaced frequency response over 20 Hz - 20 kHz (D-09)."""
    f_lo, f_hi = 20.0, 20000.0
    freqs = [f_lo * (f_hi / f_lo) ** (i / (n_points - 1)) for i in range(n_points)]
    coeffs = [_biquad_coeffs(b.filter_type, b.freq_hz, b.q, b.gain_db + preamp_db)
              for b in profile.bands]
    out = []
    for f in freqs:
        total_db = 0.0
        for c in coeffs:
            total_db += _biquad_magnitude_db(*c, f)
        out.append((f, total_db))
    return out
```

**Notes:**
- `_FS = 48000.0` is a reasonable display default. The actual sample rate at
  playback time can differ (44.1k HTTP streams common); the difference over
  the audible band is negligible for display purposes.
- This is a DISPLAY approximation of what `equalizer-nbands` does. The real
  element's IIR topology is "direct form" and may differ by small dB fractions
  from the cookbook. Good enough for a visual response graph.
- Zero-gain bands contribute 0.0 dB — no need to special-case.
