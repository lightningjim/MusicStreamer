## Implementation

### musicstreamer/eq_profile.py (NEW)

- `EqBand` dataclass: `filter_type: Literal["PK","LSC","HSC"]`, `freq_hz`, `gain_db`, `q` (all floats).
- `EqProfile` dataclass: `preamp_db: float = 0.0`, `bands: list[EqBand]`.
- `parse_autoeq(text)`:
  - Module-level compiled `_PREAMP_RE` and `_FILTER_RE` (mirrors `aa_import.py` style).
  - Case-insensitive tokens; `VERBOSE` regex for readability.
  - Skips OFF rows, blank lines, and `#` comment lines.
  - Raises `ValueError("AutoEQ file contains no ON filters")` when no ON rows parsed.
  - Preserves canonical uppercase form (`.upper()` on match).
- `serialize_autoeq(profile)`: emits `Preamp: X.X dB\n` then one `Filter N: ON TYPE Fc F Hz Gain G dB Q Q` line per band, using `:g` for frequency (compact) and `:.1f`/`:.2f` for gain/Q to match common AutoEQ output.
- Zero Qt / `gi` imports verified by `grep`.

## Deviations from Plan

### [Rule 3 - Blocking] Acceptance criterion for rcc plaintext grep is infeasible

- **Found during:** Task 3 verification.
- **Issue:** Plan acceptance criterion states `grep -c "multimedia-equalizer-symbolic" musicstreamer/ui_qt/icons_rc.py` must return ≥1. In practice, `pyside6-rcc` compiles resource paths into a binary hash-indexed name table — filenames are NOT stored as plaintext. Every existing icon in `icons_rc.py` also returns 0 for plaintext grep (`grep -c "media-playback-start" icons_rc.py` → 0), confirming this is how PySide6's rcc emits.
- **Fix:** Replaced the plaintext-grep verification with the functional check: `QFile(':/icons/multimedia-equalizer-symbolic.svg').open(QFile.ReadOnly)` returns True with size=698 bytes. This is the semantically correct way to verify the resource compiled in; the plan's grep acceptance criterion was a spec bug.
- **Files modified:** None (verification approach only).
- **Commit:** N/A (documentation-only).

### Regression test (`tests/test_headless_entry.py`) fails pre-existing — out of scope

- **Found during:** Task 3 regression check.
- **Issue:** `uv run --with pytest --with pytest-qt pytest tests/test_headless_entry.py` fails with `ImportError: No module named 'gi'` because GStreamer's Python bindings (`gi`) are a system package (`python3-gi` on Linux) and are not installable via `uv --with`. This test needs a system-Python invocation or a devcontainer that includes gi.
- **Verification:** Ran `git stash && pytest ... ; git stash pop` to confirm the failure is pre-existing, not caused by this plan. Confirmed pre-existing.
- **Scope:** Logged but NOT fixed — outside this plan's scope (Rule: "Only auto-fix issues DIRECTLY caused by the current task's changes"). Functional verification of the icon resource was performed directly via `QFile` + `musicstreamer.ui_qt.icons_rc`, which did pass.
- **Files modified:** None.
