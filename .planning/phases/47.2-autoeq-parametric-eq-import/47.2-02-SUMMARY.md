## Implementation

### musicstreamer/player.py (MODIFIED, +~109 LoC)

- **Import**: `from musicstreamer.eq_profile import EqBand, EqProfile, parse_autoeq` (no circular-import workarounds needed).
- **Pipeline block** (after buffer-size, before bus wiring):
  ```python
  self._eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
  if self._eq is not None:
      self._eq.set_property("num-bands", 10)   # placeholder
      self._pipeline.set_property("audio-filter", self._eq)
  ```
- **Runtime state fields** (end of `__init__`): `_eq_enabled: bool = False`, `_eq_preamp_db: float = 0.0`, `_eq_profile: EqProfile | None = None`.
- **Public API** (after `set_volume`):
  - `set_eq_enabled(enabled)` — stores the bool, calls `_apply_eq_state()`.
  - `set_eq_profile(profile)` — stores the profile, triggers rebuild if `len(profile.bands)` differs, calls `_apply_eq_state()`.
  - `set_eq_preamp(preamp_db)` — stores the float, calls `_apply_eq_state()`.
- **Private helpers** (end of class):
  - `_EQ_BAND_TYPE = {"PK": 0, "LSC": 1, "HSC": 2}` (class-level constant map, Pitfall 3).
  - `_apply_eq_state()` — writes `freq`, `bandwidth = freq/Q` (Pitfall 4), `gain = b.gain_db + self._eq_preamp_db` (Pitfall 5 ADDITIVE), `type` per band. Bypass path zeros every band gain (D-05).
  - `_rebuild_eq_element(num_bands)` — pipeline → READY → wait, create new element with new num-bands, re-assign to audio-filter (D-04, Pitfall 1). Mirrors the `pause()` NULL-transition idiom.
  - `restore_eq_from_settings(repo)` — reads `eq_active_profile`, `eq_enabled`, `eq_preamp_db` from the settings table; opens the named file under `paths.eq_profiles_dir()`; parses via `parse_autoeq`; silent on OSError/ValueError. Call site wired in Plan 03.

## Deviations from Plan

### [Rule 3 — Blocking] Added module-scope `_gst_init` fixture

- **Found during:** First RED run (`AttributeError: 'NoneType' object has no attribute 'set_property'` from `playbin3 = None`).
- **Root cause:** Production `musicstreamer/__main__.py` calls `Gst.init(None)` before instantiating `Player`. Tests that construct a real pipeline don't go through `__main__`, and `conftest.py` only stubs the bus bridge — it does not initialize GStreamer.
- **Fix:** Added `@pytest.fixture(scope="module", autouse=True) _gst_init` at the top of the new EQ test block. Guards with `Gst.is_initialized()` so the fixture is idempotent.
- **Files modified:** `tests/test_player.py` only (added 11 lines).
- **Commit:** Bundled into the RED commit (22c60f7) since the tests would not even collect without it.
- **Why not centralized in conftest.py:** The other player tests mock out the pipeline (`make_player(qtbot)` helper) so they do not need Gst.init. Adding it unconditionally in conftest could mask problems in those unit tests. Scoping per module keeps the blast radius to the 8 new tests.

### Pre-existing `yt_dlp` ImportError in test_player_failover.py — OUT OF SCOPE

- **Found during:** Regression sweep (`pytest tests/test_player_failover.py`).
- **Confirmed pre-existing:** `git stash && pytest` still fails with `ModuleNotFoundError: No module named 'yt_dlp'` on HEAD before this plan.
- **Scope:** Not caused by this plan; same class of issue as Plan 01's `test_headless_entry.py` deviation (system-python lacks packages that `uv --with` provides). Not fixed.
- **Files modified:** None.
