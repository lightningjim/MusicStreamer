## EQ Engine Choice

### Q: Which EQ engine should drive the pipeline?

| Option | Description | Selected |
|--------|-------------|----------|
| equalizer-nbands | GStreamer built-in nband equalizer; zero deps; snap bands to configured frequencies; approximate AutoEQ | ✓ |
| biquad chain (true parametric) | Chain of audioiirfilter/biquad elements; exact AutoEQ reproduction; needs scipy or manual coefficient math | |
| equalizer-10bands | Fixed 10-band ISO equalizer; simplest; coarsest approximation | |

**User's choice:** equalizer-nbands (Recommended)

### Q: How should AutoEQ filter rows map to bands?

| Option | Description | Selected |
|--------|-------------|----------|
| 1 band per AutoEQ filter | N bands where N = number of PK filters in the file | ✓ |
| Fixed 10 bands, nearest-frequency snap | Always 10 bands; nearest-freq rounding; simpler and lossier | |
| Not applicable (biquad chain) | Skip if biquad chain was chosen | |

**User's choice:** 1 band per AutoEQ filter (Recommended)

### Q: How to handle LS (low-shelf) and HS (high-shelf) rows in equalizer-nbands?

| Option | Description | Selected |
|--------|-------------|----------|
| Approximate as a wide PK band | Treat LS/HS as peaking filters at corner freq with wide bandwidth | ✓ |
| Skip LS/HS rows, log warning | Only apply PK rows; some profiles will sound worse | |
| Escalate to biquad chain if LS/HS present | Hybrid: nbands for PK-only; biquad for shelves | |

**User's choice:** Approximate as a wide PK band (Recommended)

### Q: When can EQ be toggled/changed?

| Option | Description | Selected |
|--------|-------------|----------|
| Hot-toggle while playing | Build element once; bypass via zeroed band gains; live tweaks | ✓ |
| Apply on next play only | EQ set before PLAYING; changes require stop/play | |

**User's choice:** Hot-toggle while playing (Recommended)

### Q: What happens if ParametricEQ.txt fails to parse?

| Option | Description | Selected |
|--------|-------------|----------|
| Toast error + keep previous profile | Show toast; retain prior active profile; user can retry | ✓ |
| Toast error + disable EQ | Toast and turn EQ off; discards prior config on bad file | |

**User's choice:** Toast error + keep previous profile (Recommended)

## UI Surface

### Q: Where does the EQ UI live?

| Option | Description | Selected |
|--------|-------------|----------|
| Dedicated EQ Dialog from hamburger menu | New 'Equalizer' entry; modal QDialog; matches existing pattern | ✓ |
| Section in a Settings dialog | Requires creating a new Settings dialog; larger scope | |
| Inline panel always visible in main window | Persistent EQ strip; clutters main window | |

**User's choice:** Dedicated EQ Dialog from hamburger menu (Recommended)

### Q: Should there be an EQ enable/disable button in the now-playing panel?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, small toggle in controls_box | Icon-only toggle next to Star/Pause/Stop; quick A/B | ✓ |
| No, toggle only inside the EQ dialog | Keeps NP panel clean; worse A/B UX | |

**User's choice:** Yes, small toggle in controls_box (Recommended)

### Q: Visualize bands inside the EQ dialog how?

| Option | Description | Selected |
|--------|-------------|----------|
| Vertical slider per band + frequency label | Classic graphic-EQ look; cheap to build | |
| Frequency-response curve graph (QPainter) | Compute combined filter response; custom painting | ✓ |
| Table of band parameters | QTableWidget listing freq/gain/Q per row | |

**User's choice:** Frequency-response curve graph (QPainter)

### Q: How involved should the response curve be?

| Option | Description | Selected |
|--------|-------------|----------|
| Static curve from parsed bands | ~256 log-spaced points; biquad magnitude formulas; redraw on change | ✓ |
| Static curve + manual band sliders below | Curve + editable slider row; two representations | |
| Live spectrum analyzer overlaid on curve | GStreamer spectrum tap + FFT overlay | |

**User's choice:** Static curve from parsed bands (Recommended)

### Q: Is the curve interactive or display-only?

| Option | Description | Selected |
|--------|-------------|----------|
| Display-only | No mouse interaction; preamp/bands editable via separate controls | ✓ |
| Draggable band points on the curve | Drag peaks to change gain/freq; pro-tool feel | |

**User's choice:** Display-only (Recommended)

## Profile Storage + Switching

### Q: How should imported AutoEQ profiles be stored?

| Option | Description | Selected |
|--------|-------------|----------|
| Copy file into eq-profiles/ dir, parse on load | Copy to ~/.local/share/musicstreamer/eq-profiles/; survives restart | ✓ |
| Parse on import, store bands as JSON in DB | Single source of truth in SQLite; less user-inspectable | |
| Reference external file by absolute path | Fragile; file could move/delete | |

**User's choice:** Copy file into eq-profiles/ dir, parse on load (Recommended)

### Q: Single active profile, or library with switcher?

| Option | Description | Selected |
|--------|-------------|----------|
| Library of profiles + switcher dropdown | Multiple profiles (different headphones); switch without re-import | ✓ |
| Single active profile, replace on import | Import overwrites current; simplest | |

**User's choice:** Library of profiles + switcher dropdown (Recommended)

### Q: What EQ state persists across restarts?

| Option | Description | Selected |
|--------|-------------|----------|
| Active profile + enable toggle + preamp | Three settings rows: eq_active_profile, eq_enabled, eq_preamp_db | ✓ |
| Active profile only; EQ enabled always on | Simpler; can't disable EQ across sessions | |

**User's choice:** Active profile + enable toggle + preamp (Recommended)

### Q: Can users delete imported profiles from the dialog?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, trash icon per profile + confirm | Same pattern as Favorites trash; removes file + clears active if needed | ✓ |
| No, users manage files manually outside the app | Smaller scope | |

**User's choice:** Yes, trash icon per profile + confirm (Recommended)

### Q: Include AutoEQ profiles in the settings export/import ZIP (Phase 42)?

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, include eq-profiles/ in ZIP | Users moving between machines keep their EQ setup | ✓ |
| No, EQ profiles are machine-local | Narrower scope; re-import needed on the work PC | |

**User's choice:** Yes, include eq-profiles/ in ZIP (Recommended)

## Band Editing + Preamp Behavior

### Q: How does the AutoEQ preamp value flow into the EQ?

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-apply from file + editable slider | Parse 'Preamp:' header and set slider; user can nudge | ✓ |
| Auto-apply only, read-only | User sees value but can't change; AutoEQ-purist | |
| Ignore file preamp, user adjusts manually | Risk of clipping if user forgets | |

**User's choice:** Auto-apply from file + editable slider (Recommended)

### Q: When an AutoEQ profile is loaded, can users tweak individual band gains?

| Option | Description | Selected |
|--------|-------------|----------|
| Read-only when profile loaded; 'Manual mode' unlocks | Display-only by default; toggle to edit; derives custom profile | ✓ |
| Always editable; 'Reset to profile' button | Bands always editable; Reset restores imported values | |
| Completely read-only, no manual editing | Only preamp slider editable | |

**User's choice:** Read-only when profile loaded; 'Manual mode' unlocks (Recommended)

### Q: If the user tweaks bands in Manual mode, how are changes persisted?

| Option | Description | Selected |
|--------|-------------|----------|
| Save as new profile with user-prompted name | Write new file to eq-profiles/; clean separation from AutoEQ originals | ✓ |
| Overwrite the active profile file | In-place edit; risk losing AutoEQ source | |
| Don't persist manual edits (session-only) | Tweaks die on app close; user rebuilds each session | |

**User's choice:** Save as new profile with user-prompted name (Recommended)

### Q: Preamp implementation detail — where does the gain apply?

| Option | Description | Selected |
|--------|-------------|----------|
| Uniform offset subtracted from every band gain | Stays within EQ element; composes with existing volume slider cleanly | ✓ |
| Separate volume compensation on playbin3 | Multiplies playbin3 volume; conflicts with AUDIO-01 slider arbitration | |

**User's choice:** Uniform offset subtracted from every band gain (Recommended)
