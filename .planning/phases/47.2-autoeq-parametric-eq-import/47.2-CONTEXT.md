# Phase 47.2: In-app parametric EQ with AutoEQ profile import - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Insert a parametric EQ into the GStreamer playback pipeline (`playbin3.audio-filter` slot) and provide a UI to import/activate AutoEQ `ParametricEQ.txt` profiles. Primary driver: the user's work PC where Equalizer APO is blocked by IT — MusicStreamer's in-app EQ is the only path to headphone correction on that machine.

**Already decided (carried forward from roadmap split 2026-04-17):**
- Profile scope: per-user (one active EQ), not per-station.
- Pipeline insertion point: `playbin3.audio-filter` (set before `set_state(PLAYING)`).
- EQ composes upstream of `playbin3` volume — no conflict with AUDIO-01 volume slider.
- Hamburger-menu pattern for secondary dialogs (matches Discover/Import/Accent/Cookies/Accounts).

**Out of scope** (deferred — flagged in SEED-007 and roadmap):
- Bundled AutoEQ profiles (no shipped corpus; user imports).
- Per-station EQ override.
- Dynamic range compression, convolution/FIR, spatial audio, other DSP filters.
- Auto-detect connected headphone (USB/BT) → auto-select profile.
- Live spectrum analyzer / FFT overlay on the curve graph.
- Click/drag interaction on the response curve.

</domain>

<decisions>
## Implementation Decisions

### EQ Engine
- **D-01:** Use GStreamer `equalizer-nbands` (from `gstreamer-plugins-good`) as the audio-filter element. Zero new dependencies. Cross-platform (Linux + Windows).
- **D-02:** Map AutoEQ filter rows 1:1 onto bands — `N bands` where `N` = number of filter rows parsed from `ParametricEQ.txt` (typically 5–10). Each band is configured with the AutoEQ row's `frequency`, `bandwidth` (derived from `Q`: `bw = freq / Q`), and `gain`.
- **D-03:** Treat AutoEQ `LS` (low-shelf) and `HS` (high-shelf) rows as wide peaking (`PK`) bands at the corner frequency — imperfect but keeps the engine single-path. Log a debug note when shelves are encountered so sound-quality regressions can be traced.
- **D-04:** Rebuild the `equalizer-nbands` element whenever the band count changes (new profile with different filter count). Within a single profile, mutate band properties on the existing element rather than recreating.

### EQ Lifecycle
- **D-05:** Hot-toggle while playing — construct the `equalizer-nbands` element once at pipeline setup and leave it resident in `audio-filter`. Enable/disable is implemented by zeroing every band gain (bypass) rather than rebuilding the pipeline. Profile swaps and preamp/band tweaks update band gains live.
- **D-06:** On parse failure, show a toast (`Failed to parse EQ profile: <reason>`) and keep the previous active profile intact. Never auto-disable EQ on a bad file.

### UI Surface
- **D-07:** New `EqualizerDialog` (Qt) launched from a "Equalizer" hamburger-menu action, inserted alongside Discover/Import/Accent/Cookies/Accounts.
- **D-08:** Add a small icon-only "EQ" toggle button in `now_playing_panel.controls_box` (next to Star/Pause/Stop) so the user can A/B-compare corrected vs. flat without opening the dialog. Button reflects `eq_enabled` state visually.
- **D-09:** EQ dialog centerpiece is a **static frequency-response curve** drawn with QPainter. Compute the combined magnitude response of all bands over a fixed grid of log-spaced points (20 Hz – 20 kHz, ~256 points) using standard biquad peaking-EQ magnitude formulas. Redraw on profile load, preamp change, or band edit (Manual mode).
- **D-10:** Curve is **display-only** — no click/drag interaction. Controls for preamp and (in Manual mode) per-band gain live as separate widgets below the curve.
- **D-11:** Dialog layout (top to bottom): profile switcher dropdown + Import + delete-trash icon for active row · active profile name · response-curve canvas · preamp slider (with dB label) · per-band slider row (read-only unless Manual mode toggled) · Manual/Reset toggle · Save-as-new-profile button (Manual mode only) · Close.

### Profile Storage
- **D-12:** Imported profiles are **copied into** `{user_data_dir}/eq-profiles/<safe_headphone_name>.txt` on import. Parse at load time. Survives restart and is user-inspectable. Uses the existing `paths.py` / `platformdirs.user_data_dir("musicstreamer")` root.
- **D-13:** Library model with switcher — the dialog lists all files in `eq-profiles/` in a dropdown; user picks active. Active profile is a single setting (see D-15). Multiple profiles (different headphones) supported from day one.
- **D-14:** Trash-icon delete per row in the switcher, with confirmation — deletes the file and clears `eq_active_profile` if the deleted one was active.
- **D-15:** Persist EQ state in SQLite `settings` table via the existing `set_setting`/`get_setting` pattern:
  - `eq_active_profile` — filename (basename only; full path reconstructed from `paths.user_data_dir() / eq-profiles /`)
  - `eq_enabled` — `"1"` / `"0"`
  - `eq_preamp_db` — float as string (e.g., `"-6.5"`)
- **D-16:** Extend Phase 42 settings export/import ZIP to include an `eq-profiles/` folder. `settings_export.py` writes the directory into the ZIP; `settings_import_dialog.py` + importer restores it alongside `settings.json` and `logos/`. Preserves the v2.0 "manual settings sync between machines" goal — user can move their EQ setup from Linux to the work PC.

### Band Editing + Preamp
- **D-17:** Auto-apply the `Preamp: X.X dB` header from the AutoEQ file into the preamp slider on import. User can adjust afterward. Preamp persists per session in `eq_preamp_db` (D-15). No per-profile preamp history (tweaks ride the currently-active profile).
- **D-18:** Preamp implementation: **uniform offset subtracted from every band's gain** before the gain is written to the `equalizer-nbands` band. Keeps all attenuation inside the EQ element — no conflict with `playbin3.volume` (AUDIO-01 slider).
- **D-19:** Per-band sliders are **read-only when an AutoEQ profile is loaded**. A "Manual" toggle in the dialog unlocks them for editing; toggling Manual on does NOT detach from the active file automatically — edits become pending until the user hits "Save as new profile".
- **D-20:** "Save as new profile" prompts for a profile name (`QInputDialog.getText`), writes a new `ParametricEQ.txt`-formatted file to `eq-profiles/<name>.txt`, and switches the active profile to the new file. Original imported profile is untouched. Manual tweaks are NEVER written back over the source file.

### Parser
- **D-21:** AutoEQ parser lives in a new module — `musicstreamer/eq_profile.py` — with two pure functions:
  - `parse_autoeq(text: str) -> EqProfile` — raises `ValueError` on malformed input.
  - `serialize_autoeq(profile: EqProfile) -> str` — for D-20 save-as-new.
- **D-22:** `EqProfile` is a dataclass: `{ preamp_db: float, bands: list[EqBand] }` where `EqBand = { filter_type: Literal["PK","LS","HS"], freq_hz: float, gain_db: float, q: float }`.
- **D-23:** Parse regex handles the canonical AutoEQ line format: `Filter N: ON PK Fc <freq> Hz Gain <gain> dB Q <q>` (and `OFF` filters are skipped). Case-insensitive on `PK`/`LS`/`HS`. Tolerate leading/trailing whitespace. Reject files with zero `ON` filters.

### Claude's Discretion
- Exact Qt geometry (slider lengths, dialog width, curve canvas height). Planner/executor to match the visual weight of other dialogs (`DiscoveryDialog`, `ImportDialog`).
- Icon choice for the NP-panel EQ toggle — pick a suitable icon from `icons/` or add a new one; match existing monochrome style.
- Whether the curve canvas is a custom `QWidget` subclass or `QGraphicsScene`-backed — QWidget + `paintEvent` is simpler and recommended.
- Threshold below which a band gain is rounded to zero (to avoid noisy sub-0.1 dB tweaks on the curve). Suggest ±0.05 dB.
- Test file layout — suggest `tests/test_eq_profile.py` (parser round-trip + malformed inputs) and `tests/test_equalizer_dialog.py` (widget smoke tests with pytest-qt).

### Folded Todos
None.

- **`Gst.ElementFactory.make` + playbin3 property-set pattern** at `player.py:77-85` — `audio-filter` slots in identically to the existing `audio-sink` assignment.
- **`set_setting`/`get_setting`** at `repo.py:348/354` — all three EQ settings (D-15) ride this existing contract.
- **Hamburger menu action + `_on_open_*` slot** at `main_window.py:120+` — mechanical copy for the Equalizer entry.

- **Settings ZIP extension points** — Phase 42's `settings_export.py` + `settings_import_dialog.py` are the clean seams to add `eq-profiles/`.

### Established Patterns
- Dialogs are standalone `QDialog` subclasses under `musicstreamer/ui_qt/`, opened from `main_window` via `_on_open_<name>` slots that construct-and-`exec()`. Follow for `EqualizerDialog`.
- Settings rows are string-typed via `set_setting`/`get_setting`. Cast at read site (e.g., `float(get_setting("eq_preamp_db", "0.0"))`).
- Parsers live in top-level `musicstreamer/` modules (`aa_import.py`, `yt_import.py`); follow for `eq_profile.py`.
- Pure-function modules get table-driven unit tests under `tests/` (e.g., `tests/test_aa_import.py`); `eq_profile.py` parser tests follow the same shape.
- User-data paths use `paths.user_data_dir()` — do NOT hardcode `~/.local/share/musicstreamer/...`.

### Integration Points
- **Pipeline:** one `set_property("audio-filter", equalizer_nbands_element)` call in `Player.__init__` between the existing `audio-sink` assignment and `buffer-duration`.
- **Settings:** three new keys in the existing settings table — zero schema changes.
- **Main window menu:** one `self._menu.addAction("Equalizer")` + slot — follows the existing group ordering (grouped with Discover/Import/Accent/Cookies/Accounts).
- **Now-playing controls_box:** one new `QToolButton` added to the existing horizontal layout.
- **Settings ZIP:** `eq-profiles/` directory added to the export writer and import parser alongside `settings.json` + `logos/`.

<specifics>
## Specific Ideas

- **One-click A/B:** NP-panel EQ toggle button (D-08) is explicitly called out in SEED-007 as a UX win for testing new profiles — worth shipping on day one.
- **Preamp-as-offset** (D-18) avoids arbitration with the existing volume slider (AUDIO-01). If a future phase adds a `Gst.ElementFactory volume` element between EQ and sink, preamp can migrate there; not now.
- **Biquad magnitude formula** for the curve graph (D-09): use the standard RBJ cookbook peaking-EQ formulas. No scipy dependency.
- **Deletion confirmation** (D-14) matches the existing Favorites trash UX.
- **Work-PC first** (per SEED-007): Windows parity matters. `equalizer-nbands` is cross-platform — no Windows-specific plumbing. Verify with `gst-inspect-1.0 equalizer-nbands` during research.

</specifics>

</deferred>
