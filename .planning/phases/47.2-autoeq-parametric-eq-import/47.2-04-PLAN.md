## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-47.2.04-01 | Tampering | ZIP `eq-profiles/../evil.txt` traversal (Pitfall 8) | mitigate | Reuse `_validate_zip_members` (lines 62-82) at the existing call site (preview_import line 211 + commit_import line 282). New extraction loop operates on pre-validated `zip_names` set. Defense verified by `test_eq_profiles_path_traversal_rejected`. |
| T-47.2.04-02 | Tampering | Non-`.txt` ZIP member under `eq-profiles/` | mitigate | Extraction conditional requires both `name.startswith("eq-profiles/")` AND `name.endswith(".txt")`; all other entries skipped. Directory entries (empty basename) skipped. |
| T-47.2.04-03 | Information Disclosure | EQ profiles in exported ZIP | accept | Profiles are the user's own headphone-correction data. No PII, no credentials. Credentials already excluded by `_EXCLUDED_SETTINGS`; EQ profiles do not fall into that category by design (D-16 — they SHOULD travel with the export). |
| T-47.2.04-04 | Denial of Service | Large or duplicate eq-profiles in ZIP | accept | Profiles are kilobytes. Python `zipfile` handles de-dup silently (last-write-wins). Not exploitable. |
| T-47.2.04-05 | Tampering | eq_enabled setting manually edited to something other than "1"/"0" | mitigate | NowPlayingPanel reads via `== "1"`; anything else evaluates False. Player's `set_eq_enabled(bool)` coerces via `bool()`. No code path depends on exotic values. |
