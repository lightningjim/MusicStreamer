## Interaction Contract

### ART-01: Bulk Import Logo Download

1. User clicks "Import Stations" on the AA tab — `_aa_import_btn` (existing trigger).
2. Progress label becomes visible; stack transitions to "importing" (spinner visible).
3. Label shows `Importing stations… (N/total)` as stations are inserted.
4. After insert loop completes, label transitions to `Fetching logos…` while parallel logo downloads run in a thread pool.
5. When all downloads settle (success or silent fail), label transitions to `Done — N imported, M skipped` and spinner stops.
6. Dialog remains open; user can close manually.

### ART-02: Editor Focus-Out Auto-Fetch

1. User pastes an AA stream URL into `url_entry` and tabs out (focus-out fires `_on_url_focus_out`).
2. If stored API key exists: `_art_stack` transitions to "spinner" immediately (same as YouTube path). Logo fetch runs in daemon thread. On callback: stack returns to "pic" with logo loaded, or "pic" with previous image if fetch failed (silent).
3. If no stored API key: "Fetch from URL" button behavior — popover appears on button click (not on focus-out). Popover contains single text entry + "Fetch" confirm button. On confirm: key saved to `repo` settings, logo fetch proceeds. On failure: popover closes, no error shown.
4. "Fetch from URL" button click always triggers logo fetch for AA URLs (same as focus-out path, after key check).

### Spinner / Stack Toggle

Use existing `_art_stack` / `_art_spinner` in `edit_dialog.py` — no new widgets. Pattern:
- Show spinner: `_art_stack.set_visible_child_name("spinner")` + `_art_spinner.start()`
- Hide spinner: `_art_spinner.stop()` + `_art_stack.set_visible_child_name("pic")`
