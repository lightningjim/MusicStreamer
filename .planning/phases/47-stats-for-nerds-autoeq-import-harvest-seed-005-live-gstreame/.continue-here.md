## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Extending `repo.insert_station()` signature for bitrate | Naive fix would widen `insert_station` to accept streams-with-bitrate. Existing callers (6+) would break. | advisory | Mirror `aa_import.import_stations_multi` post-insert fix-up: call `insert_station()` unchanged, capture returned `station_id`, then call `update_stream(station_id, ..., bitrate_kbps=N)` per stream. This is explicitly called out in 47-PATTERNS.md critical callout #4 and 47-RESEARCH.md G-2. |
| Silent-drop on settings-export serialization | Adding `bitrate_kbps` to StationStream WITHOUT updating `_station_to_dict` means exported ZIPs silently lose the field on round-trip. | advisory | `settings_export.py::_station_to_dict` at line 101 must emit `"bitrate_kbps": s.bitrate_kbps`. `_insert_station` + `_replace_station` tuples must be 8 columns. Import path must use `.get("bitrate_kbps", 0)` for forward-compat with pre-47 ZIPs. |
| Backwards-compat via try/except-only | Schema migration solely via try/except ALTER TABLE is fine (idempotent), but forgetting to also add the column to CREATE TABLE means fresh DB installs skip it. | advisory | Plan must modify BOTH CREATE TABLE in db_init AND add the ALTER TABLE migration block. |
| Empty QLineEdit → int() parse error | QIntValidator allows `""` as an intermediate state. `int(cell.text())` on an empty cell raises ValueError. | advisory | Use `int(text or "0")` per CONTEXT D-14. Mirror the existing defensive integer coerce at `edit_station_dialog.py:677-680`. |
