<objective>
Fix UAT gap 1: clearing a Bitrate cell and pressing Enter/Tab must persist as bitrate_kbps=0 instead of reverting to the prior value. Root cause: `_BitrateDelegate` (at `musicstreamer/ui_qt/edit_station_dialog.py:155`) defines only `createEditor` — it inherits `QStyledItemDelegate.setModelData`, whose default behavior for a `QLineEdit` editor writes the editor's text back via the USER property. With an empty string the default path may be inconsistent (the QLineEdit USER property returns an empty QString, which some Qt paths treat as "no change"). An explicit `setModelData` that unconditionally writes `editor.text()` to `Qt.EditRole` guarantees empty text lands in the table item, after which the existing save path (`int(bitrate_text or "0")` at line 717 — D-14, already shipped in 47-03) coerces to 0.

Purpose: Close UAT gap 1 (severity: major). The empty-cell → 0 contract was ALREADY codified by `test_empty_bitrate_saves_as_zero` (PB-17), but that test bypasses the delegate by calling `item.setText("")` directly. The real user flow goes through the delegate's setModelData, which silently drops the empty string. This plan adds the delegate override and a delegate-path regression test.

Output: One override method on `_BitrateDelegate`; one new regression test that drives the full delegate edit cycle.
</objective>
