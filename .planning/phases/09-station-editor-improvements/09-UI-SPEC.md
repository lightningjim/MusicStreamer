## Color

This project has no custom CSS file. All colors are Adwaita system tokens.

| Role | Adwaita Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `@window_bg_color` | Dialog window background |
| Secondary (30%) | `@card_bg_color` | Form area, chip scroll container |
| Accent (10%) | `suggested-action` class | Save button in HeaderBar only |
| Destructive | `destructive-action` class | Delete Station button in HeaderBar only |

Accent (`suggested-action`) reserved for: **Save button only.**

Destructive (`destructive-action` / `Adw.ResponseAppearance.DESTRUCTIVE`) reserved for:
- Delete Station button (HeaderBar)
- "Delete" response in the delete confirmation `Adw.MessageDialog`

Chip active state: Adwaita default `Gtk.ToggleButton` active appearance (no custom color —
uses system `@accent_color` automatically when toggled). No override needed.

New-provider entry: plain `Gtk.Entry` with placeholder text — no accent color. Source:
CONTEXT.md D-02, RESEARCH.md architecture section.

## Component Inventory

### Provider Row (replaces `provider_entry` at grid row 2)

| Widget | Configuration |
|--------|---------------|
| `Adw.ComboRow` | `title="Provider"`, `model=Gtk.StringList` (blank + existing providers), `set_enable_search(True)` |
| `Gtk.Entry` (below ComboRow) | `placeholder_text="Or type new provider name…"` — used only when creating a new provider not in the list |

Layout: ComboRow attached to `form` grid at row 2, column 1. The new-provider Entry
is appended to a `Gtk.Box(orientation=VERTICAL, spacing=4)` that also contains the
ComboRow, and this Box replaces `provider_entry` in the grid cell.

Pre-select current provider by scanning model items and calling `set_selected(i)`.
Prepend `""` (blank) as index 0 to represent "No provider."

Case-insensitive match on save: compare `new_provider_entry.get_text().strip().casefold()`
against existing provider names; if match found, use existing provider ID. This is
Claude's discretion (CONTEXT.md).

### Tag Multi-Select Panel (replaces `tags_entry` at grid row 3)

| Widget | Configuration |
|--------|---------------|
| `Gtk.ScrolledWindow` | `set_policy(AUTOMATIC, NEVER)`, `set_min_content_height(36)` |
| `Gtk.Box` (chip_box, inside scroll) | `orientation=HORIZONTAL`, `spacing=8` |
| `Gtk.ToggleButton` (per tag) | `label=tag_name`, `set_active(tag in _selected_tags)` |
| `Gtk.Entry` (below scroll) | `placeholder_text="New tag…"` |
| Outer `Gtk.Box` (tags_box) | `orientation=VERTICAL`, `spacing=6` |

Layout: `tags_box` contains `chip_scroll` (top) and `new_tag_entry` (bottom).
`tags_box` replaces `tags_entry` in the grid at row 3, column 1.

Chip scroll vertical margins: `set_margin_top(4)`, `set_margin_bottom(4)`,
`set_margin_start(8)`, `set_margin_end(8)` — matches main_window chip scroll pattern
(source: `main_window.py` lines 136-151).

Chip box spacing: `spacing=8` — matches `_provider_chip_box` and `_tag_chip_box`
in `main_window.py`.

### YouTube Title Fetch (async, no new visible widget)

No new visible widget. On URL focus-out for a YouTube URL, name field is populated
silently if empty or equals "New Station". A spinner is not shown for the title fetch
(only thumbnail fetch shows spinner). The name entry update is the only feedback.

| State | Feedback |
|-------|----------|
| Title fetch in progress | None visible (background only) |
| Title fetched, name was blank | Name entry populated with fetched title |
| Title fetched, name already set | No change |
| Title fetch failed | No change, no error shown |

Source: CONTEXT.md D-05, D-06.

## Interaction Contracts

### Provider ComboRow

| Interaction | Behavior |
|-------------|----------|
| Open dialog | ComboRow pre-selected to current provider (or blank if none) |
| Select from list | New-provider Entry remains empty; ComboRow selection used on save |
| Type in new-provider Entry | Entry value takes precedence over ComboRow selection on save |
| Leave new-provider Entry blank | ComboRow selection is used |
| New-provider value matches existing (case-insensitive) | Treated as existing provider — no duplicate created |
| Select blank item in ComboRow | Provider set to None on save |

### Tag Chip Panel

| Interaction | Behavior |
|-------------|----------|
| Open dialog | All existing tags shown as chips; current station tags shown as active (toggled) |
| Click active chip | Tag removed from `_selected_tags`; chip deactivated |
| Click inactive chip | Tag added to `_selected_tags`; chip activated |
| Type in new-tag Entry + press Enter (or save) | Tag appended to final tag set |
| Save with empty new-tag Entry | Only `_selected_tags` persisted |

Guard: `_rebuilding` flag wraps bulk chip mutations to prevent spurious
`_on_tag_chip_toggled` calls during initialization (mirrors `_rebuild_filter_state`
pattern in `main_window.py`).

### Delete Station (existing, unchanged)

| State | Behavior |
|-------|----------|
| Station currently playing | `Adw.MessageDialog` — "Cannot Delete Station" / "Stop playback before deleting this station." / "OK" |
| Station not playing | `Adw.MessageDialog` — "Delete {name}?" / "This station will be permanently removed." / "Keep Station" (default) + "Delete" (destructive) |

Source: existing `edit_dialog.py` lines 172–203. No change in Phase 9.
