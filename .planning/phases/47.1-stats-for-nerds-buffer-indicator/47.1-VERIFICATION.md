### Human Verification Required

Phase 47.1 involves visual layout, real-time GStreamer behavior, cross-restart persistence, and menu-group placement — none programmatically verifiable in the headless test environment.

#### 1. Visual fidelity — bar sits at label baseline, 120px wide (D-01/D-02)

**Test:** Start the app. Enable "Stats for Nerds" via hamburger menu. Play a live network stream.
**Expected:** 120px progress bar sits at the baseline of the "Buffer" row label, with `{N}%` label immediately to the right, inside the center column below the controls row.
**Why human:** Layout geometry + visual baseline alignment cannot be measured programmatically in a headless test run.

#### 2. Live bar fill during real playback

**Test:** Tune to a SomaFM (or similar) live stream and observe the buffer bar on first play.
**Expected:** Bar advances 0→100% over a few seconds during initial buffer fill (queue2 low/high watermark crossings), then stays at 100% once steady state is reached.
**Why human:** Requires a running GStreamer pipeline and live network I/O; not reproducible in the test harness.

#### 3. Persistence across app restart (D-04)

**Test:** Toggle Stats for Nerds ON → close app → relaunch. Then toggle OFF → close → relaunch.
**Expected:** ON state → action starts checked, row visible on launch. OFF state → action starts unchecked, row hidden on launch.
**Why human:** Tests persistence across full-process lifecycle, not just repo `set_setting`/`get_setting` round-trip (which is covered by unit tests).

#### 4. Muted label color renders correctly in both themes (D-10)

**Test:** With Stats for Nerds visible, inspect the "Buffer" row label in both light and dark system themes.
**Expected:** Row label reads noticeably dimmer than adjacent Name/ICY/elapsed labels, but still legible.
**Why human:** Color perception / theme rendering verification cannot be asserted without a visual comparison.

#### 5. Menu group isolation (D-03)

**Test:** Open the hamburger menu.
**Expected:** "Stats for Nerds" sits in its own group with a separator above (after Accounts) and below (before Export Settings).
**Why human:** Visual separator placement confirmation.

#### 6. Cross-stream sentinel reset (D-14 + Pitfall 3)

**Test:** Play stream A until buffer bar reaches 100%; switch to a different station (stream B).
**Expected:** Bar resets and fills again for stream B. Does NOT stick at A's last value.
**Why human:** Requires two real streams and live transition behavior; de-dup reset unit test covers the code path, but only human testing confirms the observable end-to-end behavior.
