# Phase 47.1: Stats for nerds — buffer-fill indicator - Context

**Gathered:** 2026-04-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Wire a live GStreamer buffer-fill percent indicator (0–100) into the now-playing
panel so the user can see stream health at a glance. Source of truth: GStreamer's
`message::buffering` bus signal on the shared `playbin3` pipeline. Surface: a new
"stats for nerds" row inside `NowPlayingPanel`, gated by a hamburger-menu toggle.

**In scope:**
- `player.py`: subscribe to `bus.connect("message::buffering", self._on_gst_buffering)`;
  emit a class-level Qt `Signal(int)` carrying the percent (0–100) for the UI layer.
- `now_playing_panel.py`: add an extensible "stats for nerds" row that renders the
  buffer percent as a mini progress bar + `{N}%` label. Hidden by default; visibility
  driven by the `show_stats_for_nerds` setting.
- `main_window.py`: add a checkable "Stats for Nerds" action to the hamburger menu;
  persist state via `repo.set_setting("show_stats_for_nerds", "0|1")`; wire the
  toggle to `NowPlayingPanel` via an existing Qt signal or method call.

**Out of scope (ROADMAP-deferred):**
- Codec label, sample rate, ICY bitrate, stream URL, failover status row. Captured
  as follow-up in the extensible row container, but not populated this phase.

</domain>

<decisions>
## Implementation Decisions

### Display Form
- **D-01:** Buffer percent renders as **mini progress bar + numeric label** side-by-side
  (preview: `[███████░░░] 70%`). Progress bar uses `QProgressBar` with `range(0, 100)`
  and `setTextVisible(False)`; the `{N}%` is a separate `QLabel` to the right. Both
  share a single row inside the stats widget.
- **D-02:** Progress bar width is small and fixed (~120 px) so the row doesn't dominate
  the center column. Height matches the adjacent label baseline.

### Visibility Toggle
- **D-03:** Toggle mechanism is a **checkable `QAction`** in the existing hamburger
  menu, placed below the "Accounts" item in a new group (new separator, new group).
  Label: "Stats for Nerds". Checked state reflects the current setting on startup.
- **D-04:** Persistence key: `show_stats_for_nerds` in the `settings` key/value table
  via `repo.get_setting(...)` / `repo.set_setting(...)`. Values: `"0"` (hidden,
  default) and `"1"` (visible). Read at panel construction; written on toggle.
- **D-05:** Default state is **hidden** (`"0"`). First-time users don't see the row
  until they opt in. Matches the roadmap "hidden by default" note.
- **D-06:** No settings dialog required for this phase. If future stat rows need
  per-row toggles, that's a follow-up phase's concern.

### Panel Placement
- **D-07:** Stats row lives in the **center column of `NowPlayingPanel`, below the
  existing play/pause/controls row**. Sits as the last item in the center `QVBoxLayout`.
  When `show_stats_for_nerds == "0"`, the whole stats widget is `setVisible(False)` —
  no layout shuffle, no disclosure triangle.
- **D-08:** The stats widget is always **constructed** (not lazily created) so toggling
  is instant. It's just hidden by default.

### Future-Stat Hook (Extensibility)
- **D-09:** The stats widget uses a **`QFormLayout`** (label → value rows) so future
  stats (codec, URL, failover count, etc.) are a 1-line addition: `form.addRow("Codec",
  codec_label)`. The buffer row is the only populated row this phase.
- **D-10:** Row labels use a **muted/secondary color** (from the theme tokens
  introduced in Phase 46 — `palette().color(QPalette.Disabled, ...)` or whatever the
  existing secondary-text token is). Researcher should confirm the exact token.
- **D-11:** The buffer label is named so it can be updated from the player signal
  slot: one `QLabel` bound to the slot, one `QProgressBar` bound to the slot. Both
  updates happen in a single slot method.

### Player → UI Wiring
- **D-12:** `Player` exposes a new class-level Signal: `buffer_percent = Signal(int)`
  (beside existing `title_changed`, `elapsed_updated`, etc.). Emits 0–100.
- **D-13:** `MainWindow` connects `self._player.buffer_percent` to a
  `NowPlayingPanel.set_buffer_percent(int)` slot (bound method, no lambda) at the
  same wiring site where other player signals get connected.
- **D-14:** Buffer percent parsing uses `msg.parse_buffering()` on the `GstMessage`
  inside `_on_gst_buffering`. Emit the Signal with the returned int. Do not emit
  redundant values — track last emitted and skip if unchanged (avoids UI churn on
  GStreamer's chatty bus).

### Claude's Discretion
- **Update throttling:** GStreamer emits `message::buffering` whenever the fill level
  crosses a threshold (default every 1%). D-14's "skip if unchanged" de-dup handles
  the redundant case. No further throttling (e.g., `QTimer.singleShot` coalescing)
  needed — the signal is cheap enough. Planner may revisit if UI feels laggy.
- **Hide-when-full behavior:** Whether the row should auto-hide/gray when buffer
  reaches 100% is not locked. Claude defaults to "always visible when toggle is on,
  showing 100%" — a steady-state 100% is itself informative (stream is healthy).
- **Exact row sizing / fonts / margins:** standard Qt defaults; follow patterns from
  existing rows in `NowPlayingPanel`.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **Class-level Qt Signals on `Player`**: pattern already established at
  `player.py:60-68` — add `buffer_percent = Signal(int)` the same way.
- **`bus.connect("message::*", handler)` pattern**: already present at `player.py:91`
  for `message::error` and `message::tag`. `message::buffering` slots in identically.
- **Hamburger menu checkable actions**: `main_window.py:119` menu builder — easy to
  add another `QAction` with `setCheckable(True)`.
- **`repo.get_setting/set_setting`**: key/value string storage already used for
  `accent_color` and volume — same pattern for `show_stats_for_nerds`.

### Established Patterns
- Bound-method signal connections (no self-capturing lambdas) per QA-05 — keep this
  for the new wiring.
- `ICY` label uses `Qt.PlainText` for untrusted content — buffer percent is
  integer-only so no escaping concerns.

### Integration Points
- `Player._on_gst_buffering` is a new instance method (parallel to `_on_gst_error`)
  that parses the message and emits `buffer_percent`.
- `MainWindow` wiring site (wherever `title_changed`, `elapsed_updated` are connected)
  gains one more `.connect(...)` call.
- `NowPlayingPanel.__init__` builds the stats widget (always); reads
  `show_stats_for_nerds` at construction to set initial visibility.
- `NowPlayingPanel.set_buffer_percent(int)` is the new slot bound to the Signal.
- `NowPlayingPanel.set_stats_visible(bool)` (or equivalent) is called from
  `MainWindow` when the hamburger action toggles.

</code_context>
