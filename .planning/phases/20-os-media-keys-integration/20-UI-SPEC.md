## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Vertical spacing in center box (`spacing=4`) |
| sm | 8px | Filter box spacing, margin top/bottom on filter rows |
| md | 16px | Panel margin top/bottom |
| lg | 24px | Panel margin start/end |
| controls | 6px | `controls_box` horizontal spacing between buttons |
| label | 12px | Label margin start in station rows |

Exceptions:
- `controls_box` uses `spacing=6` — not a standard 4-point multiple, matches existing stop/star button gap. The new pause button follows this same 6px gap (no change).
- Touch target minimum: icon-only buttons inherit GTK4 default minimum 34px; no explicit override needed.

Source: codebase grep on `main_window.py` margin/spacing calls.

## Interaction Contract

### Pause Button

| State | Icon | Tooltip | Sensitive |
|-------|------|---------|-----------|
| No station playing | `media-playback-pause-symbolic` | "Pause" | False |
| Playing | `media-playback-pause-symbolic` | "Pause" | True |
| Paused | `media-playback-start-symbolic` | "Resume" | True |
| Stopped (after stop_btn) | `media-playback-pause-symbolic` | "Pause" | False |

Button position: inserted in `controls_box` between `star_btn` and `stop_btn`.

Source: CONTEXT.md D-01, D-02, D-03, D-05.

### Now-Playing Panel Visibility

- Playing → panel visible, all metadata shown
- Paused → panel stays visible, all metadata unchanged (no "Paused" overlay, no dimming)
- Stopped → panel hidden (existing behavior, unchanged)

Source: CONTEXT.md D-03.

### Stop Button

Behavior unchanged. Calling stop from paused state resets pause silently — no confirmation needed (not a destructive action from user data perspective).

Source: CONTEXT.md D-07.

### MPRIS2 D-Bus (no visible UI)

No visual elements added. MPRIS2 is a background service. The only user-visible effect is that OS media keys (keyboard play/pause, stop) trigger the same state transitions as the in-app buttons.
