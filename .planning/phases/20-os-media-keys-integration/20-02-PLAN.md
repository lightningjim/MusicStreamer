## Trust Boundaries

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03 | Spoofing | D-Bus method calls | accept | Session D-Bus is user-scoped; any same-user process can call MPRIS2 methods. This is standard MPRIS2 behavior for all Linux media players. |
| T-20-04 | Tampering | PlayPause/Stop dispatch | mitigate | All D-Bus method handlers dispatch to GTK main thread via GLib.idle_add() — never directly modify widget state from D-Bus thread |
| T-20-05 | DoS | Rapid PlayPause calls via D-Bus | accept | Each call is lightweight (pipeline state change). No amplification. Same-user access only. |
| T-20-06 | Information Disclosure | Metadata properties | accept | Station name and ICY title are not sensitive. Exposed only on session bus (same user). |
</threat_model>
