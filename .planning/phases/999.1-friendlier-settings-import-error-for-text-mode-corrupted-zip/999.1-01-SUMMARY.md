# Phase 999.1 Plan 01: Text-mode corruption detection + unit tests Summary

**One-liner:** Two-tier BadZipFile handling with raw-byte CRLF detection (PK magic + 0x0D0A) raising typed TextModeCorruptionError for text-mode transfers, generic ValueError+hint for all other bad ZIPs.

## What Was Built

**`musicstreamer/settings_export.py`:**
- `TextModeCorruptionError(ValueError)` — typed exception class with docstring noting LF->CRLF transfer corruption and binary-mode recovery. Subclasses ValueError so existing broad handlers still treat it as an import-validation error (D-03).
- `_is_text_mode_corrupted_zip(path: str) -> bool` — raw-byte detection helper. Opens in binary mode, reads full file, returns `data[:4] == b"PK\x03\x04" and b"\x0d\x0a" in data`. Returns False on OSError. No decompression, no zipfile parsing, no byte-rewriting (D-01, T-999.1-01, T-999.1-02).
- Extended `except zipfile.BadZipFile` block now covers both `ZipFile()` open and `zf.read()` member reads. Two-tier: text-mode detection → `TextModeCorruptionError("TEXT_MODE_ZIP")`; otherwise → `ValueError("Not a valid ZIP archive — the file may be incomplete or corrupted.")` (D-03).

**`tests/test_settings_export.py`:**
- Extended import line to include `TextModeCorruptionError` and `_is_text_mode_corrupted_zip`.
- `_make_crlf_mangled(src_bytes)` helper: replaces lone 0x0A with 0x0D 0x0A to simulate text-mode transfer.
- 4 new headless tests (no Qt imports): `test_crlf_detection_text_mode_zip` (DET-01), `test_crlf_detection_truncated_zip` (DET-02), `test_crlf_detection_non_zip` (DET-03), `test_preview_raises_text_mode_error` (RAISE-01).

**Final test count:** 39 tests (35 existing + 4 new). All green.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Extended try/except scope to cover member-read BadZipFile**
- **Found during:** Task 2 — RAISE-01 test
- **Issue:** A CRLF-mangled zip can open successfully at `zipfile.ZipFile(zip_path, "r")` (the central directory at the end of the file is intact enough for Python's zipfile to parse it), but fails with `BadZipFile` when reading a member (`zf.read("settings.json")`) because the local file headers are corrupted. The original except block only wrapped the `ZipFile()` call, so the member-read `BadZipFile` propagated uncaught.
- **Fix:** Moved `with zf_handle as zf:` block inside the try block, with `except (ValueError, KeyError): raise` before `except zipfile.BadZipFile:` to pass through deliberate ValueError/KeyError raises from the content validation logic unchanged.
- **Files modified:** musicstreamer/settings_export.py
- **Commit:** 59152da7
