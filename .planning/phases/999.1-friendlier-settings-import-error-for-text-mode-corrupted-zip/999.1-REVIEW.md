This phase adds text-mode-corruption detection to the settings import path: a new `TextModeCorruptionError(ValueError)` exception class, an `_is_text_mode_corrupted_zip()` detection helper, a branched `except zipfile.BadZipFile` handler in `preview_import`, a sentinel-discriminated `except` clause in `_ImportPreviewWorker.run()`, and a `QMessageBox.warning` dialog branch in `_on_import_preview_error`.

The core logic is correct. ## Warnings

### WR-01: Unbounded file read in `_is_text_mode_corrupted_zip` can exhaust RAM on large non-ZIP input

**File:** `musicstreamer/settings_export.py:66`

**Issue:** `data = fh.read()` reads the entire file into memory without any size limit. This function is called after `zipfile.BadZipFile` is raised, which means the caller already confirmed the file is not a valid ZIP archive. A user who accidentally selects a large non-ZIP file (e.g., a 4 GB ISO or disk image) via the `QFileDialog.getOpenFileName` filter (which is enforced weakly on Linux) will cause the process to attempt to read the entire file into RAM before the detection returns `False`.

The DEFLATE guard described in the docstring ("Settings ZIPs are small, typically <500 KB") applies to legitimate settings exports, but the check is called on any file that fails to open as a ZIP, regardless of origin. The `QFileDialog` filter `"ZIP Archive (*.zip)"` is advisory on Linux/macOS and does not prevent large non-ZIP files from being passed.

**Fix:** Cap the read at a safe upper bound. The detection only needs the first 4 bytes for the magic check; scanning for `0x0D0A` requires reading further, but a few megabytes is more than sufficient for any realistic settings ZIP. A streaming approach is ideal:

```python
_MAX_DETECTION_BYTES = 4 * 1024 * 1024  # 4 MB ceiling

def _is_text_mode_corrupted_zip(path: str) -> bool:
    try:
        with open(path, "rb") as fh:
            data = fh.read(_MAX_DETECTION_BYTES)
    except OSError:
        return False
    return data[:4] == b"PK\x03\x04" and b"\x0d\x0a" in data
```

This preserves 100% detection fidelity for any real settings ZIP (which fit in well under 4 MB) while bounding memory use for pathological inputs. If a future requirement to handle very large ZIPs arises, a chunked scan can replace the bounded read at that point.

## Info

### IN-01: No test asserts `TextModeCorruptionError` is a subclass of `ValueError`

**File:** `tests/test_settings_export.py` (new tests block, lines 1444–1492)

**Issue:** The docstring for `TextModeCorruptionError` (settings_export.py:29–37) documents a specific contract: "Subclasses `ValueError` so existing broad handlers still treat it as an import-validation error." This is a load-bearing property — if the inheritance were accidentally changed (e.g., to inherit only from `Exception`), the broad `except ValueError` handlers in callers outside the test suite would silently stop catching it. None of the four new tests verify this relationship.

**Fix:** Add a one-line assertion to the detection test file:

```python
def test_text_mode_corruption_error_is_value_error():
    """TextModeCorruptionError must subclass ValueError per D-03 contract."""
    assert issubclass(TextModeCorruptionError, ValueError)
```

This guards the inheritance contract against future refactoring without requiring Qt or any I/O.

## Adversarial Pass — Refuted Candidates

The following candidate findings did not survive adversarial refutation and are excluded from the report:

**"TextModeCorruptionError caught by `except (ValueError, KeyError)` before reaching caller"** — Refuted. `TextModeCorruptionError` is raised inside the `except zipfile.BadZipFile` handler body, not inside the `try` block. Python does not re-route exceptions raised inside an `except` clause through sibling `except` clauses in the same compound statement. The exception exits the entire `try-except` and propagates normally. No ordering hazard exists.

**"Sentinel string `TEXT_MODE_ZIP` could be confused with a real error message"** — Refuted. The sentinel is matched with `==` (not `in`), so a genuine error message beginning with those characters would need to be exactly `"TEXT_MODE_ZIP"` with no other text. No exception in the code path produces that exact string other than the intentional emit at line 124. The pattern mirrors the pre-existing `"auth_expired"` idiom.

**"RAISE-01 test is tautological or fragile"** — Refuted. `pytest.raises(TextModeCorruptionError)` correctly distinguishes the subclass from its parent `ValueError` — it would not pass if only a plain `ValueError` were raised. The mangled ZIP produced by `_make_crlf_mangled` on real DEFLATE-compressed content reliably triggers `zipfile.BadZipFile` because CRLF injection corrupts the ZIP's internal byte-offset fields.

**"DET-02 test data might accidentally contain `0x0D0A`"** — Refuted. The filler bytes `\x00\x01\x02\x03\x04\x05\x06\x07\x08\x09\x0b\x0c\x0e\x0f` explicitly skip both `0x0A` (10) and `0x0D` (13). Neither byte appears in the data, so `b"\x0d\x0a" in data` is `False` by construction.

**"Except ordering in `_ImportPreviewWorker.run()` is wrong"** — Refuted. `except settings_export.TextModeCorruptionError` appears at line 120, before `except Exception` at line 125. The more specific subclass handler is first, which is correct Python style and prevents the generic handler from swallowing the sentinel-bearing exception.
