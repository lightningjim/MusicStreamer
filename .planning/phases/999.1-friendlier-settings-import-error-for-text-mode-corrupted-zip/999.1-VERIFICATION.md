# Phase 999.1: Friendlier Settings-Import Error for Text-Mode Corrupted ZIP — Verification Report

**Phase Goal:** When a settings backup `.zip` fails to open (`zipfile.BadZipFile` in `settings_export.preview_import`), detect the signature of a text-mode/CRLF-corrupted transfer (starts with `PK` zip magic but raises `BadZipFile`, with injected `0x0D 0x0A` runs) and surface an actionable message: "This backup looks corrupted by a text-mode file transfer. Re-copy it in binary mode and try again." Scope is DETECTION + GUIDANCE ONLY — no auto-repair. File-picker settings-import path only; Flatpak wizard excluded.
