## Layout Contract (Phase-Specific)

This phase is exclusively a layout fix â€” the primary deliverable is replacing horizontal ScrolledWindow chip containers in the main window filter bar with wrapping FlowBoxes.

### Problem

`_provider_scroll` and `_tag_scroll` (`Gtk.ScrolledWindow` + horizontal `Gtk.Box`) expand horizontally when many providers or tags exist, overflowing past adjacent buttons (Add Station, Edit, Clear). Chips scroll off-screen rather than wrapping.

### Required Fix: FlowBox (Decision D-01 + D-02 from CONTEXT.md)

**Decision:** Replace both `Gtk.ScrolledWindow` + `Gtk.Box` pairs with `Gtk.FlowBox`. Same pattern as Phase 24 edit dialog fix. Keep provider chips and tag chips in separate FlowBoxes (D-02).

**FlowBox properties (both `_provider_flow` and `_tag_flow`):**
- `orientation = Gtk.Orientation.HORIZONTAL`
- `selection_mode = Gtk.SelectionMode.NONE` (selection handled by ToggleButton state, not FlowBox)
- `column_spacing = 8`
- `row_spacing = 4`
- `homogeneous = False`
- `hexpand = False` (critical â€” prevents overflow past adjacent buttons)
- `margin_top = 4`, `margin_bottom = 4`, `margin_start = 8`, `margin_end = 8` (preserve existing scroll margins)
- Min height: not set (natural height from rows)

**Remove:** `_provider_scroll`, `_tag_scroll` (both `Gtk.ScrolledWindow`), and `_provider_chip_box`, `_tag_chip_box` (both horizontal `Gtk.Box`). Replace with `self._provider_flow` and `self._tag_flow` as `Gtk.FlowBox` appended directly to `chip_container`.

**chip_container** remains a vertical `Gtk.Box` with `hexpand=True` appended to `filter_box` between Edit and Clear buttons â€” no change to outer structure.

### Rebuild Method Changes

`_rebuild_provider_chips` (line 385) and `_rebuild_tag_chips` (line 401): update iteration target from `_provider_chip_box`/`_tag_chip_box` to `_provider_flow`/`_tag_flow`. Logic (get_first_child/remove/append pattern) is identical.

## Interaction Contract

| Trigger | Response |
|---------|----------|
| Click unselected provider chip | ToggleButton becomes active; `_on_provider_chip_toggled` fires (unchanged) |
| Click selected provider chip | ToggleButton becomes inactive; `_on_provider_chip_toggled` fires (unchanged) |
| Click unselected tag chip | ToggleButton becomes active; `_on_tag_chip_toggled` fires (unchanged) |
| Click selected tag chip | ToggleButton becomes inactive; `_on_tag_chip_toggled` fires (unchanged) |
| `_rebuild_provider_chips` called | `_provider_flow` children cleared and re-appended |
| `_rebuild_tag_chips` called | `_tag_flow` children cleared and re-appended |
| Window resize | Both FlowBoxes re-wrap chips at new available width (GTK native behavior) |
| Many providers/tags present | Chips wrap to multiple lines; filter bar expands vertically; adjacent buttons remain at fixed position |

No new keyboard, focus, or accessibility interactions introduced.
