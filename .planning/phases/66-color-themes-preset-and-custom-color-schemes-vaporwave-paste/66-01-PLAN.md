  <rationale>
    **db_connect hoist trade-off (checker Warning 5 — startup-contract change):**

    Hoisting `db_connect()` from its original position (after single-instance acquire +
    Node runtime check) to immediately after `app.setDesktopFileName(...)` means
    second-instance forwards (the path where `single_instance.acquire_or_forward()`
    forwards activation to the running primary and exits) now incur a SQLite
    connect/close cycle before exiting.

    This trade-off is acceptable because:
    1. **Latency:** the connect/close cycle is <5ms on local FS (SQLite open is a fopen
       + a few page reads on the local block device — measured on similar hardware).
       Below the threshold of human perception for app launch.
    2. **Single-source repo:** avoids two separate `db_connect` call sites (one for
       theme, one for the main app), keeping repo construction at exactly one place
       inside `_run_gui`. If we instead lazy-connected inside `apply_theme_palette`,
       we would either pass a connect closure (uglier signature) or open a second
       connection (heavier).
    3. **Simpler signature:** `theme.apply_theme_palette(app, repo)` takes the actual
       Repo, not a closure. Mirrors Phase 59's `apply_accent_palette(app, hex)` shape.
    4. **Documented for future revisit:** if startup latency becomes a concern (e.g.,
       network-mounted home dirs add fopen latency on second-instance forwards), the
       fix is to gate the hoist behind `if not is_second_instance`. Not done here
       because (a) latency is currently irrelevant, (b) we have no benchmark suggesting
       a regression, (c) gating adds a branch in the bootstrap path which the threat
       model currently considers a single linear sequence.

    **Threat model update:** This is a minor change in startup contract — second-instance
    forwards now touch the SQLite file before exit. Captured in this plan's threat
    register as a documented design decision rather than a new STRIDE entry (no new
    trust boundary; the SQLite file is in the same trust domain as the process).
  </rationale>
