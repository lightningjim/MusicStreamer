| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Vertical margins on filter_box rows (set_margin_top/bottom: 4) |
| sm | 8px | Horizontal margins on filter_box (set_margin_start/end: 8), chip row internal spacing |
| md | 16px | Default widget spacing within sections |
| lg | 24px | Not used in this phase |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Group header labels: margin_top=8, margin_bottom=4, margin_start=12, margin_end=8 (existing, unchanged)
- Chip row (Gtk.ScrolledWindow + Gtk.Box): internal Gtk.Box spacing=8, container margins match filter_box: top=4, bottom=4, start=8, end=8
- Chips: minimum width unset (size determined by label); height follows ToggleButton natural height (~32px)

Source: existing filter_box and group label margins observed in main_window.py lines 117-121, 273-276.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Adwaita window background | Main window, list background |
| Secondary (30%) | Adwaita headerbar/toolbar surface | Filter bar strip, now-playing panel |
| Accent (10%) | Adwaita `accent-bg-color` (blue) | Active/selected chip state only |
| Destructive | Adwaita `destructive-action` (red) | Not used in this phase |

Accent reserved for: selected Gtk.ToggleButton chips (active state, CSS class `suggested-action` is NOT used here — default ToggleButton active state uses accent automatically in Adwaita).

Note: Per CONTEXT.md D-01, chips are `Gtk.ToggleButton`. When toggled-on (get_active() == True), Adwaita applies accent color automatically. No custom CSS required.

Chip deselect (×) button: uses `flat` CSS class to appear borderless within the chip, same surface color as the button, no accent color.

Source: Adwaita theming behavior; deferred chip color-coding (per CONTEXT.md Deferred section) explicitly excluded from this phase.

## Component Inventory

### Chip Strip (new)
- Container: `Gtk.ScrolledWindow` with `hscrollbar-policy=AUTOMATIC`, `vscrollbar-policy=NEVER`
- Inner box: `Gtk.Box(orientation=HORIZONTAL, spacing=8)` with margin top=4, bottom=4, start=8, end=8
- Each chip: `Gtk.Box(orientation=HORIZONTAL, spacing=4)` containing:
  - `Gtk.ToggleButton(label="{value}")` — connects to `_on_filter_changed`
  - `Gtk.Button(icon_name="window-close-symbolic")` with CSS class `flat` — clicking calls `toggle.set_active(False)` then `_on_filter_changed`
- Provider chips row: one strip for providers
- Tag chips row: one strip for tags
- Both strips share the filter_box vertical stack, replacing `provider_dropdown` and `tag_dropdown`

Decision source: CONTEXT.md D-01, D-02, D-03.

### Retained Widgets (unchanged)
- Search entry (`Gtk.SearchEntry`) — unchanged
- Add button, Edit button — unchanged
- Clear button — extended to deselect all chips (D-04)

## Interaction Contract

### Chip Toggle
- Click chip body: toggles active state; `_on_filter_changed` fires
- Click × button: sets chip inactive; `_on_filter_changed` fires
- Multiple chips can be active simultaneously (no radio behavior)
- No chip selected = filter category is inactive (show all for that dimension)

### Filter Composition
- Provider chips: OR within providers (D-08)
- Tag chips: OR within tags (D-09)
- Provider AND tag: AND between dimensions (D-10)
- Any chip or search active: Recently Played section hidden (D-12)
- Any provider chip active: flat list mode (D-05, D-06)
- Tag chips in grouped mode: filter applies within groups (D-07)

### Clear Button
- Deselects all provider chips, all tag chips, clears search entry (D-04)
- Result: full grouped station list shown

### Chip Strip Overflow
- Horizontal scroll via Gtk.ScrolledWindow when chip count exceeds strip width (D-02)
- No ellipsis or "more" button
