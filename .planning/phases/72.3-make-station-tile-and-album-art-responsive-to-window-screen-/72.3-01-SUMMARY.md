   ```
   - [ ] **LAYOUT-03**: When the now-playing panel's width changes, both
     `logo_label` and `cover_label` retarget to a width-driven tier value drawn
   ```

The first failure cites the cover_label default (currently 180 once the
mock-medium tier is implicit, although technically the panel defaults are
180 logo + 160 cover — Plan 02 reconciles both ends). At Wave 0 there is no
`panel._current_art_tier` attribute and the cover label is still 160, so
TBD-01..TBD-08 and TBD-11 all fail on either an AttributeError or a
label-size mismatch. TBD-09 fails because `_set_cover_pixmap` hard-codes
QSize(160, 160). TBD-10 fails because `_show_station_logo` hard-codes
QSize(180, 180), so a 1400-px panel still gets a 180-px wide pixmap.

## Decisions Made

- **D-01..D-07 from CONTEXT encoded in test names** per the LAYOUT-03 must_haves
  checklist:
  - D-02 single-source-of-truth -> TBD-07 (pixmap dims track tier)
  - D-03 three render sites refactored -> TBD-09 (cover-art ready after resize matches tier)
  - D-04 tier values 140/180/240 -> TBD-01 / TBD-02 / TBD-03
  - D-05 logo == cover at every tier -> TBD-06
  - D-06 panel-width trigger via resizeEvent -> TBD-11 (compact vs expanded parity)
  - D-07 no lambdas -> TBD-13
  - D-09 inheritance no-persistence -> TBD-12
- **Threshold lock** at 700 and 1100 (inclusive on the lower edge of the next
  tier) encoded in TBD-04 and TBD-05.
- **Simpler FakeRepo** (no `streams_by_station` extension) per RESEARCH
  §Wave 0 Gaps — tier behavior is stream-count-agnostic.
- **Tolerance 1% for 16:9 letterbox aspect-ratio assertion** per RESEARCH
  Open Question 3 (Qt rounds 180 * 9/16 = 101.25 -> 101, actual ratio 1.782
  vs theoretical 1.778).
- **`panel.show()` + `qtbot.waitExposed(panel)` in the fixture** to mitigate
  Pitfall 6 (Wayland resize events not dispatching to unexposed widgets).
