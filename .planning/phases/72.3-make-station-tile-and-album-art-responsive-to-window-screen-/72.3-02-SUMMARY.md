# Phase 72.3 Plan 02: Wave 1 GREEN — Helpers, Sentinels, Cover Default Bump Summary

Lands the two new bound methods (`_current_art_tier_size` pure compute and
`_apply_art_tier` cached-diff orchestrator), the two `Optional[T]=None`

## Decisions Made

- **Docstring rephrased to avoid literal `"lambda"` and `"repo.set_setting"` substrings** inside `_apply_art_tier`. The QA-05 / D-09 invariants are documented as "Bound method only (QA-05)" and "No SQLite persistence (D-09 inheritance)". This keeps the source-grep test `test_no_lambda_in_tier_wiring` and the negative `test_no_repo_set_setting_for_tier_state` clean while preserving the same documentation intent. Sibling-plan precedent: Plan 01 SUMMARY (lines 137-149) documented the same lambda-substring tension.
- **Banner placement** — the new Phase 72.3 helper banner block is placed AFTER the entire 72.1 helper section ends (immediately after `_reflow_stream_picker_row`), not alongside it. This keeps Phase 72.1 / 72.3 sections cleanly separated and follows Pattern S-6 (section-banner-per-phase chronology).
- **Bare integer literals 140/180/240** in `_current_art_tier_size` (no named constants). Plan / PATTERNS quote the values verbatim and the truth table is documented inline in the docstring; introducing named constants would diverge from the verbatim PATTERNS.md §H source quote and add no value at two call sites.
