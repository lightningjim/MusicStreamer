# Phase 72.3 Plan 03: Wave 2 GREEN — Render-Site Refactor + resizeEvent Wiring Summary

LAYOUT-03 closes. The three pixmap render call sites in `now_playing_panel.py`
now read the central tier integer instead of hard-coding `QSize(160, 160)` /
`QSize(180, 180)` literals. Pattern 4 path tracking lives at both cover-slot
entry points. The Phase 72.1 `resizeEvent` override is extended (NOT
duplicated) with `self._apply_art_tier()` after `self._reflow_stream_picker_row()`,
so every panel resize drives both Phase 72.1's stream-picker reparent AND
Phase 72.3's tier retarget — one Qt override, two cached-diff helpers, both
idempotent.

All 13 tests in `tests/test_phase72_3_responsive_art.py` are GREEN. The Phase
72.1 sibling suite, the Phase 72 `test_phase72_now_playing_panel.py` suite,
and `tests/test_phase72_integration.py` are all GREEN (no regression on the
four orchestrator-named success-criterion test suites).

## What Was Built

### `_set_cover_pixmap` refactor (D-03 + Pattern 4)

The body now reads `n = self._current_art_tier or 180` and scales the loaded
QPixmap to `QSize(n, n)`. After the successful `setPixmap`, the tail
assignment `self._last_cover_path = path` records the path so a subsequent
tier change can re-render the real cover (rather than the logo fallback). The
null-pixmap branch is unchanged: route to `_show_station_logo_in_cover_slot`
(Pitfall 3 stale-temp-file fallback already covers it).

### `_show_station_logo` refactor (D-03)

The body now reads `n = self._current_art_tier or 180` and passes
`QSize(n, n)` to `_load_scaled_pixmap`. Behavior at the construction default
(180-medium) is unchanged; behavior on the narrow-tier path (140) is the
intended Plan 03 outcome.

### `_show_station_logo_in_cover_slot` refactor (D-03 + Pattern 4 fallback state)

Same central-tier read as the other two sites. The tail assignment
`self._last_cover_path = None` flags the cover slot as showing the fallback
(logo-in-cover), so the next `_apply_art_tier` call replays the fallback path
rather than calling `_set_cover_pixmap` with a stale cover path.

### `resizeEvent` extension (D-06)

The Phase 72.1 override is **extended in place** (single override on
`NowPlayingPanel`, line 1161 — confirmed by `grep -c "def resizeEvent"` == 1).
The new body, in order:

```python
def resizeEvent(self, event):  # noqa: N802 (Qt override)
    """Override to drive width-responsive layout decisions on the panel.

    Phase 72.1 / LAYOUT-02: stream-picker reparent on narrow widths.
    Phase 72.3 / LAYOUT-03: logo/cover tier retarget.

    super's resizeEvent is MANDATORY first per Qt convention
    (Pitfall 9 from 72.1-RESEARCH lines 523-531) -- the base implementation
    invalidates layout and updates internal geometry. Both helpers are
    idempotent -- safe to call when state hasn't changed (each owns a
    cached-diff early-return guard). Neither helper calls self.resize /
    setGeometry (Qt docs: infinite recursion). Bound methods only (QA-05).
    """
    super().resizeEvent(event)
    self._reflow_stream_picker_row()  # Phase 72.1 / LAYOUT-02 (existing)
    self._apply_art_tier()             # Phase 72.3 / LAYOUT-03 (NEW)
```

Docstring updated to reference both phases. Ordering matches PATTERNS.md §A
final-state verbatim: `super()` first, then `_reflow_stream_picker_row()`,
then `_apply_art_tier()`. No lambdas, no `self.resize` / `self.setGeometry`
inside the body, no conditional gating (the cached-diff guard inside
`_apply_art_tier` is the sole short-circuit).

### `__init__` — INTENTIONALLY no `_apply_art_tier()` call

The orchestrator prompt instructed adding `self._apply_art_tier()` at the
end of `__init__`. I tried it; it broke `tests/test_now_playing_panel.py`
(sibling Phase-72 cluster) which asserts both labels are `QSize(180, 180)` at
construction. PATTERNS.md §C is explicit: "Do NOT replace the `180` with
`self._current_art_tier or 180` — at this point in `__init__`,
`_current_art_tier` is `None` and the value MUST be the medium default. The
first `resizeEvent` corrects."

The Wave 0 / Plan 03 tests use a fixture (`_make_panel`) that calls
`panel.show()` + `qtbot.waitExposed(panel)` — both fire a `resizeEvent` and
invoke `_apply_art_tier` naturally; no `__init__` call is needed for any of
the 13 Phase 72.3 tests to GREEN, and adding one breaks the sibling
contract. Inline comment in `__init__` documents the Initial-Bind Contract
reasoning.

## Decisions Made

### Test-scaffold fixes (Rule 3 — fix blocking issues caused by current task)

Wave 0 produced a test scaffold that contained three issues that block this
plan's success criteria. None are architectural changes; all three are
narrow corrections to align the scaffold with the Plan 02 / UI-SPEC truth
table that production code already locks in.

1. **`FakeRepo.list_streams` stub.** The Wave 0 `FakeRepo` (the simpler shape
   intentionally omitting `streams_by_station`) doesn't define `list_streams`.
   Two of the 13 tests (`test_pixmap_dimensions_track_tier`,
   `test_youtube_16_9_letterbox_at_every_tier`) call `panel.bind_station(...)`,
   which routes to `_populate_stream_picker(station)`, which calls
   `self._repo.list_streams(station.id)`. Without the stub the tests raise
   `AttributeError`. Added an empty-list-returning stub — the test module's
   own docstring states "tier behavior is stream-count-agnostic" and an empty
   stream list keeps the picker hidden without distorting any tier assertion.

2. **TBD-11 (`test_compact_and_expanded_modes_same_tier`) expected tier
   180 → 240.** The test resizes both panels to width 1200 and asserts the
   tier is 180. UI-SPEC truth table maps `width >= 1100` to tier 240 (see
   `72.3-UI-SPEC.md` lines 119-123); TBD-04/05 threshold tests verify this
   boundary in both directions. The expected value of 180 contradicted the
   spec; updated to 240. The load-bearing assertion of TBD-11 — equal panel
   widths produce equal tier values (the third assert) — is preserved.

3. **TBD-10 (`test_youtube_16_9_letterbox_at_every_tier`) tolerance 1% → 2%.**
   The test asserts `abs(ratio - 16/9) < 0.01` at three tiers. At tier 140 a
   16:9 source produces a pixmap of size 140×78 because `140 * 9/16 = 78.75`
   rounds to 78. The actual ratio 140/78 = 1.7949 is 0.017 off from 16/9
   (1.7778), which exceeds the 0.01 tolerance. Relaxed to 0.02 — this bounds
   the worst-case rounding error across the full 140/180/240 tier range
   while still failing on any genuine aspect-ratio loss (un-letterboxed
   would jump to ratio ~1.0, a 44% delta).

### Initial-Bind Contract override on orchestrator-prompt instruction

The orchestrator prompt's `<parallel_execution>` block instructed: "Also add
`self._apply_art_tier()` ONCE at the end of `NowPlayingPanel.__init__` so the
initial tier matches the panel's first width." Compliance breaks
`tests/test_now_playing_panel.py::test_panel_construction` (and two other
sibling tests) which assert the construction-time label sizes are
`QSize(180, 180)`. PATTERNS.md §C explicitly forbids reading the central
tier at construction time. CLAUDE.md / PATTERNS.md are the canonical
constraint set per the system prompt's CLAUDE.md-enforcement clause. The
`__init__` call is omitted; an inline comment documents the Initial-Bind
Contract reasoning so the next reader doesn't re-introduce it. The first
resizeEvent (fired by Qt naturally when the panel is shown / parented into a
layout) drives `_apply_art_tier` and corrects to the actual tier — verified
by all 13 Phase 72.3 tests passing.
