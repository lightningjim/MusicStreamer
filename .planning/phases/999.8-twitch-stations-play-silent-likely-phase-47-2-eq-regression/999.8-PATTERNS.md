```python
def _apply_eq_state(self) -> None:
    if self._eq is None:
        return
    if self._eq_profile is None or not self._eq_enabled:
        for i in range(self._eq.get_children_count()):
            self._eq.get_child_by_index(i).set_property("gain", 0.0)
        return
    for i, b in enumerate(self._eq_profile.bands):
        if i >= self._eq.get_children_count():
            break
        band = self._eq.get_child_by_index(i)
        band.set_property("freq", float(b.freq_hz))
        band.set_property("bandwidth", float(b.freq_hz) / max(float(b.q), 0.01))
        band.set_property("gain", float(b.gain_db) + self._eq_preamp_db)
        band.set_property("type", self._EQ_BAND_TYPE.get(b.filter_type, 0))
```
This works unchanged because `self._eq` continues to refer to the inner equalizer-nbands element (now a child of `self._eq_bin`). RESEARCH §Pattern §2 verified `band.set_property` round-trips through `get_child_by_index` on an element living inside a bin.

## Shared Patterns

### Graceful-degrade contract (preserved across player.py changes)

**Source:** `musicstreamer/player.py` line 102 (current `if self._eq is not None:` guard) + `_apply_eq_state` line 601 (`if self._eq is None: return`) + `set_eq_enabled` / `set_eq_profile` / `set_eq_preamp` (analogous guards exist in lines 540-580 region).
**Apply to:** `_build_eq_bin` helper, `__init__` block, `_rebuild_eq_element`.
**Contract:** When ANY required GStreamer factory returns None, `_build_eq_bin` returns `(None, None)`. Both `self._eq_bin` and `self._eq` end up None. Every existing `if self._eq is None` guard continues to short-circuit correctly. RESEARCH Pitfall 7 verifies this.

```python
# Helper: returns (None, None) on any missing element — graceful degrade.
def _build_eq_bin(num_bands: int):
    eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
    ac_in  = Gst.ElementFactory.make("audioconvert",  "eq_ac_in")
    ac_out = Gst.ElementFactory.make("audioconvert",  "eq_ac_out")
    if eq is None or ac_in is None or ac_out is None:
        return (None, None)
    # ... wire bin, return (bin_, eq) ...
```

### Inner-equalizer reference caching (`self._eq` semantic)

**Source:** New convention introduced this phase, codified by RESEARCH §Pattern §2 + Pitfall 4.
**Apply to:** `Player.__init__`, `_rebuild_eq_element`, ALL existing call sites that read/write `self._eq.get_child_by_index(...)` or `self._eq.get_property("num-bands")`.
**Rule:** `self._eq` is ALWAYS the inner `equalizer-nbands` element (cached at construction). `self._eq_bin` is the wrapper bin assigned to `audio-filter`. Every existing 47.2-02 EQ test that inspects `self._eq.get_child_by_index(i)` keeps working without modification because the inner element is the cached reference.
