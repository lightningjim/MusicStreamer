# Phase 999.8: Twitch stations play silent — likely Phase 47.2 EQ regression — Research

**Researched:** 2026-04-24
**Domain:** GStreamer audio-filter caps negotiation (`equalizer-nbands` + `playbin3` + AAC HLS variants)
**Confidence:** HIGH (smoking-gun reproduced via local Python probes against the installed stack)

## Summary

The CONTEXT.md hypothesis is empirically confirmed against the dev-machine GStreamer 1.26.6 / gst-plugins-good 1.26.5 install: `equalizer-nbands` accepts only `audio/x-raw, format=(string){S16LE,F32LE,F64LE}, layout=(string)interleaved` on its sink/src pad templates. Anything else fails caps negotiation at the audio-filter slot. The Twitch path (streamlink HLS → playbin3 → AAC demux/decode via `avdec_aac`) produces `format=F32LE, layout=non-interleaved` — the layout mismatch is the silencer. The YouTube-live path (yt-dlp resolved HLS → Opus or AAC, but routed through `decodebin3` which composes its own audioconvert in some plugin builds) and the ShoutCast/MP3 path (`mpegaudioparse` → `mpg123audiodec`) happen to emit a layout/format the bare equalizer accepts, which is why those work today by accident.

The CONTEXT D-07 fix shape (wrap `equalizer-nbands` in a `GstBin` containing `audioconvert ! equalizer-nbands ! audioconvert` and assign the bin to `playbin3.audio-filter`) was probed and works for both interleaved S16LE and non-interleaved S32LE inputs. The inner `audioconvert` resolves both `format` AND `layout` (interleaved/non-interleaved) — the equalizer sees `F32LE, interleaved` regardless of upstream. This is the GStreamer-canonical remedy for this class of bug.

**Primary recommendation:** Execute D-06 first (in-app `set_eq_enabled(False)` runtime toggle on the live Twitch URL) to confirm the production reproduction matches the synthetic probe. Then implement D-07 with the inner-equalizer reference cached on `Player` so child-by-index band writes still work without `bin.get_by_name("eq")` lookups on every gain change. `audioresample` per D-08 is **not** triggered by current evidence — `avdec_aac` produces 48000 Hz, `audioconvert` does not require resampling for that, and `pulsesink` accepts arbitrary rates.

### Pattern §1: GstBin wrapper construction (D-07 canonical)

**What:** Replace the bare `equalizer-nbands` assignment to `playbin3.audio-filter` with a `GstBin` containing `audioconvert ! equalizer-nbands ! audioconvert`. Expose ghost pads so playbin3 sees the bin as a single element.

**When to use:** Once, in `Player.__init__`, replacing player.py:102-105. Same pattern reused inside `_rebuild_eq_element` for band-count changes.

**Source:** [VERIFIED] Probed on the dev machine via `Gst.Bin.new(...)`, `Gst.ElementFactory.make(...)`, `Gst.GhostPad.new(...)`, `pb.set_property("audio-filter", bin_)`. The `audio-filter` slot accepts the bin, the inner equalizer remains addressable via `bin.get_by_name("eq")` AND via the cached Python reference to the inner element.

```python
# player.py — replace lines 97-105 with this block.
def _build_eq_bin(self, num_bands: int) -> tuple["Gst.Bin | None", "Gst.Element | None"]:
    """Build (bin, inner_eq) where inner_eq is the equalizer-nbands element.

    Returns (None, None) if any element is missing — graceful degrade.
    """
    eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
    ac_in  = Gst.ElementFactory.make("audioconvert",  "eq_ac_in")
    ac_out = Gst.ElementFactory.make("audioconvert",  "eq_ac_out")
    if eq is None or ac_in is None or ac_out is None:
        return (None, None)

    eq.set_property("num-bands", num_bands)

    bin_ = Gst.Bin.new("eq-bin")
    bin_.add(ac_in)
    bin_.add(eq)
    bin_.add(ac_out)
    if not ac_in.link(eq):
        return (None, None)
    if not eq.link(ac_out):
        return (None, None)

    bin_.add_pad(Gst.GhostPad.new("sink", ac_in.get_static_pad("sink")))
    bin_.add_pad(Gst.GhostPad.new("src",  ac_out.get_static_pad("src")))
    return (bin_, eq)

# In Player.__init__, replacing the four lines that today say
#     self._eq = Gst.ElementFactory.make("equalizer-nbands", "eq")
#     if self._eq is not None: ...
self._eq_bin, self._eq = self._build_eq_bin(num_bands=10)  # placeholder; rebuilt per profile
if self._eq_bin is not None:
    self._pipeline.set_property("audio-filter", self._eq_bin)
# self._eq is None when graceful-degrade kicked in; all set_eq_* methods
# already check `if self._eq is None: return` (Pitfall 7 contract).
```

**Why two `audioconvert` elements (one before AND one after):**

1. **`ac_in`:** absorbs whatever raw format playbin3's decoder emits (e.g., `F32LE non-interleaved` from Twitch's AAC) and presents `S16LE | F32LE | F64LE, interleaved` to the equalizer.
2. **`ac_out`:** absorbs whatever the equalizer emits (always interleaved S16LE/F32LE/F64LE) and lets the downstream sink pick its own preferred format. Without the trailing `audioconvert`, a sink that wants e.g. non-interleaved S32LE again would reject the equalizer's interleaved-only output.

**Why NOT just one `audioconvert`:** A leading-only `audioconvert` would fix the input side but leave the output side at `interleaved S16LE/F32LE/F64LE` — fine for `pulsesink`, but Windows `wasapisink` and `directsoundsink` have their own preferred formats. The trailing `audioconvert` keeps the audio-filter slot format-agnostic on both sides.

### Pattern §3: Caps capture via pad probes (D-02 evidence)

**What:** Attach `BUFFER` pad probes to the bin's sink and src ghost pads (or to the inner equalizer's sink and src pads) to capture the negotiated caps at first buffer flow. Serialize via `Caps.to_string()` for grep-able output.

**When to use:** D-04 manual harness, on every cell.

**Source:** [VERIFIED] Pad probe pattern reproduced on dev machine; first-buffer caps captured for both bin-sink and bin-src.

```python
def make_caps_probe(label: str, bucket: dict[str, str]):
    """Returns a pad-probe callback that records the first observed caps."""
    def _cb(pad, info):
        if label in bucket:
            return Gst.PadProbeReturn.OK
        caps = pad.get_current_caps()
        if caps is not None:
            bucket[label] = caps.to_string()
        return Gst.PadProbeReturn.OK
    return _cb

# Attach inside the harness AFTER the bin is constructed and BEFORE PLAYING:
caps = {}
bin_.get_static_pad("sink").add_probe(
    Gst.PadProbeType.BUFFER, make_caps_probe("eq_bin.sink", caps)
)
bin_.get_static_pad("src").add_probe(
    Gst.PadProbeType.BUFFER, make_caps_probe("eq_bin.src", caps)
)
# Optionally probe the inner equalizer pads too — useful for confirming
# that audioconvert delivered an accepted format:
inner = bin_.get_by_name("eq")
inner.get_static_pad("sink").add_probe(
    Gst.PadProbeType.BUFFER, make_caps_probe("eq_inner.sink", caps)
)
inner.get_static_pad("src").add_probe(
    Gst.PadProbeType.BUFFER, make_caps_probe("eq_inner.src", caps)
)
```

```python
# tests/test_player.py — NEW test. Place AFTER the existing 47.2-02 EQ tests
# so it lives next to the other equalizer-nbands tests.
import gi
gi.require_version("Gst", "1.0")
from gi.repository import Gst
import pytest

@pytest.mark.parametrize("input_caps,label", [
    ("audio/x-raw,format=S16LE,rate=44100,channels=2,layout=interleaved",     "shoutcast-mp3-like"),
    ("audio/x-raw,format=F32LE,rate=48000,channels=2,layout=interleaved",     "youtube-opus-like"),
    ("audio/x-raw,format=F32LE,rate=48000,channels=2,layout=non-interleaved", "twitch-aac-like"),
])
def test_eq_bin_passes_arbitrary_input_caps(input_caps, label):
    """D-03: the audio-filter bin must accept every realistic decoder output
    format and pass buffers through to a downstream sink in bypass mode."""
    if not Gst.is_initialized():
        Gst.init(None)

    # Build the same bin Player builds — call the helper under test.
    from musicstreamer.player import Player
    bin_, inner = Player._build_eq_bin.__func__(None, num_bands=10)  # static-style
    assert bin_ is not None, f"[{label}] eq-bin construction failed"

    pipe = Gst.Pipeline.new(f"test-{label}")
    src  = Gst.ElementFactory.make("appsrc",   "src")
    sink = Gst.ElementFactory.make("fakesink", "sink")
    sink.set_property("sync", False)
    sink.set_property("async", False)
    sink.set_property("signal-handoffs", True)
    handoff_count = [0]
    sink.connect("handoff", lambda *_: handoff_count.__setitem__(0, handoff_count[0] + 1))
    src.set_property("caps", Gst.Caps.from_string(input_caps))
    src.set_property("format", Gst.Format.TIME)

    for el in (src, bin_, sink):
        pipe.add(el)
    assert src.link(bin_), f"[{label}] appsrc → eq-bin link failed (caps mismatch)"
    assert bin_.link(sink), f"[{label}] eq-bin → fakesink link failed"

    pipe.set_state(Gst.State.PLAYING)
    # Push a handful of silent buffers
    for _ in range(10):
        buf = Gst.Buffer.new_wrapped(b"\x00" * 1024)
        src.emit("push-buffer", buf)
    src.emit("end-of-stream")

    bus = pipe.get_bus()
    msg = bus.timed_pop_filtered(2 * Gst.SECOND, Gst.MessageType.EOS | Gst.MessageType.ERROR)
    pipe.set_state(Gst.State.NULL)

    assert msg is not None, f"[{label}] no EOS/ERROR within 2s"
    assert msg.type == Gst.MessageType.EOS, (
        f"[{label}] expected EOS, got ERROR: {msg.parse_error()[0].message}"
    )
    assert handoff_count[0] > 0, f"[{label}] zero buffers reached the sink"
```

**Static-method note:** `Player._build_eq_bin` is currently shown as an instance method in Pattern §1 for namespace clarity. The test calls it via `__func__(None, num_bands=10)` to avoid constructing a Player. If the planner prefers, refactor `_build_eq_bin` to a `@staticmethod` (or a module-level free function) — that is cleaner and removes the `__func__(None, ...)` awkwardness. Either way works.

**Bypass-state caveat:** The test does NOT set any band gains and does NOT load a profile. The default `equalizer-nbands` band gain is 0.0 dB on every band — that's bypass. The test verifies caps negotiation, not DSP correctness.

**`audio/mpeg` AAC variant note:** CONTEXT D-03 mentions a `audio/mpeg` AAC caps variant. **Do not feed `audio/mpeg` directly to the bin** — the equalizer (and its inner audioconverts) only accept `audio/x-raw`. AAC is a compressed codec; it must be decoded first. The Twitch-realistic test case is `format=F32LE, layout=non-interleaved` (the AAC decoder's output), not `audio/mpeg`. Surface this to the planner as a **clarification to D-03**: the third caps row should be the post-`avdec_aac` raw form, not the pre-decode encoded form.

### Pattern §5: Manual matrix harness (D-04)

**What:** Python script that iterates the 5×3 + preamp×3 + bands×4 cells, plays each URL through a real `playbin3` with the bin-wrapped eq, captures caps at the eq-bin boundaries, and asserts the audio-sink received non-zero buffer counts. Exits non-zero on any silence.

**When to use:** On demand, before any release that touches `player.py` EQ code or the audio-filter slot.

**Source:** [VERIFIED] Pattern probe — `playbin3` + bin audio-filter + pad probes + `level` element for buffer-count instead of audibility metering all work in isolation.

```python
# scripts/eq_audit.py  (or tests/integration/audio_filter_smoke.py)
"""D-04 manual harness — local-only, NOT pytest-collected.

Usage:
    python scripts/eq_audit.py --urls eq_audit_urls.txt
```

### Pattern §6: Diagnostic gate runtime toggle (D-06)

**What:** While Twitch is silently "playing," call `player.set_eq_enabled(False)` from the running app and listen for audio. If audio returns within ~one buffer cycle (10s with current `BUFFER_DURATION_S`), the EQ is the silencer. If audio does NOT return, the bug is elsewhere (sink negotiation, buffering, HLS segment boundary).

**When to use:** Phase 1 of the plan, before any fix code is written.

**Source:** [VERIFIED] `set_eq_enabled(False)` zeroes every band gain on the existing element via `_apply_eq_state` (player.py:594-616). Zero-gain bands are unity passthrough — but the equalizer ELEMENT is still in the path. **Important caveat:** zero-gain bypass does NOT remove the element from the audio-filter slot, so it does NOT change caps negotiation. If the bug is caps negotiation (which the probe strongly suggests), zero-gain bypass will NOT restore audio.

**Implication for D-06:** The test "toggle EQ off" should be augmented with a SECOND test that actually drops the element from the slot:

```python
# Diagnostic helper (TEMPORARY — for D-06 only, do NOT ship):
player._pipeline.set_state(Gst.State.READY)
player._pipeline.get_state(Gst.CLOCK_TIME_NONE)
player._pipeline.set_property("audio-filter", None)
player._pipeline.set_state(Gst.State.PLAYING)
```

If audio returns when `audio-filter=None` but does NOT return when only `set_eq_enabled(False)` is toggled, the hypothesis is **confirmed** and D-07 is the right fix. If audio still does not return when `audio-filter=None`, the bug is elsewhere and research must widen.

**Surface this to the planner:** Update D-06's diagnostic procedure to include both toggles (zero-gain bypass + audio-filter detach). The CONTEXT wording "toggle `set_eq_enabled(False)` at runtime" should be expanded.

### Anti-Patterns to Avoid

- **Setting `audio-filter` to a NEW bin while PLAYING.** The slot is honored at NULL→READY transition. Always `set_state(READY)` and `get_state(CLOCK_TIME_NONE)` before reassigning. (Existing `_rebuild_eq_element` already does this — preserve the pattern.)
- **Calling `bin.get_by_name("eq")` on every band write.** Cache the inner equalizer reference at construction time (`self._eq = inner`). Repeated lookups walk the bin's child list every call.
- **Forgetting GhostPads.** A bin with no ghost pads has no externally visible pads. `playbin3.set_property("audio-filter", bin)` will silently NOT link the bin into the pipeline. Symptom: bin is in the slot but no buffers flow through it. Verify with a sink-side caps probe.
- **Assuming `audio-filter=None` is the same as "EQ disabled."** Setting the property to None DETACHES the element entirely — that is a different code path from the in-element gain bypass. Use detach for diagnosis only.
- **Linking `equalizer-nbands` to anything but interleaved S16LE/F32LE/F64LE.** Plugin pad templates are non-negotiable. ALWAYS go through `audioconvert`.
- **Adding `audioresample` "to be safe."** Without evidence, it adds an extra resampling step (and latency) that the equalizer's wide rate range `[1000, 2147483647]` does not need.

## Common Pitfalls

### Pitfall 1: Forgetting GhostPads on the bin
**What goes wrong:** `playbin3.set_property("audio-filter", bin)` accepts the property write, but no buffers flow through the bin (it has no externally visible pads). Symptom: silent audio, no error message, bin shows up in `GST_DEBUG` traces with zero buffer counts.
**Why it happens:** A `Gst.Bin` is just a container; without `GhostPad`s, its internal elements' pads are inaccessible from outside.
**How to avoid:** ALWAYS call `bin.add_pad(Gst.GhostPad.new("sink", inner.get_static_pad("sink")))` and the matching src after `bin.add(...)` and after `link(...)`.
**Warning signs:** "playbin3 is silent and there's no error in the log."

### Pitfall 2: Reassigning `audio-filter` while PLAYING
**What goes wrong:** Property write succeeds, but the new filter is not picked up; OR a pad-renegotiation race causes a silent failure mid-stream.
**Why it happens:** `audio-filter` is honored at the NULL→READY transition.
**How to avoid:** Always `set_state(READY)` + `get_state(CLOCK_TIME_NONE)` before reassigning. The existing `_rebuild_eq_element` does this — preserve the pattern when extending it for the bin.

### Pitfall 3: Layout (interleaved vs non-interleaved) is part of caps
**What goes wrong:** Bare equalizer accepts `format=F32LE` but rejects `layout=non-interleaved`. The Twitch AAC decoder emits `non-interleaved`. Negotiation fails silently (the audio-filter slot receives no buffers); from the user's perspective the playback "works" because playbin3 still ticks the elapsed timer and the buffer fill counter, but no audio reaches the sink.
**Why it happens:** Equalizer pad template specifies `layout: interleaved` only.
**How to avoid:** `audioconvert` adapts both `format` AND `layout`. The wrapper bin is the only fix — there is no per-element configuration that makes bare `equalizer-nbands` accept non-interleaved input.
**Warning signs:** "Audio works for some streams but not others, and the failing streams are AAC-decoded."

### Pitfall 4: Losing the inner-equalizer reference
**What goes wrong:** After wrapping in a bin, `self._eq` either points to the bin (wrong — `get_child_by_index` returns bin children, not equalizer band children) or to nothing.
**Why it happens:** Sloppy refactor of `Player.__init__` and `_rebuild_eq_element`.
**How to avoid:** `_build_eq_bin` returns `(bin, inner_eq)`; assign both: `self._eq_bin, self._eq = self._build_eq_bin(n)`. Every existing call site that uses `self._eq.get_child_by_index(i)` and `self._eq.get_property("num-bands")` then works unchanged.

### Pitfall 5: Rebuild path forgets to rebuild the bin
**What goes wrong:** `_rebuild_eq_element` rebuilds only the inner equalizer but assigns `self._eq` (the inner) to `audio-filter` instead of `self._eq_bin` (the wrapper). Reverts to the pre-fix bare-equalizer state.
**Why it happens:** The rebuild method's body looks at the bare-element pattern and copies it.
**How to avoid:** `_rebuild_eq_element` calls `_build_eq_bin(num_bands)` and reassigns BOTH `self._eq_bin` and `self._eq`. Then `set_property("audio-filter", self._eq_bin)`.
```python
def _rebuild_eq_element(self, num_bands: int) -> None:
    self._pipeline.set_state(Gst.State.READY)
    self._pipeline.get_state(Gst.CLOCK_TIME_NONE)
    self._eq_bin, self._eq = self._build_eq_bin(num_bands)
    if self._eq_bin is not None:
        self._pipeline.set_property("audio-filter", self._eq_bin)
```

### Pitfall 6: Assuming `set_eq_enabled(False)` removes the EQ from the path
**What goes wrong:** D-06 diagnostic concludes EQ is innocent because zero-gain bypass did not restore audio.
**Why it happens:** Zero-gain bypass keeps the equalizer ELEMENT in the path; only its DSP coefficients become unity. Caps negotiation is unaffected.
**How to avoid:** D-06 must include a second toggle: temporarily set `audio-filter=None` (with the surrounding READY/PLAYING dance) to fully detach the element. Only that toggle isolates the caps-negotiation hypothesis.

### Pitfall 7: Graceful-degrade contract changes shape
**What goes wrong:** When `equalizer-nbands` plugin is missing on Windows (Phase 43 verified it ships, but the contract must still hold), `_build_eq_bin` returns `(None, None)` and `self._eq_bin = None`. Existing code at every EQ touchpoint checks `if self._eq is None` — that check still works because `self._eq` is also `None` from the same return.
**Why it happens:** Refactor introduces a new field (`self._eq_bin`) and the `is None` check on `self._eq` still gates correctly only because we set both to None together.
**How to avoid:** `_build_eq_bin` MUST return `(None, None)` when ANY of the three elements is missing. Document this in the helper's docstring. Add a unit test mirroring `test_player_eq_handles_missing_plugin` (already exists) but with the `audioconvert` factory mocked to None — this catches future drift.

### Pitfall 8: `appsrc` test forgets to push EOS
**What goes wrong:** Unit test hangs at `bus.timed_pop_filtered` waiting for EOS that never arrives because the test forgot `src.emit("end-of-stream")`.
**Why it happens:** AppSrc keeps the pipeline in a "more buffers possible" state until told otherwise.
**How to avoid:** ALWAYS call `src.emit("end-of-stream")` after pushing the test buffers. The 2-second timeout is a safety net, not the success signal.

### How D-03 Unit Test Verifies Caps Negotiation

The unit test parametrizes three input caps strings — one per realistic decoder output — and asserts that:
1. `appsrc.link(eq_bin)` succeeds (proves the bin's sink ghost pad accepts these caps).
2. EOS reaches the bus within 2 s (proves no negotiation error mid-stream).
3. `fakesink` `handoff` signal fires at least once (proves at least one buffer flowed end-to-end).

If caps negotiation failed, link() would fail (immediate test fail, exact same error as the §1 probe) OR the bin would silently swallow buffers and `handoff_count` would stay at 0 (also a test fail). Either way the test catches it.

### How D-04 Harness Verifies End-to-End Audibility

For each cell in the 5×3 matrix (plus preamp/bands axes against the `enabled-with-profile` row):
1. Construct a real `Player`, drive it through the cell's EQ state via `set_eq_profile/set_eq_enabled/set_eq_preamp`.
2. Replace the audio sink with `fakesink signal-handoffs=True` chained after a `level` element (or just count handoff buffer sizes).
3. Play the cell's URL for a fixed window (e.g., 15 s — slightly more than `BUFFER_DURATION_S` so post-buffer audio actually flows).
4. Capture caps at `eq_bin.sink`, `eq_inner.sink`, `eq_inner.src`, `eq_bin.src` via pad probes.
5. Assert `total_bytes_to_sink > some_threshold` (e.g., 100 KB — a sanity floor that distinguishes "audio flowing" from "silent stream"). Below threshold = `[FAIL] silent`.
6. Append the cell result + caps strings to a markdown evidence file in the phase directory.

## Open Questions

1. **Should `_build_eq_bin` be a `@staticmethod` or stay an instance method?**
   - What we know: It uses no `self` state; pure factory.
   - What's unclear: The project does not have a strong precedent either way.
   - Recommendation: `@staticmethod` for clarity. Trivial change; lowest cognitive load at call sites.

2. **D-03 wording references an `audio/mpeg` AAC caps variant — should the unit test feed encoded AAC?**
   - What we know: The bin's input pad accepts only `audio/x-raw`. Feeding `audio/mpeg` would require an in-test decoder, which is not what the test should be doing.
   - What's unclear: Whether the CONTEXT author meant "AAC-decoded raw form" (F32LE non-interleaved) or "AAC encoded form" (audio/mpeg).
   - Recommendation: Use the AAC-decoded raw form. If the CONTEXT intent was the encoded form, that's a separate test (decoder negotiation) and not what D-03 is testing.

3. **Should the manual harness also test the band-count rebuild path (1, 5, 10, 20+ bands axis) under each stream type, or only against the "enabled-with-profile" cell?**
   - What we know: CONTEXT D-02 says the bands axis is "tested against the enabled-with-profile state on each stream type."
   - What's unclear: Whether the rebuild path triggers caps re-negotiation (it should NOT, because bin topology is unchanged — only the inner equalizer is replaced).
   - Recommendation: Test against enabled-with-profile only per CONTEXT. If a regression slips through, add a dedicated test then.

4. **Where should the harness URL list live — in-repo or out?**
   - What we know: The list contains user-specific account URLs (AudioAddict listen keys, etc.) that may include credentials.
   - What's unclear: Whether `eq_audit_urls.txt` should be gitignored or shipped as a template (`eq_audit_urls.example.txt`) with placeholders.
   - Recommendation: Ship `eq_audit_urls.example.txt` as a template; gitignore `eq_audit_urls.txt`. Mirror the precedent set by `.env.example` patterns even though MusicStreamer doesn't currently use one.
