# Phase 999.8 Diagnostic Report

**Date:** 2026-04-24
**Verdict:** REFUTED
**Actual root cause:** streamlink 8.x API regression (unrelated to Phase 47.2 EQ)
**Fix commit:** `9df84de` — `fix(streamlink): use prefixed session.set_option for Twitch OAuth header`

## Setup

- App launch: `bash run_local.sh`
- Twitch station: live from station list
- Two-stage probe (Ctrl+Shift+D / Ctrl+Shift+E) was **not run** — diagnosis collapsed earlier than expected.

## What actually happened

While preparing to launch the diagnostic shortcuts, the user attempted to play a Twitch station and observed an unrelated traceback in the foreground log:

```
File "musicstreamer/player.py", line 558, in _twitch_resolve_worker
    session.set_plugin_option(
AttributeError: 'Streamlink' object has no attribute 'set_plugin_option'
```

`streamlink` 8.x dropped `Streamlink.set_plugin_option(plugin, key, value)`. The modern equivalent is `session.set_option(f"{plugin}-{key}", value)`. Confirmed via local probe: the prefixed key round-trips through `session.get_option` and is read by the Twitch plugin under the same key.

The crash occurred **inside the resolution worker thread**, before `session.streams(url)` ran and before any URI was set on `playbin3`. Result: playbin3's elapsed timer ticked against an empty/stale URI while the user heard silence — the exact symptom that motivated Phase 999.8.

After applying the one-line fix (`session.set_plugin_option("twitch", "api-header", v)` → `session.set_option("twitch-api-header", v)`), Twitch stations play with audio end-to-end.

## Verdict: REFUTED

The Phase 47.2 equalizer-nbands element on `playbin3.audio-filter` is **not** silencing Twitch. The hypothesis was built on a symptom (silent playback while timer ticks) that is also produced by a worker-thread crash before URI assignment. Once the AttributeError is fixed, audio plays through the full EQ chain correctly.

## Why the original CONTEXT.md hypothesis was wrong

- The CONTEXT.md narrative assumed streamlink resolved a valid HLS URL and `gst-play-1.0 <url>` played it standalone. That control was likely run with the URL already cached or extracted from an earlier, pre-token state — at which point the AttributeError didn't fire (the broken call is gated by `if token:`).
- With a real OAuth token loaded (post-Phase 999.3), the broken `set_plugin_option` call always raises before reaching `session.streams(url)`, so playbin3 was being asked to play nothing.
- Because the worker is a `daemon=True` thread, the traceback printed to stderr but did not surface as an `playback_error` signal — only the bare timer-tick was visible in the GUI.

## Implications for Plans 02 and 03

- **Plan 02** (GstBin wrapper around equalizer-nbands per D-07): not needed. The EQ never had a problem with Twitch's caps. Wrapping for defensive caps tolerance is cosmetic against a misdiagnosis.
- **Plan 03** (manual matrix harness across 5 stream types × 3 EQ states + axes per D-04): also not needed for this bug. Could stand on its own merits later if EQ regression risk grows, but per single-user pragmatic policy: not built speculatively.

Both plans deleted from the phase directory along with `999.8-VALIDATION.md`. See `999.8-SUMMARY.md` for the closure record.
