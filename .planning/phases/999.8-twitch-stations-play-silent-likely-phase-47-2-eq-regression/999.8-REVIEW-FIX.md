## Fixed Issues

### WR-03: `QTimer.singleShot` invoked from worker thread (twitch resolve)

**Files modified:** `musicstreamer/player.py`
**Commit:** 79186ae
**Applied fix:** Added class-level `_try_next_stream_requested = Signal()`
alongside the existing `_cancel_timers_requested` / `_error_recovery_requested`
cross-thread marshaling signals. Wired it in `__init__` with
`Qt.ConnectionType.QueuedConnection` to `_try_next_stream`. Replaced both
`QTimer.singleShot(0, self._try_next_stream)` calls in the Twitch worker's
`NoPluginError` and `PluginError` branches with `self._try_next_stream_requested.emit()`.
The worker thread has no Qt event loop, so the prior `singleShot` calls
silently no-op'd; the queued signal auto-marshals to the main thread.

This fix was applied first because WR-01 uses the same signal in its
backstop path.

### WR-01: `_twitch_resolve_worker` unguarded daemon-thread code paths

**Files modified:** `musicstreamer/player.py`
**Commit:** 63d4482
**Applied fix:** Wrapped the entire worker body (after the narrow streamlink
imports) in a top-level `try/except Exception as e` backstop that emits
`playback_error` with the repr and calls `self._try_next_stream_requested.emit()`
(using the WR-03 queued signal). The existing inner `NoPluginError` /
`PluginError` handlers are unchanged. Now any `AttributeError`/`TypeError`
from a future streamlink API rename (the exact 999.8 failure mode), a
constructor failure, or a property access crash on `streams["best"]`
surfaces as a user-visible error and advances the failover queue instead
of silently killing the daemon thread.

### WR-02: `_youtube_resolve_worker` unguarded prologue/epilogue

**Files modified:** `musicstreamer/player.py`
**Commit:** b2b9c67
**Applied fix:** Wrapped the entire worker body in a top-level
`try/except Exception as e` backstop that emits
`youtube_resolution_failed` with the repr. Covers ImportError on `yt_dlp`,
filesystem errors in the cookies corruption check
(`paths.cookies_path()`, `os.path.exists`, `cookie_utils.is_cookie_file_corrupted`,
`constants.clear_cookies()`), and `PermissionError`/`OSError` from
`cookie_utils.temp_cookies_copy()` `__enter__`. The existing inner
`extract_info` except-block is preserved. The `youtube_resolution_failed`
signal is already connected via `QueuedConnection` and its handler
(`_on_youtube_resolution_failed`) advances the failover queue on the main
thread.
