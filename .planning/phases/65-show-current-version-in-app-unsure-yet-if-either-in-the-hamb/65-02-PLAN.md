## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source `.spec` file → PyInstaller build | The .spec is Python interpreted by `pyinstaller MusicStreamer.spec` at build time. The build runs on Kyle's Win11 VM (developer-controlled, trusted). No untrusted input crosses this boundary. |
| `copy_metadata("musicstreamer")` → bundle dist-info | `copy_metadata` reads from the install's `site-packages/musicstreamer-*.dist-info/` directory and copies its files into the bundle. | Bundled dist-info → runtime importlib.metadata lookup | At app startup, `importlib.metadata.version("musicstreamer")` reads the bundled METADATA file. | T-65-03 | Information Disclosure | dist-info METADATA shipping in bundle | accept | METADATA contents (`Name`, `Version`, `Summary`, `Home-page`, `License`, `Requires-Dist`) are public information already in `pyproject.toml` and (when published) PyPI. No PII, no secrets. Threat surface: zero. |

A user installing a "successful" build is guaranteed to have a valid dist-info; a developer will catch the failure at PyInstaller time, not in production. |
