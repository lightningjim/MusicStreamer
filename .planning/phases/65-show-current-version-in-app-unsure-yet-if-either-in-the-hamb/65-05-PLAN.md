# Phase 65 Plan 05: Replace `uv pip` with `python -m pip` in build.ps1 step 3c so Win11 VM spike conda env can run the build (VER-02-J retest gate)

<objective>
Close the new gap surfaced by today's UAT retest of VER-02-J: `build.ps1` step 3c "PRE-BUNDLE CLEAN" crashes on the Win11 VM with `uv : The term 'uv' is not recognized as the name of a cmdlet, function, script file, or operable program.` The crash aborts the build before reaching either `PRE-BUNDLE CLEAN OK` or `POST-BUNDLE ASSERTION OK`, so the v2.1.65 fix shipped by Plan 65-04 cannot be verified end-to-end.

**Why uv is missing on the Win11 VM:** Per the project's `Skill("spike-findings-musicstreamer")` (`references/windows-gstreamer-bundling.md` lines 7-19), the validated build environment for Windows is a conda-forge env created via:

```powershell
conda create -n spike -c conda-forge `
    python=3.12 `
    pygobject pycairo `
    pyinstaller pyinstaller-hooks-contrib `
    gstreamer gst-plugins-base gst-plugins-good gst-plugins-bad gst-plugins-ugly
conda activate spike
```

`uv` is not in that recipe. It IS on the Linux dev host (where the project's top-level `uv.lock` is the source of truth), but the Windows build host intentionally uses conda for PyGObject (PyPI has only sdist for PyGObject; conda-forge is the only viable path on Windows per the spike findings). Plan 65-04's step 3c assumed `uv` would be on PATH everywhere; on the Win11 VM it isn't, and the `Invoke-Native` wrapper surfaces the cmdlet-not-found error verbatim before the failure branch can fire.

**Replacement:** `python -m pip uninstall musicstreamer -y` + `python -m pip install -e ..\..\` — `pip` is bundled with every conda env and every uv-managed venv, so it works on BOTH hosts (Linux dev AND Win11 VM) without conditional logic. This is also the existing project precedent at step 3 (lines 102-108): `python -m pip install --upgrade "pyinstaller>=6.19" "pyinstaller-hooks-contrib>=2026.2" "pygobject>=3.50"` already runs in the same script when not skipping the pip install. The new step 3c just adopts the same idiom.

**Cross-cutting concern:** Plan 65-04 added two drift-guard tests in `tests/test_packaging_spec.py` that lock the literal `uv pip uninstall musicstreamer` and `uv pip install -e` / `uv sync --reinstall-package musicstreamer` substrings. If we change build.ps1 without updating those assertions, the drift-guards turn RED and the test suite breaks. If we silently relax the assertions to `pass` we destroy the regression-lock value and a future maintainer could remove step 3c entirely without the guards firing. The mitigation is to update the assertions in lockstep — replace the `uv pip` substrings with `python -m pip` substrings, keeping the SAME intent (the step exists, it uninstalls musicstreamer, it reinstalls musicstreamer, it exits 8 on failure with the documented BUILD_FAIL reason, the failure path uses Write-Host not Write-Error per WR-01). The legacy `uv pip install -e` and `uv sync --reinstall-package musicstreamer` substrings stay accepted in the reinstall acceptance set, purely for forward-compat in case a future maintainer reintroduces a uv-managed env on Windows; the primary `python -m pip` substring is the one that must be present after this plan ships.

**Scope discipline (from `<planning_directives>`):**

- Scope is exactly: (1) make build.ps1 step 3c run on the Win11 spike conda env without `uv`; (2) update drift-guards in lockstep; (3) verify cross-platform via pytest on Linux + provide explicit Win11 VM retest instructions.
- Out of scope: broader build refactors, conda env provisioning automation, VER-02-J h1/h2/h3 hypotheses (h1/h2/h3 already shipped via Plan 65-04 step 3c + 4a; they cannot be re-verified until the build runs).
- The retest gate for VER-02-J is reused (NOT a new validation_id) — the same UAT prompt the user already has open re-runs after this plan ships.

Purpose: Close the new VER-02-J build-script gap so the Win11 VM rebuild can execute end-to-end, unblocking retest of Plan 65-04's already-shipped defenses (step 3c clean + step 4a assertion).
Output: build.ps1 step 3c rewritten with `python -m pip` + drift-guards updated in lockstep.

### packaging/windows/build.ps1 — current step 3c shape (lines 131-162)

```powershell
    # --- 3c. Pre-bundle dist-info clean (VER-02-J defense) -------------
    # PyInstaller's `copy_metadata("musicstreamer")` (MusicStreamer.spec
    # line 41) picks up EVERY musicstreamer-*.dist-info directory it
    # finds on sys.path. If the build env has both the current
    # 2.1.{phase}.dist-info AND a stale older dist-info (e.g. from a
    # previous v1.x or v2.0.x editable install that was never cleaned),
    # BOTH ship into dist/MusicStreamer/_internal/, and at runtime
    # importlib.metadata.version("musicstreamer") returns whichever
    # appears first on sys.path -- typically the older one (e.g. 1.1.0
    # observed on Kyle's Win11 VM during Phase 65 UAT, gap VER-02-J).
    #
    # Defense: uninstall + reinstall musicstreamer right before
    # pyinstaller runs, guaranteeing exactly one fresh dist-info
    # matching pyproject.toml [project].version exists when
    # copy_metadata scans. Cheap (~3-5s); makes the build resilient to
    # whatever historical state the build env happens to be in.
    #
    # DO NOT REMOVE without first updating both:
    #   - tests/test_packaging_spec.py (drift-guard test)
    #   - .planning/phases/65-.../65-04-PLAN.md (this plan's rationale)
    Write-Host "=== PRE-BUNDLE CLEAN: uv pip uninstall + reinstall musicstreamer ==="
    Invoke-Native { uv pip uninstall musicstreamer -y 2>&1 | Out-Host }
    # Note: uninstall exit code is intentionally NOT checked -- uv pip
    # uninstall returns non-zero if the package isn't installed, which
    # is fine on a fresh build env. We only care about the install
    # below succeeding.
    Invoke-Native { uv pip install -e ..\.. 2>&1 | Out-Host }
    if ($LASTEXITCODE -ne 0) {
        Write-Host "BUILD_FAIL reason=pre_bundle_clean_failed hint='uv pip install -e ..\..\\ failed; check uv install + pyproject.toml validity'" -ForegroundColor Red
        exit 8
    }
    Write-Host "PRE-BUNDLE CLEAN OK -- fresh musicstreamer dist-info materialized in build env"
```
