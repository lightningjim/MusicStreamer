## Decisions Made

- **on_title_changed ordering is load-bearing** — the six-step ordering matches RESEARCH §Pitfall 5 lines 339-350. Swapping no-downgrade guard and setText would allow the downgrade before the guard runs. Swapping `_gbs_label_source = 'icy'` flip and `_update_star_enabled` would show star inconsistently for one frame. No alternatives considered — the ordering spec was pre-locked in PATTERNS §E.
- **Bridge-window check uses `_gbs_label_source != 'ajax'` not `is None`** — after on_title_changed's own flip, the flag is 'icy'. 'icy' correctly qualifies as bridge window (cover-art suppressed). Using `is None` would suppress cover-art only on the first arrival and not on subsequent ICY arrivals that arrive before /ajax stamps. See PLAN §Sub-change A rationale block.
- **Kick is last in on_title_changed** — any side effects from `_on_gbs_poll_tick()` (spawning a new `_GbsPollWorker`, connecting signals) cannot reach back and affect the local writes above. Direct method call — no `QTimer.singleShot`, no `Signal.connect` (per RESEARCH §Anti-Patterns and PATTERNS §"Bound-method connection NOT used by 60.3").
- **test_gbs_auth_expired_relaxes_bridge_gate does NOT assert star_btn.isEnabled()** — the cookies file is present in this test (to simulate auth_expired during a live session), so `_is_gbs_logged_in()` returns True and the bridge gate is still active. The D-08 star-enabled relaxation is cleanly verified by `test_gbs_logged_out_icy_writes_normally` (no cookies file). Two-test split is intentional and documented per PLAN §"Important nuance for test_gbs_auth_expired_relaxes_bridge_gate".
