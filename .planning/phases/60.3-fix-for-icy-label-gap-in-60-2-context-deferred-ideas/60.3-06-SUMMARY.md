## Decisions Made

- **Extract a single helper rather than duplicate the favourites block.** The original CR-03 defect was that two writers (`on_title_changed` and `_apply_gbs_icy_label`) each owned a copy of "refresh the star surface" — and one drifted. The fix is structural: there is now exactly one writer, callable from both sites. This is the recommendation in REVIEW.md lines 167-168.
- **Preserve existing per-test `gbs_station.icy_disabled = False` overrides.** A naive CR-04 fix would have removed all 12 explicit overrides in Plans 02-05 tests after setting the factory default. We did NOT do this for three reasons: (1) defence-in-depth — if a future regression flips the factory default, those tests fail loud instead of silent; (2) the explicit override pattern is the BLOCKER #1 invariant called out in Plan 03's docstrings — surface visibility is a feature; (3) the diff stays minimal and CR-04-focused.
- **Two named tests, not one parametrized.** The favourited vs un-favourited scenarios use asymmetric `is_favorited` side-effect dispatchers (`track_title == "Artist - Track"` vs `track_title == "Track"`) — parametrization would obscure the load-bearing piece of each test. Two clearly named tests aid grep and match the file's `test_gbs_logged_out_*` / `test_gbs_auth_expired_*` sibling-test convention.
- **Drop the redundant `self._station is not None` clause from the cover-art block.** The new WR-01 provider guard at the top of `_apply_gbs_icy_label` already guarantees `self._station is not None` is true by the time execution reaches the cover-art block. Keeping the redundant check would be defensive-but-misleading — it would imply the WR-01 guard might not have run.
