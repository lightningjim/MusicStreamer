## Gaps Summary

Two blocking gaps prevent the full D-01..D-08 contract from being met:

**Gap 1 — D-05 over-fires across track changes (CR-01):** The no-downgrade guard at `on_title_changed:542-543` (`if is_gbs and self._gbs_label_source == 'ajax': return`) uses a coarse source-flag check with no track-identity component. When Track A is /ajax-stamped (`_gbs_label_source = 'ajax'`), the flag is never reset before Track B's ICY tag arrives. The guard fires on Track B as if it were a same-track ordering race, blocking both the label write AND the D-03 kick. The user sees Track A's label for up to 15 s before the periodic timer fires. The CONTEXT doc required either a title-suffix match or entryid coupling; neither was implemented. Fix: reset `_gbs_label_source = None` in `_on_gbs_playlist_ready` when `track_changed is True`, OR split the kick out of the early-return block so it always runs in GBS context.

**Gap 2 — D-08 auth-expired does not relax bridge gate when cookies persist (CR-02):** `_on_gbs_playlist_error('auth_expired')` flips `_gbs_label_source = 'icy'`, but the bridge-gate predicate in `_update_star_enabled` and `on_title_changed` also checks `_is_gbs_logged_in()` (cookie file existence). The file is not deleted on auth_expired, so both checks remain True and the gate stays active. Star is disabled and cover-art is suppressed even though /ajax is no longer possible. `test_gbs_auth_expired_relaxes_bridge_gate` explicitly documents this with prose rather than an assertion. Fix: delete the cookie file in the auth_expired branch, OR introduce a `_gbs_ajax_disabled` flag and fold it into the gate predicate.

The REVIEW (60.3-REVIEW.md) accurately identifies both of these as BLOCKER-level findings (CR-01, CR-02) along with WARNING-level findings CR-03 (star icon not refreshed on /ajax upgrade) and CR-04 (existing test contaminated by MagicMock truthy icy_disabled).
