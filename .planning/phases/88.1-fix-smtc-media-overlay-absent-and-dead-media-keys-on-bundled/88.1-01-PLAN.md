<objective>
Make the Windows media-keys backend selection LOUD and DIAGNOSABLE, and add the
headless backend-identity check the Windows build's smoke guard will invoke.

This plan covers the application/runtime-harness half of Phase 88.1 (gap G2):
- D-03: backend-selection logging in the `media_keys` factory so the silent
  degradation that hid G2 can never ship undiagnosed again.

  `__main__.py` that the (plan 88.1-02) `build.ps1` SMTC smoke guard calls.
- D-03 test coverage: extend `test_media_keys_smtc.py` to lock in the INFO
  success log and the WARNING fallback log (with exception type).

This half runs and verifies entirely on Linux. The frozen-exe behavior of
`--check-mediakeys` is VM-deferred to the consolidated 88-03 session (D-06/D-07).
