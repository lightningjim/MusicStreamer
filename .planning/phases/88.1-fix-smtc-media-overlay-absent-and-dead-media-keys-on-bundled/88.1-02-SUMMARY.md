# Phase 88.1 Plan 02: Windows spec winrt collect_all + SMTC smoke guard Summary

Fix the root-cause PyInstaller bundling defect (gap G2) by adding `collect_all` for five pywinrt distributions to `MusicStreamer.spec`, adding a step-4c SMTC smoke guard to `build.ps1` (exit 11), and creating `tests/test_packaging_winrt_spec.py` as a Linux static drift-guard.

## What Was Built

### Task 1 — collect_all for 5 winrt distributions (MusicStreamer.spec)

Added five `collect_all` triplets after the existing `charset_normalizer`/`streamlink`/`yt_dlp` block, following the `_<abbrev>_datas/_binaries/_hiddenimports` naming convention:

- `_wr_datas, _wr_binaries, _wr_hiddenimports = collect_all("winrt-runtime")`
- `_wmp_datas, _wmp_binaries, _wmp_hiddenimports = collect_all("winrt-Windows.Media.Playback")`
- `_wm_datas, _wm_binaries, _wm_hiddenimports = collect_all("winrt-Windows.Media")`
- `_wss_datas, _wss_binaries, _wss_hiddenimports = collect_all("winrt-Windows.Storage.Streams")`
- `_wf_datas, _wf_binaries, _wf_hiddenimports = collect_all("winrt-Windows.Foundation")`

All five sets of outputs are wired into `Analysis(binaries=..., datas=..., hiddenimports=...)` via `+` concatenation. The four old `"winrt.windows.*"` hiddenimports-only strings are removed (collect_all subsumes them). `upx=False` is unchanged.

**Commit:** `1a6fdca6`

### Task 2 — Step-4c SMTC smoke guard (build.ps1)

Updated the exit-codes header comment to document `11=smtc backend not loaded in frozen bundle (Phase 88.1 / WIN-02)`. Inserted a new step 4c block between step 4b (plugin guard) and step 5 (smoke):

- `Write-Host "=== SMTC SMOKE GUARD: assert WindowsMediaKeysBackend in frozen bundle (Phase 88.1 / D-05 / WIN-02) ==="`
- `Invoke-Native { & "..\..\dist\MusicStreamer\MusicStreamer.exe" --check-mediakeys 2>&1 | Out-Host }`
- On `$LASTEXITCODE -ne 0`: `Write-Host "BUILD_FAIL reason=smtc_backend_not_loaded hint='...'" -ForegroundColor Red; exit 11`
- `Write-Host "SMTC SMOKE GUARD OK"` on success

WR-01 compliant: uses `Write-Host` (not `Write-Error`) so the `exit 11` actually fires under `$ErrorActionPreference = "Stop"`.

**Commit:** `0b7f7a2d`

### Task 3 — tests/test_packaging_winrt_spec.py (6 tests, 221 lines)

New Linux static drift-guard file mirroring `tests/test_packaging_spec.py`'s idiom:

| Test | What It Asserts |
|------|----------------|
| `test_spec_collects_winrt_runtime` | `collect_all("winrt-runtime")` present |
| `test_spec_collects_winrt_namespace_packages` | All 4 namespace distributions present |
| `test_spec_wires_winrt_binaries_into_analysis` | `_wr_`/`_wf_` binaries/datas/hiddenimports wired |
| `test_spec_removes_old_winrt_hiddenimports` | Old `winrt.windows.*` strings absent (strip comments) |
| `test_build_ps1_smtc_smoke_guard_present` | `SMTC SMOKE GUARD` header + `--check-mediakeys` + `BUILD_FAIL reason=smtc_backend_not_loaded` |
| `test_build_ps1_smtc_smoke_guard_exit_11` | `exit 11` + `11=smtc` header doc + `Write-Host` WR-01 compliance |

All 6 tests green on Linux. Full packaging suite (49 tests across 3 files) passes.
