<objective>
Fix the root-cause PyInstaller bundling defect (gap G2): pywinrt 3.2.x ships
compiled `.pyd` extensions at the `winrt/` namespace root that `hiddenimports`
cannot collect, so the frozen exe's winrt import fails and the factory silently
degrades to NoOp. The fix is `collect_all` for the five pywinrt distributions
(mirroring the existing charset_normalizer/streamlink/yt_dlp pattern), a
build-time SMTC smoke guard in `build.ps1`, and a Linux static drift-guard test.

This plan covers the packaging/CI half of Phase 88.1. All assertions are
Linux-runnable source-text checks — no PyInstaller or Windows build required.
The actual Windows-build smoke pass and UAT-3/UAT-7 re-test are VM-deferred to
the consolidated 88-03 session (D-06/D-07).

Purpose: Close the Linux-side of WIN-02 / VER-02-J gap G2's bundling fix + guards.
Output: Patched spec, build.ps1 SMTC guard, new static spec test.
</objective>
