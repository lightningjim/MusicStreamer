### `musicstreamer/repo.py` — Backfill pass inside `db_init`

**Analog:** No direct backfill analog in `db_init` today (all prior migrations are schema-only). The SQL subquery shape follows RESEARCH.md §Finding 2 and is confirmed safe by analysis of `updated_at TEXT DEFAULT (datetime('now'))` — ISO-8601 lexicographic ordering guarantees `MAX(updated_at)` is the most-recently-updated row.

**Backfill pattern** (goes immediately after the providers ALTER block):
```python
# Phase 89.1 D-01/D-02/D-03: one-time idempotent backfill.
# Copy the most-recently-updated per-station PNG to the provider-keyed location
# and record the path in providers.avatar_path. Old per-station files deleted
# only after providers.avatar_path is committed (Pitfall 3 — shutil.copy2 not
# os.rename; source file survives a mid-backfill crash).
import os as _os, shutil as _shutil
from musicstreamer import paths as _bfpaths  # safe: no Qt dependency

_avatar_dir = _bfpaths.channel_avatars_dir()
_bf_rows = con.execute(
    """
    SELECT s.provider_id,
           s.channel_avatar_path,
           s.id AS station_id
    FROM   stations s
    WHERE  s.channel_avatar_path IS NOT NULL
      AND  s.provider_id IS NOT NULL
      AND  s.updated_at = (
               SELECT MAX(s2.updated_at)
               FROM   stations s2
               WHERE  s2.provider_id = s.provider_id
                 AND  s2.channel_avatar_path IS NOT NULL
           )
      AND  NOT EXISTS (
               SELECT 1 FROM providers p
               WHERE p.id = s.provider_id AND p.avatar_path IS NOT NULL
           )
    ORDER BY s.provider_id, s.id DESC
    """
).fetchall()
_seen: set = set()
for _row in _bf_rows:
    _pid = _row[0]
    if _pid in _seen:
        continue  # take only the first (best) row per provider
    _seen.add(_pid)
    _src = _os.path.join(_bfpaths.data_dir(), _row[1])
    if not _os.path.isfile(_src):
        continue  # avatar missing on disk — skip this provider
    _dst = _os.path.join(_avatar_dir, f"{_pid}.png")
    _prov_rel = _os.path.relpath(_dst, _bfpaths.data_dir())
    try:
        _os.makedirs(_avatar_dir, exist_ok=True)
        _shutil.copy2(_src, _dst)
    except OSError:
        continue  # skip provider on copy failure — don't abort db_init
    con.execute(
        "UPDATE providers SET avatar_path = ? WHERE id = ?",
        (_prov_rel, _pid),
    )
    con.commit()

# Clean up per-station files whose provider now has an avatar (D-01).
# OSError swallowed — file may already be gone (idempotent).
for _cr in con.execute(
    """
    SELECT s.channel_avatar_path
    FROM   stations s
    JOIN   providers p ON p.id = s.provider_id
    WHERE  s.channel_avatar_path IS NOT NULL
      AND  p.avatar_path IS NOT NULL
    """
).fetchall():
    try:
        _os.unlink(_os.path.join(_bfpaths.data_dir(), _cr[0]))
    except OSError:
        pass  # already gone — idempotent
```

### `musicstreamer/assets.py` — `write_provider_avatar` (new sibling function)

**Analog:** `assets.py:32–60` (`write_channel_avatar`)

**Exact analog** (lines 32–60):
```python
def write_channel_avatar(station_id: int, data: bytes) -> str:
    """Write avatar PNG bytes atomically to the channel-avatars directory.

    Returns path relative to paths.data_dir(), e.g. 'assets/channel-avatars/12.png'.
    Uses tempfile.mkstemp in the same directory so os.replace is atomic
    (same filesystem — RESEARCH.md A2 / D-12).

    station_id is an int from SQLite — no user-controlled string, no traversal
    risk (T-89-02).
    """
    dst_dir = paths.channel_avatars_dir()
    os.makedirs(dst_dir, exist_ok=True)
    dst = os.path.join(dst_dir, f"{station_id}.png")
    fd, tmp = tempfile.mkstemp(dir=dst_dir, suffix=".png.tmp")
    try:
        os.write(fd, data)
        os.close(fd)
        os.replace(tmp, dst)
    except Exception:
        try:
            os.close(fd)
        except OSError:
            pass
        try:
            os.unlink(tmp)
        except OSError:
            pass
        raise
    return os.path.relpath(dst, paths.data_dir())
```

    Uses tempfile.mkstemp + os.replace for atomicity (same as write_channel_avatar).
    provider_id is an int from SQLite — no traversal risk (T-89.1-01).
