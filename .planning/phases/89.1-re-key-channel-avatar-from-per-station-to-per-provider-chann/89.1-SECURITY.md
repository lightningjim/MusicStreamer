## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| SQLite row → filesystem path | `provider_id` (int from DB) is formatted into `{provider_id}.png` | integer PK, non-sensitive |
| Backfill copy/delete during db_init | files copied and old per-station files deleted during the one-time migration | local PNG bytes (prior network fetch) |
| DB row (`Station.provider_avatar_path`) → Qt render | a relative path string from SQLite resolved against `data_dir()` and loaded as a QPixmap | relative path string |
| EditStationDialog worker thread → filesystem/DB | the fetch worker writes a PNG; the main thread persists the provider path | PNG bytes + relative path |

### Additional defense (code-review CR-01)

A NULL `provider_id` guard (`edit_station_dialog.py:1312–1316`) fires before the fetch worker is constructed for a providerless station, preventing a junk `None.png` orphan and a silent 0-row `UPDATE … WHERE id = NULL`. This reinforces the T-89.1-01 int-keyed-path boundary. Fixed in commit `a2d1b4be`.
