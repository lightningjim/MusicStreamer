# Phase 89.1 Plan 02: Consumer Re-wire — provider_avatar_path Summary

**One-liner:** Rewired three avatar consumers (_channel_avatar_lookup, bind_station, EditStationDialog) to read and persist via the per-provider provider_avatar_path established in Plan 01, with D-07 reuse-on-open gate and D-08 force-refresh bypass.

## What Was Built

Plan 02 consumed the per-provider avatar infrastructure from Plan 01 across three consumer sites:

1. **`cover_art._channel_avatar_lookup`** (D-05): Single attribute rename from `channel_avatar_path` to `provider_avatar_path`. WR-04 contract (never-raise, no thread, no QPixmap) fully preserved — only the field read changed.

2. **`now_playing_panel.bind_station`** (D-04/D-06): Replaced the `channel_avatar_path` read with a guarded `provider_avatar_path` resolution:
   - `getattr(station, "provider_avatar_path", None)` reads the per-provider path
   - `os.path.isfile(os.path.join(_p.data_dir(), _provider_rel))` guards against stale DB entries (Pitfall 4 / T-89.1-05)
   - NULL provider or missing file falls through to the station logo already shown by `_show_station_logo_in_cover_slot()` (D-06)

3. **`edit_station_dialog.py`** — Six coordinated changes:
   - `_AvatarFetchWorker.__init__`: added `provider_id: int | None = None` param, stored as `self._provider_id`
   - `_AvatarFetchWorker.run`: calls `write_provider_avatar(self._provider_id, data)` instead of `write_channel_avatar`
   - `_on_url_timer_timeout` (YouTube branch): D-07 reuse-on-open gate — if `provider_avatar_path` is set and `_force_avatar_refresh` is False, show "Avatar loaded from channel (shared)" status and return without network fetch
   - `_on_avatar_fetched`: sets `self._station.provider_avatar_path = rel_path` in-memory, shows D-08 shared-effect hint `"Avatar saved — all {provider_name} stations updated"`, persists via `update_provider_avatar_path(provider_id, rel_path)` (not `update_channel_avatar_path`)
   - `_refresh_avatar_preview`: reads `provider_avatar_path` (not `channel_avatar_path`)
   - `_on_refresh_avatar_clicked`: sets `_force_avatar_refresh = True` in try/finally so Refresh always re-fetches; `__init__` initializes the flag as `False`

4. **`tests/test_cover_art_avatar.py`** (TDD RED→GREEN):
   - RED: Updated three existing lookup tests to mock `provider_avatar_path` with `spec=["provider_avatar_path"]` guard, preventing MagicMock from auto-creating `channel_avatar_path` (Pitfall 7). Added `test_channel_avatar_lookup_null_provider_avatar_calls_cb_none` for D-06 fallback 2.
   - GREEN: Updated production code passed all 10 tests.
