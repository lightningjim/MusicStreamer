```python
def _on_eq_ramp_tick(self) -> None:
    state = self._eq_ramp_state
    if self._eq is None or state is None:
        self._eq_ramp_timer.stop()
        self._eq_ramp_state = None
        return
    state["tick_index"] += 1
    k = state["tick_index"]
    n = self._eq.get_children_count()
    target = state["target_gain"]
    start = state["start_gain"]
    if k >= self._EQ_RAMP_TICKS:
        # Final tick: commit exact target (no lerp residual).
        for i in range(n):
            t_i = target[i] if i < len(target) else 0.0
            self._eq.get_child_by_index(i).set_property("gain", t_i)
        self._eq_ramp_timer.stop()
        self._eq_ramp_state = None
        return
    t = float(k) / float(self._EQ_RAMP_TICKS)
    for i in range(n):
        s_i = start[i] if i < len(start) else 0.0
        t_i = target[i] if i < len(target) else 0.0
        g = s_i + (t_i - s_i) * t
        self._eq.get_child_by_index(i).set_property("gain", g)
```

Composition / sequencing rationale (D-12 + D-15):
- Phase 52's EQ ramp is asynchronous (QTimer ticks across 40ms total). Naive port to pause() would mean: pause() arms the QTimer and returns; QTimer ticks fire across 40ms; on the final tick (when audible volume reaches 0), set_state(NULL) is called. This is the "smoothing-then-NULL" shape.
- During those 40ms, Plan 57-03's bus-message handler is dormant (no PLAYING-arrival yet — the pipeline is still PLAYING, draining toward NULL).
- After NULL, pause() returns. User clicks Play -> play()/play_stream() -> _try_next_stream() -> _set_uri() -> set_state(PLAYING). playbin3 transitions NULL -> READY -> PAUSED -> PLAYING. On the PLAYING transition, Plan 57-03's bus-message handler fires and writes self._volume to playbin3.volume. The user perceives: ~40ms fade-out, brief silence (the NULL+rebuild gap), then audible audio at the slider's value.
- No double-write: ramp ticks only run while pipeline is PLAYING (pre-NULL); bus-message handler only fires post-NULL on PLAYING-arrival. The two write windows are disjoint by construction.

Final-tick behavior choice (write 0 OR skip the write before NULL):
- Phase 52 final-tick writes the EXACT target gain. For pause volume ramp, target = 0 (silence). The final tick writes 0 to playbin3.volume, then set_state(NULL) is called inside the same tick callback — so the user hears silence right before the NULL teardown, which is exactly the "no pop" behavior we want.
- DO NOT skip the final 0 write — partial-fade-then-NULL still pops because the audio level isn't zero at NULL-time.
