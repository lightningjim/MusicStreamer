## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-11 | Denial-of-Service | `_on_title_changed_for_media_keys` called on every ICY update | accept | D-05 establishes emissions are rare (2-5 min typical). Cover-art write is to a stable per-station path (D-04). No debouncing needed. |
| T-41-12 | Tampering | `_on_media_key_stop` can be called by any session-bus client | accept | Single-user desktop threat model. Worst outcome is stream stopping — user can resume. Matches v1.5 acceptance. |
| T-41-13 | Race condition | MainWindow close while backend is mid-publish | mitigate | `closeEvent` calls `backend.shutdown()` synchronously before calling `super().closeEvent`. The backend's shutdown is idempotent and only touches D-Bus connection state. publish_metadata in flight (if any) completes on the same main thread before close event processes (Qt event loop is cooperative). Test 12 (Plan 02) confirms idempotency. |
| T-41-14 | Information Disclosure | Cover-art PNG on disk (in user cache dir) | accept | Same trust zone as the app's SQLite db + cached logos. File permissions follow user's umask. Not a credential or secret. |
</threat_model>
