## Interaction Contract

This is the only dimension with substantive Phase 55 content. Two distinct event boundaries with two distinct contracts.

### Boundary A — Save / mutation refresh (this phase's fix)

**Triggers:** Any caller of `MainWindow._refresh_station_list()` →
`StationListPanel.refresh_model()`. Five callers per CONTEXT D-01:
1. Edit-existing-station save (`main_window.py:467`)
2. New-station save (`main_window.py:447`)
3. Station delete (`main_window.py:476` via `_on_station_deleted`)
4. Discovery import-complete (`main_window.py:633`)
5. Settings import-complete (`main_window.py:648` / `:653`)

**User-perceivable contract** (testable, measurable):

| Pre-Save state of provider group `G` | Post-Save state of `G` | Source decision |
|--------------------------------------|------------------------|-----------------|
| Expanded, still exists post-save | **Expanded** (preserved) | CONTEXT D-04 / SC #1 |
| Collapsed, still exists post-save | **Collapsed** (preserved) | CONTEXT D-04 / SC #2 |
| Did not exist pre-save (brand-new provider) | **Expanded** (D-06 default) | CONTEXT D-06 |
| Existed pre-save, no longer exists post-save (last station moved/deleted) | **Silently dropped** — captured state for the gone group is pruned by intersect | CONTEXT §"Empty-source-group cleanup" |
| Destination group of a cross-provider move, was collapsed pre-save | **Collapsed** (preserved — not auto-expanded) | CONTEXT D-07 |
| Any group, while a search/chip filter is active | **Same captured state** — filter does not change the contract | CONTEXT D-04 |

**Negative contract** (must NOT happen on save):
- `refresh_model()` must NOT call `self._sync_tree_expansion()` (CONTEXT D-03; spy-based test required per RESEARCH §Validation Architecture).
- `refresh_model()` must NOT call `self.tree.expandAll()` or `self.tree.collapseAll()` directly.
- `refresh_model()` must NOT issue any expand/collapse on a provider group whose name is in neither the captured-expanded set nor the brand-new-difference set.

**Non-tree side effects of `refresh_model()` are unchanged** (CONTEXT §Claude's Discretion):
- `self._populate_recent()` still runs after the model reset.
- `self._build_chip_rows()` still runs after the model reset.
- Neither touches tree expansion; neither is affected by this fix.

**Animation / motion contract:** None. Qt's `tree.expand(idx)` is instantaneous (no animation). The user perceives the save as happening with the tree visually frozen at their pre-save state — except for the rows of the actual saved station, which the model reset re-renders. There is no transition, no fade, no scroll re-anchor introduced by this phase.

**Race / ordering contract:**
1. Capture (build `expanded_pre: set[str]` and `all_pre: set[str]`) MUST run **before** `self.model.refresh(...)` — capture reads the live view-side `tree.isExpanded(...)` state, which is invalidated by `endResetModel`.
2. `self.model.refresh(...)` runs (wraps `beginResetModel`/`endResetModel` per existing implementation).
3. Restore walks the post-refresh source model and applies `tree.expand(proxy_idx)` for matching names, guarded by `proxy_idx.isValid()` (RESEARCH Pitfall #2).
4. `self._populate_recent()` then `self._build_chip_rows()` run last.

This ordering matches the existing line ordering at `station_list_panel.py:316-319` for the post-refresh side-effect calls; the only structural change is the new capture-before / restore-after wrapping around line 316 (now replacing line 317's deleted `_sync_tree_expansion()` call).

### Boundary B — Filter-change (unchanged behavior)

**Triggers:** Any of:
- `_on_search_changed` (line 424) — user types into / clears the search box
- `_on_provider_chip_clicked` (line 428) — user clicks a provider chip
- `_on_tag_chip_clicked` (line 438) — user clicks a tag chip
- `_clear_all_filters` (line 453) — user clicks the "Clear all" button

**Contract (preserved verbatim from current behavior — CONTEXT D-05):**

| Filter state after change | Tree action |
|---------------------------|-------------|
| Any filter active (search non-empty OR ≥1 provider chip selected OR ≥1 tag chip selected) | `self.tree.expandAll()` |
| All filters cleared (back to no-filter state) | `self.tree.collapseAll()` |

**This boundary intentionally CLOBBERS user manual expansion state.** That is the existing project decision (Phase 38 / earlier), not a Phase 55 change. The user's mental model: "if I'm filtering, show me everything that matches; if I'm browsing, give me a clean collapsed list."

**Regression-lock required** (RESEARCH §Validation Architecture point 2): a spy-based pytest-qt test must assert that `_on_search_changed` (and at least one chip-click handler + clear-all) STILL invoke `_sync_tree_expansion()` after this phase ships. Without this lock, a future "simplification" could remove the auto-expand-on-filter behavior silently.

### Boundary separation guarantee

The single-line load-bearing invariant the executor must preserve:

> **Save events do not call `_sync_tree_expansion()`. Filter-change events do.**

This is the entire visible behavioral delta of Phase 55. Stated as a code-level test: deleting line 317 (`self._sync_tree_expansion()` from inside `refresh_model`) and adding the capture/restore wrap is the minimal sufficient change.
