<objective>
Close the CI-runnable half of WIN-02-B and the WIN-05 guard-regression half on
Linux. Create `tests/test_aumid_string_parity.py` with two concerns:

1. A static 3-way AUMID parity assertion: the literal in
   `musicstreamer/constants.py::APP_ID`, the value used by
   `musicstreamer/__main__.py::_set_windows_aumid`, and the
   `AppUserModelID:` literal in the Inno `[Icons]` directive of
   `packaging/windows/MusicStreamer.iss` are all byte-identical to
   `org.lightningjim.MusicStreamer`. This catches DRIFT between the three
   sources of truth forever, in normal Linux CI, with no installed `.lnk`.

2. An AAC-guard regression assertion for WIN-05: import
   `tools.check_bundle_plugins.REQUIRED_PLUGIN_DLLS`, assert it still maps
   `gstlibav.dll -> avdec_aac` (gst-libav) and `gstaudioparsers.dll -> aacparse`,
   and assert `check_bundle_plugins.main()` returns exit code 10 against a
   tmp_path bundle that is missing `gstlibav.dll` (simulating a missing
   gst-libav rebuild).

Purpose: The actual installed-`.lnk` AUMID readback (`System.AppUserModel.ID`)
can only run on a Windows VM (handled by the Plan 03 UAT). This plan delivers
the automatic, forever-green drift guard plus the AAC guard regression that
DO run on Linux CI, so any future edit that desyncs the AUMID or weakens the
AAC plugin guard fails the suite immediately.

Output: `tests/test_aumid_string_parity.py`.
