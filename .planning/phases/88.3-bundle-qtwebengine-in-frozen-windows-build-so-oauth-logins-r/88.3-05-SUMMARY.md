## Deviations from Plan — Two Gap-Closure Fixes Found at UAT

The UAT initially failed on two distinct defects; both were root-caused and fixed during the session (the gate's explicit "describe the failure for gap closure" path), then re-validated by a VM rebuild.

**1. [Blocker] Helper `oauth_helper.exe` exit=2 — `DLL load failed while importing QtWebEngineWidgets: the specified procedure could not be found`**
- **Root cause:** `build.ps1` froze the pip-only helper with conda's `Library\bin` still on `$env:PATH` (step 0 prepends it for the main bundle's GStreamer introspection). PyInstaller resolves each collected DLL's transitive imports via the build-time PATH, so it baked conda's `Qt6Core.dll` into the helper bundle next to pip's WebEngine `.pyd` → ABI mismatch in every environment (the G6 signature). Presence checks passed because the DLLs existed; they were just the wrong (conda) Qt.
- **Fix (commit `45c3ce7c`):** Strip `conda|miniforge|miniconda`, `\Library\bin`, and `$GstRoot` from PATH for the helper pip-install + PyInstaller (`--clean`), restore for the Inno step. Add a SHA256 **ABI guard** asserting bundled `Qt6Core.dll` == the venv's pip `Qt6Core.dll` (`BUILD_FAIL helper_qt6core_abi_mismatch` otherwise). Gitignore the recreated-every-build helper venv/work dirs + `full-build.log`.
- **Memory:** `helper-build-needs-conda-stripped-from-path`.

**2. [Blocker] Twitch "Browser not supported - integrity error" (Kasada)**
- **Root cause:** Twitch's Kasada anti-bot rejected the Qt 6.10 helper — its embedded Chromium 134 (revealed via `navigator.userAgentData` / `Sec-CH-UA`) is an integrity mismatch against the spoofed Chrome/140 UA. The Linux dev build (PySide6 6.11.0, newer Chromium) clears Kasada with identical helper code.
- **Fix (commit `46f30049`):** Bump helper `PySide6-Essentials`/`PySide6-Addons` 6.10.1 → 6.11.0. Cookie contract preserved — `oauth_helper` serializes via its own `_cookie_to_netscape` (stable `QNetworkCookie` accessors), and the helper and conda main app exchange only stdout text + exit codes (no shared Qt/ABI), so the helper may run a different PySide6 than the 6.10.1 conda main bundle. Helper-requirements drift-guard test updated to the new pins.
- **Memory:** `twitch-kasada-needs-newer-chromium`.
