## Current Test
<!-- OVERWRITE each test - shows where we are -->

[RESOLVED 2026-06-13 — all four tests pass on the two-artifact B1 build (installer `MusicStreamer-2.2.86-win64-setup.exe`, v2.2.86). Two root causes were found and fixed at this UAT session:
1. **Helper exit=2 (DLL-load)** — `build.ps1` froze the isolated helper with conda's `Library\bin` on the build PATH, so PyInstaller baked conda's `Qt6Core.dll` into the pip-WebEngine helper → ABI mismatch → "the specified procedure could not be found" in every environment. Fixed by sanitizing PATH around the helper pip-install + PyInstaller (`--clean`) and restoring it for Inno, plus a SHA256 **ABI guard** asserting the bundled `Qt6Core.dll` == the venv's pip `Qt6Core.dll`.
2. **Twitch "Browser not supported - integrity error"** — Twitch's Kasada anti-bot rejected the Qt 6.10 helper: its embedded Chromium 134 (revealed by `navigator.userAgentData` / `Sec-CH-UA`) is an integrity mismatch against the spoofed Chrome/140 UA. The Linux dev build (PySide6 6.11.0, newer Chromium) passes Kasada with identical code, so the helper was bumped to **PySide6 6.11.0** to match. GBS/Google were never affected.]
