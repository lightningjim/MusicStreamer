# Phase 88.3 Plan 01: WebEngine Bundle + Drift-Guards Summary

QtWebEngine hiddenimports + `--check-webengine` frozen-bundle guard + build.ps1 step-4e WEBENGINE GUARD (exit 13) + PySide6-Addons README preconditions, all verified by five G6 source-text drift-guard tests.

## What Was Built

This plan delivers all Linux-runnable source-text changes for Phase 88 UAT gap G6: the oauth_helper subprocess crashes exit=2 because QtWebEngine is not in the build env or bundled in MusicStreamer.spec. The main app deliberately never imports QtWebEngine (130MB startup avoidance), so PyInstaller's modulegraph is blind to it. Two tasks fixed this:

**Task 1: Wave 0 drift-guard tests + spec WebEngine hiddenimports**

- Created `tests/test_packaging_webengine_spec.py` with five G6 drift-guards (mirrors `test_packaging_winrt_spec.py` structure: module-scoped fixtures, `_strip_comments()` helper for negative gates, accepts single/double quotes)
- Edited `packaging/windows/MusicStreamer.spec`: added `"PySide6.QtWebEngineWidgets"` and `"PySide6.QtWebEngineCore"` to `Analysis()` `hiddenimports=` list alongside existing `PySide6.QtNetwork` / `PySide6.QtSvg` entries
- hiddenimports-only approach (RESEARCH OQ1): the hiddenimport triggers `hook-PySide6.QtWebEngineCore` which calls `get_qt_webengine_binaries_and_data_files()` to collect `QtWebEngineProcess.exe` / `*.pak` / locales / `qt.conf` — no `collect_all` needed or correct

**Task 2: --check-webengine guard + build.ps1 step-4e + preflight + README**

- Added `_run_check_webengine()` to `musicstreamer/__main__.py`: import-only guard (no `QCoreApplication`) — tries `from PySide6.QtWebEngineWidgets import QWebEngineView`; prints `WEBENGINE_GUARD OK` to stderr and returns 0 on success; prints `WEBENGINE_GUARD FAIL ...` to stderr and returns 13 on ImportError
- Added `--check-webengine` argparse flag + dispatch in `main()` (after `--oauth-helper`, consistent with existing order)
- Updated `build.ps1`:
  - Exit-codes header: appended `13=webengine not bundled in frozen bundle (Phase 88.3 / G6)`
  - Added `-SkipWebEngineGuard` switch param (mirrors `-SkipOauthGuard`)
  - step-0 preflight: `python -c "from PySide6.QtWebEngineWidgets import QWebEngineView"` before PyInstaller; `exit 1` with `BUILD_FAIL reason=pyside6_webengine_missing` + install hint
  - step-4e WEBENGINE GUARD: runs `MusicStreamer.exe --check-webengine`; on failure `Write-Host BUILD_FAIL reason=webengine_not_bundled` then `exit 13` (WR-01: `Write-Host` not `Write-Error`)
- Updated `packaging/windows/README.md`: Phase 88.3 note after conda recipe documenting `pip install "PySide6-Addons==$(python -c '...')"` exact version pin, ICU ABI hazard explanation, and warnings against unpinned install / `conda install qt6-webengine`
