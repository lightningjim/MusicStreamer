# Phase 70 — UI Design Contract

> Visual and interaction contract for the Hi-Res / Lossless quality indicator across five surfaces of the MusicStreamer Qt UI. Generated by gsd-ui-researcher 2026-05-11. Consumed by gsd-ui-checker, gsd-planner, gsd-executor, gsd-ui-auditor.

This is a **native Qt application** — vocabulary throughout is Qt-idiomatic (QLabel, QPushButton, QStyledItemDelegate, QPalette.ColorRole, QStyleOptionViewItem, QPainter primitives, QSS for widget chrome only). There is no web frontend, no shadcn, no Tailwind. The "design system" is the Phase 66 theme architecture (preset QPalette palettes + user-editable Custom) plus the Phase 68 LIVE-badge / filter-chip QSS templates, plus the Phase 60.4 `_theme.py` color tokens.

**Sample sizeHint math (delegate-painted tree-row pill, `HI-RES` text, 6 characters):**

Assume QFont().pointSize() = 9 pt at DPR=1.0 (Wayland primary target). At a typical Qt rendering ratio of ~1.3 px per point for bold sans-serif, "HI-RES" measures ≈ 50 px wide (use `QFontMetrics.horizontalAdvance("HI-RES")` at paint() time — do NOT hardcode). Add inner horizontal padding of 6 px on each side → pill width ≈ 62 px. Pill height = `QFontMetrics.height()` + 2 × 4 px ≈ 22 px. Tree row currently advertises width = `base.width() + 24 px` (star 20 + margin 4). New width = `base.width() + pill_width + 8 px (pill-to-star gap) + 24 px`. Concrete example for a "HI-RES" pill: `base.width() + 62 + 8 + 24 = base.width() + 94 px`. Height: `max(base.height(), STATION_ICON_SIZE=32, pill_height=22) = 32 px` (unchanged — pill fits inside existing row height).

The delegate MUST use `QFontMetrics(option.font).horizontalAdvance(text)` for pixel-accurate sizing; the 50-px estimate above is for sizeHint-budgeting documentation only.

## Color

Phase 70 uses **only QPalette tokens and one optional Phase 60.4 _theme.py constant**. **No new hex literals.** No `setBackground(QColor(...))` calls with inline hexes.

### Palette role usage

| Role | QPalette source | Phase 70 Usage |
|------|-----------------|----------------|
| Dominant (60% surface) | `QPalette.Base` / `QPalette.Window` | Inherited from parent widgets — unchanged from Phase 66. |
| Secondary (30% chrome) | `QPalette.Button` / `QPalette.AlternateBase` | Inherited — unchanged. |
| Accent (10% — quality badge fill + chip-selected fill + pill fill) | **`QPalette.Highlight`** (resolved via `palette(highlight)` in QSS, `option.palette.color(QPalette.Highlight)` in delegate paint) | (1) `_quality_badge` background; (2) `_hi_res_chip` selected-state background; (3) tree-row pill fill (delegate-painted) |
| Accent text | **`QPalette.HighlightedText`** (`palette(highlighted-text)` / `option.palette.color(QPalette.HighlightedText)`) | (1) `_quality_badge` text color; (2) `_hi_res_chip` selected-state text; (3) tree-row pill text |
| Error (destructive) | `_theme.ERROR_COLOR_HEX = "#c0392b"` | Not used in Phase 70 (no destructive actions). |
| Warning (amber) | `_theme.WARNING_COLOR_HEX = "#d4a017"` | Not used in Phase 70 (no warning state for the badge). |

**Accent reserved for:** the Phase 70 `_quality_badge` (QLabel in `icy_row`), the `_hi_res_chip`'s **selected** state, and the tree-row quality pill painted by `station_star_delegate.paint()`. Lossless and Hi-Res share the same `palette(highlight)` accent — the **two tiers are differentiated by text alone (`LOSSLESS` vs `HI-RES`)**, not by color. Rationale: the Phase 68 LIVE badge already occupies the highlight color; introducing a third tier-specific accent would break the cohesion. The user's library has ~10–20 stations active per week and visual scan-ability is dominated by text + icon, not color difference.

### Tree-row pill QPalette role under selection (CRITICAL — OPEN DECISION #3 from prompt)

The tree-row pill is painted in `station_star_delegate.paint()` AFTER `super().paint(painter, option, index)` (which renders the standard row chrome including the selected-row highlight band). When the row is selected, Qt fills the row background with `palette(highlight)`. If the pill ALSO uses `palette(highlight)` for its fill, it becomes invisible against the selected-row background.

**LOCKED DECISION (recommended):** Swap pill colors under selection.

```python
if option.state & QStyle.State_Selected:
    # On selected rows the row background is palette(highlight);
    # pill background must contrast. Use palette(highlighted-text)
    # as the fill and palette(highlight) as the text — color inversion
    # of the unselected state. Both come from the same palette pair so
    # theme switches still propagate.
    pill_fill = option.palette.color(QPalette.HighlightedText)
    pill_text = option.palette.color(QPalette.Highlight)
else:
    pill_fill = option.palette.color(QPalette.Highlight)
    pill_text = option.palette.color(QPalette.HighlightedText)
```

This honors all 7 Phase 66 themes without inventing a third color. It is a paint-time-only swap; no new tokens, no new palette roles.

**Why not a fixed accent color?** A fixed color would break the Vaporwave/Overrun theme intent (those themes deliberately substitute the highlight role) and would require a new `_theme.py` constant. Rejected.

**Why not leave the pill invisible on selection?** Rejected — fails Dimension 2 (Visuals) of the gsd-ui-checker: a visible affordance that disappears on a common interaction is a bug, not a design choice.

### Delegate paint setup (load-bearing for Dimension 3 Color)

```python
# In station_star_delegate.paint(), AFTER super().paint(painter, option, index)
# and BEFORE the star icon paint:
painter.save()
painter.setRenderHint(QPainter.Antialiasing, True)
painter.setPen(Qt.NoPen)                       # no border outline by default
painter.setBrush(QBrush(pill_fill))            # see swap logic above
painter.drawRoundedRect(pill_rect, 8, 8)       # 8 px corner radius — matches Phase 68 LIVE
# Then draw the pill text:
text_font = QFont(option.font)
text_font.setBold(True)
painter.setFont(text_font)
painter.setPen(pill_text)
painter.drawText(pill_rect, Qt.AlignCenter, tier_text)
painter.restore()
```

**No `setStyleSheet` on the delegate** — QSS does not flow into `QPainter` drawing. The Phase 68 LIVE QSS string is for QLabel only; the delegate hand-rolls the equivalent via `QPainter` primitives.

### Theme compatibility attestation (OPEN DECISION #10 from prompt)

I attest the following for the seven preset themes (System, Vaporwave, Overrun, GBS.FM, GBS.FM After Dark, Dark, Light) and the user-editable Custom palette:

- **All seven preset themes render `palette(highlight)` / `palette(highlighted-text)` as a deliberately-contrasted pair** — Phase 66 verified this for the LIVE badge (Phase 68 UAT) and for the chip-selected state. Phase 70 reuses the same QPalette pair on the same Qt widgets (QLabel) with the same QSS pattern, so the contrast guarantee transfers verbatim.
- **The delegate-painted pill uses the same QPalette pair**, so the same contrast attestation applies — modulo the selection-state inversion described above (which keeps the SAME pair, only swapping fill/text roles).
- **The user-editable Custom palette is a risk surface** — a user can pick `Highlight = #000000` and `HighlightedText = #010101` and produce illegible chrome. Phase 66 accepted this as a "user owns the bullet they buy" trade-off for the Custom slot; Phase 70 inherits that posture. No new mitigation in this phase.
- **WCAG AA is informational only** — this is a single-user desktop app, not a published web product. Phase 70 does not impose a formal contrast-ratio assertion.

**Recommendation:** No theme-specific mitigation needed. Phase 70 ships with the same theme guarantees as Phase 68.

Phase 70 has **no Primary CTA**, **no Empty-state heading**, **no Error state**, **no Destructive confirmation** — by design. ### Failure-mode visual contract (OPEN DECISION #13 from prompt)

If GStreamer never emits caps (e.g., the stream errored before PLAYING was reached), the badge state is determined solely by the user-authored `codec` field:

- `codec ∈ {FLAC, ALAC}` + `sample_rate_hz=0, bit_depth=0` → tier = `"lossless"` → badge shows `LOSSLESS` (D-03 default).
- `codec = ""` or lossy codec → tier = `""` → badge hidden.
- The tooltip says `"Lossless — playback caps not yet detected"` so the user knows the value is provisional.

If codec is mislabeled (user wrote `FLAC` but caps say `audio/mpeg`), DS-03 forbids auto-correction. The badge stays `LOSSLESS` until the user edits the codec field. This is the documented trade-off: badge accuracy is "best knowledge from codec + cached caps," not "live measurement."

## HiDPI / DPR Considerations

Deployment target: Linux Wayland DPR=1.0 (no HiDPI rig). Phase 70 introduces:

- **`QFontMetrics(option.font).horizontalAdvance(text)` for pill width** — DPR-aware natively (Qt scales font metrics by `devicePixelRatio`). Safe on any DPR.
- **`drawRoundedRect(rect, 8, 8)` with integer radius** — Qt scales the rect by `devicePixelRatio` automatically; the round-rect remains crisp at DPR=2.0 because `QPainter` rasterizes in logical pixels.
- **Hardcoded `pill_to_star_gap = 8 px` and `_STAR_MARGIN = 4 px`** — these are logical pixels (Qt scales them on HiDPI). No special handling needed.

**No decisions in Phase 70 behave differently on a HiDPI rig.** If a future HiDPI test surfaces an issue (e.g., pill clips against star on DPR=1.5), planner can revisit the `pill_to_star_gap` token. **No HiDPI pre-ship test required.**
