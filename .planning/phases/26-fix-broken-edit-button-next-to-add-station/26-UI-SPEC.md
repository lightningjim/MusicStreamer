### New Now-Playing Edit Button Spec

```
icon:        document-edit-symbolic
css_class:   flat
tooltip:     "Edit station"
sensitive:   False by default; True when _current_station or _paused_station is not None
visibility:  Always visible (same container as star_btn); sensitivity toggles with play state
position:    Appended to controls_box before pause_btn
connect:     clicked → _open_editor(_current_station.id or _paused_station.id)
```

**Rationale:** Using `flat` (not `suggested-action`) keeps the edit button visually subordinate — it's a utility action, not the primary playback action. Consistent with per-row edit buttons.

### Filter Bar Layout After Removal

After removing `edit_btn`, filter_box contains: `add_btn` | `chip_scroll` (hexpand=True) | `clear_btn`. The chip_scroll already has `hexpand=True` (line 244) so it fills the vacated space naturally. No layout adjustments needed.

**Source:** CONTEXT.md D-05; codebase scan line 244.

## Interaction Contract

### Now-Playing Edit Button — Sensitivity States

| App State | edit_btn sensitive | edit_btn visible |
|-----------|-------------------|-----------------|
| Idle (no station) | False | True |
| Playing | True | True |
| Paused | True | True |
| Loading/buffering | False | True |

**Source:** CONTEXT.md D-03.

### Edit Button Click Behavior

- When `_current_station` is set: call `_open_editor(_current_station.id)`
- When `_paused_station` is set and `_current_station` is None: call `_open_editor(_paused_station.id)`
- When both are None: button is insensitive — click unreachable
