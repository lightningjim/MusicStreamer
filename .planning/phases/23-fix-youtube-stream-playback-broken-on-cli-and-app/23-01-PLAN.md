<objective>
Fix YouTube stream playback regression where yt-dlp overwrites the user's imported cookies.txt, corrupting it for subsequent playback attempts.

Purpose: yt-dlp's --cookies flag both reads AND writes the cookie file, overwriting the user's clean imported cookies with Chrome-epoch timestamps and all-domain cookies. This causes all subsequent YouTube playback to fail.

Output: Both player.py and yt_import.py use temporary copies of cookies.txt for every subprocess invocation. Original file is never touched. Retry-without-cookies logic handles already-corrupted files.
</objective>
