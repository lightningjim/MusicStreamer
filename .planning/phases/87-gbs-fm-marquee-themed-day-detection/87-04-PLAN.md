<objective>
Plan 87-04 implements the themed-day correlator and the logo-swap UI wiring. Themed-day detection runs ONCE per app session (D-09 / D-17) at the first GBS bind: fetch `logo_3.png`, SHA-256 hash it, compare to `GBS_LOGO_BASELINE_HASHES`, and if drift IS detected AND the current full marquee text contains any keyword from `GBS_THEMED_DAY_KEYWORDS` (D-12), emit the decoded QPixmap to NowPlayingPanel for a session-only logo override. D-12's fallback path is critical: hash drift WITHOUT a keyword match STILL applies the themed logo (graceful catching of new themed days) and emits a structured INFO log line so a future codebase pass can extend the keyword list.

This plan also resolves Pitfall #2 by wiring the cadence state machine through `NowPlayingPanel.bind_station` + `NowPlayingPanel.on_playing_state_changed` (existing surface used by AA poll lifecycle) — NOT through the phantom `Player.state_changed` / `Player.station_bound` signals that CONTEXT D-16 references but Player does not expose.
