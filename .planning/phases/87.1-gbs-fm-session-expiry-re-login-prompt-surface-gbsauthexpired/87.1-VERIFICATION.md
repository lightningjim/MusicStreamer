# Phase 87.1: GBS.FM Session Expiry Re-Login Prompt Verification Report

**Phase Goal:** When a GBS.FM session cookie has expired, the app surfaces a clear, actionable "GBS session expired — please log in again" inline prompt (in the now-playing panel where the active playlist renders) that launches the existing in-app GBS login (`oauth_helper --mode gbs`) and refreshes on success, instead of the active playlist silently failing to load. Provides a SHARED expiry→re-login handler reusable by the Phase 87 marquee poller and Phase 87b zero-token add. No silent dead-ends on cancel.

### Human Verification Required

#### 1. Real GBS Login from Inline Prompt

**Test:** With a deliberately expired or cleared GBS cookie, play a GBS.FM station. Verify the inline "GBS session expired" prompt appears in the now-playing panel (in the area where the active playlist renders). Click "Log in again". Confirm the oauth_helper WebEngine login window opens. Complete a real GBS login. Verify the expiry prompt hides and the active playlist resumes populating within one poll interval (≤15s).

**Expected:** The expiry prompt disappears; `_gbs_playlist_widget` becomes visible with real playlist data; the poll timer restarts at 15s; the GBS marquee updates on the next cadence tick.

**Why human:** Spawns a real `oauth_helper --mode gbs` subprocess with a live QtWebEngine browser window pointed at GBS.FM; requires real GBS credentials; OAuth flow cannot be simulated headlessly. This is the end-to-end GBS-AUTH-EXP-01 proof that the subprocess → cookie-write → poll-resume chain works with real infrastructure.

#### 2. Cancel Leaves Clear Inline State with Audio Live

**Test:** Trigger GBS session expiry (expired/cleared cookie while GBS station is active). When the inline prompt appears, click "Log in again". When the oauth_helper login window opens, close it without logging in (click the window's X or press Escape). Verify: (a) the expiry prompt remains visible, (b) the "Log in again" button is re-enabled, (c) audio playback continues uninterrupted, (d) ICY now-playing metadata continues updating.

**Expected:** Inline prompt stays shown; "Log in again" button is enabled for retry; audio output is uninterrupted; ICY title and artist metadata in the now-playing header continue to update; no spinner, toast, or empty-state appears.

**Why human:** Requires the real oauth_helper login window receiving an actual user cancel gesture; cannot be driven headlessly. This validates GBS-AUTH-EXP-03 (no silent dead-end on cancel) with real subprocess lifecycle and the D-03 invariant (audio unaffected by expiry).
