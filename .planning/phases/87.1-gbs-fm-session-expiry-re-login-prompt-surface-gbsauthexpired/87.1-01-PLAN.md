<objective>
Create the shared `GbsReloginHandler` QObject — the single reusable seam that catches GBS session expiry from any consumer (active-playlist poller, marquee worker, future Phase 87b zero-token add) and launches one `oauth_helper --mode gbs` subprocess with single-flight de-duplication (GBS-AUTH-EXP-02).

Purpose: Without a shared handler, each GBS call site would re-implement subprocess launch + cookie-write + success-detection, and concurrent expiry from multiple workers would spawn multiple `oauth_helper` windows. This plan delivers the handler in isolation (no panel/marquee wiring yet) so its contract — `notify_expiry_detected()` in, `relogin_succeeded` / `relogin_failed` signals out — is fixed before Wave 2 consumers connect to it.

Output: `musicstreamer/ui_qt/gbs_relogin_handler.py` + `tests/test_gbs_relogin_handler.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| oauth_helper subprocess → handler | Untrusted stdout (Netscape cookie text) + stderr (JSON events) cross into the main process |
| handler → cookies-jar file | Handler writes a credential file to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87.1-01 | Elevation of Privilege | `notify_expiry_detected` spawning oauth_helper | mitigate | Single-flight guard: `if self._login_proc is not None: return` blocks concurrent subprocess spawns (Pitfall 1/5). Test: `test_single_flight_dedup`. |
| T-87.1-02 | Denial of Service | `QProcess` FailedToStart | mitigate | `errorOccurred` handler resets `self._login_proc = None` before emitting, so a missing helper never permanently jams re-login (Pitfall 2). Test: `test_failed_to_start_resets_guard`. |
| T-87.1-03 | Tampering | cookies-jar file written by handler | mitigate | Write ONLY to `paths.gbs_cookies_path()` (no path traversal), `os.makedirs(exist_ok=True)`, `os.chmod(path, 0o600)` immediately after write (T-40-03). Test: `test_relogin_succeeded_on_valid_cookies` asserts mode 0o600. |
| T-87.1-04 | Tampering | subprocess stdout treated as valid login | mitigate | 4-condition success gate (exit_code==0 AND non-empty AND stderr category Success AND `_validate_gbs_cookies`) before treating login as succeeded; reject otherwise via `relogin_failed`. |
| T-87.1-05 | Information Disclosure | logging of credentials | mitigate | No GBS cookie/credential bytes are passed to any logging call; logs reference URLs, ProcessError enums, and categories only. Source review in acceptance criteria. |
| T-87.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — all primitives (PySide6.QtCore.QProcess, gbs_api, subprocess_utils, paths) already present (RESEARCH §Environment Availability). No package-legitimacy gate required. |
</threat_model>
