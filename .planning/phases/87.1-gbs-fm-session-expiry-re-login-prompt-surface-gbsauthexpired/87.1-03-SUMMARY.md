# Phase 87.1 Plan 03: Wire GBS Expiry Prompt into NowPlayingPanel Summary

**One-liner:** Inline non-dismissive `_gbs_expiry_widget` wired into `NowPlayingPanel` with `GbsReloginHandler` ownership, three relogin slots, `_refresh_gbs_visibility` extensions, and `GbsMarqueeWorker.auth_expired` connected via `Qt.QueuedConnection` — turning the silent expiry path into an actionable "GBS session expired" prompt (GBS-AUTH-EXP-01/02/03).

## What Was Built

### `musicstreamer/ui_qt/now_playing_panel.py`

**`__init__` additions:**

- `self._gbs_expiry_widget` — `QWidget` with `QVBoxLayout` (contentsMargins 0,4,0,4; spacing 4) containing:
  - Primary `QLabel("GBS session expired")` with `setTextFormat(Qt.TextFormat.PlainText)` (T-40-04)
  - Sub-line `QLabel("Log in again to keep the playlist updating")` with `setTextFormat(Qt.TextFormat.PlainText)` (T-40-04)
  - `self._gbs_relogin_btn = QPushButton("Log in again")` connected via bound method `self._on_gbs_relogin_clicked` (QA-05)
  - Widget starts hidden (`setVisible(False)`), added to center layout immediately after `_gbs_playlist_widget`
- `self._gbs_relogin_handler = GbsReloginHandler(parent=self)` constructed and owned; `relogin_succeeded` and `relogin_failed` signals connected to bound-method slots (QA-05)

**`_on_gbs_playlist_error` amendment (auth_expired branch):**

- Preserves all existing D-03/D-04 behavior (`_gbs_poll_timer.stop()`, `_gbs_playlist_widget` hide/clear, `_gbs_label_source = 'icy'`, `_gbs_ajax_disabled = True`)
- Adds `_gbs_expiry_widget.setVisible(True)` (D-01)
- Adds `self._gbs_relogin_handler.notify_expiry_detected()` guard-call (GBS-AUTH-EXP-02)

**Three new slots:**

- `_on_gbs_relogin_succeeded`: hides expiry widget, re-enables button, resets `_gbs_ajax_disabled = False` and `_gbs_label_source = None`, calls `_refresh_gbs_visibility()` (D-04 timer resume), force-polls marquee
- `_on_gbs_relogin_failed`: logs warning, re-enables button for retry; prompt stays visible and timer stays stopped (D-02/GBS-AUTH-EXP-03)
- `_on_gbs_relogin_clicked`: disables button on click (Pitfall 5 single-flight), calls `notify_expiry_detected()`

**`_refresh_gbs_visibility` additions:**

- `should_show=True` branch: `_gbs_expiry_widget.setVisible(False)` (safety-valve hide on GBS re-select); `_gbs_ajax_disabled` NOT reset here (WR-02 preserved)
- `should_show=False` branch: `_gbs_expiry_widget.setVisible(False)` (navigate-away hide, D-02)

**`attach_gbs_marquee_worker` addition:**

- `worker.auth_expired.connect(self._gbs_relogin_handler.notify_expiry_detected, _Qt.QueuedConnection)` — Pitfall 9 / GBS-AUTH-EXP-02 cross-thread wiring

### `tests/test_now_playing_gbs_expiry.py` (new file)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] WR-02 clobber in _refresh_gbs_visibility should_show=True branch**

- **Found during:** Task 3 regression check
- **Issue:** The PATTERNS.md §`_refresh_gbs_visibility` additions snippet said "In the should_show=True branch ensure `self._gbs_ajax_disabled = False` is set (safety-valve reset on station re-select)." Adding this broke `test_gbs_refresh_gbs_visibility_preserves_ajax_disabled_on_rebind` (WR-02 invariant: same-station re-entry must NOT clobber the flag).
- **Fix:** Removed `self._gbs_ajax_disabled = False` from `should_show=True` branch. The reset now happens exclusively in `_on_gbs_relogin_succeeded` (the explicit relogin success path) and `should_show=False` (leaving GBS context). The `_gbs_expiry_widget.setVisible(False)` safety hide was kept.
- **Files modified:** `musicstreamer/ui_qt/now_playing_panel.py`
- **Commit:** `708e9213`

**2. [Rule 1 - Bug] isVisible() vs isHidden() in tests for unrealized widgets**

- **Found during:** Task 3 test run
- **Issue:** `_gbs_expiry_widget.isVisible()` returned `False` even after `setVisible(True)` because the panel is an unrealized (off-screen) widget. This is the same issue documented at `test_now_playing_panel.py:1367`.
- **Fix:** Changed "widget is shown" assertions to `isHidden() is False` (matches existing project pattern). "Widget is hidden" assertions (`isVisible() is False`) are unaffected — they work correctly for unrealized windows.
- **Files modified:** `tests/test_now_playing_gbs_expiry.py`
- **Commit:** `708e9213`
