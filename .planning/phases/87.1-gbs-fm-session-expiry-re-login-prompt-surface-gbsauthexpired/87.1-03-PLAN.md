## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GBS server text → expiry prompt labels | Operator-controlled but untrusted text could reach QLabels |
| marquee worker thread → panel main thread | `auth_expired` Signal crosses the QThread boundary |
| stale poll token → wrong-station UI | A late playlist_error from a previous station could mis-target the prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87.1-08 | Tampering | `_gbs_expiry_widget` QLabels | mitigate | Both labels use static literal strings AND `Qt.TextFormat.PlainText` (T-40-04) — no HTML/CSS injection even if wording is later sourced from GBS. Acceptance grep enforces PlainText. |
| T-87.1-09 | Spoofing | stale poll token shows prompt on wrong station | mitigate | `if token != self._gbs_poll_token: return` guard preserved at top of `_on_gbs_playlist_error` (Pitfall 4); `_refresh_gbs_visibility(should_show=False)` hides the prompt on navigate-away. Test: `test_gbs_expiry_prompt_hides_on_station_change`. |
| T-87.1-10 | Elevation of Privilege | "Log in again" button re-click spawning subprocesses | mitigate | Button `setEnabled(False)` while in-flight (Pitfall 5) AND handler single-flight guard (Plan 01) — two layers prevent multiple oauth_helper spawns. |
| T-87.1-06 | Tampering | marquee→handler cross-thread call | mitigate | `worker.auth_expired.connect(..., Qt.QueuedConnection)` marshals the call onto the main event loop (Pitfall 9); no direct worker-thread call into the main-thread handler. Acceptance grep enforces the QueuedConnection wiring. |
| T-87.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs — UI wiring against already-present PySide6 + Plan 01/02 artifacts. No package-legitimacy gate required. |
</threat_model>
