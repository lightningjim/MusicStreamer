## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Modifying `musicstreamer/` source during a spike | Spikes are throwaway experiments on build packaging; source changes belong to Phase 44. Mixing them poisons the spike's "what's the minimum PyInstaller config that works" signal. | advisory | D-14 in 43-CONTEXT.md forbids it. Wave 1 plan `files_modified` lists only files under `.planning/phases/43-gstreamer-windows-spike/` — never expand. |
| Running PyInstaller on Linux to "pre-validate" the .spec | PyInstaller builds for the host OS; a Linux-built spec proves nothing about Windows bundling. Bytes flowing on Linux ≠ souphttpsrc SSL working on Win11. | advisory | Wave 2 is `autonomous: false`. The iteration loop ONLY runs via paste-back from the user's Windows 11 VM. Do not attempt to shortcut with a Linux build. |
| Hand-curating the plugin list on iteration 1 | Breaks playbin3's autoplug discovery — fails with cryptic "no element factory" errors that look like DLL issues. | advisory | Research mandates "start-broad, prune-later": let stock `hook-gi.repository.Gst.py` collect via live registry on iteration 1; subtract via `exclude_plugins` fnmatch only after SPIKE_OK. |
| Committing raw smoke logs or test_url.txt | AudioAddict listen key leaks. | advisory | `.gitignore` in the phase dir blocks `test_url.txt`, `artifacts/`, `build/`, `dist/`. Smoke test redacts URL query to `?<redacted>` before any logging. Negative-grep gate on 43-SPIKE-FINDINGS.md commit. |

5. Repeat
