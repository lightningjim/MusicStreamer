    class _GbsArtistWorker(QThread):
        """Phase 60.1 / GBS-01e drill-down: gbs_api.fetch_artist_songs on a worker thread.

        Mirrors _GbsSubmitWorker's typed-signal shape — finished emits a single payload
        (the drilled artist's songs as a list[dict] with keys
        {songid, artist, title, duration, add_url} matching _SongRowParser output).

        Per CONTEXT.md D-06: own typed `finished` signal, dedicated dialog slot
        (_on_artist_drilled). NO multi-mode signal, NO metadata_ready (drilled pages
        have no <p class="artists"> blocks per Pitfall 4).
        """
        # finished payload: list of result dicts
        finished = Signal(list)
        # error: 'auth_expired' sentinel OR raw msg
        error = Signal(str)

        def __init__(self, artist_id: int, artist_name: str, cookies, parent=None):
            super().__init__(parent)
            self._artist_id = artist_id
            self._artist_name = artist_name  # carried for breadcrumb rendering
            self._cookies = cookies

        @property
        def artist_name(self) -> str:

        def run(self):
            try:
                out = gbs_api.fetch_artist_songs(self._artist_id, self._cookies)
                self.finished.emit(list(out.get("results", [])))

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| QListWidgetItem UserRole href → URL construction | `_on_artist_link_activated` reads `item.data(Qt.ItemDataRole.UserRole)` which originates from gbs_api.search()'s artist_links/album_links output. While that's first-party data today, treating it as untrusted input (regex-match before `int()`-coercion) is defense in depth — protects against future markup changes or compromised search responses. |
| Worker thread → main thread (Qt signal queue) | `_GbsArtistWorker.finished.emit(list)` crosses thread boundary. The slot runs on the main thread. The `list` payload has been parsed by Plan 02's defensive parser (already mitigates HTML injection via PlainText invariant). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-60.1-09 | Tampering / URL injection | _on_artist_link_activated / _on_album_link_activated | mitigate | Module-level `_ARTIST_HREF_RE = re.compile(r"^/artist/(\d+)$")` and `_ALBUM_HREF_RE = re.compile(r"^/album/(\d+)$")` regex-match the href BEFORE `int(m.group(1))`. Hrefs that don't match (e.g., `javascript:...`, `//evil.example/...`, `/artist/abc`, anything off-shape) cause an early return with no fetch. The `int()` coercion is a second line of defense in `gbs_api.fetch_*_songs` (Plan 02 T-60.1-06). |
| T-60.1-10 | Information Disclosure (HTML injection in breadcrumb) | _breadcrumb_label | mitigate | UI-SPEC Typography mandates PlainText: `self._breadcrumb_label.setTextFormat(Qt.TextFormat.PlainText)` set at widget creation. Breadcrumb text is built via plain f-string `f"Viewing artist: {artist_name}"`. T-40-04 PlainText invariant ensures `&`, `<`, `>` in artist names render as literal characters, NOT as HTML. |
| T-60.1-11 | Tampering (state corruption) | _pre_drill_state snapshot | mitigate | Pitfall 8 single-shot guard: `if self._pre_drill_state is None: self._pre_drill_state = self._snapshot_pre_drill_state()`. | T-60.1-12 | Race condition (Back during fetch) | _back_btn / _on_*_drilled | mitigate | Pitfall 9 mitigation Approach A: `_back_btn.setEnabled(False)` in `_kick_*_drill_worker` before `worker.start()`.
