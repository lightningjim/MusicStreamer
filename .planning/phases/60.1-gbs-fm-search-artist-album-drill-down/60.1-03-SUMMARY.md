## Decisions Made

- **Auth-expired during drill: toast + relock + leave drill chrome with re-enabled `_back_btn`.** Per UI-SPEC Interaction States, no automatic retreat. The user gets feedback (toast) AND the option to retreat (Back button now re-enabled). Cleaner than auto-popping which would surprise the user.
- **`_render_results(*, clear_panels: bool = True)` kwarg propagation rather than inlining the loop into drilled slots.** Single-line propagation through `_clear_table(clear_panels=clear_panels)`; existing callers unchanged. Cleaner than maintaining two render code paths.
- **Threat ID T-60.1-09 (not T-60.1-06) for URL injection in dialog tier.** Plan 03's prose mentions `T-60.1-06` once but the threat-register table uses T-60.1-09 for the dialog-tier URL-injection. T-60.1-06 is the Plan 02 `int()` coercion in `gbs_api.fetch_*_songs`. Both lines of defense are present; source comments use T-60.1-09 to match the table.
- **Worker classes duplicated rather than factored into a base class.** Per RESEARCH §Pattern 2 + project minimum-diff convention. The two classes are ~30 LOC each; a shared base would save ~15 LOC at the cost of one more level of indirection. Keep flat.
