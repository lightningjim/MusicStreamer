# Phase 13 — UI Design Contract

> Visual and interaction contract for the Radio-Browser discovery dialog.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | GTK4 + libadwaita (Adw 1.x) |
| Icon library | GNOME symbolic icon theme (built-in) |
| Font | System default (Cantarell on GNOME) |

This is a GTK4/Python desktop app. All design tokens are expressed as GTK margin/spacing
integer values (pixels), not CSS custom properties. Adwaita provides color, typography, and
widget defaults — the contract below declares only values that deviate or supplement it.

---

## Dialog Structure

**Widget hierarchy** (source: CONTEXT.md D-01, existing `EditStationDialog` pattern):

```
Adw.Window (modal, transient_for=main_window)
  set_default_size(700, 560)
  Adw.ToolbarView
    Adw.HeaderBar
      title: "Discover Stations"
      [close button — auto via Adw.Window]
    Gtk.Box (vertical, spacing=8, margins: top=12 bottom=12 start=12 end=12)
      Gtk.Box (search row, horizontal, spacing=8)
        Gtk.SearchEntry (hexpand=True, placeholder="Search by name…")
        Gtk.DropDown (tag/genre filter, width=140)
        Gtk.DropDown (country filter, width=140)
      Gtk.Stack
        "loading"  → Gtk.Spinner (centered, pixel_size=32)
        "results"  → Gtk.ScrolledWindow → Gtk.ListBox (results list)
        "empty"    → Gtk.Box (centered label)
        "error"    → Gtk.Box (centered label + retry button)
```

**Dialog size:** 700 × 560 px (wider than EditStationDialog's 560 × 480 to accommodate
the subtitle metadata without truncation).

## Result Row Layout

**Widget:** `Adw.ActionRow` per result (source: CONTEXT.md D-02)

| Slot | Content |
|------|---------|
| Title | Station name |
| Subtitle | `{country} · {tags} · {bitrate}kbps` — e.g. "US · Jazz, Blues · 128kbps" |
| Prefix | `audio-x-generic-symbolic` icon, pixel_size=32 |
| Suffix | Play/Stop icon button + Save icon button |

**Subtitle format rule:** Omit any component that is absent. If all three are absent,
subtitle is empty (Adwaita handles gracefully). Separator is ` · ` (middle dot with spaces).

**Suffix buttons:**
- Play button: `media-playback-start-symbolic` (idle) / `media-playback-stop-symbolic`
  (when this row is the active preview). Icon-only button with tooltip "Preview" / "Stop preview".
  Source: CONTEXT.md discretion — toggle icon when row is actively previewing.
- Save button: `list-add-symbolic`. Icon-only button with tooltip "Save to library".
  Disabled (insensitive) after a successful save for that row to prevent double-adds.

## Spacing Scale

All values are multiples of 4 (8-point grid, GTK pixel values).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding |
| sm | 8px | Row suffix button spacing, filter dropdown gap |
| md | 12px | Dialog content margins (top/bottom/start/end) |
| lg | 16px | Search row internal spacing |
| xl | 24px | Not used in this dialog |
| 2xl | 48px | Not used in this dialog |
| 3xl | 64px | Not used in this dialog |

Exceptions: 12px (`md`) — project-established dialog margin, inherited from EditStationDialog.

**Established project spacings preserved** (from `main_window.py` / `edit_dialog.py`):
- Panel margins: 16px top/bottom, 24px start/end (now-playing panel — not used in dialog)
- Dialog content spacing: 12px (EditStationDialog — reused here)
- Filter chip box spacing: 8px (reused for filter dropdowns)

## Color

Adwaita provides the full color system. No custom palette needed.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Adwaita window background | Dialog background, list background |
| Secondary (30%) | Adwaita card/row background | ActionRow backgrounds |
| Accent (10%) | Adwaita accent (blue) | Save button `suggested-action` class |
| Destructive | Adwaita red | Not used in this dialog |

Accent reserved for: the Save button only (add `suggested-action` CSS class).
Play/Stop preview button is neutral (no accent class) — it is a transient control, not a
commitment action.

**Duplicate URL error:** Shown as `Adw.MessageDialog` with heading "Station Already in Library"
and body "[station name] is already saved. No changes were made." Single "Got It" response.
Source: CONTEXT.md D-09 — block with error message, not silent skip.

## Interaction Contract

### Search debounce
- 500ms debounce after last keystroke before firing API request (source: CONTEXT.md D-03).
- While waiting: no spinner. On request start: switch Stack to "loading" child.
- On results: switch Stack to "results" child and populate ListBox.
- On empty results: switch Stack to "empty" child.
- On network/API error: switch Stack to "error" child.

### Filter dropdowns
- Tag dropdown and country dropdown pre-populated via separate Radio-Browser API calls
  on dialog open (parallel, non-blocking). Dropdowns show "Any genre" / "Any country" as
  index-0 default item.
- Selecting a filter re-fires the current search immediately (no debounce — user explicitly
  chose, so treat as committed input).
- Source: CONTEXT.md D-04, D-05.

### Preview playback
- Clicking play on a row calls `player.play(url)`. Main window saves prior station state
  for resume on dialog close.
- Active preview row: play button switches to stop icon. No other row shows stop icon.
- Closing the dialog triggers auto-resume of the previously playing station.
- Source: CONTEXT.md D-06, D-07.

### Save to library
- Calls `repo.create_station()` with name, url, tags, provider name derived from
  Radio-Browser `homepage`/`language`/`countrycode` metadata.
- On success: save button becomes insensitive (grayed), tooltip changes to "Saved".
- On duplicate URL: show `Adw.MessageDialog` (see Color section above).
- Main window `reload_list()` called after successful save.
- Source: CONTEXT.md D-08, D-09, D-10.
