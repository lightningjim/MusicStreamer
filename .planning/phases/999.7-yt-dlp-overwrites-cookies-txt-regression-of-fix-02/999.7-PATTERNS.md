### 1. `musicstreamer/cookie_utils.py` (NEW — utility, transform)

**Analogs:**

**Imports pattern** (model after `musicstreamer/constants.py` lines 13–15 + stdlib tempfile idiom from `edit_station_dialog.py`):


**Core detection pattern** — model after `constants.clear_cookies()` simplicity (pure function, no UI, returns `bool`, swallows harmless `OSError`):

From `musicstreamer/constants.py` lines 36–42:
```python
def clear_cookies() -> bool:
    """Delete cookies.txt if it exists. Returns True if file was removed."""
    p = paths.cookies_path()
    if os.path.exists(p):
        os.remove(p)
        return True
    return False
```

Apply same shape to `is_cookie_file_corrupted(path: str) -> bool`:
- Pure function, no side effects
- Swallow `OSError` (return `False`) — mirrors `clear_cookies()`'s existence-check posture
- One-line docstring explaining the return contract

**Tempfile + copy2 pattern** — model after `musicstreamer/ui_qt/edit_station_dialog.py` lines 81–83:

```python
# Existing codebase idiom (edit_station_dialog.py:81-83, 112-114)
fd, tmp = tempfile.mkstemp(suffix=".jpg")
os.close(fd)
urllib.request.urlretrieve(thumb, tmp)
```

Apply same `mkstemp → os.close(fd) → work → unlink` shape for the cookies copy, but wrap in `@contextlib.contextmanager` + `try/finally`. Use `prefix="ms-cookies-"` per D-04. The `try/finally: os.unlink(tmp_path)` block must swallow `OSError` (best-effort cleanup — matches project's forgive-cleanup posture seen in `closeEvent` `try/except Exception` at `main_window.py:293-296`).

**No analog for `@contextlib.contextmanager`.** This decorator is new to the codebase. The stdlib doc pattern is:

```python
# Reference: docs.python.org/3/library/contextlib.html#contextlib.contextmanager
@contextlib.contextmanager
def temp_cookies_copy() -> Iterator[Optional[str]]:
    # setup
    ...
    try:
        yield value_or_none
    finally:
        # cleanup
        ...
```

**Key contract (from RESEARCH.md Pattern 1 + D-04/D-06):** yields `None` when (a) canonical file absent or (b) `shutil.copy2` raises `OSError`. Yields `tmp_path` otherwise. Always unlinks on exit via `finally`, even if yt-dlp raised (D-04).

```python
class FakeYDLSaveCookies:
    """FakeYDL that mimics yt_dlp.YoutubeDL.__exit__'s save_cookies() side effect:
    writes the yt-dlp header into cookiefile on __exit__ (only when set)."""
    def __init__(self, opts):

        return {"entries": []}
```

Critical: `Signal` declarations MUST be at class scope. Existing comment on line 61 (`"Pitfall 4 -- MUST be at class scope, not instance"`) locks this. Placing it inside `__init__` as `self.cookies_cleared = Signal(str)` silently breaks emission.

### Pure filesystem helper (single-purpose, swallow-harmless-OSError)

**Source:** `musicstreamer/constants.py` lines 36–42 (`clear_cookies`)
**Apply to:** `cookie_utils.is_cookie_file_corrupted` (pure predicate, returns `False` on `OSError`)

Pattern: touch `paths.cookies_path()` through the accessor (never cache), use `os.path.exists` / `os.remove` / `open()`, swallow OS-level errors into a sensible default (`False` for predicates, `False` return for actions).

### Worker-thread Qt signal emission (cross-thread marshaling)

**Source:** `musicstreamer/player.py` `_youtube_resolve_worker` (lines 462–493) emits `youtube_resolved` / `youtube_resolution_failed`; connected at lines 146–151 with `Qt.ConnectionType.QueuedConnection`
**Apply to:** `Player.cookies_cleared.emit(…)` from inside `_youtube_resolve_worker` → wired in `main_window.py:220-223` area

Qt auto-selects `QueuedConnection` across threads. The existing connection at `main_window.py:223` (`self._player.playback_error.connect(self._on_playback_error)`) does not explicitly pass `Qt.QueuedConnection` — it relies on `Qt.AutoConnection`. New `cookies_cleared` connection follows the same posture.

### Direct callback from QThread worker (no Signal needed)

**Source:** `musicstreamer/ui_qt/import_dialog.py` line 170 (`self._toast = toast_callback`) + lines 410, 416, 487, 505, 514 (direct call sites)
**Apply to:** `scan_playlist(url, toast_callback=...)` inside `_YtScanWorker.run()`

The callback is `MainWindow.show_toast`, which uses `QTimer.singleShot(0, …)` internally — thread-safe. No Signal-wrapping required for the yt_import path.

### mkstemp + os.close(fd) + explicit unlink

**Source:** `musicstreamer/ui_qt/edit_station_dialog.py` lines 81–83, 112–114
**Apply to:** `cookie_utils.temp_cookies_copy()` context manager body

```python
fd, tmp_path = tempfile.mkstemp(prefix="ms-cookies-", suffix=".txt")
os.close(fd)
# ... work with tmp_path ...
# in finally: os.unlink(tmp_path)  — best-effort, wrapped in try/except OSError
```

Do NOT use `tempfile.NamedTemporaryFile(delete=True)` (RESEARCH Anti-Pattern 2 — Windows can't reopen an open handle; blocks v2.0 port).

### Path monkeypatch for tests

Never assign `paths._root_override = ...` directly — `monkeypatch.setattr` restores on test failure; direct assignment leaks into other tests (RESEARCH Pitfall 5).
