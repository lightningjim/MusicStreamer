## Claude's Discretion

- **Temp-copy mechanism:** `tempfile.mkstemp` + explicit `shutil.copy2` + `try/finally os.unlink`. Cross-platform safe (NamedTemporaryFile open-handle issue on Windows avoided). Factored into a shared context manager so both call sites stay one-line changes.
- **Copy-failure fallback:** Silent `no-cookies` fallback on `OSError`, matching v1.5 Phase 23 behavior.
- **Test strategy:** Adapt `tests/test_cookies.py` from v1.5 Phase 23. Monkeypatch `yt_dlp.YoutubeDL` to capture `opts`; assert `opts["cookiefile"]` ≠ canonical path and file bytes unchanged before/after. Add new tests for the corruption-detection auto-clear path.

## Deferred Ideas

- Startup-time corruption detector (rejected — read-site detection is sufficient).
- Dialog-based re-import prompt (rejected in favor of toast + auto-clear).
- Audit of OAuth import / cookie_import_dialog write paths (not believed to be source; revisit only if corrupted files reappear post-fix).
