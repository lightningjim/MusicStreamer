# Phase 999.7: yt-dlp overwrites cookies.txt — regression of FIX-02 — Research

**Researched:** 2026-04-24
**Domain:** yt-dlp library API + tempfile-wrapped cookie I/O (Python, cross-platform)
**Confidence:** HIGH

## Summary

Plan 35-06 replaced the mpv/yt-dlp subprocess path with `yt_dlp.YoutubeDL` and wired `opts["cookiefile"] = paths.cookies_path()` directly. The v1.5 Phase 23 temp-copy wrapper was dropped in the transition. The regression is the same one Phase 23 fixed, now resurfaced on the library-API path.

**Root cause verified in yt-dlp source (2026.03.17):** `YoutubeDL.__exit__` → `close()` → `save_cookies()`, which calls `self.cookiejar.save()` whenever `params.get('cookiefile') is not None`. The save writes the full `YoutubeDLCookieJar` with header `# Netscape HTTP Cookie File\n# This file is generated by yt-dlp.  Do not edit.\n\n` — clobbering any externally-written content. This fires on every `with yt_dlp.YoutubeDL(opts) as ydl:` block exit, even for pure `extract_info(download=False)` calls.

**Primary recommendation:** Port the v1.5 temp-copy wrapper to the two library-API read sites via a single shared context manager in a new `musicstreamer/cookie_utils.py` module. Add corruption-detection at each read site that auto-clears and toasts. Reuse `constants.clear_cookies()` for the delete branch. Adapt the existing `tests/test_cookies.py` fixtures (already pure-Python fake-YDL, no mpv) to capture `cookiefile` points at a temp path and the canonical file stays byte-identical across calls.

**Recovery of already-corrupted cookies.txt**
- **D-01:** If `cookies.txt` content starts with the yt-dlp marker line (`# This file is generated by yt-dlp.  Do not edit.` or similar — detect by first non-blank line containing `generated by yt-dlp`), treat it as corrupted.

- **D-03:** Detection runs at the two yt-dlp read sites — inside `_youtube_resolve_worker` (before building `opts`) and inside `scan_playlist` (before building `opts`). No startup-time check, no duplicate check elsewhere. Triggers only when the user actually exercises YouTube.

**Temp-copy mechanism (Claude's Discretion → settled)**
- **D-04:** Use `tempfile.mkstemp(prefix="ms-cookies-", suffix=".txt")` + `os.close(fd)` + `shutil.copy2(paths.cookies_path(), tmp_path)`. ### Pattern 1: `temp_cookies_copy` context manager

**What:** A contextmanager that yields a per-call temp-copy path, or `None` if copy fails / canonical file absent.

**When to use:** Every yt-dlp invocation that sets `cookiefile`.

**Example:**
```python
# musicstreamer/cookie_utils.py
import contextlib
import os
import shutil
import tempfile
from typing import Iterator, Optional

from musicstreamer import paths


_YTDLP_MARKER = "generated by yt-dlp"


def is_cookie_file_corrupted(path: str) -> bool:
    """Return True if the file at ``path`` was written by yt-dlp.

    Detects the yt-dlp save header by scanning the first ~4 non-blank lines
    for the substring 'generated by yt-dlp'. Conservative predicate — avoids
    matching a user-edited file that happens to have the words in a value.
    """
    try:
        with open(path, "r", encoding="utf-8", errors="replace") as fh:
            head = fh.read(512)
    except OSError:
        return False
    for line in head.splitlines()[:4]:
        if _YTDLP_MARKER in line:
            return True
    return False


@contextlib.contextmanager
def temp_cookies_copy() -> Iterator[Optional[str]]:
    """Yield a path to a temp copy of paths.cookies_path(), or None.

    Yields None when:
      - canonical cookies.txt does not exist
      - shutil.copy2 raises OSError (disk full / perms)

    The temp file is unlinked in ``finally`` regardless of exception.
    """
    canonical = paths.cookies_path()
    if not os.path.exists(canonical):
        yield None
        return

    fd, tmp_path = tempfile.mkstemp(prefix="ms-cookies-", suffix=".txt")
    os.close(fd)
    try:
        try:
            shutil.copy2(canonical, tmp_path)
        except OSError:
            # Disk full / perms — caller proceeds without cookies.
            yield None
            return
        yield tmp_path
    finally:
        try:
            os.unlink(tmp_path)
        except OSError:
            pass  # best-effort cleanup
```

### Anti-Patterns to Avoid

- **Don't nest `temp_cookies_copy()` inside `yt_dlp.YoutubeDL(opts)` in the wrong order.** The yt-dlp `__exit__` writes cookies during its own cleanup; if the temp file is unlinked first, yt-dlp raises `FileNotFoundError` on close. Correct order: `with temp_cookies_copy() as tmp: with YoutubeDL(opts) as ydl: ...`
- **Don't use `tempfile.NamedTemporaryFile(delete=True)`.** Windows keeps the handle open, preventing yt-dlp from reading. Use `mkstemp` + `os.close(fd)` + manual unlink.
- **Don't perform corruption detection at startup.** D-03 locks this to the read sites; startup detection would scan on every app launch and complicate the cold path.
- **Don't call `show_toast` directly from the player worker thread.** Use a Qt queued Signal. `yt_import.scan_playlist` runs in a `QThread` so a direct callback is acceptable there — but the callback itself must not touch widgets (the `show_toast` entry point uses `QTimer.singleShot` internally which is thread-safe).
- **Don't re-add retry-without-cookies logic.** That was mpv-specific. The library API raises synchronously on cookie failure — retry is unnecessary and would complicate the code path.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Atomic temp file creation with unique name | Custom `open(f"/tmp/ms-cookies-{uuid}")` | `tempfile.mkstemp` | Handles race conditions, correct perms, cross-platform temp dir |
| File metadata preservation | `open().read()` + `open().write()` | `shutil.copy2` | Preserves mtime (useful for test assertions); single atomic copy |
| Resource cleanup (tempfile) | Manual `try/except/finally` in two places | `@contextlib.contextmanager` | Guarantees cleanup even under exception; one-line call site |
| Cookie corruption detection | Parse full Netscape format and validate fields | Simple substring check on first 4 lines | Marker is a stable yt-dlp header constant (`cookies.py:1284`); simple predicate is less fragile than a parser |

**Key insight:** All the pieces already exist in stdlib + the project. The temptation is to get clever with format validation — the locked decision is a simple marker check because the yt-dlp serialization is deterministic and the predicate is purely for regression detection.

## Common Pitfalls

### Pitfall 1: Context-manager nesting order

**What goes wrong:** If `temp_cookies_copy` exits before `yt_dlp.YoutubeDL` exits, yt-dlp's `save_cookies()` fires after the temp path is unlinked, raising `FileNotFoundError` inside `YoutubeDL.__exit__`.
**Why it happens:** Writing the two context managers as siblings instead of nested.
**How to avoid:** Always nest `YoutubeDL` inside `temp_cookies_copy`. Review in code review: the `with yt_dlp.YoutubeDL(opts) as ydl:` must appear inside the `with cookie_utils.temp_cookies_copy() as cookiefile:` block.
**Warning signs:** `FileNotFoundError` on first live YouTube playback; test for it explicitly (see Validation Architecture).

### Pitfall 2: Worker thread toast marshaling (Player path)

**What goes wrong:** Emitting a toast directly from `_youtube_resolve_worker` (a plain daemon `threading.Thread`, not a `QThread`) via `main_window.show_toast(...)` touches Qt widgets from a non-Qt thread.
**Why it happens:** Daemon threads have no Qt event loop; the bus-bridge thread regression from Phase 43.1 is the same family.
**How to avoid:** Add `cookies_cleared = Signal(str)` as a class-level Signal on `Player` (next to `playback_error`). Connect it in `main_window.py` to `self.show_toast` — Qt auto-selects queued connection cross-thread. Pattern is identical to the existing `playback_error` wiring.
**Warning signs:** Test crash, or `QObject::startTimer: Timers can only be used with threads started with QThread`.

### Pitfall 3: False-positive corruption detection

**What goes wrong:** A user pastes yt-dlp-formatted cookies into the Paste tab of CookieImportDialog. Auto-clear triggers on first YouTube play.
**Why it happens:** The marker substring exists in user content.
**How to avoid:** Accepted risk per single-user scope. Mitigation: restrict the scan to the first 4 non-blank lines (the header region). User's own cookies would have the marker only if they explicitly pasted a yt-dlp-saved file — in which case auto-clearing is arguably correct (they should re-harvest via Google login). Document this in commit message.
**Warning signs:** User reports "my cookies keep getting cleared" — investigate their source format.

### Pitfall 4: Windows tempfile cleanup on crash

**What goes wrong:** Process crashes between `yield tmp_path` and `finally: unlink`. Temp file leaks.
**Why it happens:** `try/finally` does not fire if the Python interpreter is killed.
**How to avoid:** Accepted. Windows `%TEMP%` is auto-cleaned by the OS; Linux `/tmp` is cleaned on reboot. Temp file size ≪ 10 KB, no accumulation risk in normal use. Same disposition as v1.5 Phase 23 T-23-03.
**Warning signs:** Only relevant under crash storms — not expected.

### Pitfall 5: Test fixture bleed across cookie-path tests

**What goes wrong:** Tests that monkeypatch `paths._root_override` leave the override set if the test fails mid-run.
**Why it happens:** `monkeypatch` fixture correctly restores, but manual `paths._root_override = ...` does not.
**How to avoid:** Always use `monkeypatch.setattr(paths, "_root_override", str(tmp_path))` — existing `tests/test_cookies.py` already does this.
**Warning signs:** Unrelated test failures reading from `~/.local/share/musicstreamer/`.

## Open Questions

1. **Should the corruption-detection toast be a new `Player.cookies_cleared` signal or reuse `playback_error`?**
   - What we know: `playback_error` already routes to `show_toast` with prefix `"Playback error: …"` (main_window.py:327-330). Reusing it would show the wrong prefix.
   - What's unclear: whether a user-facing error category already exists for "advisory" toasts vs "error" toasts.
   - Recommendation: add a new `cookies_cleared = Signal(str)` on Player, connect directly to `self.show_toast` in MainWindow. For `yt_import.scan_playlist`, accept an optional `toast_callback: Callable[[str], None]` and plumb it from `_YtScanWorker.__init__` → existing `ImportDialog._toast`.

2. **Should `cookie_utils.py` be a new module or an extension to `paths.py`?**
   - What we know: D-05 leaves the choice open. `paths.py` docstring says "This module is **pure**" — adding tempfile logic violates that.
   - Recommendation: new module `musicstreamer/cookie_utils.py`. Keeps `paths.py` pure; `cookie_utils` depends on `paths`, not the reverse.

3. **Does the corruption predicate need to survive a yt-dlp header rename in a future version?**
   - What we know: `_HEADER` has been `# Netscape HTTP Cookie File\n# This file is generated by yt-dlp.  Do not edit.\n` in yt-dlp for at least 2 years. No deprecation notice upstream.
   - What's unclear: whether yt-dlp might one day omit the "generated by" line.
   - Recommendation: Accept the risk. If yt-dlp renames the header, false negatives are safe (user manually re-imports); we add a new marker string if it ever happens. Document the marker in the helper docstring.
