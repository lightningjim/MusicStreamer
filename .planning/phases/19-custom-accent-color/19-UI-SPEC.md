## Color

This phase introduces a *user-configurable* accent. The contract defines the injection mechanism and defaults, not fixed hex values.

| Role | Value | Source | Usage |
|------|-------|--------|-------|
| Dominant (60%) | `@window_bg_color` | Libadwaita system token | App window background |
| Secondary (30%) | `@card_bg_color` | Libadwaita system token | Now-playing panel, dialog surface |
| Accent (10%) | `@accent_bg_color` (default `#3584e4`) | D-01/D-03 (CONTEXT.md) | Suggested-action buttons, links, sliders, toggles, checkboxes, progress bars |
| Destructive | `@destructive_color` | Libadwaita system token | Not used in this phase |
| Error indicator | `@error_color` | Libadwaita `error` CSS class | Hex entry row invalid-state border only |

Accent reserved for: suggested-action buttons, hyperlinks, range sliders, toggle switches, checkboxes, progress bars — all via Libadwaita's built-in `@accent_bg_color` resolution. The accent is NOT applied to arbitrary backgrounds or decorative elements.

CSS injection mechanism (from D-01):
```css
@define-color accent_bg_color #RRGGBB;
```
Injected via a dedicated `Gtk.CssProvider` at `STYLE_PROVIDER_PRIORITY_USER` priority, separate from `_APP_CSS`.

Preset palette (from D-04):

## Component Inventory

New components introduced in this phase:

### AccentDialog (`musicstreamer/ui/accent_dialog.py`)

- Type: `Adw.Dialog` subclass (modal)
- Layout: vertical `Gtk.Box`
  1. Swatch grid: `Gtk.FlowBox` or `Gtk.Grid` (8 columns), each cell is a 40×40px `Gtk.Button` filled with the preset color, no label
  2. `Gtk.Separator` (horizontal)
  3. Hex entry: `Adw.EntryRow` with label "Hex color", placeholder `#3584e4`
- Width: 320px preferred (`set_default_size(320, -1)`)
- Swatch selection state: selected swatch gets `suggested-action` CSS class to show active ring; unselected swatches have no extra class
- Hex entry error state: add `error` CSS class to the `Adw.EntryRow` on invalid input; remove on next keystroke

### Header bar accent button (in `main_window.py`)

- Type: `Gtk.Button` (icon-only)
- Icon: `preferences-color-symbolic` (fall back to `color-select-symbolic` if unavailable — resolve at implementation)
- Position: `header.pack_end()` after Import button
- Tooltip: `"Accent color"`
- Click handler: `self._open_accent_dialog()`

## Interaction Contract

### Opening the dialog

Trigger: click accent icon button in header bar.
Result: `AccentDialog` opens as modal over the main window.

### Selecting a preset swatch

Trigger: click any of the 8 colored swatches.
Result:
1. CSS provider is updated immediately (`css_provider.load_from_string("@define-color accent_bg_color #RRGGBB;")`).
2. `repo.set_setting("accent_color", "#RRGGBB")` called.
3. Hex entry field updates to show the selected color's hex value.
4. Previously selected swatch loses `suggested-action` class; newly selected swatch gains it.
5. Dialog stays open (no auto-close).

### Entering a custom hex value

Trigger: user types in hex entry row and presses Enter or unfocuses.
Validation: match `/^#([0-9a-fA-F]{3}|[0-9a-fA-F]{6})$/`.
- Valid: apply immediately (same as swatch), clear `error` class, persist.
- Invalid: add `error` class to entry row, preserve previous color, do NOT persist.

### Error indicator lifecycle

- `error` class added: on validation failure (Enter or focus-out).
- `error` class removed: as soon as user starts typing again (`changed` signal).

### Persistence (from D-02/D-03)

- Key: `"accent_color"`, stored as lowercase hex string.
- Default: `"#3584e4"`.
- On app start: load from `repo.get_setting("accent_color", "#3584e4")`, inject before `win.present()`.
