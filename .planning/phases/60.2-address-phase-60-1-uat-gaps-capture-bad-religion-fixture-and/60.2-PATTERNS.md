#### Recommended evolution shape (cite RESEARCH Pattern 3 lines 528-602)

The new `_render_results(*, clear_panels=True, group_by_album=False)` adds two helpers:

1. `_group_rows_by_album(results: list[dict]) -> list[tuple[str, list[dict]]]` — preserve fixture order; consecutive same-album rows form a group; empty-string albums form their own group (rendered without a header per D-11).
2. `_insert_album_section_header(row: int, album: str, song_count: int) -> None` — appends a 4-cell row with `QFont().setBold(True)` on column 0, `QStandardItem.setFlags(Qt.ItemFlag.ItemIsEnabled)` on **all four** items (header text in col 0, three empty placeholder items in cols 1-3), then `setSpan(row, 0, 1, columnCount())`. Header text format: `f"{album} ({song_count} songs)"` (D-03 verbatim — plain UTF-8 string, no HTML).

The song-row loop is extracted into `_render_song_rows(rows, *, base_idx, original_idx_offset=0)` so both the flat path and the grouped path call it with the right offsets.

**Caller branch (`_on_artist_drilled` only — Pitfall 6):**
- `_on_artist_drilled` calls `self._render_results(clear_panels=False, group_by_album=True)`.
- `_on_album_drilled` keeps the existing `self._render_results(clear_panels=False)` (default `group_by_album=False`). Album drill is intentionally flat — `_AlbumPageParser` rows do not carry an `album` field; only an `artist` field per row.
- Search-result paths (`_on_search_finished`, `_on_back_clicked`, paginations) use the default `group_by_album=False` — unchanged.
