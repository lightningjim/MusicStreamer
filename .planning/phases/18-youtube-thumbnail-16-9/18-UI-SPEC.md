## Art Slot Contract

This phase changes how content is rendered inside the existing 160×160 art slots. This section replaces a generic "visuals" section for this GTK-native phase.

### Slot dimensions

Both `logo_stack` and `cover_stack` remain `set_size_request(160, 160)`. No layout change.

### Widget per station type

| Station type | Widget | ContentFit | Fallback |
|---|---|---|---|
| YouTube (`youtube.com` or `youtu.be` in `st.url`) | `Gtk.Picture` | `Gtk.ContentFit.CONTAIN` | `Gtk.Image` ("audio-x-generic-symbolic", pixel_size=160) |
| Non-YouTube | `Gtk.Image` + `GdkPixbuf.new_from_file_at_scale(path, 160, 160, False)` | n/a | `Gtk.Image` ("audio-x-generic-symbolic", pixel_size=160) |

Source: CONTEXT.md D-02, D-03, D-05.

### YouTube CONTAIN letterbox appearance

- Image rendered at 160×90 px centered within the 160×160 slot.
- Approximately 35px transparent (background-colored) bands appear top and bottom.
- No cropping. Full image width visible. No distortion.
- Band color matches the widget background (Adwaita default — no explicit fill needed; `Gtk.Picture` renders transparent areas naturally).

### Non-YouTube square art appearance

- Image scaled to exactly 160×160 via GdkPixbuf (`preserve_aspect_ratio=False`).
- No letterboxing. No change from current behavior.

### iTunes cover art (_on_cover_art)

- Only fires for non-YouTube ICY streams. No change needed.
- Continues to replace `cover_stack` content with a 160×160 scaled pixbuf.

### Overflow

Both stacks keep `set_overflow(Gtk.Overflow.HIDDEN)` — no change.

### Detection

```python
is_youtube = "youtube.com" in st.url or "youtu.be" in st.url
```

Applied at play time in `_play()`. Same pattern as player.py and yt_import.py.

## Interaction Contract

| Trigger | Behavior |
|---------|----------|
| Station play (YouTube URL) | Both logo_stack and cover_stack switch to Gtk.Picture child with ContentFit.CONTAIN showing the thumbnail letterboxed |
| Station play (non-YouTube URL) | Both slots use GdkPixbuf scale 160×160 as before |
| Art file missing or load error | Silent fallback to "fallback" stack child (symbolic icon) — no user notification |
| ICY TAG fires mid-playback (non-YouTube) | cover_stack replaced with iTunes art pixbuf — unaffected by this phase |
| Stop | Both stacks reset to "fallback" child — unchanged |

## Implementation Scope Guard

Changes are confined to `musicstreamer/ui/main_window.py`:
- `_build_now_playing_panel()` — add `Gtk.Picture` as a named child alongside existing `Gtk.Image` child in both stacks, or replace conditionally at play time.
- `_play()` — YouTube branch uses `Gtk.Picture` + CONTAIN; non-YouTube branch keeps existing GdkPixbuf path.
- No other files change in this phase.

Out of scope (deferred, do not touch):
- Slot width change
- Any non-now-playing art display
- Station list row art
- Discovery dialog art
