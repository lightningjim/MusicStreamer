    RED phase — write tests first in tests/test_player_failover.py:

    - test_preferred_stream_first: Given station with streams [pos1/low, pos2/hi, pos3/med] and preferred_quality="hi", the first stream attempted should be pos2/hi. Remaining queue should be [pos1/low, pos3/med] in position order.
    - test_no_preferred_quality_uses_position_order: Given no preferred_quality (empty string), streams are tried in position order.
    - test_preferred_stream_not_duplicated: The preferred stream appears exactly once in the queue (not at position 0 AND its natural position).
    - test_gst_error_triggers_failover: Calling _on_gst_error pops next stream from queue and calls _play_next internally.
    - test_timeout_triggers_failover: When _on_timeout_cb fires (simulating 10s with no audio), it triggers _try_next_stream.
    - test_tag_received_cancels_timeout: Calling _on_gst_tag (ICY metadata received) cancels the failover timer via GLib.source_remove.
    - test_all_streams_exhausted: When _streams_queue is empty and _try_next_stream is called, on_failover is called with None and pipeline is set to NULL.
    - test_failover_callback_called_on_attempt: When failover occurs (not exhaustion), on_failover is called with the failed stream info.
    - test_timer_cancelled_on_stop: Calling stop() cancels any pending failover timer.
    - test_timer_cancelled_on_pause: Calling pause() cancels any pending failover timer.
    - test_new_play_cancels_previous_failover: Calling play() while failover is in progress clears _streams_queue and cancels timer.
    - test_play_stream_bypasses_queue: Calling play_stream(stream) clears the queue and plays that specific stream (D-08).
    - test_youtube_failover_polling: For YouTube streams (mpv), failover is detected via GLib.timeout_add polling of _yt_proc.poll() rather than GStreamer bus.

    GREEN phase — implement in musicstreamer/player.py:

    1. Add instance variables in __init__:
       - self._streams_queue: list[StationStream] = []
       - self._failover_timer_id: int | None = None
       - self._on_failover: callable | None = None
       - self._current_stream: StationStream | None = None
       - self._current_station_name: str = ""
       - self._yt_poll_timer_id: int | None = None

    2. Change play() signature to:
       ```python
       def play(self, station: Station, on_title: callable,
                preferred_quality: str = "",
                on_failover: callable = None):
       ```

    3. In play(), build ordered stream queue:
       - Find stream matching preferred_quality (first by quality match, position order)
       - Put preferred stream first
       - Append remaining streams in position order (excluding preferred to avoid D-05 pitfall)
       - Store on_failover, call _try_next_stream()

    4. Add _try_next_stream():
       - Set pipeline to NULL
       - If _streams_queue empty: call on_failover(None) via GLib.idle_add, return
       - Pop stream from front of _streams_queue
       - Set self._current_stream = stream
       - If stream is YouTube: call _play_youtube(url, name, on_title)
       - Else: call _stop_yt_proc(), _set_uri(url, name, on_title)
       - Arm timeout: self._failover_timer_id = GLib.timeout_add(BUFFER_DURATION_S * 1000, self._on_timeout_cb)
       - If this was a failover (not first attempt) and on_failover exists: GLib.idle_add(on_failover, stream)

    5. Add _on_timeout_cb(): cancel timer ID, call _try_next_stream(), return False

    6. Add _cancel_failover_timer():
       - GLib.source_remove(self._failover_timer_id) if not None, set to None
       - Also cancel _yt_poll_timer_id if not None

    7. Modify _on_gst_error(): call _cancel_failover_timer(), then _try_next_stream()

    8. Modify _on_gst_tag(): call _cancel_failover_timer() (audio data received = stream is working)

    9. Modify stop(): call _cancel_failover_timer(), clear _streams_queue, clear _on_failover

    10. Modify pause(): call _cancel_failover_timer(), clear _streams_queue

    11. Add play_stream(stream: StationStream, on_title: callable, on_failover: callable = None):
        - Clear queue and timer
        - Set _streams_queue = [stream]
        - Set _on_failover = on_failover
        - Call _try_next_stream()

    12. For YouTube failover: in _play_youtube, after the existing time.sleep(2) retry block, arm a GLib.timeout_add that polls _yt_proc.poll() every 1000ms for up to BUFFER_DURATION_S iterations. If _yt_proc exits with non-zero, call _try_next_stream(). Store this timer as _yt_poll_timer_id.

    IMPORTANT: Do NOT use the failover timer (GLib.timeout_add with BUFFER_DURATION_S) for YouTube streams. Instead use the polling approach since YouTube uses mpv subprocess, not GStreamer pipeline.

    Use the same make_player() mock pattern from test_player_pause.py. Mock GLib.timeout_add to return a fake source ID and capture the callback. Mock GLib.source_remove. Mock GLib.idle_add to call the function immediately for test assertions.
