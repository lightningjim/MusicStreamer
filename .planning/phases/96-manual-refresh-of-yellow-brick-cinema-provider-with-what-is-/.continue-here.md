## Critical Anti-Patterns (do NOT repeat these)
- [ANTI-PATTERN]: URL-based dedup treats a restarted live stream (new video ID) as a *new*
  station → `yt_import.import_stations` skips/duplicates instead of updating. The refresh path
  must **update-in-place**, not insert. → Plan an explicit update path that bypasses
  `station_exists_by_url` dedup.
- [ANTI-PATTERN]: yt-dlp YouTube extraction breaks under `.desktop` launchers (minimal PATH,
  no node) unless a resolved node runtime is threaded in. → Thread `check_node()` →
  `scan_playlist(..., node_runtime=...)` from the new UI worker (mirror import_dialog's
  `_YtScanWorker`). (Severity: advisory — known pattern, not a this-session failure.)
