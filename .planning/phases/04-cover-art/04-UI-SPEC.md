## Component Contract

This section replaces the registry section for GTK4 native phases.

### Right Slot: cover_stack (replaces cover_placeholder)

| Property | Value | Source |
|----------|-------|--------|
| Widget type | Gtk.Stack | CONTEXT.md — mirrors logo_stack pattern |
| Size | 160×160px fixed | CONTEXT.md + existing logo_stack line 57 |
| Child "fallback" | Gtk.Image, icon=audio-x-generic-symbolic, pixel_size=160 | mirrors logo_fallback |
| Child "art" | Gtk.Image, pixel_size=160 | new — populated by iTunes fetch |
| Initial visible child | "fallback" | consistent with logo_stack initial state |
| vexpand | False | matches logo_stack |
| hexpand | False | matches logo_stack |

### State machine: cover_stack

| Trigger | Action |
|---------|--------|
| App launch / nothing playing | visible_child = "fallback" |
| Station starts playing (_play_station) | no change — cover art comes via TAG bus, not station data |
| TAG message arrives, ICY string is junk (empty/whitespace/ad copy) | visible_child = "fallback" (no API call) |
| TAG message arrives, API request in-flight | keep current visible child (no placeholder flash) |
| iTunes returns image URL, pixbuf loaded | visible_child = "art" |
| iTunes returns no result | visible_child = "fallback" |
| Stop (_stop) | visible_child = "fallback" |

### Image scaling

- Scale all fetched art to 160×160 using `GdkPixbuf.Pixbuf.new_from_file_at_scale(path, 160, 160, False)` before `Gtk.Image.set_from_pixbuf`.
- Preserve aspect ratio: False (square crop to slot — consistent with logo handling).
- Source: CONTEXT.md discretion area; existing Phase 3 logo pattern (main_window.py lines 274–275).

## Interaction Summary

Phase 4 introduces zero new user interactions. All behavior is automatic:

1. User plays a station (existing interaction — unchanged).
2. ICY TAG messages arrive — cover art updates silently in the right slot.
3. User stops playback (existing interaction — unchanged) — right slot reverts to placeholder.

The only visual change the user sees: the right `audio-x-generic-symbolic` placeholder is replaced by real track art when ICY metadata is available and iTunes returns a result.
