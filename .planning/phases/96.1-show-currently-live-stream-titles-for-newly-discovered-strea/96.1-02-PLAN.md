## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| network (YouTube) → resolve_live_title | attacker-controllable video metadata (title) fetched via yt-dlp |
| canonical cookies file ↔ yt-dlp | yt-dlp save_cookies() side effect can corrupt the canonical file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96.1-02 | Denial of Service | resolve_live_title worker-thread hang on stalled connection | mitigate | socket_timeout=10 bounds the yt-dlp network op (mirrors fetch_channel_avatar) so the worker thread cannot hang indefinitely |
| T-96.1-03 | Tampering | yt-dlp save_cookies() corrupts canonical cookies file | mitigate | yt_dlp.YoutubeDL nested inside cookie_utils.temp_cookies_copy() — writes land on the temp copy, never canonical (Phase 999.7 pattern) |
| T-96.1-04 | Tampering | oversized attacker title passed downstream | accept | repo.set_live_url_title_anchor caps at 500 chars (RESEARCH Security Domain); resolve_live_title returns the raw title and does not need to re-truncate |
| T-96.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs (RESEARCH Package Legitimacy Audit: zero external packages) |
