## What Was Built

### Task 1: is_sandboxed() shared sandbox-detection helper

Added to `musicstreamer/flatpak_first_launch.py`:
- `_FLATPAK_INFO: str = "/.flatpak-info"` — module-level constant, monkeypatchable exactly as `_HOST_DB` is (mirrors existing test pattern)
- `is_sandboxed() -> bool` — pure function returning `os.path.exists(_FLATPAK_INFO)`; documented as SINGLE source of truth for sandbox detection consumed by migration + wizard gate

Two new tests in `TestIsSandboxed` class in `tests/test_flatpak_first_launch.py`:
- `test_returns_true_when_flatpak_info_exists` — monkeypatches `_FLATPAK_INFO` to existing tmp file
- `test_returns_false_when_flatpak_info_absent` — monkeypatches `_FLATPAK_INFO` to non-existent path

### Task 2: Sandbox-aware run_migration()

Modified `musicstreamer/migration.py`:
- Added sandbox guard after `os.makedirs(dest, exist_ok=True)`, BEFORE the existing `realpath` short-circuit
- Late-imports `is_sandboxed` from `musicstreamer.flatpak_first_launch` (keeps migration import-cheap)
- When `is_sandboxed()` True: writes marker, returns — zero calls to `_copy_tree_nondestructive` or `_LEGACY_LINUX`
- `makedirs(dest)` is preserved before the guard so `DATA_DIR` always exists after `run_migration()` (buffer_log ordering invariant at `__main__.py:199`)

New test `test_migration_sandboxed_skips_copy` in `tests/test_migration.py`:
- Monkeypatches `ffl._FLATPAK_INFO` to existing tmp file (is_sandboxed() True)
- Sets divergent src with `cookies.txt` + `musicstreamer.sqlite3`
- Asserts: marker exists, dest dir exists, no `cookies.txt` in dest, no `musicstreamer.sqlite3` in dest
- `_reset_root_override` fixture extended to also save/restore `ffl._FLATPAK_INFO`
