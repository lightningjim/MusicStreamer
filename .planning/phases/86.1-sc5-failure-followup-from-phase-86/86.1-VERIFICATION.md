### Human Verification Required

The following items need runtime testing in the actual Flatpak sandbox. Automated checks (23/23 tests passing, compile checks, token-presence greps) confirm the wiring is correct in code. The gaps are purely behavioral/visual — they cannot be exercised headlessly.

#### 1. First Sandboxed Launch — Wizard Appears

**Test:** Build Flatpak, ensure `~/.local/share/musicstreamer/musicstreamer.sqlite3` exists on the host, and run `flatpak run io.github.kcreasey.MusicStreamer` for the first time (no `.flatpak-import-offered` flag in the sandbox data dir yet).

**Expected:** The FlatpakImportWizard dialog appears overlaying the shown main window within a moment of app startup.

**Why human:** `QTimer.singleShot(0, _maybe_offer_import)` schedules the modal for the event loop start. Modal rendering over a shown `MainWindow` requires a running Qt display. Cannot be exercised headlessly due to PySide6/Qt not being installed in this environment.

#### 2. Offer-Once on Relaunch

**Test:** Dismiss or complete the import wizard (or cancel it). Quit the app and relaunch.

**Expected:** The wizard is NOT offered again. The `.flatpak-import-offered` flag file should exist in `~/.var/app/io.github.kcreasey.MusicStreamer/data/` after the first launch.

**Why human:** The flag-write path (`write_offered_flag()`) is called inside `FlatpakImportWizard`'s close handlers. Verifying the wizard calls it on every close path (cancel, X button, complete) requires instantiating and exercising the Qt wizard. The gate-level test (case 3) proves that once the flag exists, the gate returns False — but the flag-creation trigger inside the wizard itself needs runtime confirmation.

#### 3. Native Linux Non-Offer

**Test:** Run `python -m musicstreamer` (native, non-Flatpak) with `~/.local/share/musicstreamer/musicstreamer.sqlite3` present.

**Expected:** No import wizard dialog appears. App launches normally.

**Why human:** This proves the `is_sandboxed()` conjunct blocks the offer at runtime. The unit test (case 2) verifies the gate logic correctly, but a runtime smoke confirms no regression path inadvertently imports or shows the wizard outside the sandbox.
