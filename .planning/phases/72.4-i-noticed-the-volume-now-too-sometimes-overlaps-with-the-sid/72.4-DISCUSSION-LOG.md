## What wraps to row 2

| Option | Description | Selected |
|--------|-------------|----------|
| Volume + compact-toggle (Recommended) | The rightmost cluster (volume slider + sidebar-toggle button) moves together. Keeps transport+actions group intact on row 1. Minimal layout disturbance. | ✓ |
| Volume slider alone | Only volume wraps; compact-toggle stays on row 1 with no neighbour — visually orphaned. | |
| Star + EQ + Volume + Compact-toggle (secondary cluster) | All four secondary-cluster widgets wrap together; row 1 narrows to just play/stop/edit. Crowds row 2 if picker is there. | |
| Compact-toggle alone | Only compact-toggle wraps; volume stays on row 1. Probably doesn't free enough width (compact = 28px; volume = 120px is the bigger offender). | |

**User's choice:** Volume + compact-toggle wrap together as a unit.
**Notes:** Cluster moves as a unit; preserves the "transport reads as a block" mental model from Phase 72.1 D-03.

## Coexistence with stream_combo on row 2

| Option | Description | Selected |
|--------|-------------|----------|
| New row 3, below the picker (Recommended) | Three explicit rows: row 1 = play/stop/edit/star/eq, row 2 = stream_combo (Expanding, full-width), row 3 = volume + compact-toggle. Independent wrap tiers, easy to test. | ✓ |
| Same row 2, alongside picker | Row 2 holds [stream_combo \| volume \| compact-toggle] side-by-side. Picker can't use full-width Expanding (breaks 72.1 D-07). | |
| Picker bumps to row 3; volume cluster takes row 2 | Reorder by priority. Inverts natural reading order; breaks 72.1's row 2 = picker invariant. | |

**User's choice:** New row 3 below the picker for the three-row case.
**Notes:** Each wrap tier remains independent; the picker's 72.1 invariant is preserved. Three-row state only appears at the narrowest widths.

## Volume slider sizing on row 2

| Option | Description | Selected |
|--------|-------------|----------|
| Stretch (Expanding) to fill the row (Recommended) | Volume slider grows to consume remaining row width; compact-toggle stays 28px fixed on the right. Improves precision on narrow displays. Mirrors 72.1 D-07. | ✓ |
| Keep 120px fixed | Volume stays 120px wide; identical visual to row 1. Wastes the wrap row's width. | |
| Smaller fixed (e.g., 80–100px) | Trade slider precision for vertical compactness. Probably wrong — wrap row has the space. | |

**User's choice:** Volume slider uses Expanding policy on its wrap row.
**Notes:** Compact-toggle stays 28×28 fixed on the right of the wrap row. No addStretch needed — volume's Expanding policy consumes free space.

## Claude's Discretion

The following items were not asked about and are left for the planner:

- **Threshold computation:** likely extend the existing `_picker_row_min_width()` into a generalized helper or add a sibling cached field. Outer-chrome calculation is reused verbatim.
- **Cache invalidation hook for the new threshold:** volume slider and compact-toggle are constructed once and never repopulated; likely cache-once, never invalidate.
- **Helper naming:** recommended `_reflow_volume_cluster`, `_move_volume_cluster_to`, `_set_volume_size_policy` to mirror the 72.1 helpers for grep-affinity.
- **Row 3 insertion site:** after `center.addLayout(self._controls_row2)` at `now_playing_panel.py:577`. Planner verifies no widget between row 2 and the stats widget that should sit above row 3.
- **Row 3 spacing and margins:** `setSpacing(8)` to mirror row 1 and row 2; no explicit margins.
- **Hysteresis dead-band:** ship without, observe, add only if real flicker is observed (mirrors 72.1's same conclusion).
- **Test file location:** extend the existing 72.1 test file with a `TestPhase72_4VolumeClusterReflow` block.
- **Order of reflow calls inside `resizeEvent`:** recommended picker reflow → volume-cluster reflow → art-tier reflow.
- **REQUIREMENTS.md entry:** planner picks the requirement code (likely `LAYOUT-04` as the continuation of LAYOUT-01/02/03).
- **Volume slider size-policy toggle implementation:** planner picks between `setFixedWidth(120)` ↔ `setSizePolicy + unset` toggle pair, or a wrapper-container approach.
