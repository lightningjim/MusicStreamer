## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] _populate_stream_picker missing _reflow_volume_cluster hook for station-change path**

- **Found during:** Task 2 — `test_multi_to_single_mid_very_narrow_rehomes_cluster` failed
- **Issue:** When a multi-stream station switches to single-stream while the panel is already
  at `_W_VERY_NARROW` (560px), the cluster was on row 3. The test called `bind_station(single)`
  then `panel.resize(560, 800)` (same dimensions). Qt does NOT fire `resizeEvent` when the panel
  size has not changed, so the cluster remained stranded on row 3 instead of moving to row 2.
  The test expected `panel._controls_row2` as the cluster's home after the station change.
- **Fix:** Added an eager `_reflow_volume_cluster()` call at the end of `_populate_stream_picker`,
  guarded by `hasattr(self, '_reflow_volume_cluster')` for `__init__`-safety. This mirrors the
  existing eager `_move_stream_picker_to(self.controls)` call already in `_populate_stream_picker`
  for the picker's single-stream re-home path. Pattern is consistent with CONTEXT D-14 (re-home
  eagerly rather than relying on the next resizeEvent tick that may never come).
- **Files modified:** `musicstreamer/ui_qt/now_playing_panel.py`
- **Committed in:** `bbbf04b` (Task 2 commit)
- **Verification:** `test_multi_to_single_mid_very_narrow_rehomes_cluster` GREEN after fix.
