<objective>
Fix the Newly-Discovered merge-target dropdown in `LiveRefreshDialog` so it offers ALL of a provider's saved stations instead of only the one(s) flagged with `live_url_syncs_from_channel = 1`. Today a user with 19 saved YBC stations sees only the single flagged station when trying to merge a newly-discovered live stream onto the station whose stream actually died.

The change is a subtraction plus a wiring change: a new flag-independent repo query (`list_stations_for_provider`), a second dialog-level station load (`self._all_provider_stations`), and one call-site swap in `_populate_rows`. The REMAP section and the `apply_refresh` path are untouched (D-02, D-03 already satisfied by existing code).

Purpose: Make merge/map usable across the whole provider library when a station's live stream goes stale (un-defers Phase 96.1's "map/merge into ANY station" idea, narrowed to same-provider).
Output: 1 new repo method, 2 dialog edits, 3 new tests, all green.
</objective>
