<objective>
Land the manual UAT script covering the three Manual-Only Verifications in 73-VALIDATION.md (real-MB-cover for a real station, CAA-250 quality at 160×160, cross-machine ZIP round-trip). Run the UAT and capture the user's verdict via the checkpoint. This is the human-side complement to the 16 automated ART-MB tests; together they satisfy the phase's verification surface.

Purpose: ART-MB-01..16 prove the code is correct against mocked MB/CAA responses. They do NOT prove that the real MB service returns sane results for the user's actual stations, nor that CAA-250 produces visually acceptable pixmaps, nor that the export ZIP works cross-machine end-to-end. The UAT script tests those three pieces by hand.

This plan is **non-autonomous**: it ends in a checkpoint where the user reports their UAT results. If any scenario fails, the orchestrator opens a gap-closure plan (Plan 06+) to fix.

Output: 73-UAT-SCRIPT.md (the script itself) + completed UAT run captured in the checkpoint resume-signal.
</objective>

    **Scenario A — MB-only cover renders for a real station** (gates real-network E2E):
    1. Pre-conditions: app builds and runs; internet access available; at least one favorite station exists with ICY metadata enabled (e.g. a SomaFM or DI.fm channel known to emit `Artist - Title` ICY).
    2. Step 1: Launch app; play the station; observe ICY title arrives within ~10 sec.
    3. Step 2: Right-click (or whatever the existing UI affordance is) → Edit Station → open EditStationDialog.
    4. Step 3: Verify the new "Cover art source:" QComboBox exists immediately below the "ICY metadata:" row, with three entries in this exact order: "Auto (iTunes → MusicBrainz fallback)", "iTunes only", "MusicBrainz only".
    5. Step 4: Select "MusicBrainz only"; click Save. Verify Save button was enabled when the combo flipped (dirty detection).
    6. Step 5: Stop and re-play the station. Wait for a fresh ICY title to arrive. Observe the cover_label slot:
       - ✅ PASS: an album cover image renders (any image — content correctness is bonus, not required for pass).
       - ❌ FAIL: cover stays as the station-logo placeholder for >30 sec after a `' - '`-shaped ICY title arrives.
       - ⚠️ NOTE: if the specific track has no MB match (uncommon for SomaFM / DI.fm popular tracks), this is still PASS — try 2-3 tracks before failing.

    **Scenario B — CAA-250 image quality at 160×160** (gates RESEARCH A1):
    1. With the same station set to MB-only and a cover successfully rendered (from Scenario A step 5).
    2. Inspect the cover slot at native pixel ratio (no zoom; Wayland DPR=1.0).
    3. Compare visually against an iTunes-source cover (set the same station temporarily to "iTunes only" and replay; the iTunes worker fetches 160x160 from itunes.apple.com).
    4. Verdict:
       - ✅ PASS: MB cover (CAA-250 → scaled to 160×160) shows no visible pixelation; comparable visual quality to iTunes 160×160.
       - ❌ FAIL: MB cover is visibly pixelated, blurry, or shows JPEG artifacts that the iTunes cover does not.
       - If FAIL: recommend in the UAT result that Phase 73.1 (gap-closure) bumps the CAA variant from `/front-250` to `/front-500` in cover_art_mb.py (single-line change). Document the recommendation; do not auto-fix in this UAT — the verdict goes to the user.

    **Scenario C — Cross-machine ZIP round-trip preserves cover_art_source** (gates ART-MB-10 end-to-end):
    1. On the active profile: set 2-3 stations to different cover_art_source values (one Auto, one iTunes only, one MB only).
    2. Use the existing hamburger menu → Settings → Export ZIP (whatever the existing affordance is — read 73-04-SUMMARY.md to confirm; settings_export.build_zip is the codepath).
    3. Save the ZIP to disk.
    4. Open a fresh profile: either (a) launch the app with `XDG_DATA_HOME=/tmp/phase73-uat-profile musicstreamer` for a clean profile on Linux, OR (b) use a different OS user account if `XDG_DATA_HOME` override is awkward.
    5. In the fresh profile, import the ZIP (hamburger menu → Import).
    6. Open EditStationDialog on each of the 3 stations; verify the cover_art_source combo reflects the same value as on the source profile.
       - ✅ PASS: all 3 stations show the expected value (Auto / iTunes only / MB only).
       - ❌ FAIL: any station's combo shows a different value than the source profile.

    **Recording the UAT result:** at the end of the script, include a "Results" block with three checkboxes (one per scenario) and a free-text "Notes" field. The user fills this in when they run the UAT.

    **Time estimate:** Scenario A ~5 min, Scenario B ~3 min, Scenario C ~10 min (most of that is the profile-switch setup). Total ~20 min wall-clock.
