## Pre-discussion scouting

Identified the root cause before asking any questions:
`accounts_dialog._launch_gbs_login_subprocess()` launches login via
`QProcess.start(sys.executable, ["-m", "musicstreamer.oauth_helper", "--mode", "gbs"])`.
On a PyInstaller-frozen build `sys.executable` is `MusicStreamer.exe` (not a Python
interpreter), so `-m musicstreamer.oauth_helper` is silently ignored and the login window
never opens. The same pattern breaks Twitch (`_launch_oauth_subprocess`, line ~383) and
Google login. Confirmed `__main__.py:main()` already argv-multiplexes `--smoke` /
`--check-mediakeys`, making a frozen re-exec dispatch the natural fix.
