## Copywriting Contract

Phase 1 introduces no new user-facing copy. All existing strings must be preserved exactly as extracted from `main.py`. The table below documents existing copy that executors must not alter during the module split.

| Element | Existing Copy | Location after extraction | Source |
|---------|---------------|--------------------------|--------|
| Now-playing label (stream playing) | `"Now Playing: {title}"` | `ui/main_window.py` — `_play_station` callback | main.py pattern (RESEARCH.md Pattern 2) |
| Now-playing label (YouTube error) | `"yt-dlp error"` | `ui/main_window.py` via `Player.play()` | main.py |
| Now-playing label (no URL) | `"(no URL set)"` | `player.py` — `Player.play()` | RESEARCH.md Pattern 2 |
| New station default name | `"New Station"` | `repo.py` — `create_station()` | main.py |
| Station subtitle format | `"{provider} • {tags}"` | `ui/station_row.py` — `StationRow.__init__` | RESEARCH.md Pattern 3 |
| Station subtitle fallback provider | `"Unknown"` | `ui/station_row.py` | RESEARCH.md Pattern 3 |

No new CTAs, empty states, error states, or destructive confirmations are introduced in Phase 1. Existing EditStationDialog destructive action (delete station) must be preserved with its current GTK confirmation pattern — no changes.

## Phase 1 Interaction Invariants

These are the specific behavioral contracts that verify the extraction succeeded. The executor must confirm all of these hold after the module split:

1. **Station list renders identically** — `StationRow` produces the same visual output as the inline row-building code in `main.py` (Adw.ActionRow title/subtitle, 48x48 prefix image if art exists).
2. **Play/stop behavior unchanged** — `Player.play()` and `Player.stop()` produce the same GStreamer state transitions as the original `_play_station` / `_stop` methods.
3. **Now-playing label updates** — `now_label.set_text()` is called with the same string format (`"Now Playing: {title}"`) as before.
4. **EditStationDialog save/cancel** — callback chain (`_save` signal → `Repo.update_station` → `MainWindow.reload_list`) produces identical behavior.
5. **App launch sequence** — `ensure_dirs → db_connect → db_init → Repo(con) → MainWindow(app, repo) → win.present()` order is preserved in `__main__.py`.
