<task type="auto">
  <name>Task 1: Fix YouTube audio-only format selection</name>
  <files>musicstreamer/player.py</files>
  <action>
In `_play_youtube`, make two changes:

1. Change the format selector on line 36 from `"best[protocol^=m3u8]/best"` to `"bestaudio[ext=m4a]/bestaudio/best"`. This tells yt-dlp to prefer an audio-only m4a stream, falling back to any audio, then to best (which may be video+audio muxed). The current selector picks DASH video-only tracks on modern YouTube, which GStreamer plays silently.

2. After `info = ydl.extract_info(url, download=False)` (line 41), add a guard before using the URL:
```python
acodec = info.get("acodec", "none")
if acodec == "none":
    print(f"yt-dlp: selected format has no audio codec (acodec={acodec})")
    on_title("(no audio track)")
    return
```
This prevents silently playing a video-only stream if the format fallback somehow selects one.
  </action>

  </verify>
