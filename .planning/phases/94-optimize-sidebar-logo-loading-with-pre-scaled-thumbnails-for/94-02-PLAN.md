<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk → Qt image decoder | _generate_thumb decodes a PNG via QImage(source_path). The source PNG was downloaded to disk by aa/soma/gbs importers, not user-supplied at render time. Malformed/truncated PNG → QImage.isNull() → callback(None), no crash. |
| derived thumb path → disk write | thumb_path is computed solely as dirname(abs_art_path(rel))/THUMB_FILENAME on the main thread; the worker writes only to that pre-computed sibling path. No path component is attacker-controlled at render. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-02 | Tampering | Thumbnail write location (path traversal) | mitigate | thumb_path = os.path.dirname(abs_art_path(rel))/THUMB_FILENAME; abs_art_path roots relative paths under paths.data_dir() (trusted). No user string is concatenated into the write path. |
| T-94-03 | Denial of Service / Info disclosure | Malformed or truncated source PNG decode | mitigate | QImage(source).isNull() guard → callback(station_id, source_path, None); no exception escapes the worker (outer try/except). Renderer keeps the fallback icon. |
| T-94-04 | Info disclosure | Torn thumbnail visible mid-write | mitigate | Atomic write: mkstemp in same dir + os.replace; reader sees old-complete or new-complete file, never partial. |
| T-94-05 | Tampering | Image decoder robustness (Qt PNG decoder on untrusted bytes) | accept | Qt's PNG decoder is the same decoder already used for the full-res Now Playing path; bytes originate from the import flow, not the render-time attacker surface. ASVS L1: no new external input surface introduced. |
| T-94-SC | Tampering | npm/pip/cargo installs | mitigate | No packages installed — zero external dependencies (RESEARCH.md Package Legitimacy Audit: N/A). No install tasks. |
