## Spacing Scale

Declared values (multiples of 4 only). Qt layouts use `setContentsMargins` and `setSpacing`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip padding-vertical, QLineEdit inner padding |
| sm | 8px | Chip padding-horizontal, button inner padding, spacing between grouped widgets |
| md | 16px | Dialog content margins, section padding, default layout spacing |
| lg | 24px | Dialog outer margin, spacing between logical sections |
| xl | 32px | Swatch grid cell size (AccentColorDialog) |
| 2xl | 48px | Swatch button hit target with padding (AccentColorDialog) |
| 3xl | 64px | not used this phase |

Exceptions:
- Swatch buttons (AccentColorDialog): 32×32px minimum visual size, 40×40px including padding, so click target is touch-safe
- QDialog minimum width: 360px (AccountsDialog, AccentColorDialog); 480px (CookieImportDialog — needs space for paste textarea)

## Typography

Source: existing codebase patterns (station_list_panel.py, now_playing_panel.py).
All sizes are point sizes via `QFont.setPointSize()` — Qt resolves to device px at runtime.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / label | 9pt | Normal (QFont.Normal / 400) | system default (~1.4) |
| Dialog body text | 10pt | Normal (400) | system default |
| Heading / ICY prominent | 13pt | DemiBold (QFont.DemiBold / 600) | system default (~1.2) |
| Small / secondary label | 9pt | Normal (400) | system default |

Notes:
- 9pt matches the "recently played" label and name/provider label pattern (existing)
- 13pt/DemiBold matches the ICY title label pattern (existing)
- New dialog labels follow the 10pt/Normal body pattern
- Section headers inside dialogs (e.g. "Twitch" section label): 10pt/DemiBold

## Color

Source: `constants.ACCENT_PRESETS`, `_CHIP_QSS`, `_SEG_QSS` from codebase.
The app uses Qt system palette throughout — no hardcoded dark/light background hex.
Only accent and destructive use explicit hex.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | palette(window) / palette(base) | Dialog backgrounds, widget surfaces — Qt system palette |
| Secondary (30%) | palette(button) / palette(mid) | Input fields, unselected chip backgrounds, groupbox backgrounds |
| Accent (10%) | ACCENT_PRESETS[0] default: #3584e4; runtime override via QSS | Chip selected state, segmented control active state, QSlider::sub-page |
| Destructive | #c64d92 (error inline) / #c0392b (status error text) | Inline error status labels only (e.g. import failure) |

Accent reserved for:
- `QPushButton[chipState="selected"]` — background + border
- `QPushButton[segState="active"]` — background
- `QSlider::sub-page:horizontal` — progress fill
- `QPushButton` filter-toggle text color

The accent color is NOT applied to: dialog buttons (OK/Cancel/Apply/Reset), QMenu items, status labels, QLineEdit borders.

`build_accent_qss(hex_value)` QSS targets (new function, Phase 40):
```
QPushButton[chipState="selected"] { background-color: {hex}; border-color: {hex}; color: palette(highlighted-text); }
QPushButton[segState="active"] { background-color: {hex}; color: palette(highlighted-text); }
QSlider::sub-page:horizontal { background-color: {hex}; }
QPushButton#filterToggle { color: {hex}; }
```

Reset behavior: `QApplication.instance().setStyleSheet("")` — safe only if no other global QSS is applied. Planner must verify no other `QApplication.setStyleSheet()` calls exist before trusting this.

## Component Inventory

### AccountsDialog (UI-08)

- Layout: `QVBoxLayout`, single "Twitch" `QGroupBox` section
- Status row: `QLabel` showing "Connected" or "Not connected" — 10pt/Normal
- Action button: `QPushButton` "Connect Twitch" or "Disconnect" depending on `os.path.exists(twitch_token_path())`
- Close button: standard `QDialogButtonBox(QDialogButtonBox.Close)`
- Modality: modal (`exec()`) — simpler given it mutates token state
- Minimum width: 360px
- Connection in progress state: button disabled + label "Connecting..." while `QProcess` is running

### CookieImportDialog (UI-09)

- Layout: `QTabWidget` with three tabs: "File", "Paste", "Google Login"
  - Tab "File": `QPushButton` "Choose File..." + `QLabel` showing selected filename (truncated to 40 chars)
  - Tab "Paste": `QTextEdit` placeholder "Paste Netscape cookie text here..." + `QPushButton` "Import"
  - Tab "Google Login": `QLabel` "Opens Google login in a browser window." + `QPushButton` "Open Google Login"
- Bottom: `QDialogButtonBox(QDialogButtonBox.Close)`
- Feedback: uses `main_window.show_toast()` for success/error (not inline) — except inline validation error label below tabs for format failures
- Inline error label: 9pt, color #c0392b, hidden by default, shown on validation failure
- Minimum width: 480px

### AccentColorDialog (UI-11)

- Layout: `QVBoxLayout`
  - Swatch grid: `QGridLayout`, 4 columns × 2 rows = 8 swatches
  - Each swatch: `QPushButton`, no text, 32×32px fixed size, `border-radius: 4px`, background-color set to preset hex, selected state shown with 2px white inner ring + 2px colored outer border
  - Hex entry row: `QLabel` "Hex:" + `QLineEdit` (max 7 chars, placeholder "#rrggbb")
  - Preview: live — swatch click or valid hex entry immediately calls `QApplication.instance().setStyleSheet(build_accent_qss(hex))`
- Buttons: `QDialogButtonBox` with "Apply", "Reset", "Cancel"
  - Apply: saves to repo + writes `accent_css_path()` + accepts dialog
  - Reset: clears repo setting + `setStyleSheet("")` + updates swatch selection to none
  - Cancel: restores `_original_qss` saved at dialog open
- Minimum width: 360px
- `self._swatches`: list of QPushButton instances (not closures) to avoid QA-05 GC

### Hamburger Menu (UI-10)

Flat `QMenu` populated in `MainWindow.__init__`, no submenus.

```
Discover Stations       (triggers DiscoveryDialog)
Import Stations         (triggers ImportDialog)
──────────────────
Accent Color            (triggers AccentColorDialog)
YouTube Cookies         (triggers CookieImportDialog)
Accounts                (triggers AccountsDialog)
──────────────────
Export Settings         (disabled, tooltip "Coming in a future update")
Import Settings         (disabled, tooltip "Coming in a future update")
```

## Interaction States

### AccountsDialog
| State | Visual |
|-------|--------|
| Not connected | Label "Not connected", button "Connect Twitch" (enabled) |
| Connecting | Label "Connecting...", button disabled |
| Connected | Label "Connected", button "Disconnect" (enabled) |
| OAuth subprocess failed (exit_code != 0) | Show QMessageBox.warning "Twitch connection failed. Try again." |

### AccentColorDialog
| State | Visual |
|-------|--------|
| No saved accent | No swatch selected, hex entry empty |
| Swatch hovered | Cursor: pointer; swatch scales to 34×34px via stylesheet |
| Swatch selected | 2px white inner ring + 2px colored outer ring via stylesheet |
| Hex entry valid | No indicator change — live preview updates immediately |
| Hex entry invalid | QLineEdit border turns red (`border: 1px solid #c0392b`) |
| Apply clicked | Dialog closes, accent persists |
| Reset clicked | All swatches deselected, hex entry cleared, QSS stripped |

### CookieImportDialog — File tab
| State | Visual |
|-------|--------|
| No file chosen | Filename label shows "No file selected" (9pt, muted) |
| File chosen (valid) | Filename shown (truncated), "Import" button appears below label |
| File chosen (invalid) | Inline error label shown in red |

### CookieImportDialog — Paste tab
| State | Visual |
|-------|--------|
| Empty textarea | Import button disabled |
| Text entered | Import button enabled |
| Validation failed | Inline error label shown below textarea |
