  <action>
Add two new functions to `musicstreamer/accent_utils.py` (per D-11):

1. `build_accent_qss(hex_value: str) -> str` — returns a Qt QSS string targeting ONLY `QSlider::sub-page:horizontal { background-color: {hex_value}; }`. This is the only selector that needs global QSS because chip/segment/filter-toggle all use `palette(highlight)` in per-widget stylesheets.

2. `apply_accent_palette(app, hex_value: str) -> None` — modifies the app's QPalette: sets `QPalette.Highlight` to `QColor(hex_value)` and `QPalette.HighlightedText` to white. Then calls `app.setStyleSheet(build_accent_qss(hex_value))`. This causes all widgets using `palette(highlight)` in their QSS to pick up the new color on next polish.

3. `reset_accent_palette(app, original_palette) -> None` — calls `app.setPalette(original_palette)` and `app.setStyleSheet("")` to restore the default accent.

IMPORTANT: The reason for palette modification instead of global QSS is that per-widget `setStyleSheet()` calls (used by _CHIP_QSS, _SEG_QSS, filter toggle) have higher specificity than `QApplication.setStyleSheet()`. Changing the palette's Highlight role is the only way to affect what `palette(highlight)` resolves to in those per-widget stylesheets.

Do NOT modify `build_accent_css()` (GTK-era, kept for backward compat).

Add tests to `tests/test_accent_provider.py` for the new functions.
  </action>
