Trigger-condition semantics (RESEARCH §_GbsLoginWindow Design lines 247-423, VERIFIED 2026-05-16):

- `csrftoken` is set by Django on first page load (anonymous GET to `/accounts/login/`).
- `sessionid` is set ONLY after successful authentication (POST to `/login` with valid credentials).
- Therefore: waiting for BOTH cookies on the gbs.fm domain is the deterministic "login succeeded" signal.
- Collect EVERY gbs.fm-domain cookie (not just the two trigger names) to be forward-compat with auxiliary cookies (Django `messages`, future CSRF rotation). The trigger-name check gates WHEN to flush, not WHAT to collect.
