## Spacing Scale

GTK4 spacing follows the same 4-point base as the web 8-point scale. All values are in logical pixels.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline label gaps, icon-to-label padding (e.g. `spacing=4` in center column Gtk.Box) |
| sm | 8px | Chip strip margins, filter box spacing, list row padding-top/bottom |
| md | 12px | Center column left margin in now-playing panel |
| lg | 16px | Panel margin-top/margin-bottom |
| xl | 24px | Panel margin-start/margin-end |
| 2xl | 32px | — (not used in this phase) |
| 3xl | 48px | — (not used in this phase) |

Exceptions:
- Now-playing panel fixed height: 160px (size_request)
- Logo and cover art images: 160×160px fixed
- Station thumbnail prefix in list rows: 48×48px
- Touch/click target minimum for trash icon button: 44×44px (apply `set_size_request(44, 44)` or let Adwaita defaults handle it)

## Typography

This is a GTK4/Adwaita app. Typography is governed by the Adwaita HIG — no explicit px values in CSS unless overriding. Declare roles as Adwaita widget classes.

| Role | Adwaita Equivalent | Weight | Line Height |
|------|-------------------|--------|-------------|
| Body (track title) | `Gtk.Label` default | 400 regular | 1.5 (system default) |
| Dim label (station name, "Nothing playing") | `Gtk.Label` + `.dim-label` CSS class | 400 regular | 1.5 |
| Row primary (track title in favorites row) | `Adw.ActionRow` title | 400 regular | 1.2 |
| Row secondary (station name + provider) | `Adw.ActionRow` subtitle | 400 regular | 1.2 |

Rules:
- All `Adw.ActionRow` title and subtitle strings MUST be escaped with `GLib.markup_escape_text(value, -1)` before assignment — ActionRow parses Pango markup.
- No custom font sizes. Match existing now-playing labels: `title_label` and `station_name_label` patterns.

Source: CONTEXT.md D-09; existing code station_row.py and main_window.py

## Color

This app uses Adwaita's named color tokens — no hex overrides except in `_APP_CSS`. The 60/30/10 model maps to Adwaita roles.

| Role | Adwaita Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `@window_bg_color` | App window background, station list area |
| Secondary (30%) | `@card_bg_color` (+ shade 1.04→0.97 gradient) | Now-playing panel (`.now-playing-panel`) |
| Accent (10%) | `@accent_color` (system) | Stop button (`.suggested-action`), star button active state |
| Destructive | `@destructive_color` (system) | Trash icon button on each favorites row — use `.destructive-action` CSS class |

Accent reserved for:
1. Stop button (`.suggested-action` — already established)
2. Star button in filled/active state (`starred-symbolic` icon, normal color is fine — no explicit accent override needed)

Destructive reserved for:
1. Trash icon button on each Favorites row only

No custom hex values needed for this phase. Existing `_APP_CSS` classes carry forward unchanged.

## Component Inventory

New components introduced in this phase:

### 1. Adw.ToggleGroup (view toggle)

- Placement: Gtk.Box row inserted above the station list scroller, inside `Adw.ToolbarView` as an additional top bar or as first child of content area
- Segments: "Stations" | "Favorites"
- Default active: "Stations"
- Width: `set_hexpand(False)`, `set_halign(Gtk.Align.CENTER)` — centered above the list
- Margins: `set_margin_top(8)`, `set_margin_bottom(8)`
- Widget hierarchy: wrap in a `Gtk.Box` with `set_halign(CENTER)` and add as `shell.add_top_bar()` after filter_box

Source: CONTEXT.md D-01, D-02, D-03; Claude's Discretion for exact hierarchy

### 2. Star button (Gtk.Button, now-playing panel)

- Icon: `non-starred-symbolic` (outline) when not favorited; `starred-symbolic` (filled) when favorited
- Position: center column Gtk.Box, inserted before `self.stop_btn` (i.e. appended to center box in order: title_label → station_name_label → star_btn → stop_btn → volume_slider)
- Visibility: `set_visible(False)` by default; shown only when `is_junk_title()` returns False and a station is actively playing
- Sensitive: always True when visible (state is optimistic, no loading)
- Size: default Gtk.Button — no explicit size_request (Adwaita handles touch target)
- CSS class: none (icon-only button, no label)
- `set_icon_name("non-starred-symbolic")` / `set_icon_name("starred-symbolic")` to toggle state

Source: CONTEXT.md D-05, D-06, D-07, D-08

### 3. Favorites list row (Adw.ActionRow per row)

- Title: track title string (escaped with `GLib.markup_escape_text`)
- Subtitle: `"Station Name · Provider"` format (both values escaped)
- Suffix widget: `Gtk.Button` with icon `user-trash-symbolic`, CSS class `.destructive-action`
  - `set_valign(Gtk.Align.CENTER)`
  - No confirmation dialog — one tap removes immediately (CONTEXT.md D-11)
- Prefix widget: none (no thumbnail for favorites rows)
- CSS class on row: `"favorites-list-row"` (new class, inherits `.station-list-row` padding pattern)
- Row padding: `padding-top: 4px; padding-bottom: 4px` (same as `.station-list-row` in `_APP_CSS`)

Source: CONTEXT.md D-09, D-10, D-11

### 4. Favorites empty state (Adw.StatusPage)

- Title: "No favorites yet"
- Description: "Star a track while it's playing to save it here."
- Child widget: none (no CTA button needed — user must go play a station)
- Reuses existing `Gtk.Stack` empty-state swap pattern already in MainWindow

## Interaction Contracts

### View Toggle Behavior

1. User taps "Favorites" segment → filter_box (provider/tag chips) hidden via `set_visible(False)` (not just insensitive — per Claude's Discretion, hide to reduce confusion)
2. Station list content replaced with favorites list (or empty state) inline — no animation, no navigation
3. Search bar in HeaderBar remains visible but search does not apply to favorites view (search_entry ignored when in Favorites mode)
4. User taps "Stations" segment → filter_box restored to `set_visible(True)`, station list re-renders with current filter state

Source: CONTEXT.md D-03, D-04

### Star Button Behavior

1. ICY title arrives via `_on_title` → call `is_junk_title()` → if clean: `star_btn.set_visible(True)`, check `repo.is_favorited()` → set icon accordingly
2. No title (stop or junk) → `star_btn.set_visible(False)`
3. User taps star when not favorited → immediately set icon to `starred-symbolic` → call `repo.add_favorite()` (synchronous DB write) → if duplicate, silently no-op (star stays filled, per D-12)
4. User taps star when already favorited → immediately set icon to `non-starred-symbolic` → call `repo.remove_favorite()`
5. Star state re-evaluated on every `_on_title` callback (handles station change or re-tune)

Source: CONTEXT.md D-06, D-07, D-08, D-12

### Trash Button Behavior

1. User taps trash icon on a favorites row → remove row from UI immediately (no animation needed) → call `repo.remove_favorite()` → if favorites list becomes empty, swap to empty state
2. No confirmation dialog

Source: CONTEXT.md D-11
