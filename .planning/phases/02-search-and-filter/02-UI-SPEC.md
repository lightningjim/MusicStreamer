| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | GTK Usage |
|-------|-------|-----------|
| xs | 4px | `set_spacing(4)`, inline icon gap |
| sm | 8px | `set_margin_*( 8)` — compact strip padding |
| md | 16px | `set_margin_*(16)` — default element separation |
| lg | 24px | Outer section padding |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Filter strip top/bottom margin: 4px (xs) — strip must be visually thin, not section-weight
- Filter strip horizontal margin: 8px (sm) — matches Adw.ActionRow side padding convention
- Dropdown minimum width: 120px — prevents label truncation on common provider/tag names

## Widget Inventory

Specific widget choices within Claude's discretion (from CONTEXT.md):

| Widget | Choice | Rationale |
|--------|--------|-----------|
| Search | `Gtk.SearchEntry` set as HeaderBar title widget (`header.set_title_widget(search_entry)`) | Always-visible, full-width center — matches GNOME Software/Files pattern; locked in CONTEXT.md |
| Filter strip container | `Gtk.Box(orientation=HORIZONTAL)` added via `shell.add_top_bar(filter_box)` | Natural second top bar in `Adw.ToolbarView` — no extra nesting |
| Provider dropdown | `Gtk.DropDown` with `Gtk.StringList` model | Simpler API than `Adw.ComboRow`; ComboRow is for settings lists, not header controls |
| Tag dropdown | `Gtk.DropDown` with `Gtk.StringList` model | Same as provider |
| Clear button | `Gtk.Button` with label "Clear" | Appears in filter strip; hidden (`set_visible(False)`) when no filters active |
| Empty state | `Adw.StatusPage` | Standard Adw empty-state widget; replaces ScrolledWindow content |
| Filter logic | `Gtk.ListBox.set_filter_func` | Cleanest fit for existing `Gtk.ListBox`; avoids migrating to `Gtk.FilterListModel` |

## Layout Contract

```
Adw.ApplicationWindow
  Adw.ToolbarView
    [top_bar 1] Adw.HeaderBar
                  pack_start: Add Station button
                  pack_start: Edit button
                  title_widget: Gtk.SearchEntry (full width)
                  pack_end: Stop button
    [top_bar 2] Gtk.Box (filter strip, HORIZONTAL, margin 4px top/bottom, 8px sides)
                  Gtk.DropDown (provider)   — label "Provider" as first item ("All")
                  Gtk.DropDown (tag)        — label "Tag" as first item ("All")
                  Gtk.Button "Clear"        — right-aligned, visible only when filter active
    [content]   Gtk.ScrolledWindow
                  Gtk.ListBox  (or Adw.StatusPage when zero results)
```

The filter strip must be visually thin — use `sm` (8px) vertical padding so it reads as a secondary bar, not a toolbar section.

## Interaction Contract

### Search (FILT-01)
- `Gtk.SearchEntry` emits `search-changed` signal — connect to `_on_filter_changed`
- Filter applies on every keystroke (real-time, no debounce required — local data only)
- Placeholder text: "Search stations…"
- When text is cleared via the built-in clear icon, `search-changed` fires with empty string — handled automatically

### Provider Dropdown (FILT-02)
- First item: "All Providers" (index 0) — selecting this removes provider filter
- Items: sorted alphabetically, derived from `Repo.list_stations()` distinct `provider_name` values
- Signal: `notify::selected` — connect to `_on_filter_changed`

### Tag Dropdown (FILT-03)
- First item: "All Tags" (index 0) — selecting this removes tag filter
- Items: sorted alphabetically, derived from normalized tag vocabulary across all stations
- Tag normalization: split on `,` and `•`, strip whitespace, deduplicate case-insensitively (store display form as title-case of first occurrence)
- Signal: `notify::selected` — connect to `_on_filter_changed`

### AND Composition (FILT-04)
- `_on_filter_changed` reads current state of all three controls
- `set_filter_func` callback returns `True` only when ALL active filters match the row's station
- A filter with "All" selected (index 0) is considered inactive — does not restrict results

### Clear (FILT-05)
- Clear button: `set_visible(True)` when any filter is active, `set_visible(False)` when all are at default
- Clicking Clear: reset SearchEntry text to "", reset both dropdowns to index 0, call `_on_filter_changed`
- Active detection: search text non-empty OR provider index > 0 OR tag index > 0

### Zero-Result State
- When filter_func returns False for every row: swap ScrolledWindow child to `Adw.StatusPage`
- When filters are cleared or results return: swap back to ScrolledWindow with ListBox
