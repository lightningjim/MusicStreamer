<objective>
Close gap G-01 (SOMA-11 / UAT-07): re-importing SomaFM emits no toast because `_SomaImportWorker.finished = Signal(int, int)` shadows the inherited `QThread.finished` (a no-arg C++ signal Qt auto-emits when `run()` returns). When the thread completes, Qt's C++ layer auto-emits `QThread::finished()` with no arguments → PySide6 dispatches into the slot expecting 2 ints → `TypeError` raised inside the Qt event dispatcher and silently swallowed. No toast.

Rename `_SomaImportWorker.finished` to `import_finished` (Signal(int, int)) and update the emit + connect sites. Apply the same rename to the three other workers in `main_window.py` that have the identical latent bug (`_GbsImportWorker.finished`, `_ExportWorker.finished`, `_ImportPreviewWorker.finished`) so the project is free of QThread.finished shadowing across all four classes.

Purpose: Restore SOMA-11 re-import toast and clear the latent bug project-wide.
