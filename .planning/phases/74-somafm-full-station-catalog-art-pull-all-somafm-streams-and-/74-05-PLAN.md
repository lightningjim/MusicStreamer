<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SomaFM relay URL → DB bitrate field | The relay URL slug is derived from the upstream SomaFM API response. A malicious or compromised SomaFM mirror could publish a URL slug like `-999999999-mp3` to inject a bogus bitrate value into the DB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74.1-01 | Tampering | `_bitrate_from_url` parsing of upstream SomaFM URL | accept | The bitrate field is an `int` used only for sort-order in `stream_ordering.py` and display labels; an arbitrarily large value cannot cause memory exhaustion (single int) or SQL injection (parameterised inserts). Upstream SomaFM is the existing trust anchor for this phase; the same trust applies to the catalog title, description, and image URL fields. Worst case: a single station ranks oddly until the user deletes it. |
| T-74.1-02 | Denial of Service | `_BITRATE_FROM_URL_RE` regex engine on adversarial input | mitigate | The regex `r"-(\d+)-(?:mp3|aac|aacp)\b"` has no nested quantifiers or backtracking traps (Python's `re` is NFA-based and the pattern is linear-time on any input). URL length is bounded by the SomaFM channels.json response — no risk of ReDoS. No additional mitigation needed. |
</threat_model>
