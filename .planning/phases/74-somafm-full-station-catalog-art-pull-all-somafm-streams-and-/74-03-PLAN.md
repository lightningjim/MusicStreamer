<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GUI click → QThread.run | The hamburger menu action triggers a worker thread that calls outbound HTTP + DB writes. The trust boundary is the `Signal(int, int)` + `Signal(str)` interface; raw exceptions never cross the thread boundary (`str(exc)` truncation in the error path is the only data shape). |
| QThread → SQLite | The worker constructs its own `Repo(db_connect())` per Phase 60 precedent; the main thread's Repo is never touched from the worker. |
| GUI text → user display | Toast strings include user-controlled error text after a `msg[:80]` truncation; no HTML rendering (toast uses plain text), so no XSS or RichText injection surface (T-40-04 invariant unchanged). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-01 | I (User-Agent missing → SomaFM may ban) | Wired through to Plan 02's `soma_import.fetch_channels` | mitigate (inherited) | Plan 02's `_USER_AGENT` constant is set on every outbound request. This plan does not add new HTTP call sites. |
| T-74-05 | D (cross-channel exception aborts entire batch) | `_SomaImportWorker.run` exception handler | mitigate | Bare `except Exception as exc: self.error.emit(str(exc))` ensures even a catastrophic catalog-fetch failure produces a clean toast rather than a UI freeze. Plan 02's per-channel try/except handles the in-loop case. |
| T-74-06 (new) | I (Information disclosure via toast) | `_on_soma_import_error` toast text | accept | `str(exc)` may contain stack-trace-like content from arbitrary stdlib exceptions. Truncation to 80 chars + `"…"` mitigates the worst case; a future hardening phase could scrub stack frames, but for a public-API importer the user-actionable signal trumps the disclosure risk. |
| T-74-07 (new) | D (worker GC mid-flight — SYNC-05 history) | `MainWindow._soma_import_worker` retention | mitigate | Click handler stores worker on `self._soma_import_worker` with `parent=self`; both done/error slots reset to None after the cycle completes. Wave-0 test #2 asserts the attribute is set immediately after click. |
</threat_model>
