## Decisions Made

- **Threat T-999.3-03 mitigation strengthened in code:** all seven `_emit_event` call sites pass short enum-like detail strings (`"state_mismatch"`, `"access_denied"`, `"120s"`, `"empty_capture"`, `""`, `str(errno)`). `self._server.twitch_error` is sourced from the `?error=` query-param value (Twitch error code like `access_denied`), not the full URL.
- **Defensive fallback in `_TwitchAuthWindow._poll`:** if `server.done` is set but neither `captured_event` nor `twitch_error` is populated, emit `InvalidTokenResponse detail="empty_capture"` and exit 1. This handles a theoretical race where the event is set without either field; the plan's main behaviors remain intact.
- **QTimer.singleShot for timeout, QTimer repeating for poll:** two separate timers, poll stopped in `_shutdown_and_exit`. Matches plan pattern without mixing cadences.
- **`sys.stdout.write(token)` instead of `print(token, end="")`:** explicit flush for reliability when QProcess consumes it. Behavior identical to Google path.

## Deviations from Plan

None — plan executed as written. The "defensive fallback" above is a natural extension of the plan's requirement that every terminal outcome emits exactly one JSON-line event; it does not alter any behavior the plan explicitly specifies.
