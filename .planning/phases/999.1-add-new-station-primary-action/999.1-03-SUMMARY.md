## Key Decisions

### D-1: Lambda-capture of `new_id` (not functools.partial)

The plan's Pitfall 2 allowed either style. `main_window.py` has no `functools` import; `_on_edit_requested` already uses a self-capturing lambda (`lambda: self._sync_now_playing_station(fresh.id)`). Following the precedent minimizes the diff and matches local convention.

### D-2: No connection to `_sync_now_playing_station` — comment phrasing

D-07c explicitly forbids auto-play. The rationale comment originally named the method directly (`"intentionally NOT wired to _sync_now_playing_station"`), but the plan's acceptance grep demanded a literal zero-count of that identifier inside the slot. The comment was reworded to `"NOT wired to the now-playing-sync slot"` to preserve intent while satisfying the grep gate. Behavior is identical.

### D-3: Test monkeypatch invokes `self.reject()`

The Wave-0 test stubbed `exec` as `lambda self: QDialog.Rejected` — a bare return that bypasses the dialog's `reject()` override. Plan 01's is_new delete-on-reject lives inside `reject()`, so the bare-return path would leave the placeholder orphaned and fail the test. Replaced with an inner helper that calls `self.reject()` first, then returns Rejected — this models a real user-clicked-Cancel flow and exercises the full cleanup path.

No code change; environment-only fix.
