<pitfalls>
- **Pitfall 1 (CRITICAL — proxy mapping):** Tree is bound to `self._proxy`, NOT `self.model`. `tree.setCurrentIndex(source_idx)` will silently fail to render the selection. MUST call `self._proxy.mapFromSource(source_idx)` and use the resulting proxy index. Research Pitfall #2 documents this exactly.
- **Pitfall 2 (tree walk):** `StationTreeModel` has no `index_for_station_id`. Walk the tree: for each provider row at `model.index(prov_row, 0)`, iterate its children at `model.index(child_row, 0, prov_idx)`, fetch `model.station_for_index(child_idx)`, compare `.id`. Return on first match.
- **Pitfall 3 (Favorites mode):** If user is on Favorites page (`self._stack.currentIndex() == 1`), setting tree currentIndex won't be visible. Call `self._on_stations_clicked()` first. Tests may or may not exercise this — include it defensively.
- **Pitfall 4 (button visibility in Favorites):** D-02 says "header row" without specifying visibility scope. Research recommends top-level header (visible in both modes) because "New Station" is globally relevant. Place the new header BEFORE the segmented control in the main `layout`.
- **Pitfall 5 (layout.insertWidget vs addWidget order):** The main layout currently adds segmented control FIRST. Inserting the new header at index 0 pushes everything down. Use `layout.insertWidget(0, header_row)` after creating it, or construct the header BEFORE the segmented control in code order.
- **Pitfall 6 (button right-alignment):** Use `addStretch(1)` BEFORE `addWidget(self._new_station_btn)` in the header's QHBoxLayout. Test `test_new_station_button_exists_and_right_aligned` verifies this via layout inspection or by checking the stretch item precedes the button.
- **Pitfall 7 (bound slot, not lambda):** QA-05 forbids self-capturing lambdas. Use a bound method `self._on_new_station_btn_clicked` that calls `self.new_station_requested.emit()`.
</pitfalls>

    ```python
    def select_station(self, station_id: int) -> None:
        """Programmatically select a station in the tree by id (D-07).

        Switches to Stations mode if currently showing Favorites so the
        selection is visible to the user. Maps source-model index through
        the filter proxy (Pitfall #2 — selection on bare source index does
        not render through the proxied view).
        """
        # Ensure Stations page is visible so the selection is user-facing
        if self._stack.currentIndex() != 0:
            self._on_stations_clicked()

        for prov_row in range(self.model.rowCount()):
            prov_idx = self.model.index(prov_row, 0)
            for child_row in range(self.model.rowCount(prov_idx)):
                child_idx = self.model.index(child_row, 0, prov_idx)
                station = self.model.station_for_index(child_idx)
                if station is not None and station.id == station_id:
                    proxy_idx = self._proxy.mapFromSource(child_idx)
                    self.tree.setCurrentIndex(proxy_idx)
                    self.tree.scrollTo(proxy_idx)
                    return
    ```
