**New file: musicstreamer/ui/streams_dialog.py (per D-03, D-04):**

Create `ManageStreamsDialog(Adw.Window)` with:

Constructor params: `(app, repo: Repo, station_id: int, on_changed=None)`

Layout:
- `Adw.ToolbarView` with `Adw.HeaderBar` (title="Streams", close button)
- Body: vertical Box containing:
  - `Gtk.ListBox` for stream rows (selection_mode=NONE)
  - "Add Stream" button at bottom

Each stream row is an `Adw.ActionRow`:
- title = label if non-empty, else truncated URL (first 60 chars)
- subtitle = quality badge (e.g. "hi") + codec if set, separated by " / "
- suffix box (horizontal): Up button, Down button, Edit button (pencil icon), Delete button (trash icon)

Up/Down buttons (per D-04 — using up/down buttons per research recommendation):
- Up: swap this stream's position with the one above, then refresh listbox
- Down: swap this stream's position with the one below, then refresh listbox
- First row's Up button is insensitive, last row's Down button is insensitive
- After swap, call `repo.reorder_streams(station_id, [ordered_ids])` and refresh

Delete button: call `repo.delete_stream(stream_id)`, refresh listbox

Edit button: populate the add/edit form at bottom with stream's current values

Add/Edit form (inline at bottom of dialog, in a `Gtk.Frame`):
- URL entry (Gtk.Entry, placeholder "Stream URL...")
- Label entry (Gtk.Entry, placeholder "Label (optional)...")
- Quality: `Gtk.DropDown` with `Gtk.StringList` containing `["", "hi", "med", "low", "custom"]`
  - When "custom" selected, show a `Gtk.Entry` for custom quality text
  - When "", "hi", "med", or "low" selected, hide custom entry
- Stream type: `Gtk.DropDown` with `Gtk.StringList` containing `["", "shoutcast", "youtube", "hls"]`
- Codec: `Gtk.Entry` with placeholder "Codec (MP3, AAC...)"
- Save button (suggested-action class) + Cancel button
- Save validates URL is non-empty, then calls `repo.insert_stream()` or `repo.update_stream()` depending on whether editing

State tracking:
- `self._editing_stream_id = None` (None = adding new, int = editing existing)
- On "Save": if `_editing_stream_id` is None, insert new stream with next position (max existing position + 1); else update existing stream
- On "Cancel": clear form, reset `_editing_stream_id = None`
- After save/cancel, refresh the listbox

`_refresh_list()` method:
- Call `repo.list_streams(self.station_id)` to get current streams
- Clear and rebuild `Gtk.ListBox` rows
- Set Up/Down sensitivity based on position

On close: call `on_changed()` callback if provided (so edit_dialog can reload station data)
