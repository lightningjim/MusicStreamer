## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Enabled-but-invisible SMTC session | Configuring `is_play_enabled`/`is_pause_enabled`/`is_stop_enabled` + subscribing to `button_pressed` is NOT sufficient — if `smtc.is_enabled` stays at its default (False) the session is created but hidden from the Win+V media overlay. Backend logs "SMTC session active" and tests pass on Linux, but on Windows the user sees nothing. | advisory | Smoke test on real Win11 VM is the only reliable gate. Mocked-winrt unit tests cannot detect this — MagicMock defaults attrs to truthy. Pattern resolved in commit 37fb557. |
| pip PySide6 in conda-forge GStreamer env | `pip install PySide6` in a conda-forge env containing GStreamer produces an unusable install: `ImportError: DLL load failed while importing QtCore: The specified procedure could not be found.` Wheels are ABI-mismatched with conda-forge's VC toolchain. `os.add_dll_directory`, VC++ redist, and version downgrades all fail to fix it. | advisory | In the Phase 44 runbook + pyproject comment: use `conda install -c conda-forge pyside6` (not pip) inside the GStreamer env. pip-installed PySide6 must be uninstalled first if present. |
