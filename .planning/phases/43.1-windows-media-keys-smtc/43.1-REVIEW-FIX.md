## Fixed Issues

### WR-01: `asyncio.run()` inside `_build_thumbnail_ref` raises if a loop is already running

**Files modified:** `musicstreamer/media_keys/smtc.py`
**Commit:** d002c71
**Applied fix:** Replaced the bare `asyncio.run(_await_store(writer))` call with a loop-aware pattern: try `asyncio.get_running_loop()`; on RuntimeError (no loop) use `asyncio.run()` as before; otherwise fall back to `asyncio.run_coroutine_threadsafe(_await_store(writer), loop).result(timeout=5.0)`. Preserves today's happy path while future-proofing against qasync/qtinter integration. Also hoisted `import asyncio` from the inline hot-path position into module-scope stdlib imports (incidentally addresses IN-01).
