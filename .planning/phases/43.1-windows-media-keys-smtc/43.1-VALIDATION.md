| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|

| SMTC overlay shows station + ICY + cover art | MEDIA-05 success criterion 4 | Requires the Windows media session overlay to render (OS-owned UI) | On Win11 VM: start playback, press Win+V to open media overlay, confirm station name + ICY track title + cover art thumbnail visible. | Overlay play/pause/stop buttons trigger Player | MEDIA-04 success criterion 3 | Requires clicking OS-rendered overlay buttons | On Win11 VM: open overlay, click overlay play/pause button, confirm audio state changes; click stop, confirm stopped |
| SMTC name NOT leaked after normal shutdown | D-09 success criterion 5 | Requires observing Windows media session registry across app lifetime | On Win11 VM: launch app, play station, close app normally (X button), open another media-aware app (e.g., Spotify), confirm MusicStreamer's SMTC session is gone |
| App launches with no winrt deps → NoOp fallback warning | D-07 success criterion 5 | Requires starting the app on a machine without the optional deps installed | On Win11 VM: create a fresh conda env WITHOUT `pip install .[windows]`, launch app, confirm it runs + logs "Windows media keys unavailable — install with pip install -e '.[windows]'", confirm playback still works (media keys simply have no effect) |
