# Phase 60.4 Plan 02: Token Counter UI (search dialog) Summary

**One-liner:** Wire `_GbsTokenWorker` + `_token_label` + 4 helper methods into `GBSSearchDialog`, surfacing the user's GBS.FM token count with three-tier color (red 0 / amber 1-3 / default 4+), optimistic decrement after submit, and a positional-request_id stale-discard guard for rapid-submit safety. Predecessor: `60.4-01-SUMMARY.md` shipped `gbs_api.fetch_user_tokens` which this plan consumes.

- New constant: `WARNING_COLOR_HEX: str = "#d4a017"` placed adjacent to `ERROR_COLOR_HEX` (D-T6 amber tier).
- Module docstring extended with Phase 60.4 D-T6 rationale (three-tier color; QSS-string consumer only; no QCOLOR companion until a `setForeground` caller surfaces).

### `musicstreamer/ui_qt/gbs_search_dialog.py` (~85 LoC additive net)
- Imports: `import logging` + `_log = logging.getLogger(__name__)` at module level. `_theme` import widened to `ERROR_COLOR_HEX, WARNING_COLOR_HEX`.
- New `class _GbsTokenWorker(QThread)`: clones `_GbsSubmitWorker` shape with two-arg signal payloads — `finished = Signal(int, int)` carrying `(request_id, count)`, `error = Signal(int, str)` carrying `(request_id, msg)`. The deviation from `_GbsSubmitWorker` is deliberate: by emitting `request_id` as the leading positional payload, the consumer-side stale-discard guard becomes a positional comparison (`if request_id != self._token_request_id: return`) that works correctly in BOTH wired Signal paths (where `self.sender()` returns the QThread) AND direct-call test paths (where `self.sender()` returns None).
- Worker run() routes:
  - `gbs_api.fetch_user_tokens(cookies) -> Optional[int]`
  - `count is None` → `error.emit(request_id, "parse_failed")` (D-T2 path)
  - `count is int` → `finished.emit(request_id, int(count))`
  - `GbsAuthExpiredError` → `error.emit(request_id, "auth_expired")`
  - Other exceptions → `error.emit(request_id, str(exc))`
- Four new `__init__` instance attributes: `_last_known_tokens: Optional[int] = None`, `_token_request_id: int = 0`, `_was_logged_in: bool = False`, `_token_worker: Optional[_GbsTokenWorker] = None`.
- New `_token_label = QLabel(...)` placed AFTER `top_row.addWidget(self._breadcrumb_label)` and BEFORE `root.addLayout(top_row)` (D-T4 placement). PlainText (T-40-04 / Pitfall H — mirrors `_breadcrumb_label`). Hidden by default; visibility toggled by `_refresh_login_gate`.
- `_refresh_login_gate` extended:
  - Reads `prev_logged_in = self._was_logged_in` early; writes `self._was_logged_in = logged_in`.
  - Toggles `_token_label.setVisible(logged_in)` (visibility only — text/stylesheet are owned by callers/slots so D-T8's `Tokens: —` stamp survives the `setVisible(False)`).
  - Kicks `_kick_token_worker(cookies)` ONLY when `logged_in and not prev_logged_in` — the False→True transition. Repeated logged-in calls (e.g., `showEvent` re-fires) do NOT spawn redundant workers.
- Four new methods:
  - `_kick_token_worker(cookies)`: increments `_token_request_id`, constructs `_GbsTokenWorker(cookies, rid)`, connects bound-method `_on_token_fetched`/`_on_token_error` (QA-05), retains `_token_worker` SYNC ref (mirrors `_submit_worker`), starts.
  - `_on_token_fetched(request_id, count)`: positional Pitfall A guard; sets `_last_known_tokens`, `_token_label.setText(f"Tokens: {count}")`, `_apply_token_color(count)`.
  - `_on_token_error(request_id, msg)`: positional Pitfall A guard; on `'auth_expired'` → flip to `Tokens: —` + clear stylesheet + `_refresh_login_gate()`; otherwise `_log.warning(...)` + `Tokens: —` + clear stylesheet (D-T2 / Pitfall 3 — silent log; NO toast).
  - `_apply_token_color(n)`: D-T6 three-tier: `n is None` → empty stylesheet; `n == 0` → `ERROR_COLOR_HEX`; `1 <= n <= 3` → `WARNING_COLOR_HEX`; `n >= 4` → empty stylesheet.
- `_on_submit_finished` success branch extended: optimistic decrement (`_last_known_tokens = max(0, _last_known_tokens - 1)`), label setText + color tier in lockstep (Pitfall B), then `_kick_token_worker(cookies)` for server-confirm refetch.
- `_on_submit_error('auth_expired')` branch extended: `_last_known_tokens = None` + `setText("Tokens: —")` + `setStyleSheet("")` BEFORE the existing toast and `_refresh_login_gate()` call (Pitfall C — flip lands BEFORE the gate's hide).

## Pitfall Mitigations Verified

- **Pitfall A** (rapid-submit token-fetch race): `_token_request_id` stamped per kick AND emitted as leading positional payload; slot guard `if request_id != self._token_request_id: return` discards stale callbacks. **No `self.sender()` use in `_on_token_fetched` or `_on_token_error`** — verified by `grep -E "self\.sender\(\)" musicstreamer/ui_qt/gbs_search_dialog.py | grep -E "_on_token_(fetched|error)" | wc -l == 0`. Direct-call test path (`test_token_stale_fetch_discarded`) exercises the same guard the wired Signal path uses.
- **Pitfall B** (color-tier boundary mid-decrement): `_on_submit_finished` success branch and `_on_token_fetched` BOTH call `setText` AND `_apply_token_color` together — never one without the other.
- **Pitfall C** (auth-expired during in-flight fetch): `_on_submit_error('auth_expired')` flips label to `Tokens: —` + clears stylesheet + nulls `_last_known_tokens` BEFORE `_refresh_login_gate()` runs. The gate's `setVisible(False)` then hides the labeled "—" cleanly.
- **Pitfall H / T-40-04** (plain-text invariant): `_token_label.setTextFormat(Qt.TextFormat.PlainText)` mirrors `_breadcrumb_label`. Grep baseline of 4 RichText/setHtml matches across `musicstreamer/ui_qt/` is **UNCHANGED** (verified).
- **Pitfall 3** (no-toast-spam on transient failures): `_on_token_error` for non-`auth_expired` msg uses `_log.warning(...)` only; no toast. `test_token_fetch_parse_miss_shows_emdash` regression-protects.
- **QA-05** (no lambdas in `connect()` calls in production dialog file): every new `worker.signal.connect(...)` uses bound-method references. Existing `test_no_self_capturing_lambdas_in_dialog` continues to pass.
- **Redundant-worker prevention** (plan-checker iter-1 WARNING 6): `_refresh_login_gate` kicks `_kick_token_worker` only on the False→True logged-in transition (`self._was_logged_in` tracker). `showEvent` re-fires while logged-in see `prev_logged_in == True` and skip the kick.
